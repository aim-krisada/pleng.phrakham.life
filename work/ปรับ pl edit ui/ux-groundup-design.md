## หลักออกแบบ (อ้างมาตรฐาน)
1. **Object + Action ไม่ใช่ Mode** — ผู้ใช้อยู่กับ "สิ่ง" (เพลง) แล้วทำ "การกระทำ" (เล่น/แก้/แชร์) · ไม่ใช่สลับ "โหมด" (Apple HIG direct-manipulation · Google Docs view→pencil)
2. **พื้นผิวเดียวต่อบริบท** — 1 หน้าจอ = 1 งานหลัก · ที่เหลือ progressive disclosure (NN/g minimalist · Material 3)
3. **Journey-first ไม่ใช่ menu-first** — วางของตามทางที่ผู้ใช้เดิน ไม่ใช่กองเมนูให้เลือกเอง
4. **touch ≠ pointer · mobile-first** · WCAG 2.2 AA · light/dark · offline (PWA)
5. **1 บริบท = 1 ปุ่มหลัก** (filled) · ปุ่มเดียวกันอยู่ที่เดียว

## 2 บริบทเท่านั้น (ทั้งแอปย่อเหลือเท่านี้)

### บริบท A — คลังเพลง (หน้าแรก) = "หา + สร้าง"
```
มือถือ                                  เดสก์ท็อป
┌───────────────────────────┐          ┌──────────────────────────────────────────┐
│ เพลง.พระคำ.ชีวิต        ☰ │          │ เพลง.พระคำ.ชีวิต              🔍ค้นหา…    ☰ │
│ ┌───────────────────────┐ │          ├──────────────────────────────────────────┤
│ │ 🔍 ค้นหาเพลง          │ │          │  📕เล่มใหญ่  📗อนุชน  📘เด็ก  ★โปรด  🎵ลิสต์ │
│ └───────────────────────┘ │          │  ─────────────────────────────────────    │
│ 📕เล่มใหญ่  📗อนุชน  📘เด็ก │          │  (bookshelf เดิม — P'Aim เคาะ คงไว้)        │
│ ★โปรด   🎵เพลย์ลิสต์      │          │                                            │
│ ── bookshelf (คงเดิม) ──  │          └──────────────────────────────────────────┘
│  🎵 12 พระเจ้าเที่ยงแท้ ★ │           ＋ = ลอย (FAB) หรือปุ่ม hero ในหน้า — "สร้างเพลงใหม่"
│              ┌──────────┐ │
│              │ ＋ เพลงใหม่│ │  ← สร้าง = อยู่ในบ้านเดียวกับหา (หน้าแรก) ไม่ bolt บนแถบ
│              └──────────┘ │
└───────────────────────────┘
```
- **หา** = ค้นหา (บนสุด) + bookshelf (คงเดิม · usage จริงคนเลือกเล่ม) + ★โปรด + 🎵ลิสต์
- **สร้าง** = ปุ่ม `＋ เพลงใหม่` เด่น *ในหน้าแรก* (FAB มือถือ / hero เดสก์ท็อป) — เจตนาต่างจากหา แต่**อยู่บ้านเดียวกัน** ไม่กระจายซ้าย-ขวาบนแถบ
- **☰** = เมนูจำเป็นเดิมที่เหลือ: คู่มือ · พระคำ↗ · เกี่ยวกับเรา · เข้าสู่ระบบ/บัญชี (ไม่แข่งพื้นที่บนสุด)
- **"รายการเพลง" ไม่ใช่เมนู** — มันคือหน้าแรกนี้ (แตะโลโก้กลับมา)

### บริบท B — หน้าเพลง = พื้นผิวเดียว (แผ่นเพลง) · ไม่มีโหมด
```
มือถือ
┌───────────────────────────────┐
│ ‹  12. พระเจ้าเที่ยงแท้  ✏️ ↗ ⋮ │  ← กลับ · ชื่อ · ✏️แก้ · ↗แชร์ · ⋮เพิ่มเติม
├───────────────────────────────┤
│   1  2  3  −  |  5  6  5  −     │
│   สรร-เส-ริญ    พระ-เจ้า         │  ← แผ่นเพลง = สิ่งที่แสดง (อ่าน/ร้องได้เลย)
│   (ไฮไลต์วิ่งตามตอนเล่น)         │
├───────────────────────────────┤
│            ▶  เล่น              │  ← Play = "ฝึกร้อง" · ปุ่มหลักเดียว (hero)
└───────────────────────────────┘
```
- **แผ่นเพลง = พื้นผิว** (ไม่ใช่ "โหมดแผ่น") · **▶เล่น = ฝึกร้อง** (เสียง+karaoke ไฮไลต์ · ไม่ใช่ "โหมดฝึก") · **✏️ดินสอ = แก้ inline** (ไม่ใช่ "โหมดแก้")
- **ไม่มี tab [ฝึก][แผ่น][แก้] อีกต่อไป**
- **⋮ เพิ่มเติม** = 🖨พิมพ์ A4 · ⬇ดาวน์โหลด · ★โปรด · ➕เข้าเพลย์ลิสต์ · 🔎เปิดเพลงอื่น · ⚙เสียง/คีย์/tempo (progressive — ใช้น้อยซ่อนไว้)
- **↗แชร์ + ✏️** โผล่บนแถบเลย (ใช้บ่อย/แกน #1) · ✏️ เปิดทุก tier (เกตที่ save เท่านั้น ตาม mission)

### บริบท B (แก้) — กด ✏️ → แผ่นเดียวกันกลายเป็นแก้ได้
```
┌───────────────────────────────┐
│ ‹ 12. พระเจ้าเที่ยงแท้  💾บันทึกแล้ว ✓ · เสร็จ │  ← สถานะบันทึก (แก้ M-6) · ปุ่ม "เสร็จ" ออกจากแก้
├───────────────────────────────┤
│   1  2 |3| − | 5  6  5  −       │  ← cursor บนโน้ต · พิมพ์ 1-7 เปลี่ยน · space เลื่อน (inline)
├───────────────────────────────┤
│ [123] [♯♭·] [คอร์ด] ... 🎼โครง │  ← แถบเครื่องมือ contextual (ตามที่เลือก · เกาะไม่บัง)
└───────────────────────────────┘
```
- แก้ = อยู่บนแผ่นเดิม (WYSIWYG) · แถบล่าง = contextual (โน้ต/สัญลักษณ์/คอร์ด) · **🎼โครง** = เปิด Drawer ลากบล็อก (ข้อ/รับ) จัดโครงเพลง
- **เสร็จ** → กลับพื้นผิวอ่าน · สถานะ "บันทึกแล้ว/ยังไม่บันทึก" ชัดตลอด

## Journey เต็ม (พิสูจน์ว่าทางเดินลื่น)
- **S (คนร้อง):** เปิดเว็บ → หน้าแรกค้น/เลือกเล่ม → แตะเพลง → **▶ เล่น** ร้องตาม (ไฮไลต์วิ่ง) → ปรับคีย์/ตัวใหญ่ (⋮/⚙) → ↗แชร์/★โปรด → (เพลย์ลิสต์) · **ไม่เจออะไรเกี่ยวกับ "แก้" มากวน**

- **M — สร้างเพลงใหม่:** หน้าแรก **＋เพลงใหม่** → แผ่นเปล่า (เข้าโหมดแก้เลย) → พิมพ์โน้ต+เนื้อ inline → **🎼โครง** ลากบล็อกจัดข้อ/รับ → 💾สถานะบันทึก → ส่งตรวจ

- **M — แก้เพลงที่มีอยู่ (วิเคราะห์เพิ่มตามพี่):** หน้าแรกค้น/เลือกเล่ม → เปิดเพลง (เห็นแผ่น = อ่านได้) → **✏️** เข้าแก้บนแผ่นเดิม → **แตะโน้ต/คำที่เห็น = cursor ไปตรงนั้น** (แก้จุดที่เห็น ไม่ใช่ไล่หา) → พิมพ์/แทรก/ลบ inline · สัญลักษณ์ผ่าน popup · โครงผ่าน 🎼Drawer → **สถานะ "บันทึกแล้ว✓/ยังไม่บันทึก" เห็นตลอด + autosave working-copy กันหาย** → **เสร็จ** ออกจากแก้ (กลับพื้นผิวอ่าน)
  - **บันทึกร่าง · ส่งตรวจ · เผยแพร่ = 3 ปุ่มแยกความหมายชัด** (เลิกปุ่มเดียวแปลว่า 5 อย่าง · แก้ M-4) · น้ำหนัก/สีต่างกัน
  - **"งานของฉัน"** (ร่าง/รอตรวจ/ส่งกลับ/อนุมัติ) = ป้ายชัดเข้าตรง ไม่ซ่อนใต้ "จัดการ" (แก้ M-7)
  - **ผู้อนุมัติ:** review banner inline (อนุมัติ/ส่งกลับ) — คงของดีเดิม
  - ⚠️ จุดเจ็บ M-edit เดิม (friction): กระโดดเข้าแก้ยาก · ปุ่มหลักคลุมเครือ · ไม่มีสถานะบันทึก → ground-up นี้แก้ครบ

## Theme สีใหม่ — อบอุ่น · กินใจ · เพลงเพราะ · สบายใจ (world-class · light+dark · WCAG AA)
**ทิศ v2 (แก้ตามพี่ — อบอุ่น + สดใส):** ปัญหาเดิม = ทองอำพันหม่น (saturation ต่ำ) เลยไม่สดใสแบบ Google/MS. **วิธี: พื้นยังอุ่น (ครีม) เพื่อความสบายใจ · แต่ accent เพิ่ม saturation+brightness เป็นชุดสีอุ่น "สดใส"** — ไม่ใช่ฟ้าเย็นแบบ Google (คงอุ่น แต่จี๊ดจ๊าดขึ้น) · **ความสดใสมาจาก accent ไม่ใช่พื้น**
| token | Light | Dark | เหตุผล |
|---|---|---|---|
| bg | ครีมอุ่น `#FDF7EE` | น้ำตาลเข้มอุ่น `#221E19` | อบอุ่น สบายตา ไม่คลินิก |
| text | น้ำตาลเข้ม `#362F28` | งาช้าง `#EFE7DA` | AA แต่ไม่แข็ง |
| **primary สดใส** | มาริโกลด์ `#F59E0B` (สว่าง+จัด) | อำพันสด `#FBBF3C` | แสงแดด/เทียนสด = warm + สดใส |
| secondary | คอรัลอุ่น `#F26B4E` | `#F98063` | มีชีวิตชีวา |
| ชุดหมวด (cheerful set · ใช้เท่าที่จำเป็น) | เขียวใบไม้ `#3FA34D` · ฟ้าอุ่น `#3E96D6` | สว่างขึ้น | หลากสีสดแบบ Google/MS แต่คุมโทนอุ่น · ใช้กับ chip/หมวด/สถานะ ไม่ใช่ทั้งจอ |
- **ข้อความบนสีจัด** = ใช้เฉดเข้มของสีนั้นเอง/ขาว ตามคอนทราสต์ WCAG (สีสว่างต้องระวังตัวอักษร)
- warm + สดใส สรุป: **พื้นอุ่น + accent สว่างจัดในตระกูลอุ่น (+ แต้มเขียว/ฟ้าเป็นหมวด)** = อบอุ่น สบายใจ แต่มีชีวิตชีวา ไม่หม่น
- ใช้เป็น **theme token จริงใน `styles.css`** (ของเดิมมี token อยู่แล้ว → เปลี่ยนค่า ไม่ hard-code) · verify คอนทราสต์จริง · light/dark ครบ
- **ไม่ทำ mockup ทิ้ง** — ถ้าพี่อยากเห็นก่อน ผม render swatch สั้น ๆ หรือ apply เป็น token บนแอปจริงให้ดู

## Brand · โลโก้ · favicon · PWA icon
- **โลโก้ปัจจุบัน = กุญแจซอล + ไม้กางเขน สีทอง** (`public/*.png` ครบชุด: favicon 16/32/ico · apple-touch 180 · android-chrome 192/512 · maskable 512 · hero) — mark ดีมาก (ดนตรี+ความเชื่อ) → **เก็บรูปทรงไว้ จำได้/มีความหมาย** · แค่ปรับสี
- **recolor → มาริโกลด์ `#F59E0B`** (โทนใหม่ สดใส) ทั้งชุด icon + brand ในแอป = สีเดียวกับธีม · ⚠️ ต้องมี **source SVG** หรือผม regenerate ชุด PNG ใหม่ · **maskable ต้องเผื่อ safe-zone** (mark อยู่ในวงกลมปลอดภัย ~80%)
- **manifest สีใหม่ (จาก clinical เดิม):** `background_color` #ffffff → **`#FDF7EE` ครีม** (splash) · `theme_color` #f8f9fa → **`#F59E0B` มาริโกลด์** (tint OS/status bar ให้ on-brand) — หรือครีมถ้าอยากซับเทิล (พี่เลือก)
- **แสดง brand — แนะนำ: icon เสมอ + wordmark บนเดสก์ท็อป** · ตอนนี้ มือถือ=icon / เดสก์ท็อป=ชื่อล้วน → **เดสก์ท็อปไม่เห็น icon = brand ไม่ consistent** · โลโก้ต้องมีเพื่อ PWA/favicon อยู่แล้ว → ใช้เป็น mark เดียวทั้งแอป: **icon(ทุกจอ) + ชื่อ(เดสก์ท็อป)** · icon = ปุ่มกลับหน้าแรก ⌂ · เหตุผล = recognition + cohesion กับ favicon/PWA (NN/g)

## UI i18n — ภาษาแอป (เผื่อ ไทย/จีน/อังกฤษ · P'Aim 22 ก.ค.)
**นี่คือภาษาของ interface (ปุ่ม/เมนู/ข้อความแอป) — คนละเรื่องกับเนื้อเพลงหลายภาษา (US-F1)** · **เผื่อตอนนี้ = ถูก+ถูกเวลา** (retrofit ทีหลังแพง)
- **ขอบเขตตอนนี้ = 3 ภาษา (th/zh/en) · อนาคตเพิ่มได้ "ไม่อั้น"** (P'Aim) → registry ต้อง **data-driven**: หย่อนไฟล์ locale ใหม่ → โผล่ในตัวสลับเอง ไม่มีเพดาน ไม่แตะโค้ด
- **externalize ข้อความ UI ทุกตัวผ่าน layer เดียว** (`t('key')` + `src/i18n/{th,zh,en}.js`) — ห้าม hard-code ไทยกระจาย · `th` = ต้นฉบับเต็ม · `zh`/`en` = โครงพร้อม เติมคำแปลได้ (label สั้น · ผมช่วยแปลชุด UI ได้ตอน string นิ่ง)
- **ตัวสลับภาษาใน ☰/ตั้งค่า — list ภาษาที่มีแบบ dynamic** (ไม่ hard-code 3) · ไทย default · detect เบราว์เซอร์ครั้งแรก · จำใน **localStorage (ไม่มี account)** ตรง mission
- **layout ยืดหยุ่นรับความยาวต่างภาษา** — จีนสั้น/อังกฤษยาวกว่าไทย → ห้ามปุ่ม/ป้ายกว้างตายตัว (สอดคล้อง responsive อยู่แล้ว)
- **font stack ครอบ ไทย + CJK (จีน) + Latin** — ให้ตัวจีนไม่กล่อง (เพิ่ม CJK fallback)
- **ship ไทยก่อน** · โครงพร้อม zh/en · ไม่ต้องแปลครบตอนนี้ (แค่ "เผื่อ" โครง)
- **กระทบทั้ง 2 สาย** — string ที่เขียนใหม่ต้องผ่าน `t()` ตั้งแต่แรก (สาย 2 กำลังเขียน UI = ต้องรู้ก่อน)

### G cross-check i18n (22 ก.ค. · ยืนยัน world-class + refinements)
- **word-break ไทย:** `word-break:keep-all; overflow-wrap:break-word;` (+`<wbr>`) ที่ label — ไทยไม่มีเว้นวรรค กันตัดกลางคำ
- **FONT = Hybrid Noto (G ฟันธง · แก้จากรอบแรก · ตรงกับที่ทีมใช้ Noto):** **UI chrome (ปุ่ม/เมนู/ข้อความแอป ทุกภาษา) = Noto self-host + subset** → brand consistent 100% ทุกเครื่อง · Noto Thai+Latin (WOFF2 subset) ~30-70KB/weight · Chinese-UI = subset เฉพาะ ~100-300 คำใน UI dict → <15KB · **รวม UI ทุกภาษา <150KB → pre-cache SW ได้ offline ชัวร์** · **แต่ "เนื้อเพลงจีน" (content อนาคต) = ใช้ system CJK** (PingFang/YaHei/Noto SC) เพราะ glyph เยอะเดายาก · full Noto SC/TC = 15-30MB ใหญ่ไป · unicode-range split ใช้ online ได้ แต่ offline PWA เดา pre-cache ไม่ได้ · fallback system กัน □ · helper ตอนทำ: @font-face + subset script + Vite PWA precache
- **localStorage fallback → `navigator.language`** ถ้าอ่านไม่ได้ (Incognito/iOS ล้าง storage)
- **lib: custom light layer ถูกแล้ว** สำหรับสเกลนี้ (<2KB · คุม dynamic registry 100%) · ระวัง pluralization → t() เผื่อ `{singular,plural}` (อังกฤษมีพหูพจน์ · ไทย/จีนไม่มี)

- **เผื่อ:** `Intl.DateTimeFormat/NumberFormat` (native · th-TH → พ.ศ. อัตโนมัติ — footer ใช้ พ.ศ. อยู่แล้ว) · **CSS logical props** (`margin-inline-start`/`text-align:start`) เผื่อ RTL อนาคต
