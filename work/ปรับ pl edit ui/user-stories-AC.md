## หลักการที่ทุก AC ต้องผ่าน (north star)
- **default ง่ายที่สุดในโลก สำหรับทุกเพศทุกวัย** · ความซับซ้อนดนตรี **ซ่อนไว้ มีให้คนที่ต้องการ** (progressive disclosure)
- **reuse ตรรกะ/เสียง/render/tests เดิม · ทำผิวหน้า inline ใหม่ที่บาง** (ไม่รื้อ engine · ไม่บวมของเดิม)
- world-class + responsive (desktop + มือถือ 360–412px) เป็นเงื่อนไขทุก story · WCAG 2.2 AA
- **แก้ที่บล็อกต้นทาง · วิวที่เห็น = ผลกาง** · แก้ทำนองที่เดียว ทุกข้อตาม (v2)

## Personas
- **S — คนร้อง/ฟัง** (คริสเตียน ทุกวัย ไม่รู้ดนตรี): เปิด → ร้อง/ฟัง/พิมพ์/แชร์ · ไม่ต้องมีคนสอน
- **M — คนทำเพลง** (พี่เปา/นักดนตรี = คอขวด · รู้ดนตรี): พิมพ์เพลงเข้าคลังเร็ว batch ได้ · ถูกต้อง

## EPIC A — เข้าโหมดแก้จากแผ่นเพลง (view → ดินสอ → edit) · [ทำต่อจาก WIP]

**US-A1** — *ในฐานะ M ฉันเห็นแผ่นเพลงฝึกร้องเป็น default แล้วกดดินสอ ✏️ เพื่อเริ่มแก้ตรงนั้น โดยไม่เด้งไปหน้าอื่น*
- AC-A1.1 default = แผ่นเพลงฝึกร้อง (โหมดอ่าน) · มีปุ่มดินสอ ✏️ เห็นชัด (FAB) — [reuse: `Studio.vue` modes, `SongViewer`/`SongSheet`; WIP `3e45ae0`]
- AC-A1.2 กดดินสอ → เข้าโหมดแก้ **บนแผ่นเพลงเดิม** (ไม่โหลดหน้าใหม่ ไม่เปลี่ยน layout กระโดด) · กดออก → กลับเป็นแผ่นเพลงเหมือนเดิม
- AC-A1.3 ดินสอเปิดให้ **ทุก tier** (แม้ไม่ล็อกอิน) — เกตเฉพาะตอน "บันทึกเข้าคลัง" [mission: เส้นแบ่ง = เก็บเข้าระบบ ไม่ใช่แก้ · WIP `c98e1ad`]
- AC-A1.4 โหมดแก้ทำงานทั้ง desktop + มือถือ · ไม่มี h-scroll · ปุ่มดินสอไม่ทับ dock เล่นเพลง [WIP `a204f1f`]

**US-A2** — *ในฐานะ M ฉันคลิก/แตะโน้ตหรือคำบนแผ่นเพลง แล้ว cursor ไปอยู่ตรงนั้นทันที เพื่อแก้จุดที่ฉันเห็น*
- AC-A2.1 คลิกโน้ต/พยางค์บน render → cursor + ไฮไลต์ไปที่ตัวนั้น (ไม่ใช่เลื่อนหาช่อง) [net-new: ต่อ hook `SongSheet` emit `seek` + `resolveContent` tag `_stanza`/`_entryIndex` → focus ต้นทาง]
- AC-A2.2 การชี้กลับต้นทางต้องนิ่ง — แก้แล้วไม่กระทบข้ออื่นผิด (index mapping source↔resolved ถูก) [ยึด `selection-driven` §3.1 · risk ที่ G เตือน]
- AC-A2.3 โน้ตจบที่รับ A/รับ B ใช้ร่วม: คลิกแก้ → เตือนว่า "กระทบทั้งคู่" ก่อนแก้ (ป้ายไฮไลต์ทุกที่ที่กระทบ)

## EPIC B — พิมพ์ inline เหมือนเอกสาร (โน้ต + เนื้อ) · [หัวใจ "ง่าย"]

**US-B1** — *ในฐานะ M ฉันพิมพ์เลข 1-7 บนแผ่นเพลงแล้วโน้ตเปลี่ยน/เพิ่มทันที เหมือนพิมพ์ Word ไม่ต้องเปิดกล่อง*
- AC-B1.1 เลือกโน้ต + พิมพ์ 1-7 = เปลี่ยนโน้ต (ทับ) · โชว์ผลบน render สด [reuse: `NoteBoxes` logic · `parseNotes` · `setSyl`]
- AC-B1.2 **แทรก (ripple)** = เพิ่มโน้ต ของข้างหลังเลื่อนขวา · **backspace = ลบดึงชิด** (default ที่พี่เคาะ) · เนื้อเลื่อนตาม [reuse: `pushSlot`/`pullSlot`]
- AC-B1.3 มี "ทับ vs แทรก" ให้สลับ (default แทรก/ripple · ทับไว้แก้โน้ตเดียวไม่ให้บรรทัดขยับ)
- AC-B1.4 แก้/แทรก/ลบ **ในทำนองที่ share → ripple ทุกข้อที่ใช้ทำนองนั้นพร้อมกัน** (เนื้อไม่เหลื่อม)
- AC-B1.5 **ลบต้อง work จริงทั้ง 2 แบบ** (ดึงชิด / เหลือช่องว่าง) — verify ด้วยการกดจริง ไม่ใช่แค่ querySelector [ชนะ DomiSol ที่ลบไม่ได้]

**US-B2** — *ในฐานะ M ฉันพิมพ์เนื้อไทยใต้โน้ตแบบต่อเนื่อง โดยแยกคำ/เอื้อน ได้ง่าย*
- AC-B2.1 `space` = พยางค์ถัดไป (คำใหม่) · `-` = แยกคำข้ามโน้ต · `_`/`~` = เอื้อน [มาตรฐานโลก · reuse `onSylKey`, `splitSyllables`]
- AC-B2.2 พิมพ์เนื้อทั้งท่อนรวดเดียว (paste ก้อน) แล้วระบบจัดลงโน้ตให้ [reuse bulk lyric textarea `EditorMode:618`]
[ความเห็นเอม - ตรงนี้อยากให้ระบบจัดการอัตโนมัติแบบที่เคยมีถ้าประโยคยาวที่มี space ก็ให้ถือเป็นแต่ละพยางค์ เติมดีมากมีกล่องให้พิมพ์ ตรวจสอบเว้นวรรค์พยางค์ แล้ว apply ไปใส่เนื้อ แบบใหม่ไม่รู้จะทำไงได้ อาจมีกล่องให้พิมพ์แยกเหมือนเดิม หรือจากข้างนอก แล้ว copy วาง แต่แบบหลังอาจแดบยากกว่าเพราะต้องออกนอกโปรแกรม ให้ช่วยดูว่าทำไงจึงจะดี ถ้าพิมพ์แก้แบบ Inline เหมือน word ได้เลย ก็อาจไม่ต่างและไม่ต้องมีแล้ว คือ copy ประโยคยาวแล้ว M สามารถกด space เพื่อแยกพยางค์และ ripple ได้ ก็จะยิ่งง่ายขึ้นไปอีก กระชับด้วย ขอสอบทานความเข้าใจ]

## EPIC C — เครื่องมือโผล่ตามสิ่งที่เลือก (selection-driven) · [ต่อยอด US selection-driven]

**US-C1** — *ในฐานะ M ฉันเห็นแค่เครื่องมือของสิ่งที่ฉันเลือก จอไม่รก (ไม่ใช่ 100 ปุ่มพร้อมกัน)*
- AC-C1.1 ไม่เลือกอะไร = จอสะอาด (เฉพาะ action ระดับเอกสาร) · เลือก scope ไหน โชว์เครื่องมือ scope นั้น [ยึด `selection-driven` §6 · เป้า <25 ปุ่ม]
- AC-C1.2 ปุ่ม verb ยุบเป็น "…สิ่งที่เลือก" (ฟัง/ลบ/ดูผล 5→1, 3→1, 3→1) [selection-driven เฟส 0]
- AC-C1.3 เครื่องมือมาในรูป **contextual popup** — **desktop เกาะ cursor** (fade ตอนพิมพ์รัว · smart-position ไม่บังโน้ต) · **mobile = แถบติดขอบคีย์บอร์ด/เหนือช่อง** (ห้าม floating ล่างจอ คีย์บอร์ดชน) [G cross-check · selection-driven §4.1]
[ความเห็นเอม - contextual popup ต้องขยับไม่บังสิ่งที่พิมพ์หรือต้องเห็น render โดยอัตโนมัติ]

**US-C2** — *ในฐานะ M ฉันใส่คอร์ด/สัญลักษณ์พิเศษ (จุด octave · #) ผ่าน popup บนโน้ตที่เลือก โดยเฉพาะบนมือถือที่คีย์บอร์ดไม่มีปุ่มพวกนี้*
[ความเห็นเอม - หรือให้กดคีย์บนคีย์บอร์ดปกติได้ก็น่าจะดี จะได้ใช้คีย์บอร์ดเลยถ้ากดถูกตัว]
- AC-C2.1 popup มีแท็บ (โน้ต/สัญลักษณ์/คอร์ด) แบบ DomiSol แต่เกาะจุด · reuse [contextual toolbox `EditorMode:328` มีแล้ว]

## EPIC D — สร้างโครงเพลงด้วยการลากบล็อก (คนไม่รู้ดนตรีทำได้) · [Drawer]

**US-D1** — *ในฐานะ M (หรือแม้แต่คนไม่รู้ดนตรี) ฉันต่อเพลงจากการ์ด "ข้อ1 · รับ · ข้อ2…" ลากจัดลำดับ โดยไม่ต้องรู้สัญลักษณ์ดนตรี*
- AC-D1.1 การ์ดโครงเพลงเปิดใน **Drawer** (ไม่ปนแผ่นเพลง) · ลากเรียง/▲▼ · เห็นผลบนแผ่นเพลง real-time [reuse: โครงเพลง rail `EditorMode:2044` · `moveRowTo`]
- AC-D1.2 **โปรแกรมสร้าง repeat/กล่อง1-2/D.C. ให้เองจากการเรียงการ์ด** — ผู้ใช้ไม่วาดสัญลักษณ์ [G ฟันธง]
- AC-D1.3 ทุกการ์ดมี badge `[ทำนอง X]` · แก้ทำนอง → ทุกการ์ดที่ใช้ทำนองนั้น **เรืองแสงเตือน** "กระทบข้อ 1,2,3"
[ความเห็นเอม - แต่ถ้าคนรู้ด้นตรีก็ควรสามารถใส่สัญลักษณ์และ Drawer สร้าง-แก้ไขตัวเอง โดยอัตโนมัติ]
[ความเห็นเอม - ถ้ามีการใส่สัญลักษณ์ทางดนตรีผิดเงื่อนไขระบบต้องเตือนอัตโนมัติ และแสดงเหตุที่ผิดให้เห็น ทั้งในภาพรวมต่อห้องหรือ อะไรก็แล้วแต่ตามมาตรฐานการนับสากล ของเดิมมี code ฮยู่ระดับหนึ่งแล้ว ทั้งที่ drawer ด้วย]

**US-D2** — *ในฐานะ M ฉันทำท่อนซ้ำ (ทำซ้ำท่อน) และก๊อป/ย้าย ห้อง/ท่อน ได้ง่าย เพื่อทำร้องรับ + จัดลำดับ + reuse ทำนอง*
- AC-D2.1 **Duplicate ท่อน = เนื้อใหม่บนทำนองเดิม** (share) [G: Default Share · reuse `duplicateBar`/`copyLine`/duplicate ท่อน]
- AC-D2.2 ปุ่ม **"แยกทำนองเป็นอิสระ (Make Unique)"** ตอนอยากให้ท่อนนั้นต่าง — clone stanza แยก [net-new? ตรวจว่ามีหรือยัง]
- AC-D2.3 copy/move/paste ห้อง/ท่อน/ข้อ ได้ [reuse: copy/paste มีแล้ว · verify ครบ scope]

[ความเห็นเอม - ตอนนี้หน้าจอใช้งานในภาพรวมยังยากอยู่ ทั้้งการเลือกเพลงเพื่่อแก้ การจะสร้างเพลงใหม่ เช่น มีปุ่มเพลง เพื่อสร้างหรือ search มันดูใช้ยาก อยากให้ออกแบบใหม่เลยให้ Minimalist และ follow design pattern ที่เป็นมาตรฐานสากลใช้ง่าย อยากให้ช่วยทวนความเข้าใจด้วยการอธิบายว่าปัจจุบันใช้ยากและงงยังไง จินตนาการ use case คนไม่เคยใช้เริ่มจากระดับ S จะใช้โปรแกรม เพื่อเลือกฟังเพลง ฝึกร้อง หรือ M จะใช้โปรแกรม เลือกเพลง แก้ไข บันทึก ส่งตรวจ อยากให้ UI UX ใหม่วิเคราะห์ครอบคลุม user journey ให้ดีกว่าที่เป็นอยู่]

[ความเห็นเอม - อยากให้มีความชัดเจนว่ารองรับการ แสดงผลบนจอ ในโหมดฝึกร้อง แบบต่อเนื่อง คือเขียนเพลงยาวไปเลย ไม่ต้องกลับมาวกท่อนใหม่ การฝึกร้องแบบนั้นดูง่ายกว่าเลื่อนลงไปเรื่อย ๆ แต่การพิมพ์ออกกระดาษอาจเลือกแบบประหยัดกระดาษแบบคริสตจักรทำอยู่ได้ ในรูปแบบที่คุ้นเคย แต่ต้อง preview ก่อนได้]

[ความเห็นเอม - อยากให้ออกแบบให้ดีเผื่ออนาคตรองรับการทำ playlist หรือ tag favorite ของแต่ละคนเพื่อใช้ฟังได้ เคยคุยกับ g ไว้ว่าจะใช้กลไก local storage และ save/ส่ง config ของแต่ละคนทาง email เมื่อกด COnfig file ใน Email แล้วเปิดหน้ามาเหมือนเดิม จะได้ไม่ต้องมีเก็บ User account management]
