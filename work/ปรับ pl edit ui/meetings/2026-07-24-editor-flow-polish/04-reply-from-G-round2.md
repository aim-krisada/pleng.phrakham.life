### G ตอบ (หลังส่วนที่ 1/1) — OK

ยืนยันเรื่อง Tab: คุณอ่านถูก 100% ครับ MuseScore 4 คืน Tab ให้ Accessibility จริง การไม่ผูก Tab กับการเดินคอร์ดในเว็บคือแนวทางที่ถูกต้องที่สุดตามหลัก WCAG (Focus Management) เขียนสเปก Space = ยืนยัน+ไปโน้ตถัดไป ได้เลยครับ

1. ความละเอียดของ "คอร์ดถัดไป" (ต่อโน้ต หรือต่อ beat?)

แนะนำ: ให้ Space = เดินทีละ cell (โน้ตถัดไป) ครับ

เหตุผล: สำหรับคนพิมพ์รัว (Power User) "ความคาดเดาได้" (Predictability) สำคัญที่สุด ถ้าระยะกระโดดไม่เท่ากัน ผู้ใช้จะกะจังหวะนิ้วพลาดและหงุดหงิด การกด Space รัวๆ 2-3 ครั้งเร็วกว่าการต้องมานั่งคิดว่าระบบจะพาเด้งไปตกที่ไหน

ทางแก้เสริม: ควรเพิ่ม Ctrl + Space หรือ Ctrl + → (ถ้าไม่ซ้ำกับ shortcut เดิม) เพื่อให้กระโดดข้ามไปที่ "หัวห้องถัดไป" (Next Measure) สำหรับจังหวะที่คอร์ดลากยาว

2. โน้ตที่ลากเสียง/หยุด (held / rest) - Space ควรหยุดไหม?

Rest (0): ต้องหยุดครับ ดนตรีร่วมสมัยมีจังหวะยก (Syncopation) หรือ "คอร์ดมาก่อนเนื้อร้อง" บ่อยมาก การใส่คอร์ดบนจังหวะหยุดเป็นเรื่องปกติ

Held (-): ควรข้าม ในทางปฏิบัติแทบไม่มีการเปลี่ยนคอร์ดกลางการลากเสียง (Tie) เว้นแต่คุณจะทำระบบแบบกวาดทุก cell เพื่อความง่ายในการเขียนโค้ด (ซึ่งถ้ายึดความง่ายในการใช้ ให้ข้าม - ไปเลยครับ)

3. ออกจากโหมดใส่คอร์ดยังไง (Esc)

Esc = ยกเลิกข้อความที่ พิมพ์ค้างอยู่ (Discard changes) แล้วปิดป๊อปอัป

Pitfall ที่มักพลาด: เมื่อกด Esc แล้วโฟกัส (Focus) มักจะหลุดไปที่ <body> หรือหายไปเลย คุณต้องเขียนดักให้ ดึง Focus กลับมาที่ cell โน้ตเดิมทันที เพื่อให้ผู้ใช้เดินลูกศรต่อได้โดยไม่ต้องใช้เมาส์คลิกใหม่

4. ลบคอร์ด (แยกยังไงให้ไม่ลบโดยไม่ตั้งใจ)

ใช้ Dirty Checking (ตรวจสอบการเปลี่ยนแปลงสถานะของ input) ครับ:

เปิดป๊อปอัปมา -> ไม่ได้แตะ/พิมพ์อะไร (Pristine) -> กด Enter = ไม่เปลี่ยน/ปิดเฉยๆ

เปิดป๊อปอัปมา -> คลุมดำลบข้อความทิ้ง (Dirty: ค่าเปลี่ยนเป็น string ว่าง) -> กด Enter = ลบคอร์ด

ข้อเสนอแนะสำหรับ Pro Desktop: ควรอนุญาตให้กด Delete หรือ Backspace ตั้งแต่ตอนที่เคอร์เซอร์อยู่บนโน้ต (โดยยังไม่เปิดป๊อปอัปคอร์ด) เพื่อลบคอร์ดของโน้ตนั้นทิ้งได้ทันที นี่คือวิธีที่เร็วและ Error-proof ที่สุดครับ

ตอนนี้เราเคลียร์เรื่องการควบคุมในโหมดป๊อปอัปแล้ว แต่เวลาคอร์ดแสดงผลจริงบนแผ่นเพลง jianpu ที่โน้ตติดกันมากๆ คุณมีวิธีจัดการ Layout Shift หรือการซ้อนทับกัน (Overlap) ของชื่อคอร์ดที่ยาวๆ (เช่น C#maj7/F) กับโน้ตข้างเคียงหรือยังครับ?
