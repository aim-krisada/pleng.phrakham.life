# รอบ 2 — ข้อความที่ส่งให้ G (ทั้งดุ้น)

ขอบคุณครับ 3 ข้อที่ผมรับทันที (ตรวจของจริงแล้วตรง): iOS ต้อง input ≥16px กัน auto-zoom ·
smooth scroll = jank ตอนพิมพ์รัว ใช้ `behavior:'auto'` · scroll ต้องรอ `visualViewport resize` ให้คีย์บอร์ดดันจอนิ่งก่อน

⚠️ **แต่ข้อคอร์ดผมเปิด handbook จริงแล้ว เจอว่าคุณพลาด 1 จุด — ยืนยันหน่อย:**
คุณบอก *"MuseScore 4: Space ข้ามโน้ต, **Tab ข้ามห้อง**"* · ผมเปิด MuseScore Studio Handbook หน้า Chord symbols แล้ว
**MuseScore 4 เอา Tab ออกจากการข้ามห้องแล้ว** — เปลี่ยนเป็น `Ctrl+→` **โดยเจตนาเพื่อคืน Tab ให้ทำงาน accessibility focus**
(Space = "move to next beat, note, or rest whichever comes first" ยังถูก) · **แปลว่าในเว็บผมยิ่งห้ามผูก Tab กับ "คอร์ดถัดไป"**
เพราะ Tab ต้องเป็น focus-traversal — ตรงกับที่ตัวแก้ผม Tab=ข้ามโน้ต + ห้าม trap focus อยู่แล้ว
**ยืนยันไหมว่าผมอ่านถูก? ถ้าใช่ ผมจะเขียนว่า Space=ยืนยัน+ไปโน้ตถัดไป · ⛔ ไม่ผูก Tab เลย**

## 4 คำถามที่ต้องล็อกก่อนเขียนสเปก

**1. ความละเอียดของ "คอร์ดถัดไป" — ต่อโน้ต หรือต่อ beat?**
เพลงเราเป็น jianpu เนื้อใต้โน้ต · 1 พยางค์ = 1 โน้ต · **ไม่ใช่ทุกโน้ตมีคอร์ด** (คอร์ดเปลี่ยนทุก ~2-4 โน้ต)
ถ้า Space เด้งไป**โน้ตถัดไปทุกตัว** ผู้ใช้ต้องกด Space รัวผ่านโน้ตที่ไม่มีคอร์ด · ถ้าเด้งไป**หัวห้องถัดไป**อาจข้ามจุดที่ต้องใส่คอร์ดกลางห้าง
**คุณแนะยังไง — Space=โน้ตถัดไป (ตรง MuseScore) แล้วผู้ใช้ปล่อยว่างข้ามเองก็พอ · หรือมีแบบที่ดีกว่าสำหรับ jianpu?**
มีมาตรฐาน/ตัวอย่างจริงของ "lead sheet chord entry ที่โน้ตไม่ได้มีคอร์ดทุกตัว" ไหม (ขอแหล่ง)

**2. โน้ตที่ลากเสียง/หยุด (held / rest) — Space ควรหยุดที่มันไหม?**
MuseScore บอก "beat, note, or rest whichever comes first" = หยุดที่ rest ด้วย · แต่ของเรา cell ลากเสียง (`-`) กับ rest (`0`)
ปกติไม่มีใครใส่คอร์ดบนนั้น · **ควรให้ Space ข้ามมันไปโน้ตจริงตัวถัดไป หรือหยุดทุก cell ตาม MuseScore?**

**3. ออกจากโหมดใส่คอร์ดยังไง — Esc ทำอะไร**
ผมวางว่า: Enter/Space = ยืนยัน · **Esc = ปิดป๊อปอัปโดยไม่เปลี่ยนคอร์ดเดิม** (คอร์ดที่มีอยู่ไม่ถูกลบ) ·
กลับไปโหมดเดินโน้ตปกติ · เคอร์เซอร์อยู่โน้ตเดิม · **มี pitfall อะไรกับ Esc ที่ผมมองข้ามไหม** (เช่น Esc ควร undo คอร์ดที่เพิ่งพิมพ์ค้างไหม)

**4. ลบคอร์ด — ที่คุณเสนอถังขยะ ≥24px บนมือถือ**
Desktop: input ว่าง + Enter = ลบ · Mobile: ปุ่มถังขยะในป๊อปอัป · **แต่ desktop input ว่าง + Enter จะกำกวมกับ "ไม่เปลี่ยน" ไหม**
(ผู้ใช้เปิดป๊อปอัปเฉย ๆ แล้ว Enter โดยไม่พิมพ์ = ตั้งใจลบ หรือตั้งใจปิด?) · **แยกยังไงให้ไม่ลบคอร์ดโดยไม่ตั้งใจ**

ตอบสั้น ๆ เป็นข้อ ๆ ได้เลยครับ ไม่ต้องเกริ่น · **ยืนยันเรื่อง Tab ก่อนเป็นข้อแรก**
