# Task B2 (octave keyboard) — สรุป + จุดที่ตัดสินเอง

**G รอบ 2 ไม่ช่วย** — ถามเรื่อง dead-key pitfall 2 ครั้ง G เข้าใจผิดทั้งคู่ (รอบแรกไล่ระบบโน้ต SPN/ABC/MML ทั่วไป · รอบสองแค่รับทราบแล้วถามว่าจะให้ทำอะไรต่อ) ไม่แตะคำถามจริง → **ตัดสินเองบนฐานที่แข็งกว่า G:**
- **ไวยากรณ์ = SSOT ของ parser** (`src/lib/notation.js:4-8`): `.5`=ต่ำ (จุดนำหน้า) · `5'`=สูง (apostrophe ตามหลัง) · `5.`=ประจุด (จุดตามหลัง) — parser แยกด้วยตำแหน่งอยู่แล้ว
- **มาตรฐาน jianpu จริง**: จุดใต้เลข=ต่ำ · จุดบนเลข=สูง (สัญลักษณ์กำกับ octave ของโน้ตตัวเลข)
- **`'`-ตามหลัง = ยกอ็อกเทฟ ยืนยันข้ามระบบ**: G รอบ 2 (ระบบ ABC/LilyPond) เอง — `c'` = ขึ้น 1 octave · ตรงกับที่เราใช้ `'`

## ของจริงวันนี้ (อ่านซอร์ส)
- `'` → `octaveSel(1)` (ยกอ็อกเทฟ) ผูกคีย์แล้ว `SongViewer.vue:448` ✅ ตรง `5'`
- `.` → `markSel('.')` = **ประจุดทันที** `SongViewer.vue:444` (อยู่ในชุด `'_.~^'`)
- **เสียงต่ำไม่มีคีย์** — มีแต่ปุ่มบนจอ `ต่ำ↓` (`NoteInputBar.vue:137` emit `octave,-1`) · คอมเมนต์ยอมรับเอง `NoteInputBar.vue:133` "octave has no keyboard key → button"
- แปลว่า **`.5` ต่ำ = ทำด้วยคีย์บอร์ดไม่ได้วันนี้** = ช่องว่างจริงที่ B2 ต้องอุด · **ของเดิม "แบบที่เคยเป็น" = textarea ดิบใน `EditorMode.vue`** (พิมพ์สตริง `.5 5'` ตรง ๆ ให้ parser แปลง) — inline editor ใหม่ยังไม่มี

## การตัดสิน (เขียนลงสเปก)
- **`'` (สูง) = คงเดิม** ผูก `octaveSel(1)` บนโน้ตที่คลุม/เพิ่งพิมพ์ — ไม่แตะ
- **`.` = dead-key prefix สำหรับเสียงต่ำ (unified 2 โหมด)**: กด `.` → pending-low · เลขถัดไป (0-7) = ต่ำ (`.5`) · ถ้าคีย์ถัดไปไม่ใช่เลข = ประจุดบนโน้ตปัจจุบัน (`5.`) → ตรง parser + muscle memory พี่เปา
- **จุดพัง (ที่ G ไม่ตอบ ผมกันเอง)**: คลุม `3` ตั้งใจประจุด กด `.` แล้วเผลอพิมพ์ `5` = ได้ `.5` แทน `3. + 5` → กันด้วย **visual pending-low indicator** (จุดจาง ๆ นำหน้า caret ตอน `.` ค้าง) ผู้ใช้เห็นสถานะก่อนพิมพ์เลข · one-shot ไม่มี timeout (ยกเลิกด้วย Esc หรือกดคีย์ที่ไม่ใช่เลข)
- **มือถือ**: ปุ่ม สูง↑/ต่ำ↓ คงอยู่ (คีย์บอร์ดล้วน = เดสก์ท็อป ไม่แทนปุ่ม) — 2 ทางอยู่ร่วมกันตามที่ PM บังคับ
- **collision check** (`SongViewer.vue:382-423`): `'` ผูกแล้ว (ไม่ชนใหม่) · `.` ต้องเปลี่ยนจาก markSel-ทันที เป็น dead-key แต่ยังเป็นคีย์เดิม ไม่ชนคีย์อื่น (ตรวจแล้ว `.` โผล่แค่ SYMBOL_CHARS learn + markSel branch) · **Space=ยืนยันคอร์ด อยู่ใน popup คอร์ด คนละ context ไม่ชน**
