**รอบ 1b — เวอร์ชันกระชับ (ส่งใหม่ เพราะรอบแรกค้าง)**

ตอบไทย · เป็นความเห็นให้เขียนกำกับ · อ้างมาตรฐานขอ URL + ชื่อ element ที่เปิดตรวจได้

ผมทำ inline editor โน้ตตัวเลข (jianpu 0-7) เว็บโน้ตเพลงคริสตจักร · เคอร์เซอร์วันนี้ = "เลือกโน้ต 1 ตัว" (block) · พิมพ์เลข = ทับโน้ตนั้น (default · ทีมล็อกเพราะงานบ่อยสุด=แก้พิตช์) · Insert=โหมดแทรก · # b n _ . ~ ^ ' คอร์ด = คำสั่งกับโน้ตที่เลือก

ผู้ใช้มือโปร (คีย์เพลงทุกวัน) เจอ 2 ปัญหา:
1. แทรกโน้ตหน้าตัวแรกในห้องไม่ได้ (block ไม่มีตำแหน่ง "หน้าโน้ตตัวแรก")
2. Del/Backspace รู้สึกกลับด้าน

จุดยืนผม (ขอให้ค้านเฉพาะที่ผิดจริง):
- เปลี่ยนเป็น **caret ระหว่างโน้ต** (N โน้ต = N+1 ช่อง · g0=หน้าตัวแรก) วาดแท่งกะพริบ → แก้ปัญหา 1
- ยังมี "โน้ต active" = ตัวซ้ายของ caret (ที่ g0 = ตัวขวา) ไฮไลต์ไว้ ให้คำสั่ง # b คอร์ด ลงถูกตัว = **selection + caret อยู่ด้วยกัน**
- พิมพ์เลข = **ทับ active (default เดิม)** · Insert = แทรกที่ช่อง caret
- **Backspace=ลบตัวซ้าย caret / Delete=ลบตัวขวา caret** (Wikipedia: Backspace ลบตัวก่อน cursor, Delete ลบตัวที่/หลัง cursor)

3 คำถาม:
Q1. โมเดลไฮบริด "caret + active=ตัวซ้าย + พิมพ์ทับ/Insert แทรก" coherent ไหม? MuseScore/Finale/Dorico ตอน note-input จัดการ "แทรกหน้าโน้ตตัวแรก" + ทิศลบ ยังไง — **ขอ URL handbook จริง อย่าเดาเวอร์ชัน**
Q2. **ฟันธง:** ควรคง overwrite-เป็น-default (แก้พิตช์เร็ว) หรือพลิกเป็น insert-default ให้เข้ากับ text-caret? มี editor จริงที่ทำ selection+caret คู่กันให้อ้างอิงไหม?
Q3. มือถือ: แตะวาง caret "ในช่องบาง ๆ ระหว่างโน้ต" ให้โดนยังไง? + จุดตายตอนใช้ทั้งวันที่ผมมองไม่เห็น (โดยเฉพาะที่ขอบห้อง/ปลายบรรทัด)
