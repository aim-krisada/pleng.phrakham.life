ผมเป็นสถาปนิกระบบ (SA) อยากปรึกษาการออกแบบ · ตอบไทย · ถ้าเป็นความเห็นให้เขียนกำกับว่าเป็นความเห็น · ถ้าอ้างมาตรฐาน/พฤติกรรมโปรแกรมอื่น ขอ URL + ชื่อ element/ปุ่มที่ผมเปิดตรวจเองได้ (ผมจะตรวจทุกข้อก่อนเขียนลงสเปก)

งาน: ผมทำ inline editor โน้ตตัวเลข (jianpu 0-7) บนเว็บโน้ตเพลงคริสตจักรไทย · เคอร์เซอร์วันนี้ = "เลือกโน้ต 1 ตัว" (block cursor) โน้ตถูกไฮไลต์ · พิมพ์เลข = ทับโน้ตนั้น (default · ทีมล็อกไว้เพราะงานที่ทำบ่อยสุด = แก้พิตช์โน้ตผิด) · มีปุ่ม Insert สลับเป็นโหมดแทรก · ปุ่ม # b n _ . ~ ^ ' และคอร์ด = คำสั่งที่ทำกับโน้ตที่เลือก

ผู้ใช้จริง (มือโปรคีย์เพลงทุกวัน) เจอ 2 ปัญหา:
1. แทรกโน้ต "หน้าตัวแรก" ในห้องไม่ได้ (โมเดล block ไม่มีตำแหน่ง "หน้าโน้ตตัวแรก" — ตำแหน่งซ้ายสุดที่เลือกได้คือตัวแรกเอง)
2. Del กับ Backspace รู้สึกกลับด้าน

จุดยืนผม (ขอให้ค้านเฉพาะจุดที่ผิดจริง ไม่ต้อง rubber-stamp):
- เปลี่ยนเป็น caret ระหว่างโน้ต (between-notes caret) — โน้ต N ตัว มี N+1 ช่องหยุด (g0 = หน้าตัวแรก) วาดเป็นแท่งกะพริบในช่องว่าง → แก้ปัญหา 1
- ยังต้องมี "โน้ต active" = ตัวทางซ้ายของ caret (ถ้า caret อยู่ g0 = ตัวทางขวา) ไฮไลต์ไว้ ให้คำสั่ง # b คอร์ด ลงถูกตัว = selection กับ caret อยู่ด้วยกัน
- พิมพ์เลข = ทับ active (คง default เดิม) · โหมด Insert = แทรกที่ช่อง caret (ที่ g0 = แทรกหน้าตัวแรก)
- Backspace = ลบโน้ตทางซ้ายของ caret + caret ถอยซ้าย · Delete = ลบโน้ตทางขวาของ caret + caret อยู่กับที่ (อ้าง Wikipedia Backspace: ลบตัวก่อน cursor · Delete: ลบตัวที่/หลัง cursor)

3 คำถาม:
Q1. โมเดลไฮบริดนี้ (caret ระหว่างโน้ต + active = ตัวซ้าย + พิมพ์ทับ active / Insert แทรกที่ช่อง) coherent ไหม หรือมีโมเดลที่สะอาดกว่า? โปรแกรมโน้ตจริง (MuseScore / Finale / Dorico) ตอน note-input จัดการ "แทรกหน้าโน้ตตัวแรก" และ "ทิศทางการลบ" ยังไง — ขอ URL handbook จริง อย่าเดาเวอร์ชัน

Q2. ฟันธงหน่อย: ควรคง overwrite เป็น default (แก้พิตช์เร็ว) หรือควรพลิกเป็น insert-default ให้เข้ากับ text-caret ที่คนคุ้น? มี editor จริงที่ทำ selection + caret คู่กัน (ทับ selection แต่แท่ง caret บอกตำแหน่งแทรก) ให้อ้างอิงไหม?

Q3. มือถือ: จะให้ "แตะวาง caret ในช่องบาง ๆ ระหว่างโน้ต" โดนยังไง? + จุดตายตอนใช้ทั้งวัน (ไม่ใช่ตอนลองครั้งแรก) ที่ผมมองไม่เห็น โดยเฉพาะพฤติกรรมที่ขอบห้อง (bar line) และปลายบรรทัด — caret ที่ท้ายห้อง/ท้ายบรรทัด กด Insert หรือ Delete ควรทำอะไร?
