**รอบ 2 — ต่อจากรอบ caret · เรื่องพิมพ์ระดับเสียง (octave) ด้วยคีย์บอร์ดล้วน**
(แชตนี้เป็นแชตใหม่ ผมสรุปบริบทให้สั้น ๆ)

## บริบทที่ตกลงกันแล้วรอบก่อน
เว็บโน้ตเพลง jianpu (เลข 0-7) inline editor · เราสรุปว่าใช้ **cursor เปลี่ยนรูปตามโหมด**:
- **โหมดทับ (default) = Block cursor** คลุมโน้ต 1 ตัว · พิมพ์เลข = ทับพิตช์โน้ตนั้น
- **โหมดแทรก = Line Caret** เส้นกะพริบระหว่างโน้ต · พิมพ์เลข = แทรกโน้ตใหม่ตรงนั้น
- Modifier (ชาร์ป/มาร์ค) กระทำกับโน้ตที่คลุม (block) หรือโน้ตที่เพิ่งพิมพ์ (caret)

## ไวยากรณ์โน้ตของระบบ (parser มีอยู่แล้ว — SSOT)
- **`.5` = เสียงต่ำ** (จุด **นำหน้า** เลข = จุดใต้โน้ต ลงหนึ่งช่วงต่อ 1 จุด)
- **`5'` = เสียงสูง** (apostrophe **ตามหลัง** เลข = จุดบนโน้ต)
- **`5.` = โน้ตประจุด** (augmentation dot ×1.5 · จุด **ตามหลัง** เลข)
คือ parser แยกความหมายของ `.` **ด้วยตำแหน่ง**: จุดก่อนเลข = เสียงต่ำ · จุดหลังเลข = ประจุด

## ปัญหา
ผู้ใช้จริง (พี่เปา · คีย์เพลงมืออาชีพทุกวัน) อยากทำเพลง **ด้วยคีย์บอร์ดล้วน ไม่ละมือไปจิ้มปุ่ม** ตาม muscle memory เดิม: `.5` = ต่ำ · `5'` = สูง
- **`5'` (สูง) ทำได้แล้ว**: กด `'` หลังพิมพ์เลข = ยกโน้ตขึ้นหนึ่งช่วง ✅
- **`.5` (ต่ำ) ทำไม่ได้ด้วยคีย์บอร์ด**: ปุ่ม `.` ตอนนี้ถูกผูกเป็น "ประจุด (aug dot)" ทันทีที่กด · เสียงต่ำมีแต่ปุ่มบนจอ (ต่ำ↓) ไม่มีคีย์

## จุดยืนผม (อยากให้ค้าน)
ทำ `.` เป็น **dead-key prefix (ปุ่มตาย/ปุ่มนำ)** แบบเดียวกับ accent dead key บนคีย์บอร์ดสากล:
- กด `.` แล้ว **ตัวถัดไปเป็นเลข 0-7** → เลขนั้นได้เสียงต่ำ (`.5`) — ใช้ได้ทั้งโหมดทับ (ทับโน้ตที่คลุมเป็น `.5`) และโหมดแทรก (แทรก `.5`)
- กด `.` แล้ว **ตัวถัดไปไม่ใช่เลข** (หรือกด `.` ตอนคลุมโน้ตที่มีพิตช์อยู่แล้ว) → เป็น **ประจุด (aug dot)** บนโน้ตนั้น
- = ตรงกับกฎ parser เป๊ะ (จุดก่อนเลข=ต่ำ · จุดหลังเลข=ประจุด) และตรง muscle memory พี่เปา
- **มือถือยังมีปุ่ม สูง↑/ต่ำ↓ อยู่** (คีย์บอร์ดล้วน = ทางเดสก์ท็อป ไม่ได้แทนปุ่ม)

## คำถาม (ขอเจาะ)
1. **dead-key prefix สำหรับ `.`** เป็นแพตเทิร์นที่ถูกต้องไหม · มี **จุดพัง**อะไร (เช่น ผู้ใช้กด `.` ตั้งใจจะประจุดโน้ตที่คลุม แล้วบังเอิญพิมพ์เลขต่อ = ได้ต่ำแทนประจุด) · จะกันยังไง หรือมี convention ที่ดีกว่า dead-key?
2. **ควรโชว์ visual ของ "pending low-octave"** ไหม (เช่น จุดจาง ๆ นำหน้า caret หลังกด `.`) เพื่อไม่ให้ผู้ใช้ลืมว่ากด `.` ค้างไว้ · หรือ dead-key ควรเงียบ?
3. มี **โปรแกรมโน้ต/IME จริง** ที่ใช้ dead-key/prefix แบบนี้กับ octave หรือ modifier ให้อ้างอิงได้ไหม (ขอ URL ถ้ามี · ถ้าเป็นความเห็นบอกว่าเป็นความเห็น)
4. **timing ของ dead-key** — ถ้าผู้ใช้กด `.` แล้วหยุดคิดนาน ๆ แล้วค่อยพิมพ์เลข ควรยังนับเป็น low-octave ไหม หรือควรมี timeout? (ผมเอนไปทาง **ไม่มี timeout** — one-shot จนกว่าจะกดคีย์ถัดไป · เห็นด้วยไหม)
