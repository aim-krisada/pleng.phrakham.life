# US — B083 จับคู่ทำนอง↔เนื้อร้อง ในโหมดแก้ไข (เสริมของเดิม)

จาก: pm7 · brief `docs/pm/brief-melody-pairing-sa.md` · visual = `docs/design/melody-pairing.html`
เคสจริง = เพลง 2 "ของขวัญ" (id `74ebe8b8…`). อ้าง `docs/ui-standards.md` + `docs/song-model-v2.md`.
⚠️ **เสริม/ซ่อมองค์ประกอบที่มีอยู่ (rail โครงเพลง · หัวท่อน · ตัวเลือกทำนอง · แผงแก้เนื้อ) — ไม่ทำจอใหม่ · KISS · โมเดล v2 ไม่เปลี่ยน** (ยกเว้นชื่อทำนอง = optional 1 field, flag PM).

## บริบทปัญหาจริง (พิสูจน์จากฐาน · เพลง 2)
- ทำนอง A รับ 91 พยางค์ (4 บรรทัด) · ทำนอง B รับ 7 (1 บรรทัด)
- ท่อน "ร้อง1" → ถูกจับทำนอง **B** (มีเนื้อ 23 → ล้น · ควรเป็น A) · "รับ" → ทำนอง A แต่เนื้อเป็น **ก้อนเดียว 1 ช่อง** · "ร้อง2" → 21/91
→ import จับสลับ + เนื้อไม่แยกพยางค์ → คนทำงานแก้ไม่ถูก

## MP1 — เปลี่ยนทำนองของท่อนแล้วแก้ต่อได้ (ไม่หาย)
ในฐานะพี่เปา ฉันอยากเปลี่ยนทำนองของท่อนที่ import ผิด แล้วยังพิมพ์/แก้เนื้อต่อได้ทันที
**AC:**
- เปลี่ยนทำนองที่หัวท่อน (ตัวเลือกเดิม `EditorMode.vue:2028`) A→B → **หัวท่อน + ช่องแก้เนื้อไม่หาย** (ปัจจุบันหายเพราะ `lensActive` ผูก `activeStanzaId` `:242`)
- พฤติกรรมใหม่: เปลี่ยนทำนอง = พาไปเปิดทำนองนั้น (activeStanza ตาม) → แก้เนื้อของท่อนบนทำนองใหม่ต่อได้เลย
- เห็นชัดว่า "ท่อนนี้ = ทำนองไหน + เนื้ออะไร" ตลอด

## MP2 — ทำนองแยกออกได้ (พรีวิว/ชื่อ)
ในฐานะคนจับคู่ ฉันอยากรู้ว่าทำนอง A/B อันไหนคืออันไหน โดยไม่ต้องเปิดดูทีละอัน
**AC:**
- ตัวเลือกทำนอง (`stanzaIdOptions :647`) + ชิป ♪ + รายการ "ทำนอง (โน้ต)" โชว์ **พรีวิว**: โน้ต 5–6 ตัวแรก + จำนวนบรรทัด + ความจุพยางค์ (เช่น "ทำนอง A · 5 1-0̲2… · 4 บรรทัด · 91 พยางค์") — **คำนวณจากทำนองที่มี ไม่เพิ่ม field**
- (optional · flag PM) ตั้ง **ชื่อทำนอง** ได้ (เช่น "ท่อนร้อง"/"ท่อนรับ") = ต้องเพิ่ม field `stanzas[].name`

## MP3 — แยกพยางค์เนื้อก้อน import
ในฐานะพี่เปา ฉันอยากเอาเนื้อทั้งท่อนที่ import มาเป็นก้อน มาแยกเป็นพยางค์ใต้โน้ตได้ง่าย
**AC:**
- ในแผง "📝 แก้เนื้อแบบย่อหน้า" (มีอยู่) มีปุ่ม **"✂ แยกพยางค์อัตโนมัติ"** → แตกก้อนเนื้อ (ตัด "," เป็นวรรค + ตัดคำไทย) เป็นช่องพยางค์ วางใต้โน้ตเรียงตาม
- ปรับละเอียดต่อด้วย ◀▶ / เว้นวรรค / Backspace-merge ที่มีอยู่ (ไม่สร้างเครื่องมือใหม่)
- เอื้อน/ตัวลาก (`-` `~` slur) = ไม่รับพยางค์ (ข้ามช่องตาม v2) · ช่องแดง = ยังไม่พอดี (นับด้วย `rowStatus` เดิม)

## MP4 — ภาพรวมการจับคู่ทั้งเพลง
ในฐานะคนตรวจ ฉันอยากเห็นในแวบเดียวว่าท่อนไหนจับคู่ทำนอง↔เนื้อผิด
**AC:**
- rail "โครงเพลง" (มีอยู่) เติม **ป้ายเตือนต่อท่อน**: เนื้อพอดีทำนอง = เขียว ✓ · ไม่พอดี = แดง got/need (เช่น "23/7 ✗") — ใช้ `rowStatus` เดิม
- เพลง 2: เปิดมาเห็น 3 ท่อนแดงทันที → จับคู่ถูกทีละท่อน ป้ายเปลี่ยนเขียว
