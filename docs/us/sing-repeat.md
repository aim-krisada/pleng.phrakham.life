## แนวคิดที่ทำให้ "ย่อหน้าจอ" กับ "ลำดับเล่นจริง" ไม่ขัดกัน

โจทย์ยากคือ: หน้าจอโชว์ท่อนรับ (hook) **ครั้งเดียว** เพื่อประหยัดที่ แต่เวลาเล่นจริงต้องวนกลับมารับหลายรอบ (ร้อง 1 → รับ → ร้อง 2 → รับ …).

**กุญแจ:** song-model-v2 แยกของสองอย่างไว้ให้แล้ว —
- **ทำนอง (stanza)** = ท่อนที่โชว์ **ครั้งเดียว** บนจอ (นี่คือ "hook ครั้งเดียว")
- **ลำดับเพลง (arrangement)** = ลำดับ **เล่นจริง** ที่รับโผล่ซ้ำหลายรอบ

เพราะฉะนั้น "tag ท่อน" = ท่อนที่โชว์บนจอ (ครั้งเดียว) · "เล่นวนกลับรับ" = เดินตาม arrangement แล้วไฮไลต์ย้อนขึ้นไปที่บล็อกรับเดิม. **ไม่ต้อง hack UI — map ตรงกับโมเดลอยู่แล้ว.**

## S0 — ยกเครื่อง control ฝึกร้องเป็น Music-player ใน bottom dock (PM #4 · directive P'Aim)
บริบท: ตอนนี้หน้าฝึกร้องมี **การ์ดควบคุมก้อนใหญ่ด้านบน** (ฟังเพลง/ความเร็ว/วนซ้ำ/เต็ม-เนื้อล้วน/ABC-IVV/พิมพ์) กินที่ · P'Aim สั่ง "ออกแบบใหม่หมด เป็น Music control ด้านล่าง ใช้ sticky mobile dock" (ภาพ `docs/pm/realuse-assets/sing-redesign-annotated.png` + ref music player)

ในฐานะคนฝึกร้องบนมือถือ ฉันอยากให้ปุ่มควบคุมทั้งหมดอยู่ในแถบล่าง sticky แบบเครื่องเล่นเพลง เพื่อให้พื้นที่จอเหลือไว้อ่านเนื้อ/โน้ต และคุมง่ายด้วยนิ้วโป้ง

**AC:**
- **ย้าย control ฝึกร้องทั้งหมดจากการ์ดบนหัว → แถบ dock ล่าง sticky** (ตัวเดียวกับ dock-core library) · ตัดการ์ดบนออก
- แถบล่าง = **music player:** (1) **แถบ progress** (timeline) มี **marker ท่อน** + เวลา · (2) **transport** ⏮ ▶/⏸ ⏭ + 🔁 วนซ้ำ · (3) control อื่นอยู่ใน **⚙ แผงตั้งค่า**
- **⚙ แผงตั้งค่า = บ้านของทุก control** (แสดงผล/คอร์ด/คีย์/ความเร็ว/ฟอนต์/ความโปร่ง/ดาวน์โหลด/พิมพ์) — **ปรับได้ตรงในแผงเลย แม้ไม่ปักขึ้นแถบ** (PM #6c · เช่น slider ความโปร่ง ไม่ต้องปัก) · แต่ละอันมี 📌 ปัก/ถอนขึ้นแถบ
- **ย้ายปุ่มดาวน์โหลดเข้า dock/แผง** ด้วย (ออกจากแถบหัว)
- แยกออกจาก B043: ระยะโน้ต↔เนื้อ/ห้อง = **B044** · hamburger แถบหัวมือถือ = **B045**

## S1 — เห็นแต่ละท่อนเป็น tag (โชว์ท่อนซ้ำครั้งเดียว)
ในฐานะคนฝึกร้อง ฉันอยากเห็นเพลงเป็น "ท่อน" ที่มีป้ายชื่อชัด (ร้อง 1 / รับ / ร้อง 2 …) โดยท่อนรับที่ซ้ำโชว์แค่ครั้งเดียว เพื่ออ่านง่ายและประหยัดที่

**AC:**
- โหมดฝึกร้องแสดงเพลงเป็น **บล็อกท่อน** แต่ละบล็อกมีป้ายชื่อ (label จาก arrangement: "ร้อง 1", "รับ", …)
- ท่อนที่เล่นซ้ำแบบ **คำเหมือนกันเป๊ะ** (เช่นรับหลังทุกข้อ) โชว์ **บล็อกเดียว** — ไม่ทวนซ้ำเต็มหน้า
- ท่อนที่ทำนองเดียวกันแต่ **คำต่างกัน** (ร้อง 1 vs ร้อง 2 ใช้ทำนอง A เหมือนกันแต่เนื้อคนละอย่าง) = **คนละบล็อก** (คนละ tag)
- เพลง v1 / เพลงท่อนเดียว / ไม่มีท่อนรับ = โชว์เป็นบล็อกเดียว (หรือรายท่อนถ้ามีป้ายท่อน) — ไม่พังของเดิม

## S1b — เพลงไม่มีโครงท่อน ก็ยังใช้ได้ (fallback)
บริบท: เพลงเก่าแบบ v1 (เช่นเพลง 1 "พระเจ้าเป็นความรัก") เก็บเป็นบรรทัดแบน ไม่มีโครงท่อน → ไม่มี tag ให้เลือก

ในฐานะคนฝึกร้องเพลงเก่าที่ยังไม่มีท่อน ฉันอยากให้เล่น/วนทั้งเพลงได้ตามปกติ ไม่ติดขัด

**AC:**
- เพลงไม่มีโครงท่อน = **ไม่มี marker/ปุ่มเลือกท่อน** · เหลือ `▶` + ปุ่มวนซ้ำ = เล่น/วนทั้งเพลง (พฤติกรรมเดิม)
- **F = เงียบ (เคาะแล้ว):** ไม่โชว์คำใบ้ "เพิ่มโครงท่อน" — เล่นทั้งเพลงเงียบ ๆ ตามปกติ

## S2 — เลือกท่อนแบบ Gmail (เลือกเสมอ · All/None · music control ทำกับที่เลือก) — PM #6
บริบท: P'Aim อยากได้พฤติกรรมแบบ Gmail (ภาพ `docs/pm/realuse-assets/gmail-select-all-none.png`) — ติ๊กเลือกได้ตลอด ไม่ต้องเข้าโหมดพิเศษ · มีปุ่มแม่ **All / None**

ในฐานะคนฝึกร้อง ฉันอยากแตะเลือกท่อนไหนก็ได้ตลอดเวลา แล้วให้ปุ่มเล่น/วน/ก่อน-ถัดไป ทำงานกับ "ท่อนที่เลือก" เพื่อซ้อมเฉพาะที่ยังไม่คล่อง

**AC:**
- **เลือกท่อนจากรายการเลื่อนได้** (เปิดจากปุ่ม "☰ เลือกท่อน" · มือถือ = แผ่นเลื่อนขึ้นมา) — แต่ละแถวมี checkbox + ชื่อท่อน · **ท่อนเยอะแค่ไหนก็เลื่อนได้ ไม่เบียดจอ** (PM #6d)
- **ปุ่ม All / None** อยู่หัวรายการ = เลือกทุกท่อน / ล้างทั้งหมด
- **เลือกจากป้ายท่อนบนแผ่นเพลงก็ได้** (ทางลัดตอนท่อนน้อย · ☐→☑ · เขียวเมื่อเลือก) — sync กับรายการ
- **แถบ progress = ตำแหน่ง/ลากหา (scrub)** · marker = จุดเล็กบอกท่อน (ไม่แบกภาระเลือก)
- **music control ทำงานกับ "ที่เลือก":** ▶/⏸ · 🔁 วนซ้ำ · ⏮/⏭ เดินตาม selection · **ไม่เลือกอะไร = ทั้งเพลง** (default)
- ตอบ "ซ้อมท่อนเดียว/รับ" ในตัว: เลือกท่อนนั้น → ▶ (เปิด 🔁 ด้วยก็วน) = ซ้อมเฉพาะท่อนนั้น (ไม่ต้องมี shortcut แยก)
- หลายท่อนที่เลือก เล่น **เรียงตามลำดับในเพลง** (เลือก {ร้อง 2, รับ} → ร้อง 2 → รับ → …; เปิด 🔁 = วน)
- ท่อนที่เลือก = **ค่าชั่วคราวของการซ้อม** (เหมือน คีย์/ความเร็ว/ฟอนต์) — **ไม่แก้เพลงต้นฉบับ** ไม่บันทึก DB
- **ความเร็ว/คอร์ด/แสดงผล = ยัง global** (ไม่ผูก selection) · อยู่ใน music dock เหมือนกัน

## S3 — ระบบเดียว: marker ท่อนบนแถบ progress = tag + เลือกวน + กระโดด (ไม่มี UI ขนาน)
บริบท: หน้าฝึกร้อง **มีแถบชิปท่อนอยู่แล้ว** (`sections` จาก `{type:'section'}` marker). ตาม S0 ชิปพวกนี้ **ยุบเข้าเป็น marker บนแถบ progress ของ music dock** (PM #1) — แถบเดียวทำทั้งบอกตำแหน่ง กระโดด และเลือกวน

ในฐานะคนฝึกร้อง ฉันอยากใช้ marker ท่อนบนแถบ progress ทั้งดูว่าอยู่ท่อนไหน กระโดดเล่น และเลือกวน ในที่เดียว

**AC:**
- **ท่อนซ้ำยุบเป็น tag เดียวสำหรับเลือก** — เลือก "รับ" = เลือกทุกจุดที่รับโผล่ (บน timeline marker ยังโผล่ตามจริงหลายจุด)
- **🔁 = วนซ้ำ เปิด/ปิด** (ไม่ใช่โหมดเลือก) — เปิด = วนสิ่งที่เล่นอยู่ (ที่เลือก หรือทั้งเพลง) · ปิด = เล่นจบแล้วหยุด
- **⏮/⏭** = ท่อนก่อน/ถัดไป (เดินใน selection ถ้ามีเลือก · ไม่งั้นทุกท่อน — decision H)
- marker ท่อนที่กำลังเล่น = เด่น + dot ตำแหน่งวิ่งตาม (marker เป็น progress ในตัว)

## S4 — เล่น/ไฮไลต์/เลื่อนจอ ตามลำดับจริงแม้จอย่อ
ในฐานะคนร้องตาม ฉันอยากให้ไฮไลต์และการเลื่อนจอไล่ตามที่ร้องจริง แม้ท่อนรับจะโชว์บล็อกเดียว

**AC:**
- เล่นถึงรอบที่วนกลับรับ → ไฮไลต์ **ย้อนขึ้นไปที่บล็อกรับเดิม** + จอ **auto-scroll กลับไปที่ท่อนนั้น** (เล็งที่ **พยางค์** ที่กำลังร้อง — เชื่อม B038)
- ไฮไลต์เดินราย **พยางค์** ตามลำดับเล่นจริง (เชื่อม B006/ps3-highlight) — บล็อกที่เล่นซ้ำ ไฮไลต์ซ้ำได้ถูกพยางค์
- มี **ป้ายบอกรอบ "รอบ N"** บนบล็อกที่กำลังเล่น (เคาะแล้ว = เอา) ให้รู้ว่าวนกลับมารอบไหน
- **แถบ progress ลากหาตำแหน่งได้ (scrub) + แตะ marker กระโดด** (เคาะ H = ทั้งสอง)
- **เล่นต่อ/เริ่มใหม่ (B042):** "เล่นต่อ" = ไปต่อจากตำแหน่งล่าสุดใน **ลำดับเล่นจริง** (ท่อน+พยางค์+รอบ) · "เริ่มใหม่" = กลับตำแหน่ง 0 ของลำดับที่กำลังใช้ (ทั้งเพลง หรือชุดท่อนที่เลือก)

## ต้องเคาะ (P'Aim ตัดสิน — มีข้อเสนอแนะแล้ว)
1. **ลำดับตอนวนหลายท่อนไม่ติดกัน** — แนะนำ: เล่นท่อนที่เลือก **เรียงตามลำดับในเพลง** แล้ววน (ยุบท่อนซ้ำที่ติดกันตรงรอยต่อ). เห็นด้วยไหม หรืออยากให้เรียงตามลำดับที่ "แตะเลือก"?
2. **ป้ายบอกรอบ ("รอบ 2")** — เอาไหม หรือแค่ไฮไลต์ย้อนขึ้นไปพอ (ไม่ต้องมีป้าย)?
3. **จำค่าท่อนที่เลือกข้ามเพลง/รีเฟรชไหม** — แนะนำ: ไม่จำ (เริ่มสดทุกเพลง เพราะท่อนคนละเพลงไม่เหมือนกัน)
