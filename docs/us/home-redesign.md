## โยงกลับ mission
รับใช้ mission ข้อ "หาเพลงเจอเร็ว เหมือนเปิดหนังสือเพลงที่คุ้นเคย" — ผู้ใช้ส่วนใหญ่จำเพลงจาก **เล่ม + เลข**
ในหนังสือกระดาษที่ถืออยู่ (เช่น "ไทยอนุชน ข้อ 5") การ landing บนลิสต์ยาวทำให้หาช้า.

## User Story
*ในฐานะ* คนของคริสตจักรที่เปิดเว็บเพื่อหาเพลงร้อง
*ฉันต้องการ* เข้าหน้าแรกแล้วเจอแค่ช่องค้นหา กับปุ่มเลือก "เล่ม" (หนังสือเพลง) 9 เล่ม
*เพื่อ* เลือกเล่มที่ถืออยู่แล้วเห็นเพลงเรียงตามเลข หรือพิมพ์ค้นเจอเพลงข้ามทุกเล่มได้ทันที — ไม่ต้องเลื่อนผ่านลิสต์เพลงยาว

**บริบท:** เล่ม = เหมือน "หนังสือ" · เพลง = เหมือน "ข้อ" (หลักการเดียวกับตัวเลือกพระธรรมของ phrakham.life
แต่ pleng มี 2 ชั้น: เล่ม → เพลง · ไม่มี "บท"). taxonomy นี้ P'Aim เคาะแล้ว.

## Acceptance Criteria (AC)
1. **Landing = ช่องค้นหาบนสุด + กริดเล่ม** — ไม่ใช่ลิสต์เพลงยาว. แสดง **9 เล่มตรงๆ** (ล·ย·ยอ·ทอ·ม·ส·สอ·อ·ว) แต่ละใบมี **ชื่อเล่ม + จำนวนเพลงในเล่ม**.
2. **เลือกเล่ม → รายการเพลงในเล่มนั้น** เรียงจากน้อยไปมากตาม **เลขในเล่ม** (`book_refs.no`) · แต่ละแถวโชว์ เลข + ชื่อเพลง (+ คีย์) · แถวเดียวจบ ไม่ห่อ.
3. **กดเพลง → เปิดหน้าเพลงเดิม** (`/song/:id`).
4. **มีทางย้อนกลับ** จากรายการเพลง → กลับกริดเล่ม (breadcrumb "← เล่มทั้งหมด").
5. **ค้นหาทำงานข้ามทุกเล่ม** — พิมพ์ในช่องค้นหา (จากชั้นไหนก็ได้) แล้วเห็นผลข้ามทุกเล่มทันที · ล้างช่อง → กลับสู่ชั้นเดิม. รองรับ ชื่อ/เลข/เนื้อ/คีย์/โน้ต/`ล.282` เหมือนเดิม (ไม่แก้ `songSearch.js`).
6. **เพลงที่ไม่มี `book_refs` ไม่หาย** — รวมในใบ "อื่นๆ / ยังไม่จัดเล่ม" ที่ท้ายกริด (แสดงเฉพาะเมื่อมีเพลงกลุ่มนี้).
7. **มือถือ-first + ผ่าน `docs/ui-standards.md`** — target ≥ 44px · คอนทราสต์ AA · โฟกัสเห็นชัด · ไม่มี horizontal overflow ที่ 375px · แถวรายการบรรทัดเดียว.

## นอกขอบเขต
- ไม่จัดกลุ่มด้วย `theme` รอบนี้ (P'Aim เลือก "เล่ม" เป็นแกนหลัก — theme note ไว้อนาคต).
- ไม่แตะ model / DB / `songSearch.js` — ข้อมูล `book_refs` พร้อมแล้ว.
- ไม่ทำ super-group ครอบเล่ม (KISS — 9 เล่มแบนๆ).
- "เล่มใหญ่" ยังไม่มี (จะเพิ่มตอน import) — กริดต้อง data-driven เพื่อรับเล่มใหม่เองโดยไม่แก้โค้ด.
