# US — ps3 Epic 5: dock ร่วม (แถบเครื่องมือล่าง) — B021 · B022 · B024 · B025

อ้างอิง prototype กดได้: `docs/design/ps3-dock-prototype.html` (เคาะโดย P'Aim ระหว่าง ps3)
ตารางไอคอนต่อโหมด: บทสนทนา ps3 (แก้ไข / ฝึกร้อง / พิมพ์) — ชื่อ Lucide ทุกตัว

**แนวคิดหลัก:** dock คือ "ที่วางเครื่องมือ" อันเดียว ใช้ทั้ง 3 โหมด (แก้ไข · ฝึกร้อง · พิมพ์)
โครง/พฤติกรรมเหมือนกันหมด ต่างแค่ "ชุดปุ่ม" ตามงาน · ต่อยอดจากเครื่องยนต์ dock ที่มีใน `EditorMode.vue` (เลือกปุ่มเอง + จำค่า + ลากย้าย ทำไว้แล้ว)

## D1 — dock โครงเดียว ใช้ 3 โหมด
ในฐานะผู้ใช้ ฉันอยากให้แถบเครื่องมือล่างหน้าตา/วิธีใช้เหมือนกันทุกโหมด เพื่อไม่ต้องเรียนรู้ใหม่
**AC:**
- โครง dock (หุบ · โปร่ง · เลือกปุ่ม · dynamic · ลากย้าย) เหมือนกันทั้ง แก้ไข/ฝึกร้อง/พิมพ์
- ต่างกันแค่ชุดปุ่ม: **แก้ไข** = คีย์โน้ต(ตายตัว) + ย้อน/ทำซ้ำ/ฟังท่อน/ฟังทั้งเพลง/หยุด/ส่งตรวจ/อนุมัติ/ดาวน์โหลด · **ฝึกร้อง** = เล่น/พัก · คอร์ด · ความเร็ว · คีย์ · แสดงผล · วนซ้ำ · ฟอนต์ · **พิมพ์** = พิมพ์ · แสดงผล · คอร์ด · คีย์ · ฟอนต์
- **คีย์โน้ต (แถวสัญลักษณ์ jianpu)** โชว์เฉพาะโหมดแก้ไข (ที่เดียวที่พิมพ์โน้ต)

## D2 — ปุ่มไอคอนล้วน + ตัวช่วยอ่าน
**AC:**
- ทุกปุ่มเป็น **ไอคอนล้วน** (คุ้นเหมือนกัน desktop+mobile) · ไอคอน = Lucide ชื่อตรงตามตาราง
- **desktop:** ชี้เมาส์ = tooltip ชื่อปุ่ม · **mobile:** กดค้าง = เห็นชื่อ
- ปุ่มที่มี "ค่าปัจจุบัน" = **คีย์** (โชว์ `E`) · **ความเร็ว** (โชว์ `92`) แสดงค่าต่อท้ายไอคอน · ปุ่มอื่น (แสดงผล/คอร์ด) ไอคอนล้วน

## D3 — ไม่ตกบรรทัด (dynamic overflow) — B025
ในฐานะคนใช้มือถือ ฉันอยากให้แถบไม่ตกบรรทัด/ไม่ดันเนื้อหาย
**AC:**
- แถบวัดความกว้างจริง → ใส่ปุ่มเท่าที่พอ · ที่เกินยุบเข้าปุ่ม **⋯** อัตโนมัติ (ไม่ใช่จำนวนตายตัว)
- กด ⋯ = ปุ่มที่เหลือเด้งเป็น**แผ่นลอยทับเนื้อ ไม่ดันเนื้อ** (แบบแถบคีย์บอร์ด Gboard/Samsung) · **ไม่มี swipe**
- **มือถือ default โหมดฝึกร้อง** โชว์ เล่น + คอร์ด + ความเร็ว (B024) ที่เหลืออยู่หลัง ⋯
- **desktop** กว้างพอ = เห็นปุ่มครบ ไม่มี ⋯ เกินจำเป็น
- คีย์โน้ต (โหมดแก้ไข) = ยืด/หดแชร์บรรทัดเดียวเสมอ ไม่มีคีย์ตกบรรทัด

## D4 — หุบ/กาง dock — B021
**AC:**
- **กดเองเท่านั้น** (ไม่มีหุบอัตโนมัติ)
- **desktop:** กดหูจับ (`panel-top-close`) → หุบเหลือแถบบาง · กดที่แถบ = กางคืน
- **mobile:** กดหูจับ → dock เลื่อนลงหาย เหลือ**แท็บเล็กริมล่าง** (`panel-bottom-open`) · กดแท็บ = ดึงกลับ (เหมือนปุ่มซ่อนคีย์บอร์ด)

## D5 — ปรับความโปร่ง — B022
**AC:**
- ปุ่ม `blend` เปิด **สไลเดอร์เต็มความกว้าง dock** · ปรับความโปร่งพื้นหลังแถบ (เห็นเนื้อ/โน้ตด้านหลังขณะพิมพ์/ร้อง)
- **จำค่าไว้ (localStorage)**

## D6 — เลือกปุ่มเอง — B024 (customizable)
**AC:**
- ปุ่ม `sliders-horizontal` เปิดแผง **เพิ่ม/เอาออก/สลับลำดับ** ปุ่มบนแถบ · **จำค่าไว้ (localStorage) แยกต่อโหมด**
- ใช้กลไก "คลัง deck-key" เดิมของหน้าแก้ไขซ้ำ (ไม่เขียนใหม่)
- ปุ่มที่ไม่เข้ากับสถานะ (เช่น "หยุด" ตอนไม่ได้เล่น · "อนุมัติ" ตอนไม่มีสิทธิ์) = ซ่อนเองอัตโนมัติ

## D7 — ตัวควบคุมฝึกร้อง (dropdown) — B024 control bar
**AC (โหมดฝึกร้อง):**
- **แสดงผล ▾** — ครบ(เนื้อ+คอร์ด+โน้ต) / เนื้อ+คอร์ด / เนื้อ+โน้ต / เนื้อล้วน / โน้ตล้วน
- **คอร์ด ▾** — คอร์ดตัวอักษร / เลขนัชวิลล์ / ซ่อนคอร์ด
- **คีย์ ▾** — โชว์คีย์ปัจจุบัน · เลื่อนขึ้น/ลง · กลับต้นฉบับ · **คีย์เดียวใช้ร่วมทุกโหมด (แบบ A)**
- **ความเร็ว ▾ (ศัพท์ดนตรี):** ตามเพลง · Grave♩40 · Largo♩50 · Larghetto♩60 · Adagio♩70 · Andante♩92 · Moderato♩110 · Allegro♩130 · Vivace♩150 · Presto♩180
- **วนซ้ำ** — ติ๊กเลือกหลายท่อน/ข้อ (ร้อง1/รับ/ร้อง2/ทั้งเพลง) แล้วเล่นวน
- **ฟอนต์** เล็กลง/ใหญ่ขึ้น · **เล่น/พัก** ปุ่มเดียว toggle ติดจอเสมอ (ผูก B016 viewer)

## Mobile + WCAG 2.2 AA
**AC:**
- ปุ่มแตะ ≥24×24px (เกณฑ์ผ่าน) · เป้าหมายจริง desktop 44 / mobile 40 · ระยะห่างพอ
- ทุกปุ่มมีชื่อให้โปรแกรมอ่าน (aria-label) · โฟกัสคีย์บอร์ดชัด · สีไอคอนตัดพื้น ≥3:1 · กดค้าง mobile = เห็นชื่อ
- โหมดพิมพ์: dock เป็น UI บนจอ · **ซ่อนตอนสั่งพิมพ์จริง** (`@media print`) ไม่ติดไปบนกระดาษ
