## 1. โจทย์ (คำของพี่เปา)

> "พิมพ์ขเบ็ด 1 ชั้น แต่โชว์เป็น 2 ชั้น เสียงถูกต้องแล้วแต่โชว์ผิด"
> (P'Aim ทวน) "มี 1 เส้นก็ให้ใส่ 1 เส้น มี 2 เส้นก็ใส่ 2 เส้น **ถ้าเส้นติดกัน จะต้อง 1 ก็ 1 จะต้อง 2 ก็ 2 — แค่ต้องลากให้ถูก**"

**ตัวอย่างจริง:** เพลง 751 "วันนี้อาณาจักรเป็นการฝึกฝนต่อข้า" ข้อ 1 ห้อง `7. 1`
โน้ตคือ `.7_.` (1 ขีด + จุด) กับ `1__` (2 ขีด) → **บนแผ่นโชว์ขีดคู่ยาวคลุมทั้งสองตัว** ทั้งที่ตัวแรกต้องมีขีดเดียว

## 2. ต้นเหตุ (PM verify จากโค้ดแล้ว)

`src/lib/notation.js:228-238` — `beamGroups()` ยุบทั้ง run เหลือธงเดียว:

```js
beams.push({ start: run[0].idx, end: run[run.length-1].idx,
             u2: run.some((t) => t.underlines >= 2) })   // ← ต้นเหตุ
```

`src/components/NoteRow.vue:161` วาดตามธงนั้น `const th = b.u2 ? 4 : 1.5` → **ทั้ง run ได้ขีดคู่ ถ้ามีตัว 2 ชั้นอยู่แม้ตัวเดียว**

หมายเหตุ: ขีดใต้ราย-ตัว (`.num.u1/.u2` CSS `NoteRow.vue:320-321`) **ถูกอยู่แล้ว** — ผิดเฉพาะ "คานคลุม" (beam overlay) ที่วาดทับ

## 3. สิ่งที่ต้องได้ (AC)

**A1 — จำนวนเส้นต้องตรงรายตัว**
ทุกโน้ตในกลุ่มต้องเห็นจำนวนเส้นเท่ากับ `underlines` ของตัวเอง (1 ชั้น = 1 เส้น · 2 ชั้น = 2 เส้น) ไม่ว่าจะอยู่ติดกับตัวไหน

**A2 — เส้นยังต่อกันเป็นคานเดียว (ไม่ถอยกลับไปเป็นขีดขาดๆ)**
เส้นชั้นที่ 1 = ลากยาวคลุมทั้ง run เหมือนเดิม (นี่คือของดีที่มีอยู่แล้ว ห้ามเสีย)
เส้นชั้นที่ 2 = ลากเฉพาะช่วงที่เป็นโน้ต 2 ชั้นจริงๆ **เท่านั้น**

**A3 — โน้ต 2 ชั้นตัวเดียวโดดในกลุ่ม**
เช่น `.7_. 1__` — ตัว `1__` ต้องได้เส้นที่ 2 เป็น **ขีดสั้น** ใต้ตัวมันเอง (partial beam / ขีดหัก ตามมาตรฐาน engraving) ไม่ใช่ลากยาวไปหาตัวข้างๆ
ทิศขีดสั้น: ชี้เข้าหาตัวโน้ตที่อยู่ใน beam เดียวกัน (ถ้าอยู่ท้าย run → ชี้ซ้าย · ถ้าอยู่หัว run → ชี้ขวา)

**A4 — ไม่ถอยหลัง**
- กลุ่มที่ทุกตัวเป็น 1 ชั้นเหมือนกัน → หน้าตาเดิมเป๊ะ
- กลุ่มที่ทุกตัวเป็น 2 ชั้นเหมือนกัน → หน้าตาเดิมเป๊ะ (คานคู่ยาวคลุมหมด = ถูกอยู่แล้ว)
- กฎการตัดกลุ่ม (จังหวะ · พยางค์ใหม่ · triplet · ตัวหยุด · `-`) **ห้ามแตะ**
- fallback ตอนวัด layout ไม่ได้ (jsdom/พิมพ์) ต้องยังทำงาน — ขีดราย-ตัวโผล่แทน

**A5 — เสียงห้ามเปลี่ยน**
`midi.js` / duration / `parseNotes` ห้ามแตะ · ทดสอบยืนยันว่า MIDI output เพลง 751 ก่อน-หลัง เหมือนกันเป๊ะ

**A6 — พิมพ์ออกกระดาษถูกด้วย**
Ctrl+P / PDF ต้องเห็นเส้นตรงกับบนจอ (beam overlay วัดใหม่ตอน `beforeprint` อยู่แล้ว — อย่าให้พัง)

**A7 — กวาดทั้งคลัง**
เขียนสคริปต์ไล่ทุกเพลงใน Supabase หา run ที่ **มีทั้ง 1 ชั้นและ 2 ชั้นปนกัน** → รายงานว่า **กี่เพลง กี่จุด** (พี่เปาบอกว่าเป็นหลายเพลง ต้องพิสูจน์ว่าหายทั้งหมด ไม่ใช่แค่ 751)
ใส่รายชื่อเพลง+เลขในรายงาน · ยืนยัน sample อย่างน้อย 5 เพลงว่าหลังแก้ถูกทุกตัว
