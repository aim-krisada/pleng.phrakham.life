## S1 — แถบบนเดียวใช้ทุกโหมด
ในฐานะผู้ใช้ ฉันอยากมีแถบบนชุดเดียวทุกหน้า/โหมด เพื่อไม่ต้องเรียนรู้เมนูใหม่ทุกที่
**AC:**
- แถบบน: `[เพลง.พระคำ.ชีวิต ▾] · [ชื่อเพลง ▾] · [เพลง ▾] · (ฝึกร้อง | แผ่นเพลง | แก้ไข) · [เข้าสู่ระบบ]`
- เมนูชื่อเว็บ = ไปหน้า: รายการเพลง · คู่มือ · เกี่ยวกับเรา · พระคำ.ชีวิต ↗ (ไอคอน = โลโก้จริง/favicon) — **ไม่มี "ทำเพลง"** (B007)
- ไอคอน (Lucide): รายการเพลง=`list-music` · คู่มือ=`book-open` · โหมดฝึกร้อง=`mic` · แผ่นเพลง=`music` · แก้ไข=`pencil`
- โหมดที่ใช้อยู่ = พื้นน้ำตาลเด่นอันเดียว

## S2 — "เพลง ▾" = สร้าง + เปิด ที่เดียว
ในฐานะคนทำเพลง ฉันอยากกดที่เดียวแล้วได้ทั้งสร้างเพลงใหม่และเปิดเพลงเดิม
**AC:**
- ปุ่ม "เพลง ▾" (แทน "เปิดเพลง") กดทีเดียวเปิด panel: **＋สร้างเพลงใหม่ (บนสุด เด่น)** → เส้นคั่น "หรือเปิดเพลงที่มีอยู่" → ช่องค้นหา + รายการเพลง
- จิ้มเพลง = เปิดเลย (**ไม่มีปุ่ม OK**) · `↑↓` เลือก · `Enter` เปิด · `Esc`/คลิกนอก = ปิด (WCAG 2.2 AA: โฟกัสชัด · แถว ≥24px · คีย์บอร์ดครบ)
- เปิดเพลง = เปิดในโหมดที่กำลังอยู่ · **ไม่มี "เลือกเพลงเพื่อแก้…" ซ้ำ** (B003)
- มือถือ: panel = **เต็มจอ** (B008/B018)

## S3 — ชื่อเพลง: คลิกแก้ + เมนู
**AC:**
- คลิกที่ตัวชื่อ = พิมพ์แก้ inline (Enter จบ) · `▾` = เมนูเพลงนี้: ตั้งค่าเพลง · ออกจากเพลงนี้ · **ลบเพลง** (แดง · **ผู้อนุมัติ Tier 2** · **confirm ก่อนลบ**)

## S4 — มือถือ 2 แถว (ไม่มี hamburger)
**AC:** (B009/B014/B017)
- แถว 1: `เพลง.พระคำ.ชีวิต ▾ · ชื่อเพลง ▾ · 👤` · แถว 2: `เพลง(📄🎵) · โหมดไอคอนล้วน (ไมค์/แผ่น/ดินสอ)`
- **ไม่มี hamburger** (▾ บอกว่าเป็นเมนูพอ) · login = ไอคอน 👤

## S4-B045 — คืนที่ให้ชื่อเพลงบนมือถือ (amend · P'Aim เคาะ 10 ก.ค. · DS `docs/ds/mobile-shell-b045.md`)

**ปัญหา:** จริง ๆ `.sb-right` (ฟอนต์ + ดาวน์โหลด + ล็อกอิน) ไปเบียดอยู่แถว 1 → ชื่อเพลงถูกตัดเหลือ `1. ...` (real-use มือถือ 9 ก.ค.) · แถว 1 ต้องเหลือแค่ `แบรนด์ · ชื่อเพลง · 👤` ตามที่ S4 ตั้งใจ

**AC:**
1. **แบรนด์คงเดิม** — ยังเป็นชื่อ `เพลง.พระคำ.ชีวิต ▾` (P'Aim: ใช้ชื่อไปก่อน · **ไม่ทำ hamburger/โลโก้** · B009 คงอยู่) · desktop brand ไม่เปลี่ยน
2. **ตัดปุ่มดาวน์โหลด (DownloadTool) ออกจากแถบบน** (P'Aim: "ตัดเลย ซ้ำซ้อน") — เพราะ dock มีดาวน์โหลดอยู่แล้ว (ฝึกร้อง = `SongViewer` viewDock `download` · แก้ไข = `EditorMode` `DOCK_DEFAULT` `download`) · โหมดแผ่นเพลงใช้ปุ่มพิมพ์ (ไม่ต้องเพิ่ม download เว้น P'Aim สั่ง)
   - ⚠️ **ก่อนตัด dev ต้องตรวจ:** DownloadTool เดิมโผล่ทุกหน้า (รวม `รายการเพลง`/คู่มือ) — ถ้าหน้าที่**ไม่มี dock** เคยพึ่ง DownloadTool ทำสิ่งที่ dock download ไม่ครอบคลุม → ต้องหาที่รองรับ **ห้ามตัดจนเสียฟีเจอร์** · ถ้า DownloadTool มีความหมายเฉพาะตอนเปิดเพลง (= มี dock อยู่แล้ว) → ตัดได้เลย
3. **ตัดปุ่มฟอนต์ (FontTool) ออกจากแถบบน → เพิ่มปุ่ม `Aa` ใน dock** (control ปรับขนาดตัวอักษร = ใช้บ่อยในการอ่าน · ต้องเข้าถึง 1 แตะ):
   - ปุ่ม `Aa` เดี่ยวใน dock โหมดที่อ่านเพลง (อย่างน้อย **ฝึกร้อง** · แผ่นเพลงถ้าเหมาะ) · แตะ ≥44×44
   - **แตะ 1 ครั้ง → popover** ลอยเหนือ dock (ไม่ดันเนื้อ · pattern เดียวกับ display/tempo/key ที่มีอยู่)
   - ใน popover = **สไลเดอร์** (P'Aim เคาะ) ปรับขนาด: ปลายซ้าย `A` เล็ก · ปลายขวา `A` ใหญ่ · ครอบช่วง + ขั้นเท่า FontTool เดิม (`readingFontScale`) · **เลื่อน = เนื้อเพลงเปลี่ยนขนาดสด** · หัวข้อ "ขนาดตัวอักษร" · ป้ายระดับปัจจุบัน (optional)
   - เข้าถึงคีย์บอร์ด + โฟกัสชัด (WCAG 2.2 AA) · slider thumb แตะได้บนมือถือ
4. **ผลลัพธ์แถว 1 มือถือ** = `แบรนด์ ▾ · ชื่อเพลง (ยาวขึ้น อ่านได้จริง) · 👤` (ตรง AC S4 เดิม) · เอา download+font ออก คืนที่ ~88px
5. **ขอบเขต/สโคป (⚠️ ให้ PM/P'Aim ยืนยัน):** download+font อยู่ใน `.sb-right` ซึ่งใช้**ทุกขนาดจอ** → การตัดออก + ย้าย Aa ลง dock **มีผล desktop ด้วย** (dock บน desktop รับ Aa/มี download อยู่แล้ว = สอดคล้อง "แถบเดียว dock เป็นบ้านของเครื่องมือเพลง") · **แบรนด์ desktop ไม่แตะ** · ถ้า P'Aim อยากคง download/font บนแถบบน desktop → ทำเป็น mobile-only (`@media max-width:760px` ซ่อน sb-right download/font) แทน — **PM เคาะ 1 บรรทัด**

**ไฟล์ (dev · หลัง rebase ฐานล่าสุด):** `ShellBar.vue` (ตัด DownloadTool/FontTool จาก default `.sb-right` slot · หรือ mobile-only ตามข้อ 5) · `SongViewer.vue` (เพิ่ม tool `Aa` เข้า viewDock + popover สไลเดอร์) · อาจ `Studio.vue`/`FontTool.vue` (reuse logic ปรับขนาด) · **ไม่ต้องแตะ** dock download (ฝึกร้อง/แก้ไข มีแล้ว) · **verify มือถือจริงผ่าน LAN** (ชื่อเพลงยาวไม่ตัด · Aa สไลเดอร์เปลี่ยนขนาดสด · ปุ่ม ≥44) · **ห้าม merge/deploy**
