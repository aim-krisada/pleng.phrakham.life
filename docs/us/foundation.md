# US — WT-0 ฐาน (foundation)

**Worktree:** WT-0 · **Branch:** `wt0-foundation` (แตกจาก `studio-shell-redesign`)
**โยงกลับ mission:** ผู้ใช้ 3 กลุ่ม (สิทธิ์) · หลักการ "เส้นแบ่ง = เก็บเข้าระบบได้ไหม" · แผนที่งาน (ต้องมีกำแพง+ประตูก่อนทำขนาน)

> **สำคัญ — อย่ารื้อของเดิม:** ส่วนใหญ่ **สร้างไว้แล้ว** บน `studio-shell-redesign` (3-tier, โหมด ดู/แผ่น/แก้, ShellBar, ปุ่มบันทึกแยกตาม tier).
> งานจริงของ WT-0 = **แกะ editor ออกจาก `Studio.vue` เป็นไฟล์แยก + เขียน "สัญญาจุดต่อ" ให้ชัด** เพื่อให้ A/B/C/D ทำขนานได้ ไม่ใช่สร้างใหม่.
> แต่ละ AC ล่างกำกับ `[มีแล้ว]` / `[ต้องทำ]`.

## US-foundation-1 — โหมดเดียว 3 มุมมอง
ในฐานะ**ผู้ใช้คนใดก็ได้** ฉันต้องการเปิดเพลงแล้วสลับ **ดู (คาราโอเกะ) / แผ่นเพลง / แก้ไข** ได้ในที่เดียว เพื่อไม่ต้องเรียนรู้หลายหน้า
- [มีแล้ว] เปิด `/song/:id` → เข้า Studio, ค่าเริ่มต้น = โหมดดู/แผ่น
- [ต้องทำ] `Studio.vue` เหลือแค่ "จัดวาง + เลือกโหมด" — โค้ดแต่ละโหมดอยู่คนละ component (`SongViewer` / `SongSheet` / `EditorMode`)
- [ต้องทำ] สลับโหมดแล้วเพลงที่กำลังทำไม่หาย (state คงอยู่)

## US-foundation-2 — เส้นแบ่งสิทธิ์อยู่ที่ "การเก็บ" ไม่ใช่ "การแก้"
ในฐานะ**เจ้าของระบบ** ฉันต้องการให้ทุกคนแก้เพลงได้ แต่ **เก็บเข้าคลังได้เฉพาะทีม** เพื่อกันคลังหลักไม่ให้ปนเปื้อน
- [มีแล้ว] Tier 0 (ไม่ล็อกอิน): ปุ่มบันทึก = **ดาวน์โหลด JSON** เท่านั้น
- [มีแล้ว] Tier 1 (editor): + บันทึก **ร่าง / ส่งตรวจ** ในระบบ
- [มีแล้ว] Tier 2 (approver): + **อนุมัติเข้าคลัง / ลบ / ย้อน**
- [ต้องทำ] รวม gating เป็นจุดเดียว (`store.tier` · `store.canStore` · `store.canApprove`) ให้ A/B/C/D เรียกใช้ ไม่กระจายเช็ก `session` ซ้ำๆ

## US-foundation-3 — เข้าระบบแบบเชิญเท่านั้น
ในฐานะ**เจ้าของระบบ** ฉันต้องการให้สมัครเองไม่ได้ ต้องถูกเชิญ เพื่อคุมว่าใครเป็น editor / approver
- [ต้องทำ/ตรวจ] หน้า login ไม่มีปุ่ม "สมัคร" — บัญชีถูกสร้างโดยเจ้าของผ่าน Supabase เท่านั้น
- [ต้องทำ] role (editor/approver) อ่านจาก session — DS ระบุว่าเก็บที่ไหน

## US-foundation-4 — สัญญาจุดต่อ (contract) สำหรับงานขนาน
ในฐานะ**ทีมพัฒนา (หลาย session)** เราต้องการให้แต่ละโหมดเป็นไฟล์ของตัวเอง มีหน้าตาการต่อ (props/events) ชัด เพื่อทำพร้อมกันโดยไม่แตะ `Studio.vue`
- [ต้องทำ] แกะ `EditorMode.vue` ออกจาก `Studio.vue`
- [ต้องทำ] ทั้ง 3 โหมดรับ prop มาตรฐาน + ส่ง event มาตรฐาน (นิยามใน DS)
- [ต้องทำ] หลัง WT-0 merge: A/B/C/D แก้เฉพาะไฟล์โหมดของตัวเอง + `jsonIO.js` — ไม่ต้องแตะ `Studio.vue`
