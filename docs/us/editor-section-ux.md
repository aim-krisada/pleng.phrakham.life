# US — โหมดแก้ไข: จัดการท่อน/ข้อ ให้เข้าใจง่ายระดับสากล

ต่อยอด `docs/us/ps3-editor.md` (E1 rail · E4 แก้เนื้อ) · visual = `docs/design/editor-section-ux.html`
เป้า: คนไม่เทคนิค (พี่เปา) เปิดโหมดแก้ไขครั้งแรก **รู้เองว่าตั้งชื่อท่อน · จัดลำดับ · แก้เนื้อ ตรงไหน** โดยไม่ต้องมีคนสอน.
⚠️ โมเดล v2 ไม่เปลี่ยน (stanza=ทำนอง · arrangement row=ท่อน) — เปลี่ยน UI/interaction เท่านั้น.

## ✅ P'Aim เคาะแล้ว (10 ก.ค.) — ล็อก
1. **ตัดบล็อก “📜 ลำดับเพลง” ล่างสุดทิ้ง** → ฟังก์ชันย้ายไปแถบซ้าย + หัวท่อน
2. **แต่ละแถวเรียก “ท่อน”** (= arrangement row = ตัวหลักที่ผู้ใช้เห็น) → เพื่อกันคำชน **melody/stanza เปลี่ยน label UI จาก “ท่อน A” เป็น “ทำนอง A”**
3. **ทำนองเป็นเบื้องหลัง** — เพิ่มท่อน = ได้ทำนองอัตโนมัติ · ใช้ซ้ำผ่านป้าย ♪
4. **จัดลำดับ = ลาก + ▲▼ ทั้งคู่**
5. 🚧 **ข้อบังคับหลัก (P'Aim ย้ำ): “ไส้ใน” การแก้ไข/การดู/สัญลักษณ์ inline ต้องใช้งานได้เหมือนเดิมทุกอย่าง** — ดู **SX7 (กำแพงกันของเดิม)** ด้านล่าง · redesign แตะเฉพาะชั้น “จัดการท่อน” (rail + แก้ชื่อ + จัดลำดับ + ตัดบล็อกล่าง + lens default) เท่านั้น

## SX1 — แถบซ้ายเป็น “โครงเพลง” รายการเดียว (แทน 3 กลุ่ม)
ในฐานะคนทำเพลง ฉันอยากเห็นลำดับท่อนทั้งเพลงเป็นรายการเดียวที่จัดการได้ครบในที่เดียว
**AC:**
- แถบซ้ายมีรายการหลัก **“โครงเพลง”** = arrangement rows เรียงตามลำดับร้อง (แทนกลุ่ม “เนื้อร้อง” + บล็อกล่าง)
- แต่ละแถว: ที่จับลาก ⠿ · เลขลำดับ · ชื่อ (แก้ในที่) · ป้ายทำนอง ♪X · ▲▼ · ⋯
- กลุ่ม **“ทำนอง (โน้ต)”** ย้ายเป็นส่วนรอง **ยุบไว้** ใต้โครงเพลง (เปิดเมื่ออยากแก้โน้ต/ใช้ซ้ำ)
- **ตัดคำ “ขั้นสูง” ทิ้ง** · **ตัดบล็อก “📜 ลำดับเพลง” ล่างสุดทิ้ง** (ฟังก์ชันย้ายมาแถบซ้าย + หัวท่อน)
- รายการ arrangement แสดง **ที่เดียว** (ไม่ซ้ำ 2 จุด) — แก้ที่ไหนก็สะท้อนทุกที่

## SX2 — แก้ชื่อท่อนในที่ (in-line rename)
ในฐานะผู้ใช้ ฉันอยากคลิกชื่อท่อนแล้วพิมพ์แก้ได้เลย ไม่ต้องไปหาช่องแก้แยก
**AC:**
- คลิกชื่อท่อน (ทั้งในแถบซ้าย **และ** หัวท่อนบนแคนวาส) → กลายเป็นช่องพิมพ์ตรงนั้น
- **Enter / คลิกที่อื่น = บันทึก** · **Esc = ยกเลิก** · แก้ที่เดียว (`row.label`) สะท้อนทุกที่ที่ชื่อโชว์ (P1/P5)
- มี affordance ชัดว่าคลิกแก้ได้ (hover เปลี่ยนพื้น/เคอร์เซอร์ text) — WCAG: คีย์บอร์ดเข้าถึงได้ มี label

## SX3 — จัดลำดับด้วยลากวาง + ปุ่ม ▲▼ (รองรับสัมผัส)
ในฐานะผู้ใช้ ฉันอยากลากท่อนสลับลำดับได้ หรือกดขึ้น/ลงถ้าลากไม่ถนัด
**AC:**
- ลากที่จับ ⠿ จัดลำดับได้ทั้ง **เมาส์และนิ้ว (touch)** · มี drop indicator เห็นชัด
- **ปุ่ม ▲▼ ยังอยู่** ที่แถวแถบซ้าย **และ** หัวท่อนบนแคนวาส = ทางเลือกไม่ต้องลาก (**WCAG 2.2 §2.5.7**)
- ลาก/เลื่อนแล้ว selection + เนื้อกลางตามท่อนที่ย้าย · ปุ่มขอบ (บนสุด/ล่างสุด) disabled

## SX4 — เลือกท่อน = เห็นเนื้อทันที (แก้ P4)
ในฐานะผู้ใช้ ฉันอยากพอเลือกท่อนแล้วเห็นช่องพิมพ์คำใต้โน้ตเลย ไม่ต้องกดเปิดอะไรก่อน
**AC:**
- เลือกท่อนในแถบซ้าย/แคนวาส → โน้ต+**ช่องคำใต้โน้ตของท่อนนั้นโผล่ทันที** (lens default = ท่อนที่เลือก ไม่ใช่ -1 ซ่อน)
- ช่องคำแดง = ยังไม่ใส่คำ · ตัวนับ “x/y คำ” คงเดิม · พาเนล “📝 แก้เนื้อแบบย่อหน้า” ยังอยู่เป็นทางเลือกรอง (E4 เดิม)

## SX5 — ทำนองเป็นสมบัติของท่อน (แก้ P8)
ในฐานะคนใหม่ ฉันไม่อยากถูกบังคับให้เข้าใจ “ข้อต้องเลือกทำนอง” ก่อนเริ่ม
**AC:**
- **เพิ่มท่อน = ได้ทำนองให้อัตโนมัติ** (สร้าง stanza ใหม่ หรือใช้ท่อนก่อนหน้า — เคาะใน DS) → เริ่มพิมพ์ได้ทันที
- การ **ใช้ทำนองซ้ำ** (เช่นท่อนรับ) = แตะป้าย ♪X เลือกทำนองที่มี/สร้างใหม่ (ไม่ใช่ dropdown เด่นในทุกแถว)
- ยังแก้ทำนอง/เพิ่มทำนองตรง ๆ ได้ในส่วน “ทำนอง (โน้ต)” ที่ยุบไว้ (power user)

## SX6 — มือถือ + PC ดีทั้งคู่
**AC:**
- **PC:** แถบซ้ายค้าง = ภาพรวม+กระโดด · หัวท่อนถือ action หลัก
- **มือถือ:** หัวท่อนบนแคนวาสถือ แก้ชื่อ + ▲▼ ในตัว (ทำได้โดยไม่เปิดลิ้นชัก) · “โครงเพลง” เต็ม = bottom-sheet/drawer เปิดเมื่ออยากกระโดด · ปุ่มแตะ ≥44px · ลาก long-press ได้

## SX7 — 🚧 กำแพงกันของเดิม (P'Aim ย้ำ: “ไส้ในต้องเหมือนเดิม”)
ในฐานะพี่เปา ฉันเรียนรู้วิธีแก้โน้ต/เนื้อ/ดูผลไปแล้ว — redesign **ต้องไม่ทำให้สิ่งที่ทำเป็นอยู่แล้วเปลี่ยนหรือหาย**
**AC (ทุกข้อต้องทำงานเหมือนเดิมเป๊ะ — เป็น regression gate ก่อน merge):**
- **การพิมพ์โน้ต** (ช่องโน้ต NoteBoxes · ripple · Enter/เว้นวรรค/←→) — เหมือนเดิม
- **แถบสัญลักษณ์ล่าง** (จุด/ขีด/โค้ง/วงเล็บ/`#`/`b` · แตะช่องแล้วจิ้ม) — เหมือนเดิม
- **การดู (สัญลักษณ์ inline / jianpu render)** — “ตัวอย่างสด” (render เหนือแต่ละห้อง) · “ดูผล” ต่อห้อง (สลับแก้⇄แผ่น) · “ดูผลทั้งเพลง” (หน้าต่างลอย/overlay) — เหมือนเดิมทุกอัน
- **คอร์ด** (เลือก/แทรก/รวม ที่โน้ต) · **ช่องคำรายพยางค์ใต้โน้ต** (แดง=ยังไม่ใส่ · เส้นประ=ลากเสียง · ◀▶ ดัน/ดึง · เกินโน้ต) — เหมือนเดิม
- **สถานะจังหวะต่อห้อง** + **ห้องต่อกัน (pickup)** + **เครื่องมือห้อง ⋯** (ย้าย/สำเนา/ลบ/‖:/:‖/volta) — เหมือนเดิม
- **โครงเพลงระดับบรรทัด** (ฮุก/ซ้ำ/คัดลอก/ลบบรรทัด · ⋯ ชื่อบรรทัด/ต่อห้อง/ป้าย · จบเพลง) · **เค้าโครง 1ห้อง/ต่อกัน** · **ฟังเสียง** (ท่อน/บรรทัด/ห้อง) — เหมือนเดิม
- **แผง “📝 แก้เนื้อแบบย่อหน้า”** · **การ์ดตั้งค่าเพลง** · **ตรวจแล้ว** · **บันทึกร่าง/ส่งตรวจ/อนุมัติ/เผยแพร่/ลบ** · **ประวัติ/งานร่าง** · **ดาวน์โหลด/อัปโหลด JSON** · **undo/redo (⌘Z)** — เหมือนเดิม
- **help:** (i) วิธีใช้ · (?) legend สัญลักษณ์ · ลิงก์คู่มือ — เหมือนเดิม
- **ผลลัพธ์:** เพลง import v2 (เช่นเพลง 100) โหลด/แก้/บันทึกได้เหมือนก่อน redesign 100% (ไม่ regress)
