# US — ปุ่ม Remix + ล็อกสูตรการบรรเลง (สุ่มได้ · ปักได้ · MP3 ตรงกับที่ฟัง)

> **สรุปภาษาคน:** เปียโน "บรรเลง" ของเราแต่งการเล่นให้ฟังเหมือนคนเล่นจริง — มีการหยอดโน้ต/ประกาย
> เล็กน้อยที่กำหนดด้วย "สูตรสุ่ม" (seed). วันนี้แต่ละเพลงมีสูตรเดียวตายตัว. งานนี้ให้ผู้ฟัง
> **กดสุ่มสูตรใหม่ (Remix)** ฟังหลาย ๆ แบบ, **ปักสูตรที่ชอบ (ล็อก)** ให้เล่นเหมือนเดิมทุกครั้ง,
> และทำให้ **ไฟล์ MP3 ที่ดาวน์โหลด = เสียงเดียวกับที่กำลังฟังเป๊ะ** (ปัจจุบัน MP3 ไม่ผ่านตัวแต่ง
> เสียงเลย → ฟังต่างจากหน้าเว็บ).

เอกสารนี้ = **requirement + AC เท่านั้น** (design/สเปกเทคนิค อยู่ที่ [`docs/ds/remix.md`](../ds/remix.md)).
ยึด **world-class by default** (`docs/sop.md §0`) + `docs/ui-standards.md` (WCAG 2.2 AA · APG · HIG/Material).

---

## เป้าหมาย (north star)
ผู้ใช้ที่ไม่รู้ดนตรี **ควบคุมความหลากหลายของการบรรเลงได้ด้วยตัวเอง** และมั่นใจว่า "สิ่งที่ได้ยิน =
สิ่งที่ดาวน์โหลด" — ไม่มีเซอร์ไพรส์.

## ผู้ใช้ + บริบท
- **ผู้ฟัง/นักร้อง (หน้าฝึกร้อง)** = ผู้ใช้หลัก. Remix ทำงานเฉพาะเมื่อเปิดสไตล์ที่มีการบรรเลง
  (บรรเลง/สงบ) — โหมด "ตรงโน้ต" (ตรวจโน้ต) ไม่มีการสุ่ม จึงไม่มี Remix.
- **หน้าแก้เพลง (พี่เปา ตรวจโน้ต)** default = ตรงโน้ต → ไม่เห็น Remix (เห็นได้ถ้าสลับไปสไตล์บรรเลง).
- **ต่อยอดของเดิม ไม่รื้อ** (`feedback_refine_not_redesign`): ใช้ seed + `arrange()` ที่มีอยู่, เสียบ
  ปุ่มในป็อปอัพ "เสียงดนตรี" ที่มีอยู่.

## ขอบเขต (scope)
- **มี:** ปุ่ม Remix (สุ่มสูตรใหม่) · ปุ่มล็อก/ปลดล็อก · ป้ายบอกสูตรปัจจุบัน + สถานะล็อก · MP3
  render ด้วยสูตร+สไตล์เดียวกับที่ฟัง · จำสูตรที่ล็อกไว้ต่อเพลง.
- **ไม่มี (คนละงาน):** เมนู "ปรับละเอียด" 12 เทคนิค (แยกงาน · Remix แค่**ประกอบร่วม** ตาม DS §7) ·
  การแชร์สูตรข้ามเครื่อง/ผู้ใช้ · แกลเลอรีสูตรสำเร็จรูป · undo history ของ remix.

## Acceptance Criteria

### AC1 — ปุ่ม Remix เปลี่ยนการบรรเลงได้ทันที
- **Given** เปิดเพลงหน้าฝึกร้อง สไตล์ = บรรเลง (หรือ สงบ) **When** กด "Remix" **Then** การบรรเลง
  เปลี่ยนเป็นสูตรใหม่ (การหยอดโน้ต/ประกาย/humanize ต่างจากเดิมอย่างได้ยิน) และถ้ากำลังเล่นอยู่ **เสียง
  เปลี่ยนทันทีต่อจากโน้ตปัจจุบัน** (ไม่กระตุก · ไม่เริ่มใหม่จากต้น) — สอดคล้องพฤติกรรม live re-schedule
  ที่มีอยู่ (คีย์/tempo/สไตล์).
- ทำนอง (pitch) และคอร์ดบนแผ่นเพลง **ไม่เปลี่ยน** — Remix ปรุงแค่ "การแสดง" ตาม GOLDEN RULE (แผ่น = SSOT).
- กด Remix ซ้ำ ๆ ได้เรื่อย ๆ โดยป็อปอัพ **ไม่ปิด** (ลูป ฟัง→สุ่ม→ฟัง ลื่น).

### AC2 — เห็นว่าตอนนี้เป็นสูตรไหน
- มีป้ายบอก **สูตรปัจจุบัน** อ่านเข้าใจโดยไม่ต้องรู้ดนตรี (เช่น "แบบเริ่มต้น" / "แบบที่ 3").
- เปลี่ยนสูตรแล้วป้ายอัปเดต และประกาศให้ screen reader รู้ (`aria-live="polite"`).

### AC3 — ล็อกสูตรที่ชอบ
- **Given** สุ่มจนได้สูตรที่ชอบ **When** กด "ล็อก" **Then** สถานะเป็น 🔒 ล็อก + ป้ายบอกว่าล็อกสูตรไหน.
- **Given** ล็อกไว้แล้ว ปิดเพลง/รีเฟรช/เปิดใหม่ **Then** เพลงนั้นเล่นด้วยสูตรที่ล็อกไว้ (จำต่อเพลง).
- **Given** ยังไม่ล็อก ปิดแล้วเปิดใหม่ **Then** กลับเป็นสูตรเริ่มต้น (แบบเดิมของเพลง).
- กด "ปลดล็อก" → กลับเป็นไม่ล็อก (ยังฟังสูตรปัจจุบันต่อได้ แต่ไม่จำข้ามการเปิดใหม่).

### AC4 — MP3 = เสียงเดียวกับที่ฟังเป๊ะ (หัวใจของงาน)
- **Given** กำลังฟังสูตรใด ๆ (สุ่มแล้ว/ล็อกแล้ว/ค่าเริ่มต้น) ที่คีย์+tempo+สไตล์+เทคนิคปัจจุบัน
  **When** ดาวน์โหลด MP3 **Then** ไฟล์ที่ได้ = **เพอร์ฟอร์แมนซ์เดียวกับรอบแรกที่ได้ยิน** (ผ่านตัวแต่งเสียง
  `arrange()` ด้วย seed + คอนฟิกเดียวกัน) — ไม่ใช่เสียงทื่อ/ต่างออกไปเหมือนปัจจุบัน.
- เล่นไฟล์ซ้ำกี่ครั้งก็เหมือนเดิม (deterministic).
- ป็อปอัพดาวน์โหลดบอกชัดว่า "ไฟล์นี้ = สูตรที่กำลังฟัง (แบบที่ N / 🔒)" — ตัด doubt.

### AC5 — เข้าถึงได้ (a11y · WCAG 2.2 AA · APG)
- ปุ่ม Remix/ล็อก = `<button>` มี `aria-label` ชัด · โฟกัสเห็นชัด (2.4.7) · ใช้คีย์บอร์ดครบ (2.1.1) ·
  target ≥ 44px (โปรเจกต์ตั้งเป้า · 2.5.8) · หลังกด Remix โฟกัสยังอยู่ที่ปุ่ม (กดรัวได้ด้วยคีย์บอร์ด).
- ปุ่มล็อกเป็น toggle: สื่อสถานะด้วย `aria-pressed` (ไม่พึ่งสีอย่างเดียว · 1.4.1) + ไอคอน 🔒/🔓 +
  ข้อความ.
- เปลี่ยนสูตร → ประกาศผ่าน `aria-live` (4.1.3).
- เคารพ `prefers-reduced-motion` (2.3.3) — ถ้ามีอนิเมชันสุ่ม ต้องปิดได้/ลดลง.

### AC6 — มือถือ + ป็อปอัพไม่ scroll
- ใช้งานได้จริงบนมือถือ 360px + 412px (per-button `right ≤ viewport` · ไม่ overflow · บทเรียน launch).
- ป็อปอัพ "เสียงดนตรี" ที่เพิ่มกลุ่ม Remix เข้าไป **ยังไม่มี scroll แนวตั้ง/แนวนอนบนจอปกติ** และไม่หลุด
  ขอบ (ui-standards §2). ปุ่มในด็อก **ไม่เพิ่มปุ่มใหม่บนแถบ** (คงหลัก "1 ปุ่มเสียงดนตรี" ของ P'Aim 13 ก.ค.).

### AC7 — สอดคล้อง/ไม่ถอยหลัง
- โหมด "ตรงโน้ต" ยังเล่นตรงโน้ตเป๊ะ ไม่มีการสุ่ม/Remix (invariant §6c ของ arranger คงอยู่).
- ค่า `variant = 0` (เริ่มต้น) = พฤติกรรมเดิมทุกอย่าง → 54 arranger tests + audio tests เดิม **ยังเขียว**.
