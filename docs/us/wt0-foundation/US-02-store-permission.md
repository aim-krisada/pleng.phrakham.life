# US-02 — สิทธิ์อยู่ที่ "การเก็บ" ไม่ใช่ "การแก้"

**Worktree:** WT-0 · **Branch:** `wt0-foundation` · **คู่ DS:** `ds/wt0-foundation/DS-02-store-permission.md`
**สถานะ:** spec (รอ dev)
**โยง mission:** เส้นแบ่งสิทธิ์ = "เก็บเข้าระบบได้ไหม"

## เล่าเป็นภาษาคน
เราอยากให้เว็บนี้ใครก็ **หยิบเพลงไปแก้เล่นได้** ไม่ต้องขออนุญาต — แต่ **คลังเพลงกลาง** ที่คนทั้งประเทศเห็น ต้องสะอาด ไม่ให้ใครก็ได้ยัดเพลงเข้าไปมั่ว
เส้นแบ่งจึงไม่ได้อยู่ที่ "ใครแก้ได้" (แก้ได้ทุกคน) แต่อยู่ที่ "**ใครเก็บเข้าระบบได้**":
คนทั่วไปเก็บงานเป็น **ไฟล์ของตัวเอง** (ดาวน์โหลด JSON) · ทีมที่ล็อกอินเก็บ **ร่าง/ส่งตรวจ** ในระบบได้ · **ผู้อนุมัติ** เท่านั้นที่ดันเข้าคลังจริง

## Story
ในฐานะ **เจ้าของระบบ** ฉันต้องการให้ทุกคนแก้เพลงได้ แต่เก็บเข้าคลังได้เฉพาะทีม เพื่อกันคลังหลักไม่ให้ปนเปื้อน

## Acceptance Criteria
- [ ] Tier 0 (ไม่ล็อกอิน): ปุ่มบันทึก = **ดาวน์โหลด JSON** เท่านั้น *(มีแล้ว)*
- [ ] Tier 1 (editor): + บันทึก **ร่าง / ส่งตรวจ** ในระบบ *(มีแล้ว)*
- [ ] Tier 2 (approver): + **อนุมัติเข้าคลัง / ลบ / ย้อน** *(มีแล้ว)*
- [ ] รวม gating เป็นจุดเดียว: `store.tier` · `store.canStore` · `store.canApprove` (เลิกกระจายเช็ก `session`)
