# US-02 — สิทธิ์อยู่ที่ "การเก็บ" ไม่ใช่ "การแก้"

**Worktree:** WT-0 · **Branch:** `wt0-foundation` · **คู่ DS:** `ds/wt0-foundation/DS-02-store-permission.md`
**สถานะ:** spec (รอ dev)
**โยง mission:** เส้นแบ่งสิทธิ์ = "เก็บเข้าระบบได้ไหม"

## Story
ในฐานะ **เจ้าของระบบ** ฉันต้องการให้ทุกคนแก้เพลงได้ แต่ **เก็บเข้าคลังได้เฉพาะทีม** เพื่อกันคลังหลักไม่ให้ปนเปื้อน

## Acceptance Criteria
- [ ] Tier 0 (ไม่ล็อกอิน): ปุ่มบันทึก = **ดาวน์โหลด JSON** เท่านั้น *(มีแล้ว)*
- [ ] Tier 1 (editor): + บันทึก **ร่าง / ส่งตรวจ** ในระบบ *(มีแล้ว)*
- [ ] Tier 2 (approver): + **อนุมัติเข้าคลัง / ลบ / ย้อน** *(มีแล้ว)*
- [ ] รวม gating เป็นจุดเดียว: `store.tier` · `store.canStore` · `store.canApprove` (เลิกกระจายเช็ก `session`)

## นอกขอบเขต
- รายละเอียด workflow ร่าง→อนุมัติ (WT-D)
