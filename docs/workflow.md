# วิธีทำงาน — เพลง.พระคำ.ชีวิต

เอกสาร "วิธีทำงานรวม" — ใช้กับทุกงาน ทุก worktree ในอนาคต (เขียนครั้งเดียว งานใหม่แค่อ้างอิงที่นี่)

## ใครทำอะไร (บทบาท)
- **พี่เอม (เจ้าของ/ลูกค้า)** — บอกว่าอยากได้อะไร เป็น **ภาษาคน** ไม่ต้องรู้เทคนิค
- **SA (Claude ใน session ที่คุยกับพี่เอม)** — ฟังพี่เอม → **แปลงเป็น spec + คำสั่งเทคนิค** (backlog · US · DS · prompt) ให้ dev
- **Dev (Claude Code session อื่น)** — อ่าน spec/prompt แล้วลงมือเขียนโค้ด

**กติกาภาษา:** คุยกับพี่เอม = ภาษาคนล้วน (ไม่มี git/โค้ดดิบๆ). รายละเอียดเทคนิคไปอยู่ในเอกสาร/prompt ที่ dev อ่าน — พี่เอมไม่ต้องอ่านส่วนนั้น. ถ้าต้องโชว์อะไรเทคนิค ให้อธิบายแนวคิดเป็นภาษาคนก่อน แล้วค่อยใส่บล็อกเทคนิคที่ติดป้าย "สำหรับ dev — พี่เอมข้ามได้"

## ทำงานหลาย session พร้อมกันยังไงไม่ให้ชน (ภาษาคน)
คิดว่าโปรเจกต์ = **สมุดเล่มเดียว** ถ้าหลายคนเขียนลงเล่มเดียวกันพร้อมกัน = เลอะทับกัน
เราเลยให้แต่ละคน **ถ่ายสำเนาสมุดไปไว้ห้องตัวเอง** (โฟลเดอร์แยกบนเครื่อง — git เรียก "worktree") เขียนในสำเนาตัวเองอิสระ ไม่เลอะกัน
แต่ละสำเนา **เปิดเว็บพรีวิวคนละช่อง** (port คนละเลข) → พี่เอมเปิดหลายแท็บ ดูงานทุกคนสดๆ พร้อมกันได้
พอใครเสร็จ ก็ **รวมงานกลับเข้าเล่มหลัก** (merge) — เพราะแต่ละคนเขียนคนละหน้า (คนละไฟล์) การรวมเลยเนียน ไม่ชนกัน

**หัวใจ 3 ข้อ:** 1 งาน = 1 สำเนา (worktree) + 1 ชุดไฟล์ของตัวเอง + 1 ช่องพรีวิว (port) — เจอกันแค่ตอน merge

## วงจรงาน (lifecycle)
```
idea → docs/backlog.md → docs/us/<epic>.md (story+AC) → docs/ds/<epic>.md (design)
     → prompt สั่งงาน → dev สร้างใน worktree → merge กลับฐาน studio-shell-redesign
     → (พี่เอมสั่งค่อยขึ้น main = เว็บจริง)
```

## สำหรับ dev session (พี่เอมข้ามส่วนนี้ได้)

เริ่มงาน 1 worktree (แทน `<ชื่อ>` `<branch>` `<port>` ตามตารางด้านบน):
```sh
git worktree add ../pleng-<ชื่อ> -b <branch> studio-shell-redesign
cd ../pleng-<ชื่อ>
npm install
npm run dev -- --port <port>
```

กติกาขณะทำ (git best practice ให้ขนานไม่ชน):
- แก้ **เฉพาะไฟล์ที่ worktree ตัวเองเป็นเจ้าของ** (ดู DS ของงาน) — ห้ามแตะไฟล์ของ worktree อื่น
- commit ในสาขาตัวเอง · **ห้าม merge `main` · ห้าม deploy**
- ถ้าฐาน (`studio-shell-redesign`) ขยับระหว่างทาง → `git rebase studio-shell-redesign` ในสาขาตัวเอง (ไฟล์ไม่ทับกัน จึงแทบไม่มี conflict)
- เสร็จแล้ว: commit + รายงานว่าแก้ไฟล์ไหนบ้าง → **รอพี่เอมสั่ง merge กลับฐาน** (SA เป็นคนรวม)
- ก่อน commit เช็ก `git branch --show-current` เสมอ (dir หลักใช้ร่วมหลาย session)
