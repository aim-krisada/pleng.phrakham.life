# ใบสั่ง (SA · ออกแบบ) — จัดการทำนอง + จับคู่ทำนอง↔เนื้อร้อง ในโหมดแก้ไข (B083)

## ปัญหาจริง (พิสูจน์แล้ว 3 ชั้น)
1. **เครื่องมือเปลี่ยนทำนองของท่อนพัง:** หัวท่อนมี ComboSelect เลือกทำนอง (`EditorMode.vue:2028`) · เปลี่ยน `lensRow.stanza` A→B ได้จริง **แต่แถบแก้เนื้อผูกกับ "ทำนองที่เปิดอยู่" (`activeStanzaId`)** → พอท่อนไป B ไม่ตรงกับ A ที่เปิด → `lensActive` false (`:242`) → `cshead` (v-if) **หายทั้งแถบ** → พี่เปาเห็นเป็น "กดเปลี่ยนไม่ได้"
2. **ทำนองไม่มีชื่อ/พรีวิว:** เป็นแค่ id `A`/`B` (`stanzaIdOptions :647`) → **แยกไม่ออกว่าทำนองไหนเป็นของท่อนไหน** เวลา import สลับมา คนทำงานไม่รู้ว่าอันไหนถูก · เปลี่ยนชื่อทำนองไม่ได้ (ไม่มี field)
3. **เนื้อ import เป็นก้อนย่อหน้า:** เพลง 2 ท่อน "รับ" มีเนื้อร้องท่อน 1 ทั้งก้อน (`"ชีวิตไม่ยาวนาน,เวลาผ่าน…"`) ยัดใต้โน้ตตัวเดียว **ไม่แยกพยางค์** + ทำนองซ้ำ + ท่อนผิด (รับควรเป็นเนื้อรับ) → จับคู่เละ

## โจทย์ SA (ออกแบบให้ "คนทำงานจับคู่ทำนอง↔เนื้อ ให้ถูก" ได้จริง)
ออกแบบ flow/mockup (ใช้เพลง 2 เป็นตัวเดิน) ครอบ:
1. **เปลี่ยนทำนองของท่อนแล้วแก้ต่อได้** (ไม่หาย) — เปลี่ยนทำนองท่อน ควรพาไปเปิดทำนองนั้นให้แก้ต่อทันที (แนวคิด: `selectStanza` ตาม) · ให้เห็นชัดว่า "ท่อนนี้ = ทำนองไหน + เนื้ออะไร"
2. **ทำนองแยกออก** — มี **ชื่อ (แก้ได้) และ/หรือพรีวิวโน้ต 2-3 ตัวแรก** แทน A/B ล้วน → เลือกจับคู่ได้มั่นใจ
3. **จับคู่/แยกพยางค์เนื้อก้อน import** — เอาเนื้อก้อนย่อหน้ามา **แยกพยางค์วางใต้โน้ตให้ตรง** ได้ง่าย (กึ่งอัตโนมัติ: มีตัวช่วยแยกพยางค์ไทย/เลื่อนจัด) · จัดการเคส "ทำนองซ้ำ/ท่อนสลับ" ได้
4. **มองเห็นภาพรวมการจับคู่** — ท่อน↔ทำนอง↔เนื้อ ทั้งเพลงในจอเดียว เห็นจุดที่ยังไม่จับคู่/ผิด
