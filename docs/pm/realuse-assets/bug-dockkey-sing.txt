ทำไม design perfect แล้ว dev ผิดเยอะ ไม่ได้เอาที่ออกแบบมาใช้เติม function call หรือ ใส่ label place holder ในลักษณะ code ของ library กลางหรอ
ไม่งั้นจะเหนื่อย fine tune design กันไปทำไม
ทุกการทำงานจากนี้อิงมาตรฐาน สากล ขั้นต่ำ wcag 2.2aa และอื่น ๆ ที่เป็น de facto standard
ห้ามตามผมอย่างเดียว ถ้ามีอะไรดีกว่าต้องคิดและแนะนำก่อนทำเสมอ

หน้าฝึกร้อง
- เวลาความยาวเพลงต้องไม่ซ้อนกับปุ่มคีย์
- การทำงานของปุ่มเลือกท่อนยังผิด เช่น กดเลือกแค่ร้องรับ แต่ highlight ใน timeline 2 ท่อนเลย
- ให้ย้าย window ที่เปิดเพิ่มมามาชิดขวาทั้งหมด ท่อน, Aa, เฟือง
- ไม่ต้องแสดง ไม่เลือก = ทั้งเพลง ใน popup ท่อน
- เปลี่ยน เลือกท่อนที่จะซ้อม เป็น เลือกท่อนที่จะฟัง
- timeline ใน row บน ให้มี margin จากขอบตามความเหมาะสม
- ไม่ต้องเขียนคำว่าดาวน์โหลด เหลือแต่ Icon
- ตอนคลิ๊ก download (เปิด download popup) แล้ว label หาย 
- ให้ทุก popup (คีย์, ท่อน, ดาวน์โหลด, ขนาดตัวอักษร) ชิดขวา เริ่มจากขอบบนของ dock key เหมือน setting 
- คีย์ ไม่ต้องมีอักษรชี้ลง เพราะทุกปุ่มกดให้เลือกได้เปิดหน้าต่างใหม่หมดไม่เห็นต้องมีลูกศรชี้ลงเลย
- หน้าต่าง setting และทุกหน้าต่างต้องไม่ scroll vertical/horizontal ตอนนี้ setting มี horizontal scroll จัดให้ fit พอดี
- popup setting ไม่ต้องแสดง ⚙ ตั้งค่า — ปรับได้ที่นี่ · 📌 = ปักขึ้นแถบ 
- margin การจัดแต่ละรายการใน setting pop เพี้ยน แก้ให้ถูก มี margin ที่ดี

ให้ทำสิ่งที่ผมตรวจเป็น check list จะเพิ่ม role tester ให้ตรวจตามเช็คลิสต์หลัง dev ส่งงาน ก่อนให้ผมดูทุกครั้ง
อย่างน้อยสิ่งนี้จะใช้ได้กับทุก dock key และ บางข้อ กับ อื่นๆ 
หลังเสร็จจะให้สร้าง tester แยก ทำ unit test dock key หน้า แผ่นเพลง และ แก้ไข ต่างหาก
