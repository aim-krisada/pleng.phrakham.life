# DockKey — Acceptance Checklist (tester ตรวจก่อน P'Aim ดูทุกครั้ง)

ที่มา: P'Aim ตรวจ DockKey หน้าฝึกร้อง 2026-07-11 (`OneDrive/Screenshots/bug dock key หน้าฝึกร้อง/`).
**วิธีใช้:** dev ส่งงาน → **tester รันเช็กลิสต์นี้ + WCAG** → ผ่านหมดค่อยให้ P'Aim ดู. §A = ทุก DockKey ทุกหน้า · §B = หน้าฝึกร้องเฉพาะ · §C = process.

## §0 หลักการทั่วไป (บังคับทุกงาน ไม่ใช่แค่ dock)
- [ ] ผ่าน **WCAG 2.2 AA เป็นขั้นต่ำ** + de-facto standard ของ UI ระดับสากล
- [ ] **ห้ามตาม P'Aim อย่างเดียว** — ถ้ามีทางที่ดีกว่า/ตรงมาตรฐานกว่า ต้อง **คิด + เสนอก่อนทำ** เสมอ

## §A ENGINE invariants — ทำใน DockKey core ครั้งเดียว ทุกหน้าได้ตาม (นี่คือหัวใจ)
- [ ] **ทุก popup (คีย์ · ท่อน · ดาวน์โหลด · ขนาดตัวอักษร · setting) เปิด "ชิดขวา" เริ่มจากขอบบนของ dock** — ตำแหน่งเดียวกันหมดเหมือน setting ปัจจุบัน (ไม่ใช่โผล่ซ้าย/ลอยคนละที่)
- [ ] **ทุก popup ต้องไม่มี scroll แนวตั้ง/แนวนอน** — จัดให้ fit เนื้อหาพอดี (ตอนนี้ setting มี horizontal scroll = ผิด · dropdown ถูกตัด)
- [ ] **margin/spacing ในทุก popup สม่ำเสมอ** (ไอคอน·ป้าย·ตัวปรับ·▲▼·📌 เรียงมี rhythm เดียวกัน ดูมืออาชีพ)
- [ ] **ปุ่มบนแถบที่กดแล้วเปิดหน้าต่าง = ไม่ต้องมีลูกศรชี้ลง (caret ▾)** — ทุกปุ่มเปิดหน้าต่างอยู่แล้ว (เช่นคีย์ตัด ▾)
- [ ] ปุ่มบนแถบ = ไอคอนกระชับ ไม่ยัดคำยาว (ดู B7)
- [ ] popup ไม่โชว์คำอธิบายวิธีใช้ในหัว (ดู B12)

## §B หน้าฝึกร้อง (จาก P'Aim 11 ก.ค.)
- [ ] B1 · **เวลา/ความยาวเพลง ไม่ซ้อนกับปุ่มคีย์** (timeline row) + timeline มี margin จากขอบ row บนตามเหมาะสม
- [ ] B2 · **bug เลือกท่อน:** กดเลือกเฉพาะ "ร้อง+รับ" แต่ timeline ไป highlight 2 ท่อน = ผิด → selection ต้อง sync กับ timeline ถูกต้อง (ท่อนที่เลือกเท่านั้น)
- [ ] B3 · window ที่เปิดเพิ่ม (ท่อน · Aa · เฟือง) **ย้ายชิดขวาทั้งหมด** (= §A ข้อ popup-ชิดขวา)
- [ ] B4 · popup ท่อน **ตัดข้อความ "ไม่เลือก = ทั้งเพลง" ออก**
- [ ] B5 · เปลี่ยนหัว popup ท่อน **"เลือกท่อนที่จะซ้อม" → "เลือกท่อนที่จะฟัง"**
- [ ] B6 · timeline row บน มี margin จากขอบ (รวมใน B1)
- [ ] B7 · ปุ่มดาวน์โหลด **ตัดคำ "ดาวน์โหลด" เหลือแต่ไอคอน**
- [ ] B8 · **bug:** คลิก download (เปิด popup) แล้ว **label หาย** → แก้
- [ ] B9 · ปุ่มคีย์ **ตัดลูกศรชี้ลง** (= §A caret)
- [ ] B10 · **setting popup ไม่มี horizontal scroll** — จัด dropdown ให้ fit (ตอนนี้ถูกตัด/ล้น) (= §A no-scroll)
- [ ] B11 · setting popup **ตัดข้อความหัว "⚙ ตั้งค่า — ปรับได้ที่นี่ · 📌 = ปักขึ้นแถบ"**
- [ ] B12 · **margin แต่ละรายการใน setting popup เพี้ยน → แก้ให้มี spacing ที่ดี** (= §A spacing)

## §C process (P'Aim 11 ก.ค.)
- [ ] dev ส่งงาน → **tester รันเช็กลิสต์นี้ก่อน** → ผ่านหมดค่อยให้ P'Aim ดู (P'Aim ไม่ต้องเป็น QA)
- [ ] เช็กลิสต์นี้ใช้กับ **ทุก DockKey** (หน้าฝึกร้อง/แผ่นเพลง/แก้ไข) · §0+§A ใช้กับงานอื่นด้วย
- [ ] หลัง DockKey เสร็จ → สร้าง **tester แยก** ทำ **unit test DockKey หน้าแผ่นเพลง + แก้ไข** ต่างหาก
