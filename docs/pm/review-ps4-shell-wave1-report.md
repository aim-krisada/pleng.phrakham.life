## ยืนยัน 4 บั๊กที่ P'Aim เจอ

### B030 — dock ไม่แสดงในโหมดฝึกร้อง → **ไม่ใช่บั๊กคลื่น 1 (เป็นงานคลื่น 2)**
- **อาการ (repro จริง):** โหมดฝึกร้อง `.sd-dock` ที่เจอใน DOM คือของโหมดแก้ไข (`display:none`) — **ไม่มี dock ที่มองเห็น** (`offsetParent=null` ยืนยัน)
- **ควรเป็นตามดีไซน์:** dock ร่วม 3 โหมด — แต่ **registry sing/print + การเมาต์ dock ระดับ Studio = คลื่น 2** (prompt คลื่น 1 กันไว้ชัด · report ข้อ 2)
- **ต้นเหตุ:** `Studio.vue` เมาต์ `StudioDock` **เฉพาะใน `EditorMode`** (view/sheet ใช้ SongViewer/SongSheet ที่ยังไม่มี dock) · แถบคีย์โน้ตเป็น edit-only โดยตั้งใจ
- **ความรุนแรง:** — (ไม่ใช่ defect · **triage: ยกไปคลื่น 2**) · P'Aim เห็น "ไม่มี dock" ถูกแล้ว แต่เป็นเพราะ **ยังไม่ build** ไม่ใช่ของเสีย

### B031 — โหมดแก้ไข แถบบนไม่เหมือนฝึกร้อง/แผ่นเพลง (ไม่มี "เพลง ▾") → **บั๊กจริง (inconsistency)**
- **อาการ (repro จริง):** ฝึกร้อง/แผ่นเพลง มีปุ่มรวม "เพลง ▾" (`.sb-open-btn` = true) · **โหมดแก้ไข ไม่มี** (`.sb-open-btn` = false) → แถบบนคนละหน้าตา
- **ควรเป็นตามดีไซน์:** `ps3-shell` S1 = "แถบบนชุดเดียวทุกโหมด" + S2 "เพลง ▾" รวม (สร้าง+ค้นหา+เปิด) ใช้ทุกโหมด
- **ต้นเหตุ:** `Studio.vue:211` teleport "เพลง ▾" ด้วย `v-if="mode !== 'edit'"` → ในแก้ไข editor teleport เมนู "เพลง"/"จัดการ" ของตัวเองแทน · การรวมให้เป็นอันเดียว = ต้องแตะ teleport ของ `EditorMode` = **คลื่น 2 (editor rebase)** ตามที่ prompt กันไว้ (report ข้อ 1)
- **ความรุนแรง:** **กลาง** — ใช้งานได้ทั้งคู่ แต่ UX ไม่สม่ำเสมอ + ผูกกับ B003 (ตัดเมนูซ้ำ) · **triage: fix ในคลื่น 2** (หรือดึงมาทำก่อนถ้า P'Aim อยากให้สม่ำเสมอทันที — จะชนสาย editor ตอน merge)

### B033 — แถวคีย์โน้ตเลื่อนแนวนอนไม่ได้ (คีย์เล็ก/แออัด) → **บั๊กจริง (usability)**
- **อาการ (repro จริง):** โหมดแก้ไขมีคีย์ **21 ตัว** ยัดบรรทัดเดียว (flex แชร์กัน) → บนเดสก์ท็อป ~26px/ตัว · **มือถือ ~14px เล็กจนกดยาก** · ไม่มี scroll แนวนอนให้เลื่อนหา
- **ควรเป็นตามดีไซน์:** `ps3-dock` D3 = "คีย์ยืด/หดแชร์บรรทัดเดียว ไม่ตกบรรทัด" — **แต่ดีไซน์/prototype คิดบนคีย์ ~11 ตัว** ไม่ใช่ 21 → พอของจริง 21 ตัวเลยแออัดเกิน
- **ต้นเหตุ:** `StudioDock.vue` `.sd-keys{flex-wrap:nowrap} .sd-key{flex:1 1 0}` (บรรทัดเดียว ไม่ scroll) + `EditorMode` ส่ง palette 21 คีย์
- **ความรุนแรง:** **สูง** — ป้อนโน้ตคือหัวใจโหมดแก้ไข ถ้ากดคีย์ไม่โดน = ใช้ไม่ได้จริงบนมือถือ
- **ต้องเคาะดีไซน์ (P'Aim/SA):** ลดชุดคีย์เริ่มต้น · หรือ 2 แถว · หรือ **ยอมให้แถวคีย์ scroll แนวนอนได้** (scroll บน "แถวคีย์บอร์ด" รับได้ ต่างจากกฎ no-swipe ของปุ่มเครื่องมือ) — *ผมไม่เขียน DS ตามกติกา · แจ้ง PM ให้จัด*

### B034 — dock หุบแล้วกดแสดงกลับไม่ได้ (toggle เสีย) → **บั๊กจริง (desktop)**
- **อาการ (repro จริง):** กดปุ่ม "หุบแถบ" → หุบ (collapsed=true) ✓ · **ปุ่มเดิม (panel-top-close) ยังโชว์อยู่ตัวเดียว · กดซ้ำ = หุบซ้ำ ไม่กาง** (ทดสอบแล้ว: คลิกปุ่มเดียวที่เห็น → ยัง collapsed=true) → **dock ค้างหุบ เปิดกลับไม่ได้**
- **ควรเป็นตามดีไซน์:** `ps3-dock` D4 = กดหุบ/กางสลับได้ · desktop คลิกแถบบางกางคืน
- **ต้นเหตุ:** `StudioDock.vue` ปุ่มหุบ (บรรทัด ~292-297) `@click.stop="collapse"` — **เรียก collapse() เสมอ ไม่มี expand** + ไม่มีคลาส `hideoncol` (เลยเป็นปุ่มเดียวที่เหลือตอนหุบ) · ทางกางจริง = `onDockClick` (บรรทัด 157) แต่ `.stop` ของปุ่มบังไว้ + ปุ่มกินพื้นที่แถบเกือบหมด → คลิกโดนแต่ปุ่ม = หุบซ้ำ
- **มือถือ:** ใช้ `.sd-tab` (panel-bottom-open) กางได้ → **บั๊กนี้เฉพาะ desktop**
- **ความรุนแรง:** **สูง** — dock เป็นเครื่องมือหลัก ถ้าหุบแล้วเรียกกลับไม่ได้ = ต้อง refresh
- **แนวแก้ (ให้ dev):** ปุ่มหุบให้ **toggle** (ตอน collapsed = expand) หรือ ตอนหุบซ่อนปุ่มหุบแล้วโชว์ปุ่ม "กาง" แทน
