# Standup — sa-log-system (B028 บันทึกประวัติ / audit log)

สายงาน: sa-log-system   ·   สถานะล่าสุด: รอพี่เอมเคาะ US (ยังเขียน DS ต่อไม่ได้)
อัปเดตล่าสุด: 2026-07-08 โดย session รอบที่ 1

## 1. คุณคือใคร
SA สายใหม่ **"บันทึกประวัติ" (audit log)** — ออกแบบระบบจดว่า *ใครทำอะไรกับเพลงในคลัง*.
ธีมนี้แยกจาก ps3sa (หน้าจอ/เมนู) และ sa-jianpu-rules (กฎโน้ต) — คนละพื้นที่ ไม่ทับกัน.

## 2. ตอนนี้ทำอะไรอยู่
เขียน **US เสร็จแล้ว** · สถานะ = **รอพี่เอม approve**. ยังไม่เริ่ม DS (กติกา SA = US ต้องผ่านก่อน).

## 3. ถึงไหนแล้ว / เหลืออะไร
- ✅ เก็บ requirement จากพี่เอม (จับ 2 มือ: คนทำเพลง + คนอนุมัติเผยแพร่ · เก็บ diff ก่อน→หลัง · ยังไม่ทำปุ่มย้อน)
- ✅ ร่าง US 5 เรื่อง + AC (ภาษาคน) · ลง backlog B028 · commit บนฐานแล้ว
- ⏳ **รอพี่เอมเคาะ US** ← จุดที่ค้างตอนนี้
- ⬜ เขียน DS (ตารางฐานข้อมูล + trigger) — **ทำต่อได้เมื่อ US ผ่าน**
- ⬜ prompt สั่ง dev + build (แผนคือ build ตอน ps4 คู่กับ WT-D รอบ2)

## 4. ไฟล์รายละเอียดอยู่ไหน
- US: `docs/us/audit-log.md` (5 stories + AC · สถานะในไฟล์ = "spec (รอ P'Aim approve)")
- DS: `docs/ds/audit-log.md` — **ยังไม่มีไฟล์** (ยืนยันแล้ว · ตั้งใจ ยังไม่ถึงขั้น)
- backlog: แถว **B028** ใน `docs/backlog.md`

## 5. คุณเป็นเจ้าของไฟล์ไหน
- เจ้าของ: `docs/us/audit-log.md` · (อนาคต) `docs/ds/audit-log.md` · แถว B028 · ไฟล์ standup นี้
- อนาคตตอน build: ตาราง+trigger ฝั่งฐานข้อมูล (Supabase) + component "ประวัติการแก้ไข" (ไฟล์ใหม่)
- branch: `studio-shell-redesign` (ฐาน) · port: ยังไม่มี (ยังไม่ถึง dev)

## 6. ติด/รออะไรอยู่
**Blocker เดียว = รอพี่เอมเคาะ US** — เป็น gate ของ SA (US ผ่าน → ถึงเขียน DS ต่อได้). ไม่ได้รอ session อื่น.

## 7. จุดต่อกับคนอื่น (handshake ตอน build — ยังไม่ถึง)
- **WT-D รอบ 2 (ps4):** เหตุการณ์ "ส่งตรวจ / อนุมัติ+เผยแพร่ / ตีกลับ" เกิดในงาน editor→library (WT-D) → log ต้องเกาะจังหวะนั้น → นัดลำดับตอน build ps4
- **ps3sa:** ปุ่ม/ทางเข้าดู "ประวัติการแก้ไข" ไปวางในหน้าจอ (พื้นที่ shell/editor ของ ps3sa) → เขียนชิ้นให้อิสระ แล้วนัดจุดวางตอน build
- เฟส design ปัจจุบัน **ยังไม่ชนไฟล์กับใคร** (แตะแต่ไฟล์ของตัวเอง + append แถว B028)
