## บั๊ก 1 — issue8: เส้นใต้ (beam) ต้องเดินตามพยางค์
**Source (อ่านก่อนเขียน spec):** `C:\Users\aimkr\OneDrive\4 Personal\pleng.phrakham.life\pleng2-pow-bug-report\issue 8\` (3 png + text.txt)

**กฎที่ P'Aim เคาะแล้ว** (หลักการเดียว: beam grouping = syllable grouping ภายในจังหวะ):
- โน้ตหลายตัว = **คำเดียวลากยาว (เอื้อน = ช่องคำว่าง/ไม่มี lyric)** → **เส้นเดียวต่อเนื่อง**
- โน้ตที่เป็น **คนละคำ (แต่ละตัวมี lyric ของตัวเอง)** → **ขีดแยกกัน (ตัดเส้นระหว่างนั้น)**

3 เคสจากภาพ (ใช้เป็น golden test — ดู DoD):
1. `1. .6 1 2 3` / ต้อง·ไขว่·คว้า·[ว่าง]·เสาะ → `คว้า` ลากถึงช่องว่าง = **เส้นต่อ** (ถูกแล้ว)
2. `1 1_ .6_ 1 2` / ใส่·อา·ภรณ์·งาม·นัก → `อา`+`ภรณ์` คนละคำ = **ต้องตัดเส้นแยก** (ตอนนี้ผิด: ต่อกัน)
3. `5 3 – 3_ 2_` / น-·ภา·[–]·ไม่·เคย → `ไม่`+`เคย` คนละคำ = **ต้องตัดเส้นแยก** (ตอนนี้ผิด: ต่อกัน)

**มาตรฐาน:** ธรรมเนียมโน้ตร้องดั้งเดิม (vocal beaming ตามพยางค์) + jianpu vocal · **ตรงหนังสืออ้างอิงไทย** (เป้าหมายหลัก pleng). *(มาตรฐานฝรั่งสมัยใหม่ Gould "Behind Bars" = beam ตามจังหวะไม่สนพยางค์ — เราไม่ใช้ เพราะอิงหนังสือ · บันทึกเหตุผลนี้ใน spec กัน future "fix" ผิดทาง)*

**สัญญาณจากข้อมูล (zero migration):** โน้ตมี lyric = พยางค์ใหม่ → ตัดเส้นก่อนหน้า · โน้ต lyric ว่าง = เอื้อน → ต่อเส้น. สัญญาณเดียวกับที่ issues2 ใช้ suppress arc → **คำนวณจาก v2 data เดิมได้เลย ไม่ต้องแก้ 120 เพลง**. SA ยืนยันว่า v2 model เก็บ lyric ต่อโน้ตครบ (blank vs มีคำ) จริง — ถ้าไม่ครบ ให้ระบุ gap.
**ต้องระบุใน spec:** renderer ต้อง **ตัด beam กลางจังหวะได้ (partial beam)** ไม่ใช่แค่ทั้งจังหวะต่อ/ไม่ต่อ (เคส เอื้อนแล้วขึ้นคำใหม่ในจังหวะเดียว).

## บั๊ก 2 — issues5: slur เปิด-ปิดคนละห้อง วาดผิด
**Source:** `...\pleng2-pow-bug-report\issues5\` (2 png + new 1.txt)
**อาการ (พี่เปา):** พิมพ์ slur `( )` ถูก แต่ **เปิดท้ายห้องหนึ่ง `(1'` ปิดต้นห้อง/บรรทัดถัดไป `6)`** → หน้าแผ่นเพลงโชว์เครื่องหมายผิด (arc วาดผิด/หาย เมื่อข้ามเส้นห้อง หรือข้ามการตัดบรรทัด).
**ให้ SA:** วิเคราะห์ว่าตอนนี้ arc-render จับคู่ open/close ยังไงเมื่อโน้ตอยู่คนละห้อง/คนละแถวที่ wrap → spec วิธีวาดที่ถูก (แนวปฏิบัติสากล: slur ข้ามห้อง/ข้ามบรรทัด = วาด **2 ส่วน** ปลายเปิดที่ท้ายห้องแรก + ปลายเข้าที่ต้นห้องถัดไป). ระบุพฤติกรรมทั้งข้ามห้องในบรรทัดเดียว **และ** ข้ามการ wrap บรรทัด.
