# Brief — B101: คัดลอก/วาง "บรรทัด" หรือ "ห้อง" ไปวางที่ไหนก็ได้ (ยืดหยุ่น)

**ฐาน:** `studio-shell-redesign` · **worktree/branch ใหม่:** `b101-copy-paste-flexible` (แตกจากฐาน · เช็ก `git branch --show-current` ก่อน commit)
**สั่งโดย:** PM (pm11) · ที่มา: P'Aim 12 ก.ค. · ต่อยอด B098 (คัดลอก/ลบ โน้ต+ห้อง ในที่เดิม ที่ขึ้น live แล้ว)

## ความต้องการ (P'Aim)
อยาก **คัดลอก "ทั้งบรรทัด" หรือ "ทั้งห้อง (bar)"** แล้ว **วางที่ไหนก็ได้อย่างยืดหยุ่น** เช่น วางเข้า **ท่อนใหม่**
พร้อม **"ช่วยคิดคำ/ป้าย (wording) ให้ออกแบบใช้ง่าย"** — P'Aim อยากได้คำที่ผู้ใช้เข้าใจง่าย

## สิ่งที่ต้องได้
- คัดลอก **บรรทัด** และ **ห้อง (bar)** → **วางได้ที่ตำแหน่ง/ท่อนอื่น** (ไม่จำกัดแค่ที่เดิมแบบ B098)
- รองรับ "วางเข้าท่อนใหม่" (paste ข้ามท่อน)
- **wording/ป้ายภาษาไทยชัด ใช้ง่าย** (เช่น "คัดลอก" → ไปวางที่ไหน ผู้ใช้ต้องเข้าใจ) — สายที่รับเลือกคำที่ดี + **อธิบายเหตุผลคำที่เลือกในรายงาน** ให้ PM รีเลย์ P'Aim

## ขอบเขต (สายที่รับ = วิเคราะห์เอง+ออกแบบ interaction+wording+แก้+verify)
- ออกแบบวิธี "วางที่ไหน" ให้ใช้ง่าย (เช่น มี clipboard ในหน่วยความจำ + จุด "วางที่นี่" ตามตำแหน่ง / หรือแนวอื่นที่ดีกว่า) — เลือกให้ KISS
- **ต่อยอดของเดิม B098 (`duplicateSegment`/`duplicateBar` ฯลฯ) ไม่รื้อ** · ถ้า interaction จะซับซ้อนมาก **ping PM ก่อนลงมือ** (เสนอแนว 1-2 บรรทัด)
- ตาม `docs/ui-standards.md` · คงมือถือ (B092) ให้ยังใช้ได้
- pointer: `EditorMode.vue` (segment/bar ops, seg-col, bar tools)

## ⚠️ ชนไฟล์ (PM เฝ้า)
แตะ `EditorMode.vue` เดียวกับ **B100** (เตือนออกหน้า) ที่จ่ายขนาน → แยก branch จากฐาน · region น่าจะคนละที่ · PM cherry-pick ทีละงาน · เจอ region เดียวกัน flag PM

- dev server **`--host`** + **Network URL** ในรายงาน
- **verify เบราว์เซอร์จริง:** คัดลอกบรรทัด → วางในท่อนใหม่ (เนื้อ+โน้ตครบถูก) · คัดลอกห้อง → วางที่อื่น · เช็กมือถือ — แนบภาพ · **สรุป wording ที่เลือก + เหตุผล** ในรายงาน
