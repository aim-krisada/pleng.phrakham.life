### PM คัดกรองคำแนะนำที่ปรึกษา — 4 ข้อที่ต้องรู้ก่อนลงมือ

**(ก) "หน่วงเวลาแบบสุ่ม" = ค้าน · วัดได้ แต่ห้ามสรุปเป็นค่าสุ่ม**
- คนเล่นไม่ได้เหลื่อมแบบสุ่ม เขาเหลื่อม **มีทิศทางและสม่ำเสมอ** → ต้องรายงาน **เครื่องหมาย (นำ/ตาม) + ความสม่ำเสมอ (SD)** ไม่ใช่แค่ค่าเฉลี่ย · ถ้า SD กว้าง = วัดไม่ได้ ให้บอกตรง ๆ
- ⚠️ **กับดักฟิสิกส์:** เชลโลมีหัวเสียงไต่ช้าตามธรรมชาติ (~30–80ms) → onset ที่เครื่องมือจับได้ **ปนฟิสิกส์ของเครื่อง ไม่ใช่เจตนาคนเล่น** · ต้องแยกให้ออก (วัด onset จาก energy rise ไม่ใช่ peak) ไม่งั้นเราจะใส่หน่วงซ้อนหน่วง
- ⚠️ **ข้อบังคับสถาปัตยกรรม:** ระบบเราต้อง **MP3 เท่ากับที่ฟังสดเป๊ะ** → ห้ามสุ่มสด ต้องผ่าน `rng.js` (seeded) เท่านั้น

**(ข) "ถอดสูตรรอยต่อคันชักจากแผ่น" = P3 · เกือบไม่มีประโยชน์**
- sample เราอัดมาแล้ว **หัวโน้ตแก้ไม่ได้** · การเอา gain envelope ไปดัดหัวโน้ต = **สิ่งที่ `celloEngine.js` ทำแล้วได้ "หวูดเรือ"** (memory `pleng-cello-piano-spike`) — อย่าทำซ้ำ
- ทางแก้จริงอยู่ที่ **legato maps ของ Karoryfer เอง** (`Programs/vc_arco_sus_legato_map.sfz` ฯลฯ = ไม่ตีหัวโน้ตใหม่) → เป็นงานของสาย `cello-bakeoff` ไม่ใช่สายนี้
- สิ่งเดียวที่คุ้มวัด = **คำถาม yes/no:** "ตอนเปลี่ยนโน้ต ความดังตกลงไหม ตกกี่ dB" (ถ้าคนจริงแทบไม่ตก = ยืนยันว่าเราต้องใช้ legato map + โน้ตซ้อนเหลื่อม ไม่ใช่ตีหัวใหม่ทุกตัว) · แค่นี้พอ

**(ค) "เอาผลไปทำ if-else" = ค้านหนัก**
- Canon spike สรุปเองแล้วว่า **static conditional logic = หุ่นยนต์ผลัดกันอ่านบท** = สาเหตุที่เชลโลพัง · เอาตัวเลขดี ๆ ใส่โครงหุ่นยนต์ = ได้หุ่นยนต์ที่ตัวเลขสวยขึ้น
- **ค่าคงที่ (constants) = เอา ✅ · โครงต้องเป็น "แผนต่อวรรค" (รู้ทั้งวรรคล่วงหน้าว่าใครนำใครตาม) ไม่ใช่ if-else ตัดสินทีละโน้ตแบบไม่รู้อนาคต ❌**
- → รายงานต้องเสนอในรูป **phrase-level role plan** (ต่อวรรค/ต่อท่อน: ใครนำ · อีกคนหลบยังไง · หนาแน่นแค่ไหน)

**(ง) n = 2 · ห้ามเฉลี่ยรวมกัน**
- 2 เพลงนี้ **สอนคนละเรื่อง**: Canon = เชลโลเดินเบสตลอด (ground bass) · A Thousand Years = เชลโลร้องนำ (ballad) → **รายงานแยกทีละเพลง** แล้วค่อยบอกว่าอะไร "ตรงกันทั้งคู่" (= น่าเชื่อ) กับอะไร "ต่างกัน" (= ขึ้นกับบทเพลง ต้องเลือกตามท่อน)
- ห้ามเฉลี่ยเป็นตารางค่าเดียว → จะได้ค่ากลางที่ไม่เหมือนสักเพลง

**เครื่องมือ:** `py -3.14` (มี ffmpeg แล้ว · **librosa/demucs ยังไม่ได้ติดตั้ง** → `pip install librosa` ก่อน) · แยกเสียงเชลโลออกจากเปียโน = `demucs` (ถ้าติดตั้งไหว ทำให้วัดแม่นขึ้นมาก · ถ้าหนักไป → รายงานว่าทำไม่ได้ อย่าเดา)
⚠️ **stem ที่แยกออกมามี artifact เสมอ** (เปียโนรั่วเข้าแทร็กเชลโล) → เลข dB/onset เป็น **ค่าประมาณ** ต้องเขียนกำกับไว้ อย่ารายงานทศนิยม 2 ตำแหน่งเหมือนเป็นความจริงสัมบูรณ์
