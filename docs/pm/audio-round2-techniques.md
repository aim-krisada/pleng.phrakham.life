## ✅ ส่งเข้า bake-off แล้ว (16 ก.ค.)

### Negative Delay (เลื่อนเชลโลให้เริ่มก่อน ชดเชยหัวเสียงไต่ช้า)

**ทำไมถึงเป็นข้อยกเว้น:** ไม่ใช่ "การจูนรสนิยม" แต่เป็น **การอุดข้อบกพร่องเชิงระบบ** — ถ้าไม่ใส่ เชลโลจะตามหลังเปียโน **ทั้ง 3 คลิปเท่ากัน** → P'Aim อาจเทหมดทั้ง 3 ตัวด้วยเหตุผลผิด (นึกว่า sample ห่วย ทั้งที่คือเราตั้งเวลาผิด) = **false negative ที่ทำให้เราปิดโปรเจกต์เชลโลผิด ๆ**

**ทำไมมันปลอดภัยกับการทดลอง:** วัดได้ ไม่ใช่รสนิยม (เปิดไฟล์ดู time-to-peak) · deterministic (ไม่สุ่ม → MP3 == live) · กระทบ 3 คลิปเท่ากัน → ไม่บิดผลเทียบ

**เข้ากับสถาปัตยกรรมเราพอดี:** วาทยกร `arrange()` คำนวณโน้ตทั้งเพลงล่วงหน้าก่อนเล่น (ไม่ใช่ระบบรับ input สด) → "แอบอ่านโน้ตล่วงหน้าแล้วขยับเวลา" = ทำได้เลย ไม่ต้องรื้อ

**วิธีที่ถูก:** วัด time-to-peak ของ sample แต่ละชุดจริง (energy rise ไม่ใช่ peak) แล้วเลื่อน onset ล่วงหน้าเท่าค่านั้น **ต่อ sample-set** (Karoryfer/SSO/Iowa หัวเสียงไม่เท่ากัน → ค่าไม่เท่ากัน) · **ห้ามยกค่ามาจากคำแนะนำ ("30–80ms") มาใส่ตรง ๆ — นั่นคือช่วงทั่วไปของเชลโลจริง ไม่ใช่ของ sample เรา**

---

## ⛔ ที่ปรึกษาขัดแย้งกันเอง — ตัดสินแล้ว

**"Attack Time Breathing" (ให้เชลโลไต่ช้ากว่าเปียโนในโน้ตแรกของวรรค)** = **ทิ้ง**
ขัดกับ "Negative Delay" ในเอกสารฉบับเดียวกันแบบตรงข้าม 100% (ข้อหนึ่งบอกเลื่อนเชลโล**ช้าลง** อีกข้อบอกเลื่อน**เร็วขึ้น**)
เชลโลไต่ช้าตามธรรมชาติอยู่แล้ว → หน่วงเพิ่ม = **ช้าซ้อนช้า** = ฟังเหมือนวงเล่นไม่พร้อมกัน · **Negative Delay ชนะ**

## 🅿️ พักไว้ — พิจารณาหลัง P'Aim เลือก sample

### 1. Stereo panning (เชลโลเอียงขวา)
- ✅ ถูกหลัก (จัดเวทีเสียงดูโอ = practice มาตรฐาน) · ถูก · เสี่ยงต่ำ
- 🔄 **แก้จากที่ปรึกษา:** เขาสั่ง "เปียโนซ้าย 15% + เชลโลขวา 20%" → **อย่าขยับเปียโน** · Splendid Grand อัดมาเป็นสเตอริโอมี image ของตัวเองแล้ว การดันซ้ายทั้งก้อน = ทำ image เบี้ยว · **ขยับเฉพาะเชลโล (mono source) ไปขวาเล็กน้อย** พอ
- ⚠️ **ผู้ใช้ส่วนใหญ่ฟังมือถือลำโพงเดียว = ไม่ได้ยินความต่าง** → คุณค่าจำกัดเฉพาะคนใส่หูฟัง → priority ต่ำ
- ตัวเลข 15/20% = ไม่มีที่มา อย่ายกมาใช้ตรง ๆ

### 2. Dynamic layer crossfading (ผสมข้าม p/mp/mf/f)
- ✅ ไอเดียถูก (โน้ตกระโดดข้ามชั้นทื่อ ๆ = ฟังออก)
- ⚠️ **ต้องผ่านทางอ้อมที่เรามีเอกสารแล้ว** — smplr scale ความดัง **สัมพัทธ์ภายในแต่ละ velRange group** → preset รวมหลายชั้นจะ **รีเซ็ตความดังที่รอยต่อ** (วัดแล้วบน Iowa: vel80 rms .0405 → vel94 rms .0209 = ดังกลับทาง) → ต้องโหลด **แยกเป็น instrument ชั้นละตัว** แล้ว blend ด้วย gain (memory `pleng-smplr-vellayer-relative`)
- ⚠️ **ความเสี่ยงที่ที่ปรึกษาไม่ได้บอก:** เปิด 2 ชั้นพร้อมกัน = **2 take สีพร้อมกัน = ฟังเป็นเชลโล 2 ตัว** (คลังเสียงโปรอัด layer มาเพื่อ crossfade โดยเฉพาะ · ของฟรีไม่ได้อัดมาแบบนั้น) + **กินแรงมือถือ 2 เท่า**
- → ลองได้ แต่ต้อง **A/B ให้ P'Aim ฟันธง** ว่าดีขึ้นจริงหรือกลายเป็น 2 ตัว

### 3. Sustain-tail bleeding (ยืดหางเสียงโน้ตท้ายวรรค +100–150ms)
- ⚠️ **เรามีอยู่แล้วเป็นส่วนใหญ่** — เปียโนกับเชลโลวิ่งผ่าน **reverb bus เดียวกัน** อยู่แล้ว = "ละลายในห้องเดียวกัน" เกิดขึ้นแล้วโดยธรรมชาติ (`golden-piano` route)
- ❌ **คำอธิบายของที่ปรึกษาผิด:** ยืด release ≠ sympathetic resonance (คนละกลไก · การยืด fade ไม่ได้ทำให้สายเชลโลสั่นตาม)
- เลข 100–150ms ไม่มีที่มา · ผลที่ได้จริงน่าจะเล็กมาก และเสี่ยง "อื้อ"
- → priority ต่ำสุด · ทำเมื่อว่างจริง ๆ และต้อง A/B พิสูจน์ว่าต่าง
