### ✅ P'Aim เคาะ: **68 ไฟล์** (ไม่ใช่ 136)
- **4 ระดับ (`p`/`mp`/`mf`/`f`) × 17 pitch = 68** ← ส่วนที่ทำให้ "ไม่แสบ"
- **คันชัก `_d`/`_g` (อีก 2 เท่า) = ยังไม่เอา** — ได้ realism นิดเดียว แต่ไฟล์โต 2 เท่า
- ⚠️ **ขนาด = ข้อจำกัดจริง** (PWA ออฟไลน์ · พี่น้องโหลดผ่านเน็ตมือถือ · memory `pleng-pwa-self-host-samples`) · ตอนนี้ 17 ไฟล์ → 68 = **โต 4 เท่า** · **วัดขนาดจริง (MB) รายงานด้วย** ให้ P'Aim ตัดสินว่าคุ้มไหม

### ⚠️ สูตรจากแผ่น **ไม่ได้บอกตรง ๆ ว่า "ใช้ชั้นไหน"**
```
แผ่นจริง → เส้นความเข้มต่อวรรค → (เราแปลงเอง) → เลือกชั้นเสียง
```

2. **เติมหัวโน้ตสั้นเฉพาะโน้ตที่ต้องเน้น** ตามดีไซน์ marcato (staccato ทับ · hold ~200ms · ปล่อยให้ `p` พาต่อ)
3. **⭐ ทำเป็นปุ่มให้ P'Aim หมุนเอง** — **"ความแรงหัวโน้ต"** (เหมือน cc110) + **"เลื่อนเวลา `p`" (0 → 200ms)** · **นี่คือหัวใจของงาน:** "แรงแค่ไหนถึงพอดี" **SA ฟังไม่ออก → ให้ P'Aim หมุนจนพอดี แล้วค่อยจำค่านั้น** (แทนการวนเดา)
4. **วัดขนาดไฟล์ที่เพิ่ม** (staccato = โฟลเดอร์ใหม่) รายงาน

### 🔬 สมมติฐาน PM (เดา · ทดสอบ · **ค้านได้** — PM เพิ่งเดาผิดไปแล้ว 1 รอบ)
**หัวโน้ตคมเคาะตรงบีต → หูยึดหัวโน้ตเป็นตัวบอกจังหวะ → `p` บวมช้าข้างล่างได้โดยไม่รู้สึกช้า → ไม่ต้องเลื่อน `p` ล่วงหน้า = อาการ "เร่ง" หายเอง**
→ ถ้าจริง **ไอเดียเดียวของ P'Aim แก้ 3 เรื่อง**: ขาดมิติ · แสบ · เร่งจังหวะ
→ **นี่คือเหตุผลที่ปุ่ม "เลื่อนเวลา p" ต้องหมุนได้** — ให้หู P'Aim หาจุดพอดี ไม่ใช่ PM กำหนด

### ⚠️ กับดักที่ต้องเลี่ยง (PM เตือนไว้ · ยังใช้อยู่)
**อย่าทำ "ดัง = ใช้ชั้นแรง" แบบตรงไปตรงมา** — โน้ตที่ P'Aim ว่า "หนัก" คือโน้ตที่ระบบสั่งให้ดัง → ถ้าแมปตรง ๆ มันจะไปหยิบ `f` (สว่างสุด = แสบสุด) = **ทำให้โน้ตที่ P'Aim ไม่ชอบ แย่ลง** · **หัวโน้ตแยกจากตัวโน้ต = ทางที่เลี่ยงกับดักนี้พอดี**

### ยังคงเดิมทุกข้อ
⛔ ไม่แตะ `dynamics.js` · ไม่แตะโค้ด deploy · ไม่ commit ไฟล์เสียง · deterministic (MP3==live) · `--host` + Network URL จริง · **ห้ามตัดสินเองว่าเพราะขึ้น** · round-robin `_d`/`_g` = **ยังพัก** (ทำหลังจากหัวโน้ตลงตัว — อย่าเพิ่มตัวแปรพร้อมกัน)

## 🏆 ขั้น 3 — **ล็อกค่าที่ P'Aim หาเจอ** + **นับโน้ตซ้ำ** (P'Aim "ทำทั้ง 2 เลย" · 16 ก.ค. ค่ำ)

### 3.1 ⭐ ล็อกค่าเป็นค่าตั้งต้นถาวร (ทำก่อน · ห้ามพัง)

**P'Aim หมุนปุ่มเองจนพอใจ = ครั้งแรกที่เชลโลผ่านหูเขา หลังพักมา 2 รอบ (ไวโอลิน + Canon spike):**

| ค่า | P'Aim ตั้ง |
|---|---|
| **ความแรงหัวโน้ต** | **5%** |
| **เลื่อนเวลาตัวโน้ต** | **10 ms** |
| **ตัวลาก** | **`p`** |
| **หัวโน้ต** | **`mp`** (P'Aim: "mp กับ mf พอได้ทั้งคู่ · **mp นุ่มกว่าหน่อย**") |

**ทำ:** เขียนค่าชุดนี้เป็น **default ของหน้า spike** (เปิดมาได้ค่านี้เลย) + **บันทึกในรายงานว่านี่คือค่าที่ P'Aim เคาะ** → **ทุกการทดลองต่อจากนี้ ถอยกลับมาจุดนี้ได้เสมอ ห้ามหาย**
**อย่าล็อกจนหมุนไม่ได้** — ปุ่มยังต้องหมุนได้ (แค่เปลี่ยนค่าเริ่มต้น)

**⚠️ ยังไม่ใช่การเอาเข้าแอพจริง** — `soundOptions.js` เชลโลยัง `disabled:true` · ไม่แตะ · นี่คือ checkpoint ของ spike เท่านั้น

### 3.2 นับโน้ตซ้ำ — **ตอบว่าคันชักคู่คุ้มไหม (นับก่อน อย่าเพิ่งสร้าง)**

**P'Aim ถาม: "คันชักขึ้นลงเพิ่มมิติไหม?"** · **PM ตอบ: ไม่ได้ "เพิ่ม" อะไร — มันไป "ลบ" ความเป็นหุ่นยนต์** (เสียงเหมือนเดิมเป๊ะ แต่โน้ตซ้ำจะไม่เหมือนกัน 100% แบบที่คนสีจริงเป็น)
**คนอัดใช้เป็น round robin ล้วน** — verified: `vc_arco_sus_map.sfz` `seq_length=2` + `seq_position=2` × 68 = **สลับ `_d`/`_g` อัตโนมัติ ไม่ได้เลือกตามจังหวะ**

**❓ คำถามที่ต้องนับ (ในเพลงจริงที่ใช้ในคลิป · ไม่ใช่ทฤษฎี):**
1. **ไฟล์เดียวกันถูกใช้ซ้ำติด ๆ กันบ่อยแค่ไหน** (โน้ตติดกัน / ภายใน ~2 วินาที)
2. ⭐ **ระวังจุดที่ PM สงสัย:** Karoryfer มีแค่ **17 pitch** ยืดครอบทั้งช่วง → **โน้ตทำนองที่ *ต่างกัน* 2-3 ตัวติดกัน อาจมาจากไฟล์เดียวกัน** (ยืดคนละระดับ) → **เป็นหุ่นยนต์โดยไม่รู้ตัว แม้ทำนองไม่ซ้ำโน้ต** → **ต้องนับที่ "ไฟล์ต้นทาง" ไม่ใช่ที่ "โน้ต"**
3. เพลงไทยนมัสการ = 1 พยางค์/โน้ต → **น่าจะซ้ำโน้ตเยอะ** (สมมติฐาน · ให้ตัวเลขตัดสิน)

**ถ้าบ่อย → คุ้ม · ถ้าไม่บ่อย → ข้าม อย่าเสียพื้นที่ฟรี** · **รายงานตัวเลข ให้ PM/P'Aim เคาะ ยังไม่ต้องสร้าง**

### 💰 80:20 — PM ลดราคาลงจาก 1.17 MB → **~0.3 MB**

dev ประเมิน RR = +1.17 MB **แต่นั่นคือทุกชั้น** · เราใช้จริงแค่ 2:

| ชั้น | ใช้จริง | ต้อง RR ไหม |
|---|---|---|
| **`p` (ตัวลาก)** | ✅ ได้ยินเต็ม ๆ | **✅ เอา — ของจริงอยู่ตรงนี้** |
| `mp` (หัวโน้ต) | ✅ **แต่แค่ 5%** | **❌ ไม่ต้อง** — 5% แทบไม่ได้ยิน จะแยกความต่างไปทำไม |
| `mf` / `f` | ❌ ไม่ได้ใช้ | ❌ ไม่ต้อง |

→ **`_g` ของ `p` อย่างเดียว = 17 ไฟล์ ≈ 0.3 MB** = ได้ผลเกือบเท่ากัน จ่าย 1/4 · **วัดขนาดจริงยืนยันด้วย**

### 🅿️ ของที่ "ดีกว่า" แต่ **ไม่ใช่ 80:20 → ไม่ทำตอนนี้**
**ใช้คันชักเชิงดนตรี** (ลง=จังหวะหนัก · ขึ้น=จังหวะเบา) = มิติจริง **แต่คนอัดเองก็ไม่ทำ (RR ล้วน)** และเราต้องรู้ก่อนว่าจังหวะไหนหนัก = **ต้องมีแผนวรรค = งานใหญ่** → เก็บไว้
