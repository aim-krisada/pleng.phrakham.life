## 🔴 ทำ **ขั้นที่ 1 ให้จบก่อน** แล้วหยุด รอ P'Aim ฟัง — ห้ามทำ 2-4 พร้อมกัน

**เหตุผล:** ถ้าใส่ 4 อย่างรวดเดียว P'Aim แยกไม่ออกว่าอะไรช่วย อะไรทำพัง = วนไม่จบ (นี่คือกับดักที่ทำไวโอลิน+เชลโลพักไปแล้ว **2 รอบ** — memory `pleng-aesthetic-audio-needs-ear`)

### ⭐ ขั้นที่ 1 — ใช้ชั้นเสียงเบาที่ **มีอยู่แล้วแต่ไม่เคยโหลดมา** (ทำก่อน · อาจจบตั้งแต่ตรงนี้)

**หลักฐาน:** `Samples/sus` ของ Karoryfer มี **136 ไฟล์** = 17 pitch × **4 ระดับ `p`/`mp`/`mf`/`f`** × **คันชัก `_d`/`_g`**
แต่ `tools/prepare-samples-cc0.mjs:33` โหลดมาแค่ **`*_mf_d.wav` = 1 ใน 8** (คอมเมนต์ในโค้ดเขียนเองว่า `one dyn/take`)

→ **ทุกโน้ตในเพลงถูกสีด้วยน้ำหนักกลางค่อนข้างหนักเท่ากันหมด = สมมติฐานหลักของ "แสบแก้วหู"**

**ทำไมนี่คือ 20/80 ตัวจริง:** เชลโลจริงสีเบา **เนื้อเสียงเปลี่ยนเป็นคนละเสียง** (นุ่ม ทึบ ไม่มีเสียงคันชักครูด) — ไม่ใช่แค่ค่อยลง · **การเอา `mf` มาหรี่ให้เบา = ได้ "เสียงสีแรงที่เบา" ซึ่งยังแสบเหมือนเดิม** · ชั้น `p`/`mp` = **ความนุ่มของจริงที่นักเชลโลสีมา** ไม่ใช่ของปลอมที่ปั้นด้วย DSP

**ส่งมอบขั้น 1:** คลิปเดิม (เพลง #1 ท่อนแรก · เปียโนทองเดิม · เชลโลร้องนำ) แต่ให้ P'Aim **สลับฟังชั้นเสียงได้**:
`p` · `mp` · `mf` (= ของเดิมที่แสบ) — **ปุ่มเดียวกันแบบ bake-off** · + MP3

**กติกาเทคนิค:**
- ⚠️ **smplr velRange gotcha** — โหลด **แยกเป็น instrument ชั้นละตัว** (velRange เต็ม) **ห้าม**ยัดเป็น velRange groups ในตัวเดียว (ความดังจะรีเซ็ตที่รอยต่อ · memory `pleng-smplr-vellayer-relative`)
- ⚠️ **negative delay ต้องวัด t50 ใหม่ "ต่อชั้น"** — ชั้น `p` คันชักบวมช้ากว่า `f` แน่นอน → ค่าไม่เท่ากัน · ใช้เครื่องมือเดิม `tools/prepare-cello-bakeoff.py` (เฟรม 5ms) · เขียนลง `preset.json → plengMeta.attackMs` เหมือนเดิม
- ⚠️ **ความยุติธรรม** — match ความดัง **จากท่อนที่เล่นจริง** ไม่ใช่ค่าเฉลี่ยทั้งไลบรารี (บทเรียนรอบก่อน: Karoryfer เสียงสูงเบากว่าต่ำ 14.7 dB → เกือบทำ P'Aim เลือกผิดตัว)
- **คำถามที่ต้องตอบในรายงาน:** ชั้นเบา **แลกอะไรไป** — เบาไปจนจมเปียโนไหม? เสียงลมคันชัก (noise floor) โผล่ไหม? → **รายงานตรง ๆ อย่าปิด**
- **ยังใช้ `Samples/sus` ตรง ๆ เหมือนเดิม** (ยังไม่ใช้ legato maps — คนละงาน)

## 🅿️ ขั้น 2-4 — **ยังห้ามทำ** รอ P'Aim ฟังขั้น 1 ก่อน (เขียนไว้ให้เห็นภาพเท่านั้น)

| ขั้น | ทำอะไร | ทำไม |
|---|---|---|
| **2** | **ขยับเชลโลให้ห่างออก (ห้องเสียง/early reflections)** | Karoryfer **close-miked unprocessed** (readme เขียนเอง) = ได้ยินคันชักครูดจ่อหู · **แผ่นจริงไมค์ห่าง 2-3 เมตร** = ระยะห่างคือตัวทำให้นุ่มที่สุด · เรามี reverb bus อยู่แล้ว |
| **3** | **EQ กดย่านแสบ — วัดเป้าจากแผ่นจริง** | **ไอเดีย P'Aim: "อ้างอิงความนุ่มจาก mp3 ที่เรามี"** = เปลี่ยนจาก "จูนให้เพราะ" (SA ทำไม่ได้) → "ทำตัวเลขให้ตรงเป้า" (SA ทำได้แม่น) |
| **4** | **สีเบา = เสียงทึบลง (brightness ตาม dynamics)** | เลียนธรรมชาติเชลโล · เสริมชั้นเสียงให้เนียนขึ้น |
| — | vibrato · legato maps · pan · crossfade | **P'Aim บอก vibrato ไม่จำเป็น** → พักทั้งหมด |

**ขั้น 3 วิธีที่ถูก (ตอนถึงคิว):** วัดสเปกตรัมเชลโลในแผ่นจริง (stem จากสายวิเคราะห์ · `claude/brave-grothendieck-977c83` มีเครื่องมือ demucs แล้ว) → วัดของเรา → **ส่วนต่าง = เส้น EQ** · ⚠️ **ใช้แค่ ~50-70% ของส่วนต่าง อย่า match เป๊ะ** (stem มี artifact + แผ่นผ่าน mastering มาแล้ว จะ bake ของแปลกปลอมเข้ามา)

## กติกาเหล็ก (ทั้ง 4 ขั้น)

1. **ทีละขั้น · ให้ P'Aim ฟังทีละขั้น · หยุดรอทุกครั้ง** — ห้ามรวบ
2. **ทุกค่ามาจากการวัด ห้ามหมุนเอาเอง** — วัดไม่ได้ → **ถามหู P'Aim อย่าเดา** · **SA ฟัง mix ไม่ออก** พิสูจน์ได้แค่ "มีเสียง/ไม่คลิป/ตรงจูน/ดังเท่ากัน" ไม่ใช่ "เพราะ"
3. **deterministic เสมอ** — MP3 ต้อง == live · สุ่มต้องผ่าน `rng.js` (seeded) เท่านั้น
4. **honest to the sheet** — เล่นโน้ตตามแผ่น · ต้องย้าย octave → **รายงาน อย่าเงียบ ๆ ทำ** (memory `feedback-audio-honest-to-sheet`)
5. **ห้ามแตะโค้ดที่ deploy แล้ว** (`midi.js` / `audioExport.js` / SongView / arranger) — spike แยกไฟล์ · **bug preset ต่ำ 1 octave = ยังไม่แก้** (นอก scope · cello `disabled:true` อยู่ = ไม่กระทบผู้ใช้)
6. **ห้าม commit ไฟล์เสียง** (.wav/.ogg/.mp3) — `.gitignore` มีแล้ว · Karoryfer = CC0 ปลอดภัย แต่ repo ต้องสะอาด
7. **`--host` + ใส่ Network URL จริงในรายงาน** (P'Aim ฟังมือถือ · IP เปลี่ยนบ่อย เช็ก vite Network line)
