ปัญหาจริงในเพลง 2 (import จับสลับ)

import อัตโนมัติจับทำนอง↔เนื้อผิด + ยัดเนื้อทั้งท่อนเป็นก้อนเดียว → คนทำงาน “แก้ไม่ถูก”. ตัวเลขจริง:

ท่อน (ชื่อจาก import)ทำนองที่ถูกจับให้ทำนองรับได้เนื้อที่มีผล
ร้อง1ทำนอง B (สั้น 1 บรรทัด)7 พยางค์23 พยางค์✗ ล้นทำนอง (ควรเป็น A)
รับทำนอง A (ยาว 4 บรรทัด)91 พยางค์1 ช่อง = ก้อนเดียว✗ ยังไม่แยกพยางค์
ร้อง2ทำนอง A91 พยางค์21 พยางค์✗ ไม่พอดี

“รับ” เนื้อจริงในฐาน = "ชีวิตไม่ยาวนาน,เวลาผ่านเร็วไว,ถ้าท่านไม่ฉวยเอาไว้,…" ยัดใต้โน้ตตัวเดียว

1เปลี่ยนทำนองของท่อน แล้ว “แก้ต่อได้” (ไม่หายทั้งแถบ)

ของเดิม: หัวท่อนมีตัวเลือกทำนองอยู่แล้ว (EditorMode.vue:2028) แต่แถบแก้เนื้อผูกกับ “ทำนองที่เปิดอยู่” (lensActive :242) → พอเปลี่ยน A→B แล้วไม่ตรง → หัวท่อน+ช่องเนื้อหายหมด = พี่เปาเห็น “กดไม่ได้”

ของเดิม — เปลี่ยน ร้อง1 เป็นทำนอง A

บั๊ก: lensRow.stanza เปลี่ยนเป็น A แต่ activeStanzaId ยังเป็น B → lensActive=false → v-if ซ่อนทั้ง cshead

ของใหม่ — เปลี่ยนแล้วพาไปเปิดทำนองนั้นให้เลย

เสริม: ตอนเปลี่ยนทำนองที่หัวท่อน ให้เรียก selectStanza(ทำนองใหม่) ต่อทันที → activeStanza ตาม → แถบไม่หาย แก้ต่อได้เลย (แก้จุดเดียวใน handler บรรทัด 2034)

👉 กดปุ่มเลือกทำนองในทั้งสองกล่องเพื่อเทียบ (ซ้าย = หาย · ขวา = อยู่ต่อ)

2ทำนองมี “พรีวิว” (แยกออกว่าอันไหนคืออันไหน)

ของเดิม: ตัวเลือกทำนองโชว์แค่ ทำนอง A / ทำนอง B (stanzaIdOptions :647) → import สลับมาก็เดาไม่ออกว่าอันไหนถูก

ของเดิม

ทำนอง A
ทำนอง B

แยกไม่ออก · ทั้งคู่หน้าตาเหมือนกัน

ของใหม่ — พรีวิวโน้ต + จำนวนบรรทัด/พยางค์ (คำนวณจากทำนอง ไม่เพิ่ม field)

ทำนอง A 5 1-0̲2 1̲ 7…4 บรรทัด · รับ 91 พยางค์
ทำนอง B (1̲2̲3̲) 5̲ 3.…1 บรรทัด · รับ 7 พยางค์

เสริม: เติมพรีวิวใน stanzaIdOptions (โน้ต 5–6 ตัวแรก + จำนวนบรรทัด/ความจุพยางค์) — derived ล้วน. ชื่อทำนองแก้ได้ (เช่น “ท่อนร้อง”/“ท่อนรับ”) = ต้องเพิ่ม 1 field stanzas[].name → flag PM (คำถาม 1)

3แยกพยางค์เนื้อ “ก้อน import” วางใต้โน้ตให้ตรง

ของเดิม: มีแผง “📝 แก้เนื้อแบบย่อหน้า” + ช่องคำใต้โน้ต + ปุ่มเลื่อน ◀▶ อยู่แล้ว — แต่เนื้อ import เป็นก้อนเดียวใต้โน้ตตัวแรก ต้องแยกมือทั้งหมด

ของใหม่ — ปุ่ม “✂ แยกพยางค์อัตโนมัติ” ในแผงแก้เนื้อ (ลองจริง)

เนื้อก้อนของ “รับ” (จากฐานจริง) — กดปุ่มแล้วดูมันแตกเป็นช่องพยางค์ใต้โน้ต:

เสริม: ปุ่มเรียก Intl.Segmenter('th',{granularity:'word'}) (มากับเบราว์เซอร์ · ไม่ต้องโหลด dictionary) ตัด “,” เป็นวรรค → เติมช่องพยางค์เดิม แล้วปรับละเอียดด้วย ◀▶ / เว้นวรรค ที่มีอยู่. ช่องแดง = ยังไม่พอดีโน้ต (ใช้ rowStatus เดิม). เอื้อน/ตัวลาก = ไม่รับพยางค์ (v2 · ข้ามช่องอัตโนมัติ)

4ภาพรวมการจับคู่ทั้งเพลง (เห็นจุดผิดในแวบเดียว)

ของเดิม: rail “โครงเพลง” ลิสต์ท่อน+ชิป ♪ อยู่แล้ว — เสริมป้ายเตือน เมื่อเนื้อไม่พอดีทำนอง (ใช้ rowStatus เดิมที่คำนวณ got/need อยู่แล้ว) → เห็น import สลับทันที

คำถามถึง P'Aim (≤3)

  1. ชื่อทำนองแก้ได้ (เช่น “ท่อนร้อง” / “ท่อนรับ”) มีประโยชน์เวลาจับคู่ — แต่ต้องเพิ่ม 1 field stanzas[].name ในโมเดล. เอาไหม หรือ เริ่มด้วยพรีวิวโน้ตอย่างเดียวก่อน (ไม่แตะโมเดล · แนะนำ)?
  2. แยกพยางค์อัตโนมัติ ใช้ตัวตัดคำไทยของเบราว์เซอร์ (Intl.Segmenter) = ตัดเป็น “คำ” ไม่เป๊ะทุกพยางค์ 100% แต่ใกล้ + ปรับต่อด้วย ◀▶ ที่มีอยู่. โอเคไหม (เร็ว·ไม่ต้องโหลด dictionary) หรืออยากได้ตัดพยางค์เป๊ะกว่า (ต้องเพิ่ม lib)?
  3. ตอนเปลี่ยนทำนองที่ทำให้เนื้อไม่พอดี (เช่น 23 พยางค์ไปทำนองรับ 7) — ให้ เตือนแต่ยอมให้เปลี่ยน (แก้ต่อได้) ใช่ไหม (ไม่บล็อก)?
