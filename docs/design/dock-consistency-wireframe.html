2 · ที่เสนอ — ไวยากรณ์เดียว ("dock grammar")

โครงเดียวกันทุกโหมด · เส้นประ = ขอบซ้ายที่ grip กับ ⚙ ตรงกัน (ตามที่ P'Aim ขอ)

R1 · grip = มุมบนซ้ายเสมอ ทุกโหมด
R2 · ⚙ ตั้งค่า = ล่างซ้าย ใต้ grip พอดี (ขอบซ้ายตรงกัน) · เป็น "บ้านเดียว" ของปุ่มรอง — ความโปร่ง/จัดปุ่ม/ดูเพิ่ม ยุบเข้ามาใน ⚙ ทุกโหมด
R3 · timeline = แถวของตัวเอง เต็มความกว้าง (เฉพาะโหมดที่มี playback) · "เลือกท่อน" ย้ายออกจากแถว timeline
R4 · ปุ่มหลัก = กึ่งกลางแถวหลัก · R5 · ปุ่มลัด (pins) = ชิดขวา · R6 · แผง ⚙ = กางขึ้นจากปุ่ม ⚙ (ปักซ้าย) · ลากได้ · ไม่หลุดขอบ — พฤติกรรมเดียวทุกโหมด

3 · คำถามให้ P'Aim เคาะ

#คำถามSA แนะนำ
Q1⚙ อยู่ ซ้ายล่าง (ตรงกับ grip) หรือ ขวาล่าง (นิ้วโป้งขวาถึงบนมือถือ)?ซ้ายล่าง — ตรงกับที่พี่ขอให้ grip+⚙ ขอบซ้ายตรงกัน · เหตุผลเดียวที่จะเลือกขวา = เอื้อมนิ้วโป้งมือถือ
Q2แผ่นเพลง/แก้ไข: ความโปร่ง + จัดปุ่ม + ดูเพิ่ม(⋯) ยุบเข้า ⚙ (บ้านเดียว) หรือแยกเป็นปุ่มขวาเหมือนเดิม?ยุบเข้า ⚙ — ให้เหมือนฝึกร้อง
Q3ปุ่มหลัก กึ่งกลาง (อ่านเป็น "เครื่องเล่น" ทุกโหมด) หรือ ชิดซ้ายหลัง ⚙ (มือคุ้นตำแหน่งปุ่มแก้ไข)? = "สม่ำเสมอ" แปลว่าเหมือนเป๊ะ หรือ ไวยากรณ์เดียว?ฝึกร้อง = กึ่งกลาง · แก้ไข = ชิดซ้าย (tools ≠ playback) — ขอพี่เคาะนิยาม "สม่ำเสมอ"
Q4timeline เต็มแถว → "เลือกท่อน" ต้องออกจากแถว timeline · ย้ายไปแถวหลัก(ขวา) หรือเข้า ⚙?ปุ่มเล็กที่แถวหลัง (ขวา) — ยังเห็นเร็ว
Q5ทำเป็น คอมโพเนนต์โครง dock เดียว (ฝึกร้องเลิกวาด chrome เอง) เพื่อการันตี R1–R6 ด้วยโครงสร้าง? (= ต้นทุน build ที่ PM ต้องวางแผน)ใช่ — ไม่งั้นกฎจะ drift อีก
