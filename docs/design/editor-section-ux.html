โหมดแก้ไข — จัดการ “ท่อน / ข้อ” ให้เข้าใจง่ายระดับสากล

SA mockup · คลิกลองได้จริง (แก้ชื่อ · ลากจัดลำดับ · ปุ่มขึ้น/ลง) · โมเดล v2 ไม่เปลี่ยน — เปลี่ยนแค่หน้าตา/วิธีใช้

⛔ นี่คือ mockup เพื่อให้พี่เอมเคาะก่อนสร้างจริง — ยังไม่แตะโค้ด/ฐานข้อมูล

เปิดโหมด “แก้ไข” ครั้งแรก พี่เปา (นักดนตรี ไม่ใช่สายเทคนิค) เจอ 3 รายการที่เล่าเพลงเดียวกัน 3 แบบ — ทำนอง / เนื้อร้อง / ลำดับเพลง — และของสำคัญไปซ่อนอยู่ล่างสุดใต้คำว่า “ขั้นสูง”

  • P1แก้ชื่อข้อได้ที่เดียว แต่ชื่อโชว์ 3 ที่. ชื่อข้อ (“ร้อง 1”, “รับ”) แก้ได้เฉพาะในบล็อก “📜 ลำดับเพลง” ล่างสุด · แต่มันโชว์ที่แถบซ้าย + เมนูเลือกข้อด้วย ซึ่งกดแก้ตรงนั้นไม่ได้ → คนใหม่ไม่รู้ว่าต้องเลื่อนลงไปแก้ที่ไหน → แก้: คลิกชื่อตรงไหนก็พิมพ์แก้ได้เลย (in-line) ทุกที่ผูกค่าเดียวกัน
  • P2จัดลำดับซ่อนอยู่ล่างสุด หลังปุ่มจิ๋ว ▲▼. ต้องเลื่อนไปสุดหน้าถึงจะเจอปุ่มเลื่อนข้อ · ไม่มีอะไรบอกว่า “ลากได้” → แก้: ลากวางจัดลำดับ (รองรับสัมผัส) + ปุ่ม ▲▼ ติดที่ท่อนที่กำลังแก้
  • P3คำว่า “ขั้นสูง” ปิดของที่ใช้บ่อยที่สุด. “ลำดับเพลง” = การเรียงท่อนทั้งเพลง กลับไปอยู่ใต้กลุ่ม “ขั้นสูง” · นักดนตรีไม่มีทางกด “ขั้นสูง” เพื่อทำเรื่องพื้นฐาน → แก้: เอาลำดับเพลงขึ้นเป็นตัวหลักของแถบซ้าย · เลิกใช้คำ “ขั้นสูง”
  • P4แก้เนื้อ 2 ที่ ไม่รู้เกี่ยวกันยังไง — และต้อง “เลือกข้อ” ก่อนถึงเห็นช่องพิมพ์คำ. ถ้ายังไม่กด ✎ เลือกข้อ จะเห็นแต่ตัวเลข ไม่มีช่องเนื้อ → “จะพิมพ์คำตรงไหน?” → แก้: เลือกท่อนไหน = โชว์ช่องคำใต้โน้ตของท่อนนั้นทันที
  • P5ลำดับเพลงโผล่ 2 ที่ (แถบซ้าย “เนื้อร้อง” + บล็อกล่าง) แต่แก้ได้ที่เดียว. รายการเดียวกันแสดงซ้ำ = สับสน → แก้: รวมเป็นรายการเดียว · ตัดบล็อกล่างทิ้ง
  • P6ไม่มี affordance ว่ากด/ลากอะไรได้. แถวในแถบซ้ายดูเหมือนปุ่มนำทาง ไม่มีที่จับลาก ไม่มีดินสอ → แก้: ที่จับลาก ⠿ + ดินสอ/คลิกชื่อ = พิมพ์แก้ (แบบ Google Docs / Notion)
  • P7มือถือ: โครงเพลงอยู่ในลิ้นชักซ้าย + บล็อกล่างอยู่ไกลจอ. จัดลำดับ/แก้ชื่อบนมือถือแทบหาไม่เจอ → แก้: หัวท่อนบนแคนวาสถือปุ่มแก้ชื่อ+เลื่อนไว้เอง ไม่ต้องเปิดลิ้นชัก
  • P8“ข้อ ต้องเลือกท่อนทำนอง” = รายละเอียดเชิงเทคนิครั่วออกมา. คนใหม่ไม่เข้าใจว่าทำไมข้อต้องผูกทำนอง → แก้: เพิ่มท่อน = ได้ทำนองให้อัตโนมัติ · การใช้ทำนองซ้ำ (ท่อนรับ) เป็นตัวเลือกเล็ก ๆ ไม่ใช่รายการหลัก
