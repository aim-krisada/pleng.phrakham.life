## สรุป 1 นาที (F60+)

พี่เปาเปิดโหมด “แก้ไข” แล้วงง เพราะหน้าจอเล่าเพลงเดียวกัน **3 รายการซ้อนกัน** (ทำนอง / เนื้อร้อง / ลำดับเพลง) และ
ของที่ใช้บ่อยสุด — **ตั้งชื่อท่อน + จัดลำดับ** — ไปซ่อนอยู่ **ล่างสุด ใต้คำว่า “ขั้นสูง”** แถมแก้ได้ที่เดียวแต่ชื่อโชว์หลายที่.

**ทางแก้ (เสนอ):** ยุบ 3 รายการเหลือ **รายการเดียว = “โครงเพลง”** ที่ **คลิกชื่อแก้ได้ทันที (in-line)** และ **ลากจัดลำดับได้ (รองรับนิ้ว)** พร้อมปุ่ม ▲▼ ติดที่หัวท่อนที่กำลังแก้ · **ตัดบล็อก “ลำดับเพลง” ล่างทิ้ง** · ทำนองกลายเป็น “ป้ายเล็ก ๆ” ที่ท่อน ไม่ใช่รายการหลักที่ต้องจัดการเอง · เลิกใช้คำ “ขั้นสูง”. นี่คือรูปแบบเดียวกับ Google Docs / Notion / Trello — เปิดมาก็รู้เองว่ากดตรงไหน.

โมเดล v2 (แยกทำนอง↔เนื้อ) **ไม่เปลี่ยน** — เปลี่ยนแค่หน้าตากับวิธีจับ.

## 1. ศึกษาของจริง (`src/components/EditorMode.vue`)

โหมดแก้ไขตอนนี้เรียงจากบนลงล่าง:

1. **แถบหัวเว็บ** — ช่องชื่อเพลง + เมนู “จัดการ ▾”
2. **การ์ด ⚙ ตั้งค่าเพลง** (ยุบได้) — เลข/ชื่อ/คีย์/จังหวะ/BPM/ธีม/หมวด + “สร้างเพลงใหม่ · เลือกเพลง” + “ตรวจแล้ว?”
3. **แถบซ้าย “ส่วนของเพลง”** (ค้างบน desktop / ลิ้นชักบนมือถือ) — 3 กลุ่ม:
   - **ทำนอง** → ท่อน A, ท่อน B… + เพิ่มทำนอง
   - **เนื้อร้อง** → ข้อ 1, ข้อ 2, รับ… + เพิ่มเนื้อ
   - **ขั้นสูง** → 📜 ลำดับเพลง
4. **แถบเครื่องมือหัวโต๊ะ** — breadcrumb (เปิดแถบซ้าย) · (i) · (?) · เค้าโครง 1ห้อง/ต่อกัน · ตัวอย่างสด · ดูผลทั้งเพลง · ฟังท่อน · ฮุก/ซ้ำ/คัดลอก/ลบบรรทัด · ⋯
5. **พื้นที่แก้บรรทัด** — ช่องโน้ต + คอร์ด + ช่องคำใต้โน้ต (โผล่เฉพาะเมื่อ “เลือกข้อ” แล้ว)
6. **ตัวนับคำ** + พาเนล “📝 แก้เนื้อแบบย่อหน้า” (ยุบได้)
7. **บล็อก “📜 ลำดับเพลง”** (ล่างสุด) — แต่ละแถว = [ท่อนทำนอง ▾][ช่องชื่อข้อ][คีย์][▲▼ ✎ ✕]

**จุดสำคัญเชิงโมเดล:** v2 แยก **ทำนอง (stanza = โน้ตล้วน)** ออกจาก **ข้อ (arrangement row = เลือกทำนอง + ใส่คำ + ชื่อ + คีย์)**.
การแยกนี้ทรงพลัง (ท่อนรับใช้ทำนองเดียวซ้ำได้หลายข้อ) แต่ปัจจุบัน UI เอา “การแยก” นี้มาโชว์ดิบ ๆ เป็น 3 รายการให้ผู้ใช้จัดการเอง.

## 2. วินิจฉัย — ทำไมงง (8 ข้อ)

| # | ปัญหา | ทำไมคนใหม่งง |
|---|---|---|
| **P1** | **แก้ชื่อข้อได้ที่เดียว แต่ชื่อโชว์ 3 ที่** | ชื่อ (“ร้อง 1”, “รับ”) แก้ได้เฉพาะ input ในบล็อกล่างสุด · แต่โชว์ที่แถบซ้าย + เมนู lens ด้วย (กดแก้ตรงนั้นไม่ได้) → ตรงกับที่ใบสั่งระบุ “แก้ 1 ที่ แต่โชว์หลายที่” |
| **P2** | **จัดลำดับซ่อนล่างสุด หลังปุ่มจิ๋ว ▲▼** | ต้องเลื่อนสุดหน้าถึงเจอ · ไม่มี affordance ว่าลากได้ · ▲▼ เล็กเกินบนมือถือ |
| **P3** | **คำว่า “ขั้นสูง” ปิดของพื้นฐานสุด** | “ลำดับเพลง” = เรียงท่อนทั้งเพลง = งานพื้นฐาน แต่ถูกจัดใต้ “ขั้นสูง” — ส่งสัญญาณ “อันตราย/ผู้เชี่ยวชาญ” นักดนตรีไม่กล้ากด |
| **P4** | **แก้เนื้อ 2 ที่ + ต้อง “เลือกข้อ” ก่อนถึงเห็นช่องคำ** | ถ้ายังไม่กด ✎ เลือกข้อ จะเห็นแต่ตัวเลข ไม่มีช่องเนื้อเลย → “จะพิมพ์คำตรงไหน?” |
| **P5** | **ลำดับเพลงโผล่ 2 ที่ แก้ได้ที่เดียว** | รายการ arrangement เดียวกันแสดงซ้ำ (แถบซ้าย “เนื้อร้อง” + บล็อกล่าง) 2 มุมมอง แก้ได้มุมเดียว = สับสน |
| **P6** | **ไม่มี affordance ว่ากด/ลากได้** | แถวแถบซ้ายดูเป็นปุ่มนำทาง · ไม่มีที่จับลาก ไม่มีดินสอ — ไม่ชวนให้ “จับของที่เห็นแล้วแก้” แบบเว็บสมัยใหม่ |
| **P7** | **มือถือ: โครงเพลงอยู่ในลิ้นชัก + บล็อกล่างไกลจอ** | จัดลำดับ/แก้ชื่อบนมือถือแทบหาไม่เจอ (พี่เปาใช้แท็บเล็ต/มือถือ) |
| **P8** | **“ข้อต้องเลือกท่อนทำนอง” = เทคนิครั่วออกมา** | คนใหม่ไม่เข้าใจว่าทำไมข้อต้องผูกทำนอง — เป็นรายละเอียดโมเดลที่ไม่ควรโผล่ในงานปกติ |

## 3. เทียบมาตรฐานสากล

| หลักการ | ใครทำ | ใช้กับ pleng ยังไง |
|---|---|---|
| **แก้ในที่ (in-line edit)** | Google Docs, Notion, Trello (ชื่อการ์ด), Airtable, Linear | คลิกชื่อท่อน (แถบซ้าย/หัวท่อน) = พิมพ์แก้ตรงนั้น · Enter บันทึก · Esc ยกเลิก |
| **ลากจัดลำดับ รองรับสัมผัส** | Notion, Trello, Todoist, iOS/Android reorder | ที่จับลาก ⠿ เห็นชัด + **ทางเลือกปุ่ม ▲▼** — **WCAG 2.2 §2.5.7 Dragging Movements** บังคับว่าทุก drag ต้องมีทางทำด้วยปอยน์เตอร์เดียวแทน (ห้าม drag อย่างเดียว) |
| **Direct manipulation** (Shneiderman) | ทั่วไป | จับสิ่งที่เห็นตรง ๆ ไม่ต้องไปหาแผงคุมแยก |
| **Progressive disclosure ที่ถูกต้อง** | ทั่วไป | ซ่อนของนาน ๆ ใช้ (Fine/D.C., volta, แก้โน้ตทำนอง) — **ไม่ใช่** ซ่อนของหลัก (ลำดับ/ชื่อ) · เลิกป้าย “ขั้นสูง” |
| **Single source of truth ต่อ 1 สิ่ง** | ทั่วไป | ชื่อท่อน = ข้อมูลชิ้นเดียว แต่คลิกแก้ได้ทุกที่ที่โชว์ (ผูก `row.label` เดียวกัน) |
| **WCAG 2.2 AA** (memory `pleng-ui-sop`) | — | ปุ่มแตะ ≥24px (§2.5.8) · โฟกัสน้ำตาลเห็นชัด · label ไม่ใช่แค่ placeholder · คีย์บอร์ดแก้ชื่อได้ |

## 4. ข้อเสนอ redesign

### หลักคิด
ยุบ **3 รายการ → 1 รายการหลัก = “โครงเพลง”** (= ลำดับการร้อง = arrangement rows) · **ทำนองเป็นสมบัติ (property) ของท่อน ไม่ใช่รายการพี่น้อง** · ทุกอย่างที่เกี่ยวกับ “ท่อนนี้” อยู่ตรงที่กำลังแก้.

### แถบซ้ายใหม่ = “โครงเพลง”
รายการเดียว เรียงตามลำดับร้อง · แต่ละแถว:
```
⠿  1.  [ร้อง 1]            ♪A   ▲▼  ⋯
⠿  2.  [ร้อง 2]            ♪A   ▲▼  ⋯
⠿  3.  [รับ]               ♪B   ▲▼  ⋯
      ＋ เพิ่มท่อน
──────────────
ทำนอง (โน้ต) ▸   ← ยุบไว้ · ปกติไม่ต้องแตะ (เพิ่มท่อนได้ทำนองเอง · เปิดเมื่ออยากแก้โน้ต/ใช้ซ้ำ)
```
- **⠿ ที่จับลาก** — ลากจัดลำดับ (รองรับนิ้ว) · **▲▼** = ทางเลือกไม่ต้องลาก (WCAG)
- **ชื่อในวงเล็บ** — คลิก = พิมพ์แก้ตรงนั้น (in-line)
- **♪A** = ทำนองที่ท่อนนี้ใช้ (แตะ = เปลี่ยน/ใช้ซ้ำ/สร้างทำนองใหม่)
- **⋯** = ของนาน ๆ ใช้ (เปลี่ยนคีย์ข้อ · ลบท่อน)

### หัวท่อนบนแคนวาส
หัวของแต่ละท่อนในพื้นที่แก้ = **ชื่อ (คลิกแก้) + ป้ายทำนอง + ▲▼/ลาก + ⋯** ในตัว →
พี่เปาแก้ชื่อ+เลื่อนได้ตรงที่กำลังพิมพ์ (ตรงกับไอเดีย P'Aim “ปุ่มเลื่อนขึ้น/ลงอยู่ตรงท่อนที่แก้”). บนมือถือ = ทำได้โดยไม่ต้องเปิดลิ้นชัก.

### แก้เนื้อ
**เลือกท่อน = ช่องคำใต้โน้ตของท่อนนั้นโผล่ทันที** (เลิกพิธี “เลือก lens” แยก · ค่าเริ่มต้น = ท่อนที่เลือกอยู่) · พาเนล “📝 แก้เนื้อแบบย่อหน้า” ยังอยู่เป็นทางเลือกรอง.

### สิ่งที่หายไป / เปลี่ยน
- **บล็อก “📜 ลำดับเพลง” ล่างสุด → ตัดทิ้ง** (ทุกอย่างย้ายไปแถบซ้าย + หัวท่อน) — ยืนยันไอเดีย P'Aim
- กลุ่ม “ขั้นสูง” → หาย · ลำดับเพลงขึ้นเป็นตัวหลัก
- กลุ่ม “ทำนอง” + “เนื้อร้อง” 2 กลุ่ม → เหลือ “โครงเพลง” 1 + “ทำนอง (โน้ต)” ยุบไว้

### มือถือ + PC ดีทั้งคู่
- **PC:** แถบซ้ายค้าง = ภาพรวม + กระโดดข้ามท่อน · หัวท่อนถือ action หลัก
- **มือถือ:** หัวท่อนบนแคนวาสถือ แก้ชื่อ+▲▼ ในตัว (ไม่ต้องเปิดลิ้นชัก) · “โครงเพลง” เต็มเปิดเป็น bottom-sheet เมื่ออยากกระโดด · ลากด้วยที่จับ ⠿ (long-press ได้)

> รายละเอียดทั้งหมด + before/after + ลองเล่นจริง อยู่ใน mockup — **แก้ชื่อได้ · ลากได้ · ▲▼ ได้จริงในหน้า**.

## 6. คำถามถึง P'Aim — ✅ เคาะครบแล้ว (10 ก.ค.)

1. **ตัดบล็อก “📜 ลำดับเพลง” ล่างสุดทิ้ง** → ✅ **ตัดได้** (ย้ายไปแถบซ้าย + หัวท่อน)
2. **คำเรียกแต่ละแถว** → ✅ **“ท่อน”** · (แก้ปมชนคำ: melody เดิม “ท่อน A” เปลี่ยน label เป็น **“ทำนอง A”** · header รายการ = “โครงเพลง”)
3. **ทำนองเป็นเบื้องหลัง** → ✅ **เอา** (เพิ่มท่อน=ได้ทำนองเอง · ใช้ซ้ำผ่านป้าย ♪ · แก้ทำนองตรง ๆ อยู่ในส่วนยุบ)
4. **จัดลำดับ** → ✅ **ลาก + ▲▼ ทั้งคู่**
