## ทำอะไรไปบ้าง (ต่อ US)
- **US-A01: ✅**
  - **AC1 กดเล่น → เสียง + ไฮไลต์ตำแหน่งปัจจุบัน** — ยืนยันของเดิมทำงาน (play engine + follow-along highlight + auto-scroll) และครอบด้วย unit test ใหม่ (`highlight follows the currently-sounding note`)
  - **AC2 หยุด / เล่นต่อ** — ของเดิมกดเล่นซ้ำแล้ว **เริ่มใหม่จากต้นเพลง** ซึ่งไม่ใช่ "เล่นต่อ" จริง → **เพิ่มระบบ pause/resume**: กดหยุดแล้วกดเล่นอีกที = เล่นต่อจากโน้ตที่ค้างไว้ (ไม่รีสตาร์ท). เริ่มเพลง/ท่อนใหม่ หรือเปลี่ยนเพลง = เริ่มจากต้นตามปกติ
  - **AC3 view-only ไม่มีปุ่มบันทึก/แก้** — ยืนยัน: ไม่มีปุ่มบันทึก/แก้ · ไม่ emit `save` · เพิ่ม prop `tier` ให้รับตาม contract WT-0 (view อ่านอย่างเดียวทุก tier)

## ไฟล์ที่แก้ (เฉพาะไฟล์ของ WT-A ตาม DS-A01)
- `src/lib/midi.js` — `playSong` รับ option `startIndex` (ข้ามโน้ตที่เล่นไปแล้วเพื่อ resume) · `onNote` ส่ง absolute index กลับมาด้วย (ให้ viewer จำตำแหน่งที่หยุด)
- `src/components/SongViewer.vue` — เพิ่ม state `playedIndex`/`pausedIndex` · `togglePlay` = หยุด(จำตำแหน่ง)/เล่นต่อ · เล่นจบเองแล้วรีเซ็ตตำแหน่ง · เปลี่ยนเพลงรีเซ็ตตำแหน่ง · เพิ่ม prop `tier` (contract)
- `src/components/SongViewer.play.test.js` — เพิ่ม 4 เทสต์ (play จากต้น · highlight ตามโน้ต · หยุด-แล้ว-เล่นต่อ resume · รับ prop `tier` + ไม่ emit save) · stub `matchMedia`/`scrollIntoView` (jsdom ไม่มี)

## ผลทดสอบ
- **unit:** `npx vitest run` → **16/16 ผ่าน** (เดิม 12 + ใหม่ 4) · ไม่มี unhandled error
- **build:** `npm run build` → ✅ ผ่าน
- **วิธี tester ลอง (ต้องฟังด้วยหู):** เปิด `http://localhost:5302` → เปิดเพลงหนึ่ง → โหมด "ดู"
  1. กด **ฟังเพลง** → ได้ยินเสียงเดินตามโน้ต + เห็นไฮไลต์วิ่งตาม + จอเลื่อนตาม
  2. กด **หยุด** กลางเพลง → เงียบ · กด **ฟังเพลง** อีกที → **เล่นต่อจากจุดที่หยุด** (ไม่กลับไปต้นเพลง)
  3. ปล่อยเล่นจนจบ → กดเล่นใหม่ = เริ่มจากต้น
  4. ยืนยันว่าไม่มีปุ่มบันทึก/แก้ในโหมดนี้

## ข้อสังเกต / คำถามถึง SA
- AC2 คำว่า "เล่นต่อ" ผมตีความว่า = **resume จากจุดที่หยุด** (ไม่ใช่แค่กดเล่นซ้ำ) จึงลงมือทำ pause/resume จริง. ถ้า SA/พี่เอมตั้งใจแค่ "กดเล่นใหม่จากต้นก็พอ" บอกได้ — ถอดออกง่าย
- resume อิงตำแหน่งเป็น "โน้ตที่กำลังดัง" ตอนกดหยุด (เล่นต่อโดยเริ่มที่โน้ตนั้น) — ระดับโน้ต ไม่ใช่ระดับวินาที ซึ่งพอสำหรับฝึกร้อง
- ปุ่มเล่น/หยุดเป็น `<button>` มีข้อความ ("ฟังเพลง"/"หยุด") เป็น accessible name อยู่แล้ว + คุมด้วยคีย์บอร์ดได้ (native) ตาม a11y ของ DS-A01
