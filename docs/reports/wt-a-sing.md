## ข้อสังเกต / คำถามถึง SA
- AC2 คำว่า "เล่นต่อ" ผมตีความว่า = **resume จากจุดที่หยุด** (ไม่ใช่แค่กดเล่นซ้ำ) จึงลงมือทำ pause/resume จริง. ถ้า SA/พี่เอมตั้งใจแค่ "กดเล่นใหม่จากต้นก็พอ" บอกได้ — ถอดออกง่าย
- resume อิงตำแหน่งเป็น "โน้ตที่กำลังดัง" ตอนกดหยุด (เล่นต่อโดยเริ่มที่โน้ตนั้น) — ระดับโน้ต ไม่ใช่ระดับวินาที ซึ่งพอสำหรับฝึกร้อง
- ปุ่มเล่น/หยุดเป็น `<button>` มีข้อความ ("ฟังเพลง"/"หยุด") เป็น accessible name อยู่แล้ว + คุมด้วยคีย์บอร์ดได้ (native) ตาม a11y ของ DS-A01

## 🔎 ประเด็นให้ SA วิเคราะห์เพื่อจ่ายงาน — ไฮไลต์ระดับคำ/โน้ต (จาก tester พี่เอม)
**อาการที่ tester เจอ:** ไฮไลต์คาราโอเกะ "ไม่ค่อยตรงจังหวะ" · พี่เอมอยากได้ **ไฮไลต์แบ่งเป็นคำ ๆ ตามโน้ต** (ไล่ทีละโน้ต/พยางค์)

**root cause:** ตอนนี้ไฮไลต์ละเอียดแค่ระดับ **segment (ช่อง)** — `onNote` ส่งกลับแค่ `{li, si}` โดย `si` = ดัชนีช่อง. หนึ่งช่องมีได้หลายโน้ต (เช่น note = "1 2 3" หรือเอื้อน) แต่ทุกโน้ตในช่องใช้ `si` เดียวกัน → ช่องทั้งช่องสว่างพร้อมกันค้างจนจบช่อง ไม่ไล่ทีละโน้ต. บวกกับตัวจับเวลา poll ทุก 100ms (มี fudge -80ms) เทียบ Web Audio clock เลยรู้สึกหน่วง

**ต้องทำอะไรถึงจะได้ระดับคำ/โน้ต:**
1. `midi.js` (WT-A ถือ) — `onNote` ส่งดัชนีระดับโน้ต/โทเคนเพิ่ม (ทำได้ในไฟล์ผม)
2. **`SongSheet.vue` + อาจถึง `NoteRow.vue`** — ต้อง render แต่ละโน้ต/พยางค์เป็นหน่วยไฮไลต์แยก → **ไฟล์กลาง ใช้ร่วมกับ WT-B (พิมพ์)** · DS-A01 ให้ WT-A ถือแค่ `SongViewer.vue`+`midi.js` → **เกินขอบเขตเดิม**

**การพึ่งพา model:** "หนึ่งพยางค์ต่อหนึ่งโน้ต" เป็นแนวคิดของ **v2 model** (แยก melody/word, syllable ผูกโน้ต 1:1). เพลง v1 ที่ผ่าน `resolveContent` อาจเป็น "1 คำ + หลายโน้ต" (เอื้อน) ที่ไม่มีจุดตัดพยางค์ชัด → per-โน้ต กับ per-คำ ไม่เท่ากันเสมอ. SA ควรดู `docs/song-model-v2.md` ประกอบตอนตัดสิน

**ทางเลือกให้ SA เคาะ:**
- (ก) ขยาย DS-A01/US-A01 ให้ WT-A ถือ `SongSheet.vue`(+`NoteRow.vue`) ช่วงงานนี้ · ต้องกันเวลาไม่ให้ชน WT-B (พิมพ์) ที่แตะ print CSS ใน SongSheet
- (ข) แยกเป็น US ใหม่ (เช่น US-A05 "ไฮไลต์ระดับคำ") ผูกกับ v2 · จ่ายทีหลังเป็น worktree เดี่ยวที่ถือ SongSheet ชัดเจน

**คำแนะนำ dev:** ทำ (ข) — แยกเป็นงานเดียวที่ถือ SongSheet/NoteRow ให้ชัด จะคุมการชนกับ WT-B ง่ายกว่า · และ **อย่าเพิ่งแก้ "จังหวะเพี้ยน" แยกตอนนี้** เพราะงานระดับคำจะเขียนตัวจับจังหวะใหม่อยู่แล้ว = แก้ซ้ำเสียเปล่า (พี่เอมยืนยันแนวนี้). งาน pause/resume (US-A01 รอบนี้) ยัง merge ได้ตามปกติ ไม่ต้องรอประเด็นนี้
