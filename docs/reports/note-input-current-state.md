## 0. สรุปสั้น

- **ตัวแยกวิเคราะห์เดียวทั้งระบบ** = `parseNotes()` ใน [notation.js:17-80](../../src/lib/notation.js#L17)
  ทุกทางเข้า (ช่องพิมพ์ · แป้นสัญลักษณ์ · ปุ่ม · JSON) ลงมาเป็น **สตริงโน้ตแบบเดียวกัน** แล้วเข้า parser ตัวนี้
- **โน้ตเก็บเป็นข้อความล้วน** ใน `content.stanzas[].lines[][].note` — คั่นโน้ตด้วยช่องว่าง (เว้นวรรค = 1 "กล่องโน้ต")
- **มี 2 ตัวแก้ที่พฤติกรรมไม่เหมือนกันเลย** — ตัวแก้เต็ม (`EditorMode`, พิมพ์ได้ทุกอักขระ) กับ
  ตัวแก้ในหน้าอ่าน (`SongViewer` inline, พิมพ์ได้แค่ตัวเลข + ♯ ♭) → **นี่คือช่องโหว่ใหญ่สุด**
- **base กับ main ตัว parser เหมือนกันเป๊ะ** — base เพิ่มแค่ `melismaSpans()` (ฝั่ง render) ไม่แตะไวยากรณ์

## 2. ทางเข้าข้อมูล — แต่ละทางรับไม่เท่ากัน

### E1 · ตัวแก้เต็ม (แท็บ "แก้") — ช่องพิมพ์ต่อโน้ต

**นี่คือทางเดียวที่ใส่ได้ครบทุกสัญลักษณ์**

### E2 · แป้นสัญลักษณ์ 21 ปุ่ม (dock ของตัวแก้)
[EditorMode.vue:998-1001](../../src/components/EditorMode.vue#L998)
```
แถว 1:  1 2 3 4 5 6 7 0 - ~
แถว 2:  . ' _ ^ ( ) { } # b n
```
แทรกที่ cursor ของกล่องที่ focus ล่าสุด — **ถ้าไม่มีกล่องโน้ต focus อยู่ ปุ่มไม่ทำอะไร**
([:1010-1011](../../src/components/EditorMode.vue#L1010) `if (!activeInput ...) return`)

### E4/E5 · ตัวแก้ inline บนหน้า "ฝึกร้อง" (SongViewer)
ตัวใหม่ · กดดินสอ ✏️ → เดินเคอร์เซอร์บนแผ่นเพลงจริง · ทุกปุ่มลงที่ `<input>` ซ่อน
[SongViewer.vue:267-299](../../src/components/SongViewer.vue#L267)
**รับแค่:** `0-7` · `#` · `b` — บวกปุ่ม สูง/ต่ำ (จุดช่วงเสียง) และปุ่มคอร์ด

**อื่นๆ ที่เจอ:**
- **`‘` (single quote โค้ง) 2 ตัว** — parser รับให้ ([notation.js:44](../../src/lib/notation.js#L44)) แต่แปลว่ามีข้อมูลที่เคยผ่าน smart-punctuation มา
- **`b` (แฟลต) มีแค่ 1 ตัวเดียวทั้งไลบรารี** เทียบกับ `#` 108 ตัว → ควรตรวจว่าเป็นเจตนาหรือคนคีย์เลี่ยง `b` เพราะพิมพ์แล้วสับสน
- **`n` (♮) ไม่มีเลยสักตัว** ทั้งที่มีปุ่มในแป้น + มีกฎ lint ทั้งกฎ
- **volta ไม่มีเลย · repeat มีแค่ 3 คู่** จาก 155 เพลง

**ไม่มีอักขระใดในข้อมูลที่ parser ไม่รู้จักโดยสิ้นเชิง** — ทุกปัญหาเป็นเรื่อง **ลำดับ** ไม่ใช่ตัวอักษรแปลกใหม่

### 7.3 ตีความ — ไม่ใช่ "โค้ดผิด" แต่ควรรู้

- **D5 (MusicXML)** ขัดกับเอกสาร แต่ **ตรงกับมติที่ P'Aim เคาะทีหลัง** (memory `pleng-ssot-scope-musicxml-not-staff`: v2 = SSOT อยู่แล้ว · MusicXML เป็น **export ปลายทาง** ไม่ใช่ core) → เอกสารอัลกอริทึมเป็นบทสนทนา *ก่อน* เคาะ ไม่ใช่ SSOT ปัจจุบัน
- **D7** เป็นความคลาดเคลื่อนของ **เอกสารสถาปัตยกรรมเอง** (เขียนว่ายังไม่มี ทั้งที่โค้ดมี) — ควรแก้เอกสาร ไม่ใช่แก้โค้ด
- **D1 (เปลี่ยนจังหวะกลางเพลง)** เป็นช่องโหว่ที่ **หนักที่สุด** ในกลุ่มนี้: ไม่ใช่แค่ใส่ไม่ได้ แต่ระบบจะ **เตือนผิด** (ขึ้น ❌ ทุกห้องในช่วงที่จังหวะเปลี่ยน) → คนกรอกจะถูกฝึกให้เพิกเฉยต่อเครื่องหมาย ❌

## 8. ความไม่ชัวร์ — สิ่งที่อ่านโค้ดแล้วยังตอบไม่ได้ (ไม่เดา)

1. **พี่เปาใช้ทางไหนจริง** — โค้ดบอกว่ามีกี่ทาง แต่บอกไม่ได้ว่าเขาพิมพ์ในกล่องเป็นหลัก
   หรือกดแป้น 21 ปุ่มเป็นหลัก · **ต้องถามเขาหรือดูเขาทำ**
2. **`n` (♮) ไม่มีใครใช้เลย — เพราะไม่ต้องการ หรือเพราะไม่รู้ว่ามี** ตอบจากโค้ดไม่ได้
3. **`b` มีตัวเดียว** — เป็นเพราะเพลงไม่ต้องใช้ หรือเพราะพิมพ์ `b` แล้วสับสนกับตัวอักษร ตอบไม่ได้
4. **6 จุดที่อ่านไม่ได้เกิดจากอะไร** — คนคีย์เอง หรือมาจาก DA import parser (`tools/parse_song.py`)
   ยังไม่ได้ตรวจฝั่งนั้น
5. **`pickup` 538 จุดมาจากไหน** — จำนวนเยอะผิดสังเกตเทียบกับ repeat 3 จุด · น่าจะมาจาก import
   แต่ยังไม่ยืนยัน
6. **ไม่ได้ทดสอบบนเบราว์เซอร์จริง** — งานนี้ read-only ทั้งหมด · พฤติกรรมคีย์ลัดที่เขียนไว้ในตาราง
   §3 อ่านจากโค้ด **ยังไม่ได้กดจริง** โดยเฉพาะ Tab ข้ามช่องพยางค์ ([:1908-1916](../../src/components/EditorMode.vue#L1908))
   ที่พึ่ง `document.querySelector('[data-slot=...]')` — อาจไม่เจอถ้าช่องพยางค์ปิดอยู่
7. **ตัวแก้ inline บน mobile** — `NoteInputBar` variant "bar" ใช้ `visualViewport` จัดตำแหน่ง
   เหนือคีย์บอร์ด · จริงเท็จบนมือถือจริงยังไม่พิสูจน์
8. **แป้น 21 ปุ่มกับ 2 แถว** — DockKey อาจ reflow ตามความกว้าง ([DockKey.vue:84-105](../../src/components/DockKey.vue#L84))
   ยังไม่รู้ว่าบนจอพี่เปาเห็นครบ 21 ปุ่มหรือถูกซ่อน
9. **เอกสาร `แปลงโน้ตเพลงเป็นอัลกอริทึม.md` เป็นบทสนทนา 2,807 บรรทัด ไม่ใช่ spec ที่ล็อกแล้ว** —
   อ่านครบเฉพาะส่วนที่พูดถึงสัญลักษณ์/โครงสร้างข้อมูล (§1, §3, และช่วง SSOT บรรทัด 640-830 · 1145-1200 ·
   1740-1900 · 2395-2470) · ส่วนที่เป็นการไล่ลำดับการร้องรายเพลง (~1,000 บรรทัด) อ่านผ่าน →
   **ถ้ามีข้อกำหนดสัญลักษณ์ซ่อนอยู่ในนั้น ผมอาจพลาด**
10. **ไม่รู้ว่าข้อไหนใน §7.2 ถูกเคาะไปแล้ว** — D5 ยืนยันได้จาก memory ว่ามีมติทีหลัง
   ส่วน D1-D4 · D9 ยังไม่เห็นมติที่ไหน จึงรายงานเป็น "เอกสารว่า vs โค้ดทำ" เฉยๆ ไม่ตัดสินว่าใครถูก
