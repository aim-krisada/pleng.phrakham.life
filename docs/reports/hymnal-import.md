## 1. เพลงอ้างอิง #32 "นมัสการพระบิดา" (PDF หน้า 20 = หน้าหนังสือ 32-33)

- **หน้า PDF 1 แผ่น = เปิดหนังสือ 2 หน้า** (ซ้าย=ต้นเพลง, บนขวา=ข้อต่อ). เพลง 32 มี **4 ข้อ**
- **ผู้แต่ง:** ทำนอง Henry T. Smart · คำร้อง P. Van den Berg (แปลจาก Dutch) · tempo Moderato
- **2 หัวข้อย่อยในเพลงเดียว:** ข้อ 1-3 = "การกำหนดล่วงหน้าของพระองค์" · ข้อ 4 = "การไถ่ของพระองค์"
- **ทำนองเว้นไว้** (โน้ตตัวเลขสแกน = เสี่ยงพลาด · ทีมเติมเองในเอดิเตอร์ B083) — import **เฉพาะเนื้อ**

### เนื้อร้องที่อ่านได้ (ตรวจแล้ว 3 รอบ zoom — ให้ P'Aim เทียบหน้าจริง)

การอ่าน **2 คอลัมน์ = อ่านสลับซ้าย-ขวาทีละบรรทัด (คู่กลอน)** ไม่ใช่ซ้ายจบแล้วขวา — ยืนยันด้วยสัมผัส
(โลกา/กายา · ผู้ใหญ่/พระทัย · เจ้า/เกลา · พระพักตร์/ประจักษ์ ล้วนคล้องกันเมื่ออ่านสลับ)

**ข้อ 1** (ใต้โน้ต)
1. สรรเสริญพระบิดาราศี, ข้าเป็นคู่หมายพระคุณ,
2. บัดนี้เพื่อพระทัยเปรมปรีดิ์, ร่วมสรรเสริญพระการุณย์,
3. ในนิรันดร์กาล, บุตรทั้งหลาย, ให้เข้าราศีวิไล,
4. ร่วมกับพระคริสต์บุตรที่รัก, เป็นคู่เคียงสุดสูงศักดิ์.

**ข้อ 2**
1. ทรงเลือกข้าก่อนสร้างโลกา / ให้วิสุทธิ์ทั่วกายา
2. กำหนดเราได้พระคุณนี้ / เป็นทรัพย์ประเสริฐราศี
3. เพื่อส่วนสิทธิแห่งบุตรา / ทรงหมายข้าไว้ล่วงหน้า
4. จนกว่าเติบโตเป็นผู้ใหญ่ / รับพรตามน้ำพระทัย

**ข้อ 3**
1. ตามพระดำรินิรันดร์กาล / เรียกโดยพระคุณโปรดปราน
2. ให้มีชีวิตพระองค์เจ้า / ขจัดบาปหายเกลี้ยงเกลา
3. พระวิญญาณมาเป็นตราหมาย / หล่อเลี้ยงด้วยสิ่งทั้งหลาย
4. ข้าเป็นของพระองค์สืบไป / รับสุขสมบูรณ์มากมาย

**ข้อ 4** (การไถ่ของพระองค์)
1. โอ นอกจากพระบิดาเจ้า / ใครอาจให้พรแก่เรา
2. ให้ข้ามีส่วนพระวิญญาณ / พระคุณชีวิตซาบซ่าน?
3. จะมีราศีไม่ช้านาน / กับพระองค์นิรันดร์กาล
4. เราร่วมเป็นอยู่เบื้องพระพักตร์ / ฉายาราศีประจักษ์!

## 2. โทเค็นที่ใช้จริง (วัดจากเพลงนี้)

วิธี "แม่นก่อน" ต่อเพลงต้อง crop ซูมหลายจุดเพื่อยืนยันคำที่กำกวม — เพลง 32 ใช้:
- render หน้าเต็ม 1 ครั้ง (dpi 110) + zoom crop 3 จุด (dpi 200-300) = **อ่านภาพ 4 ครั้ง**
- ประมาณ **~8-11K token/เพลง** เมื่อทำใน context แยก (subagent เดียว/เพลง)
  — สูงกว่าเดา 5-9K เล็กน้อย เพราะซูมยืนยันความแม่น (P'Aim เลือกแม่น > ประหยัด)
- **ข้อเสนอ:** ทำ 1 เพลง = 1 subagent สดใหม่ (ห้ามแชทเดียวยาว) ตามใบสั่ง → คงที่ ~10K/เพลง · ~470 เพลง ≈ ประเมินก่อน batch

## 4. 🚦 4 ข้อที่ต้อง P'Aim ล็อกก่อนเดินต่อ

### 4.1 category code ของเล่มนี้ ✅ ล็อกแล้ว (P'Aim 11 ก.ค.)
เล่ม anuchon (YS 120 เพลง) ใช้ `(category, number)` เป็น key. เล่มนี้เลขวิ่งถึง ~470 →
ถ้าใช้ `anuchon` เลข 32 จะทับเพลง 32 ของเล่มเดิม (คนละเพลง!) → ต้องมีโค้ดของตัวเอง.
**P'Aim: "ใช้เล่มใหญ่"** → category code = **`lem-yai`** · frontend map `lem-yai` → ชื่อโชว์ **"เล่มใหญ่"**
(⚠️ ฝาก display dev/PM เพิ่ม map `lem-yai`→"เล่มใหญ่" เหมือน anuchon). อัปเดตใน `s032.sql` แล้ว.

### 4.2 รูปแบบเก็บ = lyrics-only แบบไหน (ผมเลือก v1 — ขอยืนยัน)
ใบสั่งเขียน "v2 · เนื้อใน arrangement · stanza ว่าง". **แต่** โมเดล v2 เรนเดอร์เนื้อจาก
โน้ตใน stanza — **stanza ว่าง = หน้าเว็บโชว์ว่างเปล่า** (reader วนตาม `stanza.lines`, ไม่มีโน้ต=ไม่มีอะไร).
→ ผมเก็บเป็น **v1 `content.lines` (เนื้อล้วน, โน้ตว่าง)** แทน เพราะ:
- **เห็นเนื้อบนเว็บทันที** (P'Aim/โบสถ์อ่านได้เลย) — reader v1 โชว์ lyric แม้ note ว่าง (SongSheet.vue:350)
- **ซื่อตรง** ไม่มีทำนองปลอม
- ทีมเปิดในเอดิเตอร์ → auto-migrate v1→v2 → เติมโน้ต + จับคู่เนื้อ (B083) ได้
**ขอ P'Aim เลือก:** (ก) v1-เห็นเนื้อทันที (ผมแนะนำ) หรือ (ข) v2-stanza-ว่าง ตามใบสั่งเป๊ะ (แต่หน้าเว็บว่างจนกว่าทีมเติมโน้ต)

### 4.3 หัวข้อย่อย 2 อันในเพลงเดียว
เพลง 32 มี "การกำหนดล่วงหน้าฯ" (ข้อ 1-3) + "การไถ่ฯ" (ข้อ 4) ใต้เลขเดียว.
ผมเก็บ: title = "นมัสการพระบิดา" · หัวข้อย่อยเป็น `label` คั่นกลุ่มข้อ (เห็นในแผ่นเพลง).
โครงนี้อาจเจออีกทั้งเล่ม — **P'Aim โอเคกับวิธีนี้ไหม** (หรืออยากแยกเป็นคนละเพลง / เก็บที่อื่น)

### 4.4 ขอบเขต = เนื้ออย่างเดียวจริง (ยืนยัน)
ผม **ไม่เอา**: โน้ตตัวเลข, คอร์ด, key, time-signature (แม้หน้ามีครบ) — ทีมเติมเองพร้อมทำนอง.
เก็บ `credits` (ผู้แต่ง) + `tempo` + `source` (เล่ม/เลข/หน้า) ไว้ในเนื้อหา · `verified=false` ·
`review_flags=["melody-empty","unverified-import"]`. **ถูกต้องตามที่ P'Aim ต้องการไหม**
