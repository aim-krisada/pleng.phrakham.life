# แก้แล้ว — วาทยกรฟัง *เสียง* ด้วย ไม่ใช่ดูแค่ *เวลา*

ต่อจาก [chord-affects-playback-song33.md](chord-affects-playback-song33.md) (วินิจฉัย · ไม่แก้).
ที่นี่คือการแก้จริง + หลักฐาน.

## ต้นตอ (สรุปสั้น)

`refereeNoClash()` — "วาทยกร" ที่คุมไม่ให้ลูกเล่นทับทำนอง — ตัดสินจาก **เวลาอย่างเดียว**:
"ห่างจากหัวโน้ตทำนอง ≥ 0.4 บีต = ปลอดภัย". แต่ช่องว่าง *ก่อน* หัวโน้ตคือที่อันตรายที่สุด:
ประดับที่ลงตรงนั้น **ด้วยเสียงเดียวกับที่ทำนองกำลังจะร้อง** ไม่ได้ยินเป็นเครื่องประดับ
แต่ได้ยินเป็น **โน้ตทำนองเกินมา 1 ตัว** (เพลง 33 คอร์ด `E` → ประกาย B4 หนึ่งบีตก่อน B4 B4 B4 → หูนับได้ 4).

## ทางที่เลือก — ก. ขยายวาทยกรให้ดูเสียงด้วย (ฟันธงเอง ไม่ถามพี่เอม)

รายงานวินิจฉัยเสนอ 3 ทาง · เลือก **ก.** ด้วยเหตุผลที่วัดได้:

| ทาง | ทำไมเลือก / ไม่เลือก |
|---|---|
| **ก. วาทยกรดูเสียงด้วย** ✅ | ตัดตรงจุดที่ผิดจริง — วัดแล้วกระทบ **2 ประดับจาก 1,396 ตัวในคลัง** · ทำนอง/เบส/แตกคอร์ด **ไม่ขยับเลย** · ทดสอบ headless ได้ · ย้อนกลับได้ด้วยบรรทัดเดียว |
| ข. ห้ามประกายเข้าย่านทำนอง | เปลี่ยนเสียง **ทุกเพลง** ทั้งคลัง เพื่อแก้จุดที่ผิด 2 จุด · ประกายบางตัวจะทะลุเพดาน MIDI 86 แล้วหายไปเงียบๆ · ผิดชุดควบคุมข้อ 4 (161 เพลงต้องเหมือนเดิม) ตั้งแต่ในนิยาม |
| ค. ลดความดังประกาย | ไม่แก้ต้นเหตุ — เบาแค่ไหน ถ้ายังเล่น *เสียงเดียวกัน* *ก่อนจังหวะ* หูก็ยังผูกมันเข้ากับโน้ตที่ตามมาอยู่ดี (pre-echo เป็นเรื่องลำดับ ไม่ใช่ระดับเสียง) · แถมมีสไลเดอร์ให้ผู้ใช้อยู่แล้ว = ไม่ใช่การแก้บั๊ก |

## กติกาที่เพิ่ม (สูตรทั่วไป ไม่ใช่ patch เพลง 33)

ลูกเล่น (role `emb`) จะถูกตัดทิ้ง ถ้า **ครบทั้ง 3 ข้อ**:

1. **เสียงตรงกัน** — ยูนิซัน หรือห่างกันพอดีคู่แปด (`|Δ| % 12 === 0` และ `|Δ| ≤ 12`)
   กับ **หัวโน้ตทำนองตัวถัดไป**
2. **มาก่อนไม่เกิน 2 บีต** (`PREECHO_LOOKAHEAD`)
3. **ดังพอจะถูกเข้าใจผิดว่าเป็นทำนอง** — `gain ≥ 0.12` (`PREECHO_MIN_GAIN`)

**ทำไม 2 บีต:** ที่จังหวะนมัสการ 70–90 bpm = ~1.4–1.7 วินาที — ช่วงที่หูยังผูก
"เสียงที่ดังมาก่อน" เข้ากับ "เสียงเดียวกันที่มาลงจังหวะ" (หลักเดียวกับโน้ตนำ/ห้องขึ้นต้น
ที่ฟังเป็นของห้องถัดไป). เกินกว่านั้นหูแยกเป็นคนละเหตุการณ์ → หน้าต่างกว้างกว่านี้จะฆ่าประดับดีๆ ฟรีๆ.
วัดจริงแล้ว: 1 / 2 / 4 บีต ให้ผลอาการ = 0 เท่ากัน แต่ 4 บีตตัดประดับเพิ่มโดยไม่ได้อะไร.

**ทำไมนับคู่แปดด้วย:** ประกายถูกสร้างเป็น "เสียงบนของคอร์ด **+12**" อยู่แล้ว —
ถ้าเช็คแค่ยูนิซัน เคสคู่แปดจะรอด และมีของจริงอยู่ในคลัง (**เพลง 100** beat 146.25).
วัดแล้ว: `octaves: 0` เหลืออาการ 1 จุด · `octaves: 1` เหลือ 0 · `octaves: 2` ก็ 0 แต่ตัดประดับเพิ่มฟรี.

**ทำไมมีเพดานความดัง:** ถ้าไม่มี กติกานี้จะไปตัด `gapFill` / `chromaticApproach` / `octaveSwell`
(ดังราว 0.033–0.041 = ต่ำกว่าทำนอง ~8 เท่า) ซึ่งไม่มีใครฟังเป็นทำนองอยู่แล้ว —
วัดแล้วต่างกันมาก: **ไม่มีเพดาน = ตัด 298 ตัวใน 62 เพลง · มีเพดาน = ตัด 2 ตัวใน 2 เพลง** อาการเหลือ 0 เท่ากัน.
ค่า 0.12 = เส้นเดียวกับที่ตัววัดในรายงานวินิจฉัยใช้ → **ตัววัดกับตัวแก้พูดภาษาเดียวกัน**.

**ทิศทางเดียว (ก่อนเท่านั้น):** เสียงเดียวกันที่ดัง *หลัง* ทำนองร้องไปแล้ว = เสียงสะท้อน ไม่หลอกหู
จึงไม่ถูกตัด — ตัดด้วยจะเสียประดับดีๆ ในโน้ตลากยาวไปฟรีๆ.

โค้ด: [`src/lib/arranger/referee.js`](../../src/lib/arranger/referee.js) — `refereeNoClash()`
ปรับ config ได้ที่ `refereePreEcho` / `refereePreEchoOctaves` / `refereePreEchoMinGain` (ไม่มีปุ่มให้ผู้ใช้ — เป็นวินัย ไม่ใช่รสนิยม เหมือนกติกาวาทยกรเดิม).

### 2. ชุดควบคุม 5 ข้อ (จากรายงานวินิจฉัย) — ผ่านทั้งหมด

| # | ชุดควบคุม | ผล |
|---|---|---|
| 1 | โน้ตทำนอง (เสียง+เวลา) ทุกตัวทุกเพลง | **เปลี่ยน 0 เพลง** (170/170) |
| 2 | เบส (`bass`) และแตกคอร์ด (`inner`) | **เปลี่ยน 0 เพลง** ทั้งสอง role |
| 3 | เส้นฐาน 7.3% ของ `comp-doubles-melody.md` | **ไม่ขยับ** — ตัววัดนั้นนับเฉพาะโน้ตคลอที่ดัง *ขณะทำนองดังอยู่*; กติกาใหม่แตะเฉพาะประดับที่ดัง *ตอนทำนองเงียบ* และตัดไปแค่ 2 ตัวที่ไม่เคยถูกนับ |
| 4 | เพลงที่ไม่ถูกตัดประดับเลย ต้องเหมือนเดิมทุก byte | **168/168 เหมือนเดิมเป๊ะ** (เทียบ เสียง+จังหวะ+ความยาว+ความดัง+ไทม์ชิฟต์ ทุก event) |
| 5 | preset `ตรงโน้ต` (arranger ปิด) | ไม่มีประดับตั้งแต่ต้น · `refereeNoClash` ไม่ถูกเรียกเลยเมื่อ arranger ปิด → เปลี่ยน 0 |

**หมายเหตุที่ต้องพูดตรงๆ — 2 เพลงที่ถูกตัด (33, 100) มีผลข้างเคียงระดับ jitter ทั้งเพลง:**

| เพลง | event | ตัดออก | เพิ่ม | โน้ตที่เหลือ: gain ต่าง | timeShift ต่าง |
|---|---|---|---|---|---|
| 33 (คอร์ด E) | 563 → 562 | `emb@23.00 midi 71` | 0 | 387 ตัว (สูงสุด 12.6%) | 486 ตัว (สูงสุด 22.8 ms) |
| 100 | 603 → 602 | `emb@146.25 midi 69` | 0 | 130 ตัว (สูงสุด 11.9%) | 536 ตัว (สูงสุด 21.6 ms) |

**เสียงและจังหวะบนกริดของทุกโน้ตที่เหลือ = เดิมเป๊ะ** (ไม่มีโน้ตหาย ไม่มีโน้ตเพิ่ม นอกจาก 1 ตัวที่ตั้งใจตัด)
ที่ขยับคือ humanize — การสุ่มความดัง ±6% และเวลา ±12 ms ที่เดินตาม *ลำดับ event*
พอมี 1 event หายไป ตัวที่เหลือจึงหยิบเลขสุ่มถัดไปคนละตัว. เป็นธรรมชาติของการ "ตัดโน้ตทิ้ง" ทุกแบบ
(วาทยกรเดิมที่ตัดตามเวลาก็ทำแบบนี้อยู่แล้วทุกครั้งที่ตัด) — ไม่ใช่การเปลี่ยนดนตรี
แต่ **แปลว่าเทียบไฟล์เสียง 2 ไฟล์แบบ byte-by-byte จะต่างกันทั้งเพลง จึงพิสูจน์อะไรไม่ได้**
หลักฐานที่ใช้คือระดับ event (ตารางข้างบน) ไม่ใช่ระดับ sample.

### 5. ไฟล์เสียงให้ฟังด้วยหู

เรนเดอร์เพลง 33 **คอร์ด E ต้นฉบับ** ผ่าน `renderSongToBuffer()` (เส้นทางเดียวกับปุ่มดาวน์โหลด/ฟัง —
`arrange()` ตัวเดียวกัน) แล้วตัดเฉพาะ **บีต 14–31 = รอยต่อบรรทัด 1 → บรรทัด 2** เป็น WAV 9.4 วินาที:

```
C:\gl\pm-inbox\pleng\audio-2026-07-23-preecho\
  1-BEFORE-เพลง33-คอร์ดE-มีโน้ตผี.wav     ← โค้ดเดิม (refereePreEcho: 0)
  2-AFTER-เพลง33-คอร์ดE-แก้แล้ว.wav        ← โค้ดใหม่
```

จุดที่ต้องฟัง: ช่วงท้ายท่อน ทำนองพัก 1 บีตแล้วขึ้นบรรทัด 2 ด้วยโน้ตเดียวกัน 3 ครั้ง —
ไฟล์ **before** มีเสียงนำมาก่อน 1 ครั้ง (รวมเป็น 4) · ไฟล์ **after** ช่องพักนั้นเงียบ เหลือ 3 ตามที่เขียนไว้.

ข้อจำกัดของไฟล์ที่ต้องบอก: เรนเดอร์ด้วย **เสียงสังเคราะห์ (synth)** ไม่ใช่แซมเปิลเปียโนจริง
(ตัวโหลดแซมเปิลค้างในเซิร์ฟเวอร์ dev เพราะชื่อไฟล์มี `#` — คนละเรื่องกับบั๊กนี้ ไม่ได้ไล่ต่อ)
และ **ปิดเสียงก้อง** ไว้ เพราะ impulse ของห้องถูกสุ่มใหม่ทุกครั้งที่เรนเดอร์.
โน้ตที่ดังจึงตรงกับ `arrange()` ทุกตัว แต่ *เนื้อเสียง* ไม่ใช่ของจริงที่ผู้ใช้ได้ยิน.

## ที่ยังไม่ได้พิสูจน์

- **โหมดรวมวง (`playEnsemble`)** — เส้นทางนี้ไม่ได้วิ่งผ่าน `arrange()`/วาทยกรเลย
  (มี scheduler + ลูกรับส่งไวโอลินของตัวเองใน `midi.js`) → **กติกาใหม่ยังไม่คุ้มโหมดนี้**.
  ความเสี่ยงต่ำกว่า (ไวโอลินถูกบังคับให้อยู่ย่าน 71–86 และเข้าใน *หาง* ของโน้ตที่ลากอยู่ ไม่ใช่ในช่องว่างก่อนหัวโน้ต)
  แต่ **ยังไม่ได้วัด** — ควรเป็นใบสั่งแยก ไม่ใช่ลากเข้ามาในงานนี้ (โครงคนละชั้น).
- **หูคน** — ไฟล์เสียงเรนเดอร์แล้ว แต่ยังไม่มีใครยืนยันด้วยหูว่า "ใช่ ตัวนี้แหละที่ได้ยิน / หายแล้ว".
