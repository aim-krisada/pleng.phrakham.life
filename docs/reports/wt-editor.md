## รื้อหน้าแก้ไขให้ตรง prototype — ทำอะไรบ้าง

### B035 / B031 / B003 / E1 — แถบหัวโต๊ะ (edhead) ✅
รื้อหัวโต๊ะเดิม (ที่มี dropdown เลือก **ท่อน** + **ข้อ** ซ้ำกับแถบซ้าย) ให้เป็นแบบ prototype:
- **breadcrumb** = ปุ่มเดียว "ท่อน A · ข้อ 1" — **กดเปิด/ปิดแถบองค์ประกอบเพลง (rail)** · rail เป็น **ตัวเลือกหลักตัวเดียว** (ไม่มี dropdown ซ้ำ — มาตรฐาน master-detail) → ปิดจบ **B031 + B003 + E1**
- **(i) วิธีใช้** + **(?) สัญลักษณ์** = ปุ่มสลับ ซ่อน/แสดง (in-context help · ไม่รกจอ)
- **เค้าโครง (layout toggle)** "1 ห้อง/แถว ⇄ ต่อกัน"
- **ดูผลทั้งเพลง** (ปุ่มสลับ) + **▶ ฟังท่อน**
- **โครงเพลงด่วน (desktop)** = ท่อนฮุก · ซ้ำ · คัดลอกบรรทัด · **ลบบรรทัด (แดง + ยืนยันก่อนลบ)** — ทำงานกับ **บรรทัดที่กำลังแก้ (active line)**
- **⋯ เพิ่มเติม** = ชื่อบรรทัด · ต่อห้อง · **จบเพลง (Fine)** · ป้าย (D.C. al Fine)

### B035 — เค้าโครงห้อง (layout toggle) ✅
`1 ห้อง/แถว` (แต่ละห้องเต็มแถวของตัวเอง · ค่าเริ่มต้น ตรง prototype) ⇄ `ต่อกัน` (ห้องเรียงต่อกัน พับเมื่อยาว)

### B035 — ดูผล (per-bar) + ดูผลทั้งเพลง ✅
- แต่ละห้องมี **▶ ฟังห้อง** + **ดูผล** — กด "ดูผล" แล้วห้องนั้น **สลับจากช่องแก้เป็นโน้ตแบบแผ่นเพลง (jianpu render จริง)** แทนที่ในตำแหน่งเดิม (ไม่ซ้อนแถว) · แตะที่ตัว render กลับไปแก้ได้
- **ดูผลทั้งเพลง** สั่งทุกห้องพร้อมกัน · **ป้ายสะท้อนสถานะจริง** (`allShown()` — ถ้าทุกห้องเป็น render ป้ายเปลี่ยนเป็น "กลับไปแก้ทั้งเพลง")

### B032 — ปุ่มลบท่อน (ทำนอง) / ลบข้อ (เนื้อร้อง) ✅
ทุกแถวใน rail (ทั้ง **ทำนอง** และ **ข้อ**) มีปุ่มลบ (ถังขยะ · โผล่ตอน hover/focus) · ลบข้อ = ลบแถวลำดับเพลงให้ sync · ลบทำนองที่ถูกใช้อยู่ = ถามยืนยันก่อน (ของเดิม)

### I5 — editor อ่าน/เขียน JSON
มีอยู่แล้วจากคลื่นก่อน (`previewContent` · `applyRow` · ดาวน์โหลด/อัปโหลด JSON ใน "จัดการ") — **ไม่แตะซ้ำ**

## ทางแยกดีไซน์ที่ตัดสินเอง (จดไว้ให้ P'Aim) — เลือกทางที่ตรง prototype ที่สุด
1. **ย้ายโครงบรรทัดขึ้นหัวโต๊ะทั้งหมด → ตัดปุ่ม ⋯ ราย "บรรทัด" ออก** (เหลือแค่ ▶ฟังบรรทัด) ตาม US E3 "ทั้งหมดระดับบรรทัดในหัวโต๊ะ" · ป้ายบนหัวบรรทัด (ฮุก/ต่อห้อง/จบเพลง) ยังโชว์สถานะอยู่
2. **ราย "ห้อง" ยังเก็บ ⋯ ไว้** (ย้าย/สำเนา/ลบห้อง + ‖: :‖ + ห้องจบ volta) — prototype บอกห้องเหลือแค่ ▶+ดูผล แต่ถ้าตัดทิ้งจะ **ตั้ง volta / จัดการห้องไม่ได้เลย** และเสีย future-proof ของโมเดล → **เก็บไว้เป็น ⋯ เงียบๆ** (โมเดล v2 ห้ามลดรูป) · ขอ P'Aim เคาะว่าจะซ่อนลึกกว่านี้ไหม
3. **"ซ้ำ" (quick-struct)** = ครอบทั้งบรรทัดด้วย ‖: (ห้องแรก) + :‖ (ห้องสุดท้าย) · กดซ้ำ = ปลด · **ยังไม่มี UI ตั้ง volta ราย 1./2. ในหัวโต๊ะ** (อยู่ใน ⋯ ของห้อง) — ตาม E3 (mid-line repeat/volta = อนาคต)
4. **ไอคอนฮุก** ใช้ `anchor` (สมอ) แทน — Lucide ไม่มี `hook` ตรงๆ (DS เขียน `hook`) · ขอ P'Aim ดูว่าโอเคไหม หรืออยากได้ glyph อื่น
