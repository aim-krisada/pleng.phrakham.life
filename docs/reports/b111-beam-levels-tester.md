## A1 — แกนหลัก: วัดจำนวนเส้นที่ "คนอ่านเห็นจริง"

**วิธีของ tester (ไม่ใช้สคริปต์คนทำ):** นับ **เส้นแนวนอนที่พาดกึ่งกลางตัวเลข** จาก 2 แหล่งรวมกัน
(ก) `border-bottom` ของตัวเลขเอง (1.5px = 1 เส้น · `4px double` = **2 เส้น**)
(ข) ทุกแท่ง `.beam` ที่ช่วง x คลุมกึ่งกลางตัวเลข (แท่งเก่า `.beam-u2` นับเป็น 2 เส้น)
แล้วยุบตำแหน่ง y ที่ซ้ำกัน → ได้ "จำนวนเส้นที่ตาเห็น" · เทียบกับ `underlines` ของตัวนั้น

### เพลง 101 `(3_. 4__ 5~` — viewport 1180×900

| ตัวโน้ต | ขีดที่พิมพ์ | **base `c7f60a3`** เห็น | **fixed `e83ebcc`** เห็น |
|---|---|---|---|
| `3•` | 1 ชั้น | **3 เส้น** ❌ | **1 เส้น** ✅ |
| `4` | 2 ชั้น | **4 เส้น** ❌ | **2 เส้น** ✅ |

> base แย่กว่าที่รายงานไว้: แท่ง `.beam-u2` เก่านอนที่ `top=19.70` แต่ขีดของตัวเลขเองอยู่ที่ `22.70`
> → **เหลื่อมกัน 3px กลายเป็นเส้นซ้อน 3–4 เส้น** ไม่ใช่แค่ "2 เส้นแทน 1"

### เพลง 120 `3_. 2__ 1_. .7__` (fixed)
`3•`=1/1 ✅ · `2`=2/2 ✅ · `1•`=1/1 ✅ · `7`=2/2 ✅ · ผิด 0 (วัด 4 ตัว)

### เพลง 89 — ⛔ **วัดพิกเซลไม่ได้ บอกตรงๆ**
โหลดหน้าแล้ววัดได้ 9 โน้ตที่ถูกมัดคาน **ผิด 0** แต่ **run ที่มีปัญหา (`(1~ ~1__ 2'__ 1_`) ไม่ถูก render บนหน้าจอ**
(mixedRows = 0) → ยืนยันคำอธิบายของคนทำว่าอยู่ในแถวที่ `resolveContent` ไม่แสดง

## usability — "คนอ่านแยกออกไหมว่าขีดสั้น ≠ คานเชื่อม 2 ตัว" (ข้อสำคัญที่สุด)

**PM ตัดสินแล้วว่าคง clamp 0.5 — tester ไม่ตกเรื่องความยาว** ตรวจเฉพาะว่า *ไม่ชน / ไม่กำกวมจริง*

ตัวเลขจริง เพลง 101 (เหมือนกันทุก viewport ที่วัด: 1180 · 412 · 360):

| รายการ | ค่า |
|---|---|
| ความกว้างตัวเลข | 11.03 px |
| ช่องว่างระหว่างตัวเลข (`3•`.right → `4`.left) | 11.51 px |
| ความยาวขีดสั้น L2 | 16.78 px |
| **ช่องว่างขาวเหลือถึงตัวเลขข้างเคียง** | **5.75 px** |
| คิดเป็นสัดส่วนของช่องว่าง | **50.0 % (หยุดตรงกึ่งกลางพอดี)** |
| เทียบความหนาเส้น (1.5px) | ช่องว่าง = **3.8 เท่าของความหนาเส้น** |
| ถ้าเป็นคานเชื่อมจริง L2 จะยาว | 33.58 px = **2 เท่าของขีดสั้น** |

**คำตอบตรงๆ: แยกออก** เหตุผลเชิงวัด 3 ข้อ
1. ขีดสั้น **หยุดที่กึ่งกลางช่องว่างพอดี ไม่แตะตัวเลขข้างเคียงเลย** — คานเชื่อมจริงต้องลากถึงตัวเลข (ยาว 2 เท่า)
2. **เหนือ/ใต้ตัว `3•` ไม่มีเส้นชั้น 2 อยู่เลย** — ตัวมันมีเส้นเดียวจริงๆ ไม่ใช่เส้นจางหรือเส้นสั้น
3. ช่องว่างขาว 5.75px = 3.8 เท่าของความหนาเส้น → มองเห็นเป็น "ช่องขาด" ชัด ไม่ใช่รอยต่อ

**ข้อแม้ที่ต้องบอกตรงๆ:**
- **ไม่มีภาพ** — `computer screenshot` time out (ตรงกับที่คนทำเจอ) → คำตัดสินนี้มาจาก **เรขาคณิตที่วัดได้ ไม่ใช่การดูด้วยตา** · **ควรให้ P'Aim/พี่เปาชำเลืองยืนยันอีกที**
- **edge case เชิงทฤษฎี:** สูตรคือ `ext = max(3, min(digitWidth, gap/2))` → ถ้าช่องว่างแคบกว่า **6px** ตัว floor 3px จะชนะ แล้วขีดสั้นจะล้ำเกินกึ่งกลาง · ถ้าช่องว่าง < 3px จะแตะตัวเลขข้างเคียง = กลับไปกำกวม
  **ในคลังจริงยังไม่เจอ** (ทุกจุดที่วัด = 11.51px) แต่ฝากไว้เป็นความเสี่ยง
