## ✅ ทำอะไรไป (แผลที่ปิด)

**D3 · ตัวตนมาจากข้อมูล ไม่ใช่ประตู — หัวใจ**
`reviewingDraft` เดิมเป็น `ref` ที่ `loadDraft()` เซ็ตที่เดียว/ล้าง 5 ที่ → เปิดเพลงทางอื่น= จอเงียบ
ตอนนี้เปลี่ยนเป็น `computed` ที่ **derive จากข้อมูล**:
- `openDraft` (ref) = ร่างที่เปิดตรง ๆ เท่านั้น (navigation) — เหลือจุดนี้จุดเดียวที่ประตูแตะได้
- `openPendingDraft` / `reviewingDraft` / `pendingForThisSong` = **computed ล้วน** จาก `openDraft` + `pendingDrafts` + `editingId`
→ เปิดเพลงที่มีร่างรอตรวจผ่าน **เลือกเพลง / URL / รีเฟรช / กล่องร่าง = ได้ผลเหมือนกันทั้ง 4 ทาง** (AC-1)
→ **บั๊กขอบ #5 หายฟรี:** approver เปิดร่าง*ตัวเอง* ไม่ขึ้น "กำลังตรวจฉบับร่างของ[ตัวเอง]" อีก (เช็ก `author_id !== uid`)

**D4 · เตือนก่อนทับ**
- แถบ `.pending-alert` inline (ไม่ใช่ toast) โผล่ทันทีที่เปิดเพลงที่มีร่างของคนอื่นรอตรวจ → `⚠️ [ชื่อ]ส่งร่างของเพลงนี้มารอตรวจ` + 2 ปุ่ม `ดูร่างของ[ชื่อ]` / `แก้ฉบับเผยแพร่ต่อ` · `aria-live="polite"`
- `primaryAction()` กด "เผยแพร่" ทั้งที่มีร่างค้าง → **confirm ที่บอกผลจริง** (ชื่อคน + ผลลัพธ์) ไม่ใช่ "แน่ใจไหม?" ลอย ๆ
- **รองรับ draft ที่โหลดช้า** — `pendingForThisSong` เป็น computed → พอ `loadDrafts()` คืนค่าทีหลัง แถบเด้งขึ้นเอง (ไม่ต้อง await ก่อน loadSong)

**D2 · ปุ่มพูดความจริง + รวมเป็นแหล่งเดียว**
- ป้ายปุ่มแยกตามสถานะจริง (เผยแพร่ / เผยแพร่ทับ / ⚠️ เผยแพร่ทับ / เผยแพร่ร่างฉัน / อนุมัติร่าง / ส่งตรวจ)
- **ลบ `save()` (1414) + `primaryLabel` (1690) ทิ้ง** — SA เตือนว่า "2 ฟังก์ชันตัดสินใจคนละแบบ อย่าปะข้างเดียว" · **ผมพิสูจน์แล้วว่าทั้งคู่เป็นโค้ดตาย** (ไม่มีใครเรียกทั้งโปรเจกต์ — ปุ่มจริงวิ่ง `primaryAction` + `saveLabel`) → ลบทิ้ง = แหล่งความจริงเดียวจริง ไม่ใช่ปะข้างเดียว

## ⚖️ 2 จุดที่ของจริงต่างจาก DS — ผมตัดสินเอง (ตามที่ให้ค้านได้)

**1. DS ชี้ให้แก้ป้ายที่บรรทัด 1691 (`primaryLabel`) — แต่ปุ่มจริงไม่ได้ใช้มัน**
`primaryLabel` **ไม่มีใครอ่านเลย** · ปุ่ม dock กิน `saveLabel` (1705) ต่างหาก → ถ้าแก้ตาม DS ตรง ๆ จอจะไม่เปลี่ยนอะไร · ผมแก้ที่ `saveLabel` (ตัวจริง) + เพิ่ม `saveName` (aria/tooltip) แล้วลบ `primaryLabel` ตายทิ้ง

**2. DS §3 ให้ใส่ชื่อเจ้าของ*บนหน้าปุ่ม* ("อนุมัติร่างของโม") — ผมย้ายชื่อออกจากหน้าปุ่ม**
**เหตุผล = ตัวเลข ไม่ใช่ความรู้สึก:** วัดสดในเบราว์เซอร์จริงที่ 360px (ฟอนต์จริง 13px/600) — ปุ่ม prime อยู่ row 2 ข้าง "ฟังทั้งเพลง" · ป้าย `เผยแพร่ทับฉบับปัจจุบัน` = 135px · ชื่อ latin ยาว (`k.pituck`/`yeahwong`) ดันแถวไป **393px > 360 → ล้นจอ** = ผิด **US AC-9 (hard gate)**
**ทางออก (ยัง WCAG ครบ):** หน้าปุ่มสั้น+สม่ำเสมอ (action) · **เจ้าของงานอยู่ครบ 2 ที่:** (ก) แถบ `pending-alert`/`review-banner` ที่อยู่*เหนือปุ่มพอดี* (ข) `saveName` = aria-label + title เต็ม ("เผยแพร่ทับฉบับปัจจุบัน — มีร่างของโมรอตรวจอยู่")
→ DS กลัว "เจ้าของหายไปไหนไม่รู้" · ที่นี่เจ้าของ**อยู่ติดปุ่มเสมอ** แค่ไม่ใช่บนตัวปุ่ม · เลือก hard gate (ไม่ล้นจอ) ชนะ soft preference (ชื่อบนปุ่ม) เมื่อชนกันที่ 360px

*(ถ้า PM/SA อยากได้ชื่อบนปุ่มจริง ๆ ต้องเติม `text-overflow: ellipsis` ที่ `.dk-btn-lbl` ใน `DockKey.vue` = นอกขอบเขตเฟส 1 · ยกให้สาย UX เฟส 2)*
