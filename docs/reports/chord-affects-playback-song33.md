# เพลง 33 — "คอร์ดทำให้มีโน้ตเกิน" · หาต้นตอเจอแล้ว (วินิจฉัยอย่างเดียว · ยังไม่แก้)

พี่เปา (23 ก.ค.): บรรทัด 1 ของเพลง 33 คีย์ A "โน้ตเกินมา 1 ตัว · 3 ตัวกลายเป็น 4 ตัว
เพราะคอร์ด · บีทดันมาเล่นโน้ตตัวนั้นอีก" → เขาแก้คอร์ดท้ายบรรทัดจาก **E เป็น E7**
แล้วเสียงถูก แต่ไม่แน่ใจว่าหายเพราะ E7 หรือเพราะไปโดนอย่างอื่น

**คำตอบ: หายเพราะ E7 จริง · ไม่ใช่เพราะอย่างอื่น** — พิสูจน์ด้วยการรันเครื่องเล่นจริง
เทียบ E กับ E7 บนข้อมูลจริงจาก production

## ต้นตอ

`sparkle()` ใน [`src/lib/arranger/embellish.js`](../../src/lib/arranger/embellish.js) —
"ประกาย" ที่เปิดเป็นค่าเริ่มต้นใน preset `piano-arrangement` เล่นเสียงบนสุดของคอร์ด **ยกขึ้นหนึ่งคู่แปด**
ตรงหัวคอร์ด ด้วยความดัง `MEL_BASE × sparkleLevel` = 0.245 (ทำนองอยู่ที่ 0.31) → **เบากว่าทำนองแค่ ~20-34%
= หูได้ยินเป็นโน้ตทำนองอีกตัว ไม่ใช่เสียงประดับ**

**เสียงที่มันเล่น ถูกเลือกจากคอร์ดล้วนๆ** (`up[up.length-1] + 12`) จึงเปลี่ยนตามคอร์ดที่พิมพ์:

| คอร์ดที่พิมพ์ | เสียงบนของคอร์ด | ประกายเล่น |
|---|---|---|
| `E` | G#3, B3 | **B4** |
| `E7` | G#3, B3, **D4** | D5 |

## ทำไมหูได้ยินเป็น "เกินมา 1 ตัว"

ท้ายบรรทัด 1 = `5~ - -` \| `~5 - 0` → ทำนองลากยาว 5 บีตแล้ว**หยุดพัก 1 บีต** (บีต 23–24)
บรรทัด 2 เริ่มด้วย `2' 2' 2'` ใต้คอร์ด Bm = **B4 สามครั้ง** (บีต 24, 25, 26)

ประกายลงพอดีที่ **บีต 23** — กลางช่องพัก นำหน้าโน้ตแรกของบรรทัด 2 อยู่ 1 บีต:

| | บีต 23 (พัก) | 24 | 25 | 26 |
|---|---|---|---|---|
| **คอร์ด E (ก่อนแก้)** | **B4 ← ประกาย** | B4 | B4 | B4 |
| **คอร์ด E7 (ตอนนี้)** | D5 ← ประกาย | B4 | B4 | B4 |

**คอร์ด E → ได้ยิน B4 สี่ครั้ง ทั้งที่โน้ตเขียนไว้ 3** = "3 ตัวกลายเป็น 4 ตัว" ตรงเป๊ะ
และเพราะมันมาก่อนจังหวะ 1 บีต หูจึงรับเป็น "บีทดันมาเล่นโน้ตตัวนั้นอีก" ตรงกับคำพี่เปาทุกคำ

## ทำไมไม่มีอะไรจับได้

[`refereeNoClash()`](../../src/lib/arranger/referee.js) (วาทยกร) กันประดับไม่ให้ทับทำนอง
โดยเช็คแค่ว่า **ห่างจากหัวโน้ตทำนอง ≥ 0.4 บีต** — บีต 23 ห่างจากบีต 24 อยู่ 1.0 บีต **ผ่านฉลุย**
มันไม่เคยเช็คว่า *เสียงที่จะเล่นซ้ำกับโน้ตที่กำลังจะร้องหรือเปล่า* → ช่องโหว่คือ
"อยู่ในช่องว่าง" ถูกนับว่าปลอดภัยเสมอ ทั้งที่ช่องว่างก่อนหัวโน้ตคือที่ที่อันตรายที่สุด

lint / หน้าแผ่นเพลง จับไม่ได้เพราะ**โน้ตตัวนี้ไม่มีอยู่ในข้อมูลและไม่เคยถูกวาด** — มันเกิดตอนเล่นเสียงเท่านั้น

## หลักฐาน (รันซ้ำได้ · อ่านอย่างเดียว ไม่แตะ DB)

```
node tools/diag-chord-playback.mjs  <song33.json>   # diff E vs E7 ทั้งเพลง
node tools/diag-chord-playback2.mjs <song33.json>   # ซูมบีต 18-27 + ชุดควบคุม
node tools/diag-emb-preechoes-melody.mjs <songs.json>  # สแกนทั้งคลัง
node tools/diag-emb-risk.mjs <songs.json>              # ประชากรที่เสี่ยง
```

ผลจากการรัน `arrange()` จริง (preset ตั้งต้น `piano-arrangement` เหมือนที่ผู้ใช้ได้ยิน):

```
melody notes: E=141  E7=141          ← ทำนองไม่ขยับเลย
total events: E=563  E7=563
ONLY when E : beat 23.00  emb  B4  gain 0.204
ONLY when E7: beat 23.00  emb  D5  gain 0.204
```

**ชุดควบคุม (ต้องไม่เปลี่ยน) — ผ่านทั้งหมด:**

- โน้ตทำนองเปลี่ยน **0** ตัว (จำนวนเท่ากันเป๊ะ 141)
- เสียงเบส (`bass`) เปลี่ยน **0** ตัว
- เสียงแตกคอร์ด (`inner`) เปลี่ยน **0** ตัว
- เหตุการณ์ที่เปลี่ยนนอกช่วงคอร์ดที่สลับ (บีต 23–24) = **0**
- ตัวตรวจถูกทดสอบกับเคสบวกที่รู้คำตอบแล้ว (เพลง 33 คอร์ด E) → ยิงถูก 1 จุด ตามคาด

## เกิดกับอีกกี่เพลง

สแกนทั้งคลัง **169 เพลง** จาก production ด้วยเกณฑ์เดียวกัน:

| | จำนวน |
|---|---|
| ประดับที่ **เสียงซ้ำเป๊ะ**กับโน้ตทำนองที่กำลังจะร้อง | **0 จุด (0 เพลง)** |
| ประดับที่ **เสี่ยง** — ดังพอ + ลงในช่องว่าง ≤2 บีตก่อนหัวโน้ตทำนอง | **11 จุด ใน 8 เพลง (4.7%)** |
| ในนั้น: ห่างคู่แปด 1 · ห่างหนึ่งเสียง 1 · กัดครึ่งเสียง 0 | |

**อ่านตัวเลขนี้ยังไง:** วันนี้ไม่มีเพลงอื่นที่อาการออก **เพราะเพลง 33 เป็นเพลงเดียวที่เคยเป็น
และพี่เปาแก้ไปแล้ว** (ข้อมูลใน DB ตอนนี้คือหลังแก้) แต่กลไกยังอยู่ครบ — เสียงประดับถูกเลือก
จากคอร์ดล้วนๆ โดยไม่ดูทำนองเลย ดังนั้น **มี 8 เพลงที่ห่างจากอาการนี้แค่การแก้คอร์ดครั้งเดียว**

## คนละตัวกับ "melody เกิน 7.3%"

**คนละบั๊ก** — [`comp-doubles-melody.md`](comp-doubles-melody.md) วัดโน้ตคลอที่ดัง
**ขณะทำนองกำลังดังอยู่** (`role !== 'melody'` ทับกับโน้ตทำนองที่กำลังกินเวลาอยู่)
ตัวนี้ตรงข้าม: ประดับที่ดัง **ตอนทำนองเงียบ** จึงไม่เคยถูกนับในตัวเลข 7.3% เลยแม้แต่ครั้งเดียว
— นั่นคือเหตุผลที่หาไม่เจอมาตลอด ตัววัดที่ใช้อยู่มองไม่เห็นมัน

ทั้งสองตัวมี**รากร่วมกัน**คือ "เสียงคลอ/ประดับถูกวางโดยไม่ดูทำนอง" แต่ต้องแก้คนละจุด
และวัดคนละตัว **แก้ตัวนี้จะไม่ทำให้ 7.3% ขยับ** (และกลับกัน)

## ทางแก้ที่เสนอ (⛔ ยังไม่ได้แก้ · รอ PM เคาะ) — ครบ 3 ชั้น

### ชั้นเก็บ (ข้อมูล) — ไม่ต้องแก้ แต่มีเรื่องต้องตัดสิน

โค้ดผิด ไม่ใช่ข้อมูลผิด **`E` ที่พี่เปาลบทิ้งน่าจะเป็นคอร์ดที่ถูกตามต้นฉบับ** ส่วน `E7`
คือการแก้*ข้อมูลที่พิมพ์บนแผ่นเพลง*เพื่อไปกลบ*บั๊กเสียง* → ตอนนี้แผ่นเพลงเพลง 33
อาจไม่ตรงต้นฉบับแล้ว **ต้องถามพี่เปาว่าต้นฉบับเขียน E หรือ E7** ถ้าเป็น E ควรแก้กลับ
*หลัง*ปิดบั๊กเสียงแล้วเท่านั้น (ไม่งั้นอาการกลับมา) — ⛔ ผมไม่แตะข้อมูลตามกฎ

### ชั้นแสดง (แผ่นเพลง) — ไม่ต้องแก้

โน้ตตัวนี้ไม่เคยถูกวาดและไม่ควรถูกวาด มันเป็นเสียงประดับตอนเล่น ไม่ใช่ทำนอง

### ชั้นเล่น (เสียง) — จุดที่ต้องแก้จริง

**ก. ขยายวาทยกรให้ดูเสียงด้วย ไม่ใช่ดูแค่เวลา** (แนะนำ — ตรงจุด เปลี่ยนน้อย ย้อนง่าย)
เพิ่มกฎใน `refereeNoClash()`: ตัดประดับที่ **เสียงซ้ำกับหัวโน้ตทำนองตัวถัดไป** (เป๊ะหรือคู่แปด)
เมื่อนำหน้าไม่เกิน ~2 บีต
- ✅ แก้ที่รากของอาการนี้พอดี · กลไกมีอยู่แล้ว · ทดสอบ headless ได้
- ❌ ตัดโน้ตทิ้ง → บางจุดที่เคยมีประกายจะเงียบไป (จากการวัด: กระทบ ≤11 จุดในคลัง = แทบไม่รู้สึก)
- ⚠️ ต้องเคาะว่า "ซ้ำ" นับคู่แปดด้วยไหม และหน้าต่างกี่บีต

**ข. ห้ามประกายเข้ามาในย่านเสียงทำนอง** — บังคับให้ประกายอยู่เหนือโน้ตทำนองที่สูงสุดเสมอ
- ✅ แก้ต้นเหตุกว้างกว่า (ประกายจะไม่ปนกับทำนองอีกเลย)
- ❌ กระทบ**ทุกเพลง** เสียงโดยรวมเปลี่ยน · บางเพลงประกายจะทะลุเพดาน 86 แล้วหายไปเลย
- ⚠️ ต้องให้หูพี่เปาตัดสินก่อน

**ค. ลดความดังประกายลง** — จาก 0.245 (เบากว่าทำนองแค่ 21%) ให้ลงไปเป็นเสียงประดับจริงๆ
- ✅ ง่ายสุด มีสไลเดอร์ให้ผู้ใช้อยู่แล้ว
- ❌ ไม่ได้แก้ต้นเหตุ ถ้ายังเล่นเสียงเดียวกันก่อนจังหวะ เบาแค่ไหนหูก็ยังจับได้ว่า "มาก่อน"

### ชุดควบคุมที่ต้องไม่เปลี่ยน (ทุกทางเลือก)

1. โน้ตทำนอง (`role === 'melody'`) — จำนวน เสียง และเวลา **ต้องเท่าเดิมทุกตัวทุกเพลง**
2. เสียงเบสและแตกคอร์ด (`bass` / `inner`) — ไม่ถูกแตะ (ทางเลือก ก. และ ข. แตะเฉพาะ `emb`)
3. ตัวเลขเส้นฐาน 7.3% ของ `comp-doubles-melody.md` — **ต้องไม่ขยับ** (คนละชั้น ถ้าขยับ = แก้ผิดที่)
4. 161 เพลงที่ไม่มีประดับเข้าเกณฑ์ — เสียงต้องเหมือนเดิมเป๊ะ (diff เหตุการณ์ = 0)
5. preset `ตรงโน้ต` (arranger ปิด) — ไม่มีประดับอยู่แล้ว ต้องไม่มีอะไรเปลี่ยน

## ที่ยังไม่ได้พิสูจน์

- **ยังไม่ได้ฟังด้วยหู** — ทั้งหมดนี้พิสูจน์จากการรันเครื่องเรียบเรียงจริงและนับเหตุการณ์
  ยังไม่ได้เรนเดอร์เสียงออกมาให้พี่เปาฟังยืนยันว่า "ใช่ ตัวนี้แหละที่ได้ยิน"
- **ค่าก่อนแก้ของจริง** — ตาราง `song_revisions` อ่านได้แต่ว่างเปล่า (0 แถว) ไม่มีประวัติ
  จึง**สร้าง "ก่อนแก้" ขึ้นเองโดยเปลี่ยนแค่คอร์ดเดียว** (E7 → E) ตามที่พี่เปาบอก
  ถ้าเขาแก้อย่างอื่นด้วยในครั้งนั้น (เช่น `A#dim` ในห้องก่อนหน้า) ผลอาจต่างจากนี้
- **โหมดรวมวง (`playEnsemble`)** — ยังไม่ได้ตรวจ มันมีเส้นทางประดับของตัวเอง
  (ไวโอลินตอบท้ายวรรค) ซึ่งเลือกเสียงจากคอร์ดเหมือนกัน → น่าจะมีอาการเดียวกัน แต่ยังไม่ได้วัด
- **preset อื่น** — วัดจาก `piano-arrangement` (ค่าตั้งต้น) เท่านั้น
