## 1 · การสำรวจระบบปัจจุบัน (As-Is)

### 1.1 การเพิ่ม/แก้ไขโน้ต 1 ตัว
**โครงสร้าง:** โน้ตเก็บเป็น **string เจียนพู่ต่อ segment** (SSOT) แต่ UI แสดงเป็น **ช่อง input แยกทีละโน้ต** (`NoteBoxes.vue:11-14,130-142`) — ไม่ใช่พิมพ์รวมทั้งบรรทัด
- **แก้ค่าโน้ต:** พิมพ์ในช่องนั้นตรง ๆ · Enter/Space = ไปช่องถัดไป · ลูกศร = เลื่อนช่อง (`NoteBoxes.vue:79-121`)
- **สูง/ต่ำ (ออกเทฟ):** มีปุ่มความหมาย **▲/▼** ในกล่องเครื่องมือลอย (toolbox) — toggle `.`/`'` (`EditorMode.vue:960-968,3061-3064`) ← **นี่คือ attribute เดียวที่มีปุ่มเฉพาะ**
- **ชาร์ป/แฟลต/ค้าง/tie (`# b n . ' _ ^ - ~ ( ) { }`):** **ไม่มีปุ่มความหมายเฉพาะ** — ใส่ด้วย **แป้นสัญลักษณ์ 21 ปุ่ม** (แตะช่องโน้ตก่อน → จิ้มปุ่ม → แทรกอักขระที่ caret) (`EditorMode.vue:936-955`, `DockKey.vue:490-494`) หรือพิมพ์อักขระเอง · parse ที่ `notation.js:17-80`
- **สรุป:** มี UI ให้กด (แป้น + ▲▼ ออกเทฟ) **ผสม** การพิมพ์อักขระ — ไม่ใช่คีย์ลัดคีย์บอร์ดล้วน

### 1.2 การจัดการบรรทัด/ห้อง (line/bar)
- **โครง:** stanza → `lines[]` → `bars[]` → `segments[]{chord,note,lyric}` (`EditorMode.vue:79-96`) · ห้องเป็น object แยกจริง
- **❌ ไม่มี auto-wrap:** โน้ตล้นห้อง **ไม่ขึ้นห้อง/บรรทัดใหม่ให้เอง** · ระบบแค่ **แจ้งสถานะ** ("3/4 จังหวะ ❌" `barStatus` `EditorMode.vue:888-928`) แต่ไม่จัดโครงให้
- **ผู้ใช้กดเอง:** ปุ่ม **"+ ห้อง"** (`addBar` `:974-980,3196`) · **"+ เพิ่มบรรทัด"** (`addLine` `:1035-1037,3208`) · Enter ในช่องโน้ต = สร้างช่องโน้ตถัดไป **ไม่ใช่** ห้อง/บรรทัดใหม่
- มีย้ายห้อง/บรรทัด ซ้าย-ขวา-ขึ้น-ลง · ทำซ้ำห้อง · ต่อห้องข้ามบรรทัด (checkbox `cont`) — ทั้งหมด manual

### 1.3 การใส่เนื้อร้อง + เนื้อข้อ 2, 3
- **v2 (ปัจจุบัน): 1 พยางค์ = 1 โน้ต attack** · เนื้อร้อง **แยกเลเยอร์จากทำนอง** — ทำนองอยู่ใน stanza (ไม่มีเนื้อ) · เนื้ออยู่ใน arrangement entry เป็น `syllables[]` เรียง 1:1 กับโน้ต (`song-model-v2.md:46-66`, `songModel.js:150-159`) · เอื้อน (พยางค์เดียวลากหลายโน้ต) = โน้ตที่ตามได้ช่องว่าง ไม่ต้องเพิ่มคำ (`notation.js:143-170`)
- **ข้อ 2, 3 (ทำนองเดียวกัน): ⭐ ไม่ต้องพิมพ์ทำนองซ้ำ** — สร้าง arrangement entry หลายอันชี้ **stanza id เดียวกัน** แต่ละอันใส่แค่ `syllables[]` ของข้อตัวเอง (`song-model-v2.md:60-66`, `songModel.js:121-171`) = "แก้ทำนองที่เดียว ทุกข้ออัปเดต"
- **v1 (เก่า): ไม่มีกลไก strophic** — เนื้อผูกระดับ segment (string เดียว) · ต้อง duplicate ทั้งข้อ → ทำนอง drift (เหตุผลที่ทำ v2)

## 2 · Use Case จริง (Logical Challenges)

### 2.1 Loop & Jump (D.C. / Fine / Repeat)
- **Repeat `‖: :‖` + volta 1./2. → ✅ มีจริง เป็น attribute ของห้อง** (ไม่ใช่ text) ตั้งด้วย checkbox/select ในเมนู "⋯" ของห้อง (`EditorMode.vue:3182-3190`) หรือปุ่ม "ซ้ำ" ครอบทั้งบรรทัด (`:2871,2160-2169`) · **playback ทำตามจริง** (`midi.js:79-104` `expandRepeats`: วนกลับ + ข้าม volta ตาม pass)
- **D.C. / D.S. / Fine / Segno / Coda → ❌ ไม่มีในระบบ (เป็นแค่ข้อความ)** — ใส่ได้ทางเดียวคือ **ช่องป้ายข้อความอิสระ** ต่อบรรทัด (placeholder "Fine, D.C. al Fine" `EditorMode.vue:2885`) · **เป็น decoration ล้วน** เรนเดอร์เป็น text บนแผ่น (`SongSheet.vue:397`) · **ไม่มี parser/resolver อ่าน** → กดเล่นแล้ว **วิ่งตรงผ่าน ไม่กระโดด**
- **playback resolver:** `resolvePlayOrder`/`expandRepeats` รองรับแค่ repeat+volta+ท่อนรับ strophic · จุด D.C./D.S./Coda มีคอมเมนต์ว่าเป็น **"dispatch seam เฟส 2"** แต่ยังไม่มี case (`songModel.js:185-190`)

### 2.2 เลนลูกคู่ (Multi-Voice)
- **❌ ไม่รองรับเลย (ยืนยันเด็ดขาด)** — 1 ห้องเก็บ **โน้ตแนวเดียว + เนื้อเลนเดียว** · segment = `{chord, note, lyric}` เดียว ไม่มี field แนว 2/ประสาน/ลูกคู่ที่ไหนในโมเดล (`EditorMode.vue:79-89`) · คำว่า "ลูกคู่" = **0 ครั้ง** ทั้ง codebase
- แทรกเนื้อลูกคู่กลางห้อง = **ทำไม่ได้** (ไม่มี field) · ทางเดียวที่โมเดลยอม = ทำเป็น **ท่อน/section แยกเล่นเรียงกัน** (ไม่ใช่พร้อมกัน)
- (`voice/voices` ที่เจอในโค้ด = เครื่องดนตรี arranger เปียโน/ไวโอลิน/เชลโล — คนละเรื่องกับแนวร้อง)
- **ไม่มี design doc multi-voice ที่ไหนเลย** (backlog/us/ds)

### 2.3 ท่อนรับร่วม (Refrain) + เนื้อร้องมีเงื่อนไข
- **✅ ท่อนรับร่วมมีจริง** — refrain = stanza อีกอัน (เช่น `B`) เก็บทำนองครั้งเดียว ใช้ซ้ำผ่าน entry (`song-model-v2.md:56,64-65`)
- **"ร้องรับทุกข้อ" (เนื้อเดียว): ✅** flag `afterEachVerse` — แผ่นพิมพ์ครั้งเดียว playback แทรกให้ทุกข้อ (`songModel.js:187-220`)
- **กรณีข้อ 1-3 ใช้รับ A / ข้อ 4-5 รับเนื้อต่างกัน:** **ทำได้ แต่ต้องใส่เป็น entry แยกต่อครั้ง** (ทำนองเดิมใช้ซ้ำ · เนื้อใส่ต่อครั้ง) — auto `afterEachVerse` ทำ **ไม่ได้** เพราะมันซ้ำ "เนื้อชุดเดียว" ทุกข้อ (`songModel.js:209-218`) → เนื้อรับที่ต่างกันต้องวางแบบ explicit
- **สรุป:** ทำนองรับไม่ต้องพิมพ์ซ้ำเลย · เนื้อรับที่ "ต่างกัน" ใส่ต่อครั้ง (by design เพราะเนื้อคนละชุด)

## 3 · ข้อเสนอเชิงโครงสร้าง (Future) — ประเมินความเป็นไปได้

### 3.1 Measure-Based Grid
**เรามี "grid ตามห้อง" อยู่แล้วบางส่วน** — editor ปัจจุบันคือช่องโน้ตแยกจัดเป็นห้อง/บรรทัด ไม่ใช่ text box ยาว · สิ่งที่ยัง **ไม่มี** = การมองเป็น "ตารางบีต/ช่องตายตัวต่อห้อง" แบบ spreadsheet · **feasible** (โมเดลห้อง+segment รองรับ) แต่เป็นงานรื้อ UI edit surface พอควร ไม่ใช่ของฟรี

### 3.2 Layer / Track Management
- **มีแล้วบางส่วน:** ทำนอง (stanza) แยกจาก เนื้อร้องแต่ละข้อ (arrangement entries) เป็นคนละเลเยอร์อยู่แล้ว · แก้/ลบเนื้อข้อหนึ่งไม่กระทบทำนอง
- **ยังไม่มี:** มุมมอง "track" ชัด ๆ ให้เปิด/ปิด/ลบทีละเลเยอร์ · และ **เลนลูกคู่ = ต้องเพิ่มโมเดลใหม่** (§2.2) — นี่คือ **การตัดสินใจสถาปัตยกรรมใหญ่สุด** ถ้าจะไป MusicXML multi-voice

### 3.3 Visual Symbol Mapping (คลัง D.C./Fine/Repeat)
- **ยังไม่มี picker** — repeat/volta ตั้งผ่าน checkbox (ไม่ใช่คลัง) · D.C./Fine/Segno = **text อิสระ decoration**
- **ออกแบบไว้แล้วแต่ยังไม่สร้าง:** มี SA brief `docs/pm/brief-repeat-symbols-ui.md` (จากพี่เปา issues11 + P'Aim 17 ก.ค. "ทำเป็น UI เลือกง่าย ไม่เป็น text box") — **แต่ไฟล์ US/DS ยังไม่ถูกเขียน · ยังไม่มีโค้ด** · backlog B102 P2 เลื่อนไว้ ("เล่มอนุชนยังไม่ใช้")
- ⚠️ **คำเตือนใน brief (สำคัญ):** อย่าทำแค่ "text box สวยขึ้น" ที่สัญญา playback ที่ engine ทำไม่ได้ — ต้องทำ resolver ให้กระโดดจริงคู่กัน ไม่งั้นหลอกตา
