## 🔴 จุดที่เจอ (เรียงตามความสำคัญ)

### 1. [สูง · tablet+phone] auto-scroll ตอนร้อง = พยางค์ที่ร้องไปหลบอยู่ "หลังแถบเล่น (dock)"
**หน้า:** ฝึกร้อง · **นี่คือ gate ที่ค้างรับ B043 พอดี**
แถบเล่น (transport dock) ปักติดขอบล่างจอ กินพื้นที่ล่างสุด ~180px เป็น overlay ลอยทับเนื้อ
เวลาเล่น โค้ดใช้ `scrollIntoView({block:'nearest'})` เพื่อเลื่อนตามพยางค์ **แต่ 'nearest' นับพื้นที่เต็มจอว่า "เห็นแล้ว"
ไม่รู้ว่ามี dock บังอยู่** → พอพยางค์ที่ร้องตกลงมาโซนล่าง (หลัง dock) จอ **ไม่เลื่อน** เลย พยางค์ที่กำลังร้องเลยหายไปหลัง dock

**หลักฐานจริง (จอ 360×640, เล่นจริง):**
- dock เริ่มที่ y=459 (โซนเห็นเนื้อ = 0–459)
- 0:19/0:22 → บรรทัดสุดท้าย "ครองความ ชอบ / ธรรม" **หายทั้งบรรทัดหลัง dock** · `scrollY` ค้างที่ 0 ตลอด (ไม่เลื่อนเลย)
- ภาพ: `phone-playing-late.png` (นาที 0:19 บรรทัดสุดท้ายหาย) · `phone-playing`/log ยืนยัน scrollY=0
- ✅ ส่วนที่ **ทำงานดี**: การไล่ไฮไลต์โน้ต (`.nt-playing`) + กล่องคาราโอเกะบนพยางค์ (เช่น "เสมอ" ขึ้นกล่องน้ำตาล) เดินตามเสียงถูกต้อง — ปัญหาอยู่ที่ **การเลื่อนจอ** อย่างเดียว

**ทิศแก้ (รอบ 2 · เสนอ):** ให้เป้า scroll เผื่อความสูง dock — เช่น `scroll-margin-bottom` = ความสูง dock บน `[data-syl]`
หรือเปลี่ยนจาก `block:'nearest'` เป็นเล็งให้พยางค์อยู่ "แถบกลางค่อนบน" ของโซนที่เห็น (เหนือ dock) · ไฟล์: `src/components/SongViewer.vue` (`scrollToPlaying`, ~บรรทัด 203–229)
**กระทบ tablet ด้วย** (ไม่ใช่แค่ phone) — เพราะ dock ปักล่างทุกจอ · เพลงจริงยาว → เจอบ่อยกว่านี้

### 2. [กลาง · phone] แถบเล่นบังบรรทัดล่าง แม้ตอนหยุด
**หน้า:** ฝึกร้อง
แม้ยังไม่กดเล่น dock ก็ทับ 1–2 บรรทัดล่างสุดของเพลง และ **ไม่มี padding ล่าง** ให้เนื้อเลื่อนพ้น dock
(เป็นสาเหตุเดียวกับข้อ 1) · ภาพ: `phone-sing-viewport.png`
**ทิศแก้:** เพิ่ม padding-bottom ให้พื้นที่เนื้อ = ความสูง dock (แก้พร้อมข้อ 1 ได้)

### 3. [ต่ำ · phone] หน้าคู่มือ (Guide) ยาวมากบนจอแคบ
**หน้า:** คู่มือ · ที่ 360px สูงราว ~13,000px · ไม่ล้นแนวนอน · เนื้ออ่านได้แต่สกอลล์ยาว
อาจโยงกับ **B044 (spacing)** — ต้องให้ P'Aim ชี้จุด spacing ที่ตั้งใจหมายถึง · ภาพ: (ดู phone-guide ใน scratchpad · ไม่ commit เพราะไฟล์ใหญ่)

### 4. [ต่ำ/cosmetic · phone] placeholder ช่องค้นหาถูกตัด
**หน้า:** รายการเพลง · placeholder ยาวโดนตัด ("…หรือโน้") — เป็นแค่ข้อความ placeholder · ไม่กระทบใช้งาน · ภาพ: `phone-songlist.png`

### 5. [note · phone] หน้าแก้ไขแน่นมากบนจอพับ + แป้นโน้ตทับแถวบน
**หน้า:** แก้ไข · กล่องซ้อนเยอะ แน่นบน 360px · แป้นโน้ตตัวเลขปักล่าง (เหมือน dock ข้อ 1-2 บังแถวบนตอน scroll-top)
แต่การแก้ไข = tablet/desktop เป็นหลัก → phone ความสำคัญต่ำ · ภาพ: `phone-edit2.png` · (tablet แก้ไข = ปกติดี · `tablet-edit.png`)
