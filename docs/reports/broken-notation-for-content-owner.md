# ใบแจ้งแก้โน้ต — 7 จุดที่ระบบอ่านไม่ได้

**สำหรับ:** พี่เปา (คนดูแลเนื้อหา) — ไม่ต้องอ่านโค้ด
**ที่มา:** สแกนเพลงทั้งหมด 155 เพลงบนเว็บจริง เมื่อ 23 ก.ค. 2026
**สรุป:** เจอ 7 จุด ใน 5 เพลง · แบ่งเป็น **3 อาการ** · แก้ทีละอาการจะเร็วกว่าไล่ทีละเพลง

---

## จุดที่ผิดคืออะไร (อธิบายก่อน)

โน้ตในระบบเก็บเป็น **ข้อความ** เช่น `5` `1_` `.3` `#4` — และระบบมี **"ลำดับการเขียน"** ที่ตายตัว
ถ้าเขียนถูกตัวแต่**สลับลำดับ** ระบบจะอ่านไม่ออก แล้ว**พิมพ์ตัวอักษรดิบออกมาบนแผ่นเพลง**แทนที่จะวาดโน้ต
(เช่น เห็นจุด `.` ลอยๆ หรือขีด `_` ลอยๆ อยู่ข้างโน้ต) — และ**ตอนกดฟังก็จะข้ามโน้ตตัวนั้นไปเงียบๆ**

**ลำดับที่ถูกต้อง เขียนจากซ้ายไปขวา:**

```
~   #  b     .      เลข     '      _        .        ^
ไท  ชาร์ป  จุดล่าง  1-7   จุดบน  ขีดเขบ็ต  จุดเพิ่มค่า  ค้างเสียง
```

จำง่ายๆ: **`^` (ค้างเสียง) ต้องอยู่ท้ายสุดเสมอ** และ **`#`/`b` ต้องอยู่หน้าจุดล่าง**

# อาการที่ 1 — เขียน `^` ไว้ก่อนจุด/ขีด (4 จุด · 2 เพลง)

**ปัญหา:** `^` (เครื่องหมายค้างเสียง) ถูกเขียนไว้กลางตัว ทำให้จุดหรือขีดที่ตามหลังกลายเป็นขยะ
**วิธีแก้:** ย้าย `^` ไปไว้**ท้ายสุด**ของโน้ตตัวนั้น (ตัวอักษรเท่าเดิม แค่สลับที่)

> ✅ **มั่นใจสูงมาก** — ตรวจแล้วว่าเมื่อแก้ตามนี้ **จังหวะในห้องจะเต็มพอดี 3 เคาะ** ทุกห้อง
> ส่วนตอนนี้ที่ยังไม่แก้ ห้องเหล่านั้นจังหวะขาด/เกิน

| **แก้แล้วได้อะไร** | ห้องนี้จะเต็มพอดี 3 เคาะ (จาก 2.5) และเครื่องหมาย ❌ จะหายไป |

### ⚠️ ข้อควรตรวจของอาการที่ 1 — เฉพาะเพลง #760 ท่อนรับ

การแก้ข้างบน = **แก้ให้ระบบอ่านออกโดยรักษาสิ่งที่พิมพ์ไว้เดิมทุกตัว** และจังหวะลงพอดีทุกห้าง — เชื่อถือได้

**แต่มีเรื่องที่ผมตอบไม่ได้:** ในเพลง #760 ท่อนรับ มี `^` (ค้างเสียง) **ติดกันหลายตัวรวด**
(`(5^ #4^_ 4^)` = 3 ตัวติด และ `1^_ 2^_` = 2 ตัวติด)
ในทางดนตรี การใส่ค้างเสียงบนโน้ตเขบ็ตติดกันหลายตัวเป็นเรื่องไม่ค่อยเจอ
→ **ไม่แน่ใจว่าคนคีย์ตั้งใจใส่ค้างเสียงจริง หรือใช้ `^` แทนสัญลักษณ์อื่นในหนังสือต้นฉบับ**

**เสนอ 2 ทางเลือก — พี่เปาช่วยดูหนังสือต้นฉบับแล้วเลือก:**

- **ทาง ก** (ถ้าต้นฉบับมีเครื่องหมายค้างเสียงจริง) → แก้ตามตารางข้างบน คือ `#4_^` `1_^` `2_^`
- **ทาง ข** (ถ้าต้นฉบับไม่มีค้างเสียง คนคีย์ใส่มาผิด) → ลบ `^` ทิ้งไปเลย เหลือ `#4_` `1_` `2_`
  — จังหวะก็ลงพอดี 3 เคาะเหมือนกัน แต่จะไม่มีเครื่องหมายค้างเสียง

**ทั้ง 2 ทางจังหวะถูกเท่ากัน ต่างกันแค่มีเครื่องหมายค้างเสียงหรือไม่** — ต้องดูหนังสือถึงจะรู้
(สำหรับ #103 และ #760 จุดที่ 1 ที่เป็น `^` ตัวเดียวโดดๆ บนโน้ตยาว = ปกติมาก ไม่ต้องสงสัย)

# อาการที่ 3 — วงเล็บปิด `)` ไม่มีวงเล็บเปิด (2 จุด · 2 เพลง)

**ปัญหา:** มีเครื่องหมาย `)` (ปิดเอื้อน) อยู่ แต่ไม่มี `(` (เปิดเอื้อน) มาก่อนในบรรทัดนั้น
**ผลที่เกิด:** เส้นโค้งเอื้อนอาจไม่ถูกวาด หรือวาดคร่อมผิดที่

> ⚠️ **อันนี้ผมไม่แน่ใจว่าควรแก้เป็นอะไร — ต้องดูหนังสือต้นฉบับ**
> เพราะ**จังหวะเท่าเดิมทั้งสองทาง** เลยไม่มีตัวช่วยชี้ว่าอันไหนถูก
> เป็นไปได้ทั้ง (ก) `)` เป็นตัวเกินที่พิมพ์หลุดมา หรือ (ข) มี `(` หายไปจริง

### 3.1 · เพลง (ไม่มีเลขเพลง) "หากท่านรู้จักพระเยซู"

| | |
|---|---|
| **อยู่ตรงไหน** | ทำนอง **ท่อน A** · **บรรทัดที่ 1 จาก 4** · **ห้องที่ 5** |
| **ตอนนี้พิมพ์ไว้** | `~6) - 3` |
| **รอบข้าง** | ห้อง 4 = `6 6~ -` (มีเครื่องหมายไท `~` โยงข้ามห้องมาที่ห้อง 5 อยู่แล้ว) |
| **ทำไมระบบไม่รับ** | มีวงเล็บปิด `)` แต่ไม่มีวงเล็บเปิด `(` มาก่อนในบรรทัดนี้ |

**เสนอ 2 ทางเลือก:**
- **ทาง ก (ผมเดาว่าน่าจะใช่)** → **ลบ `)` ทิ้ง** เหลือ `~6 - 3`
  เหตุผล: การโยงเสียงข้ามห้องตรงนี้ทำด้วยเครื่องหมายไท `~` อยู่แล้ว (`6~` ในห้อง 4 → `~6` ในห้อง 5)
  จึงไม่ต้องมีวงเล็บเอื้อนซ้อนอีก
- **ทาง ข** → ถ้าในหนังสือมีเส้นโค้งเอื้อนคร่อมโน้ตหลายตัวอยู่จริง ให้เติม `(` ไว้หน้าโน้ตตัวแรกที่เส้นโค้งเริ่ม
  (ต้องดูรูปว่าเส้นโค้งเริ่มตรงไหน — ผมบอกไม่ได้)

### 3.2 · เพลง #90 "ตักน้ำด้วยความชื่นชมยินดี"

| | |
|---|---|
| **อยู่ตรงไหน** | ทำนอง **ท่อน A** (ร้อง 1) · **บรรทัดสุดท้าย (บรรทัดที่ 8 จาก 8)** · **ห้องที่ 1** |
| **ตอนนี้พิมพ์ไว้** | `1_ {1_ .6_ 1_})` ← มี `)` ห้อยท้ายสุด |
| **ทำไมระบบไม่รับ** | `{ }` (โน้ตสามพยางค์) เปิด-ปิดครบดีแล้ว แต่ `)` ที่ต่อท้ายไม่มี `(` คู่ของมัน |

**เสนอ 2 ทางเลือก:**
- **ทาง ก (ผมเดาว่าน่าจะใช่)** → **ลบ `)` ทิ้ง** เหลือ `1_ {1_ .6_ 1_}`
  เหตุผล: `)` อยู่ติดหลัง `}` พอดี ดูเหมือนพิมพ์เกินมาตอนใส่วงเล็บปีกกา
- **ทาง ข** → ถ้าในหนังสือมีเส้นโค้งเอื้อนคร่อมกลุ่มนี้จริง ให้เติม `(` ไว้หน้า `1_` ตัวแรก
  → `(1_ {1_ .6_ 1_})`

## ⚠️ ข้อควรระวังตอนแก้

1. **แก้ในทำนอง = แก้ทุกข้อพร้อมกัน** — เช่น #760 ท่อน A ใช้ร่วมกัน 7 ข้อ · #103 ใช้ร่วมกัน 6 ข้อ
   แก้ที่เดียวก็ถูกทั้งหมด **ไม่ต้องไล่แก้ทีละข้อ**
2. **แก้แล้วเช็กเครื่องหมายใต้ห้อง** — เลข "x/3 จังหวะ" ควรกลายเป็น **3/3 ✓** ในทุกห้องที่แก้ตามอาการ 1
   ถ้ายังขึ้น ❌ แปลว่ายังไม่ครบ ให้ทักกลับมา
3. **จำนวนช่องคำร้องอาจขยับ** — โน้ตที่เคยอ่านไม่ออกจะกลายเป็นโน้ตจริง
   ระบบอาจแจ้งว่าคำร้องไม่ครบ ให้ใช้ปุ่ม ◀ ▶ ใต้ช่องคำเลื่อนคำให้ตรงโน้ต
4. **⛔ ห้ามใช้วิธีนำเข้าไฟล์ทับทั้งเพลง** — ให้แก้ในหน้า "แก้" ทีละจุดเท่านั้น (ทีมกำลังทำงานบนข้อมูลชุดนี้อยู่)

---

## สิ่งที่ใบนี้ยังตอบไม่ได้ (พูดตรงๆ)

- **ไม่ได้เปิดหนังสือต้นฉบับ** — ทุกอย่างในใบนี้อ่านจากข้อมูลในระบบล้วน
  ทางเลือก "ก/ข" ทั้ง 4 จุดจึงต้องให้พี่เปาเทียบกับหนังสือ
- **ไม่รู้ว่าใครเป็นคนคีย์ผิด** — เป็นคนพิมพ์เอง หรือมาจากตอนแปลงไฟล์เข้าระบบ ยังไม่ได้ตรวจ
  (ถ้ามาจากตัวแปลงไฟล์ ควรแก้ตัวแปลงด้วย ไม่งั้นเพลงชุดหน้าจะพังแบบเดิมอีก)
- **ตรวจเจอเฉพาะที่ระบบ "อ่านไม่ออก" กับ "วงเล็บไม่ครบคู่"** — โน้ตที่ระบบอ่านออกแต่**เสียงผิด**
  (เช่น คีย์ผิดตัว ผิดช่วงเสียง) ตรวจแบบนี้ไม่เจอ ต้องใช้หูฟังเอา
