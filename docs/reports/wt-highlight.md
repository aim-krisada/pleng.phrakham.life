## ทำอะไร
ไฮไลต์ตอนเล่นเพลง เดิน **ทีละพยางค์/โน้ต** แทนที่จะสว่างทั้ง segment ค้างพร้อมกัน — คำ + ตัวเลขโน้ตที่กำลังร้องเด่นขึ้นพร้อมกัน และ **แตะคำ/โน้ต = กระโดดไปเริ่มเล่นตรงนั้น**

## หลักการ (invariant เดียวที่ทุกอย่างพึ่ง)
`midi.js` แปะ **ดัชนีช่องพยางค์ในแต่ละ segment (`syk`)** ลงบนโน้ตที่ดีด (attack) แต่ละตัว — นับทุก "กล่องโน้ต" (โน้ต / ตัวหยุด / `-`) ยกเว้นวงเล็บ จึงตรง 1:1 กับ
- span รายพยางค์ที่ `SongSheet` เรนเดอร์ (`data-syl="li-si-k"`), และ
- ตัวเลขแต่ละตัวใน `NoteRow`

ตัวหยุด/โน้ตลาก/เอื้อน (slur พิทช์เดียว) **ไม่กิน syk** → ไฮไลต์ค้างที่คำเดิมตลอดที่ลากเสียง (จัดการ melisma ถูกต้องตาม v2)

## ไฟล์ที่แก้
- `src/lib/midi.js` — `songToNotes` เพิ่ม `syk` ต่อ attack (นับ slot ต่อ segment · reset ทุก segment · รอด `expandRepeats`/`mergeTies`)
- `src/lib/songModel.js` — `resolveContent` แนบ `segment.syllables[]` (โทเคนดิบ เก็บช่องว่างไว้) คู่กับ `lyric` เดิม
- `src/components/SongSheet.vue` — เรนเดอร์ 1 span/พยางค์ (v2) · prop `playingSyl` + `interactive` · emit `seek` · v1 ไม่มี `syllables` → fallback ไฮไลต์ทั้ง segment แบบเดิม
- `src/components/NoteRow.vue` — prop `active` = ไฮไลต์ตัวเลขที่กำลังเล่น
- `src/components/SongViewer.vue` — เก็บ `playingSyl` จาก `onNote` (เฉพาะ attack ที่มี syk) · ส่ง props · `onSeek` หา index แล้วเล่นต่อจากจุดนั้น
- เทสต์ใหม่: `Highlight.syllable.test.js` (12) + SongViewer play เพิ่ม 3
