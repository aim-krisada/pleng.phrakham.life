## สรุปสั้น

**P'Aim ได้ยินถูก — เฟอร์มาตาลากเสียงน้อยมากจนแทบไม่รู้สึก แต่ต้นเหตุไม่ใช่อย่างที่ brief เดาไว้**

โค้ด `d *= 1.75` **ทำงานอยู่จริง ไม่ได้อยู่ในทางตัน** แต่มันคูณแค่ **กล่องแรกของโน้ต** กล่องเดียว —
ส่วน `-` (ขีดยืดเสียง) ถูกบวกเข้าไป **ทีหลัง** โดยไม่ได้คูณด้วย

**ผลคือบั๊กนี้กลับหัวกับการใช้งานจริงทางดนตรี:** ยิ่งโน้ตยาว เฟอร์มาตายิ่งอ่อน —
แต่เฟอร์มาตาถูกเขียนบนโน้ตยาวท้ายวรรคเป็นหลัก ซึ่งเป็นกรณีที่โค้ดเราทำงานได้แย่ที่สุดพอดี

| ที่พิมพ์ | ควรได้ (มาตรฐาน ×1.75) | **ของเดิมได้จริง** | อัตราจริง |
|---|---|---|---|
| `1^` (1 จังหวะ) | 1.75 | 1.75 | **1.750×** ✅ ถูกอยู่แล้ว |
| `1^ -` (2 จังหวะ) ← **เพลง #120** | 3.5 | 2.75 | **1.375×** ❌ ต่ำกว่าขั้นต่ำมาตรฐาน (1.5×) |
| `1^ - -` (3 จังหวะ) | 5.25 | 3.75 | **1.250×** ❌ |
| `1^ - - -` (4 จังหวะ) | 7.0 | 4.75 | **1.188×** ❌ แทบไม่ได้ยิน |

เพลง #120 ใช้ `1^ -` ทั้ง 6 ตัว → ได้ **1.375×** ทั้งเพลง = ต่ำกว่าช่วง 1.5–2× ที่พี่เปาอ้างถึง
นี่คือเสียงที่ P'Aim ฟังแล้วบอกว่า "ไม่ลากเสียงจริง"

**⛔ ไม่ได้แตะเลข 1.75 ตามที่ brief สั่ง** — แก้ที่ "ไม่ทำงาน" ไม่ใช่ที่ตัวเลข

## ต้นเหตุจริง + วิธีแก้

`src/lib/midi.js` · `songToNotes()` — ตัวจัดการกล่อง `-` (ext) บวกจังหวะดิบเข้าไปหลังการคูณ:

```js
// เดิม — บวก 1 จังหวะดิบ ไม่สนใจว่าโน้ตนี้มีเฟอร์มาตาหรือไม่
if (bn.length) bn[bn.length - 1].beats += 1 * f
```

แก้เป็น: กล่อง `-` เป็น **ส่วนหนึ่งของโน้ตตัวเดียวกัน** → เฟอร์มาตาต้องยืดมันด้วย

```js
const last = bn[bn.length - 1]
if (last) last.beats += 1 * f * (last.fermata ? FERMATA_FACTOR : 1)
```

คูณทีละกล่อง = คูณทั้งโน้ต (distributive): `(1+1) × 1.75` = `1.75 + 1.75` = 3.5 ✅
พร้อมยก `1.75` ขึ้นเป็น `export const FERMATA_FACTOR` (ค่าเท่าเดิม) เพื่อไม่ให้เลขนี้กระจายอยู่ 2 ที่แล้ว drift กัน

ตรงกับมาตรฐานที่พี่เปาส่งมาเอง: *"เฟอร์มาตาอยู่บนโน้ตตัวขาว (2 จังหวะ) → ให้ลากเพิ่มเป็น 4 จังหวะ"* — คูณ **ค่าที่เขียนไว้ทั้งตัว** ไม่ใช่แค่กล่องแรก

## ข้อเสนอเรื่องคู่มือ (ไม่ได้แก้ไฟล์ · รอ PM เคาะ)

brief บอกว่าคู่มือไม่มีเฟอร์มาตา แต่**มีอยู่แล้ว** ที่ `NotationStandard.vue:43`:

> `3^` — เฟอร์มาตา (fermata — โค้งมีจุดเหนือโน้ต) = ยืดเสียงยาวกว่าปกติ ตามผู้นำเพลง — ตอนกดฟัง เสียงจะลากยาวขึ้นให้

**ที่ยังขาด** คือคำอธิบายว่า *มันคืออะไรในความรู้สึก* ซึ่ง**พี่เปาเขียนมาเองแล้วและอธิบายดีกว่าที่เราจะเขียนได้** (เขาเป็นคนพิมพ์เพลงเข้าระบบทุกเพลง) — เสนอเพิ่มกล่องอธิบาย โดยใช้ถ้อยคำของเขาและให้เครดิตเขา:

> **เฟอร์มาตา (`^`) — เหมือนกดปุ่มสโลว์โมชั่นในหนัง**
> "มันคือจังหวะที่หนังกำลังจะข้ามไปฉากต่อไป แล้วผู้กำกับอยากให้คนดูอินกับอารมณ์ตรงนี้มาก ๆ เลยกดสโลว์ภาพให้ช้าลง หรือค้างภาพนั้นไว้แป๊บนึง เพื่อให้คนดูได้ลุ้น/ดื่มด่ำ ก่อนจะปล่อยให้หนังเล่นต่อในความเร็วปกติ"
> — คำอธิบายโดยพี่เปา
>
> จะลากยาวแค่ไหน = ตามผู้นำเพลง ไม่มีกฎตายตัว · ในแอปนี้ตอนกดฟังจะลากให้ **×1.75 ของค่าที่เขียนไว้** (โปรแกรมโน้ตสากลอย่าง Sibelius/MuseScore/Finale ใช้ 1.5–2 เท่า)

## ที่ไม่ได้แก้ (เจตนา · KISS ตาม "ทำให้จบเร็ว ไม่รื้อเกินจำเป็น")

`midi.js` มีสาย `#`-ยกทั้งห้อง รอคิวใช้ต่อ → แก้เท่าที่จำเป็น 3 บรรทัด ไม่รื้ออย่างอื่น

**edge case ที่รู้ว่ามีแต่ไม่แตะ (ยังไม่มีข้อมูลจริงในคลังสักเพลง — 0/124):**
- **tie (`~`) ข้ามห้องที่มีเฟอร์มาตา** (`1^~ | ~1`) — `mergeTies` เอา beats ของโน้ตหลังมาบวกโดยไม่คูณ 1.75
- **slur ทับโน้ตเสียงเดียวกัน (เอื้อน) ที่ตัวแรกมีเฟอร์มาตา** — ตรรกะเดียวกัน

ทั้งคู่เป็นอาการเดียวกับบั๊กนี้เป๊ะ (คูณไม่ครบทั้งโน้ต) แต่ **ยังไม่มีเพลงไหนเขียนแบบนี้** → ถ้า DA encode ties เพิ่ม (B068) ควรกลับมาดู · แจ้ง PM ไว้เป็น follow-up ไม่ใช่ scope นี้
