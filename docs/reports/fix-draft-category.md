# B104 — หมวด/เล่ม หาย แล้วถูกเปลี่ยนเป็น "อนุชน" เอง

Branch `fix-draft-category` (base = live round 33 `23f2e19`). Phase 2 = code-only, ships now.
Phase 1 = SQL prepared but **not** activated.

## อาการที่รายงาน

ผู้เขียนใส่เพลงไว้ที่ **เล่มใหญ่** · พอพี่เปาเปิดขึ้นมาแก้ กลายเป็น **อนุชน** · ถ้ากดเผยแพร่
ต่อ เพลงนั้นก็ถูกย้ายเล่มจริงในฐานข้อมูล (ข้อมูลเสียหายเงียบๆ)

## Root cause — ยืนยันแล้วจากโค้ดจริง

`song_drafts` **ไม่มี** คอลัมน์ `category` และ `theme` เลย (`db/002-draft-review-system.sql`
บรรทัด 40-52) — มีแต่ตาราง `songs`

1. `draftRow()` เก็บแค่ `song_id, number, title_th, title_en, content` → หมวดที่เลือกไว้หายทันที
   ตอนบันทึกร่าง
2. เปิดร่างกลับมา → `loadDraft()` → `applyRow(d)` → `meta.category = data.category ?? 'anuchon'`
   → แถวร่างไม่มีคีย์นี้ → **ตกไปที่ 'อนุชน'**
3. ตอนเผยแพร่ `saveDirect()` เขียน `row.category = meta.category || 'anuchon'` ทับลง `songs`
   → เพลงถูกย้ายเล่มเงียบๆ
4. เส้นทาง **ผู้อนุมัติ** (`approve()`) พังแบบเดียวกัน — ส่ง `category: meta.category || 'anuchon'`
   เข้า RPC `approve_and_publish` · และ RPC ตัวนี้ `coalesce(p_song->>'category', category)`
   (ปลอดภัยถ้าไม่ส่ง) แต่ `theme = p_song->>'theme'` **ไม่ coalesce** → ถ้าส่งค่าเดาไป ธีมหายด้วย

ตัวเลขในฐานข้อมูลวันนี้: anuchon 46 / lem-yai 29 / dek-lek 21 — ใน 46 นั้นบางเพลงน่าจะโดนย้าย
มาด้วยบั๊กนี้ (การไล่ย้อนหลัง = งานแยก ไม่อยู่ในรอบนี้)

### Phase 1 — after SQL (โค้ดที่ต้องเปิดใช้ *หลัง* รัน SQL เท่านั้น)

ยังไม่แก้ตอนนี้เพราะคอลัมน์ยังไม่มีจริง → เขียนไปจะ error ทั้งการบันทึกร่าง

1. `draftRow()` (~L1417) เพิ่ม 2 บรรทัด:
   ```js
   category: meta.category || null,
   theme: meta.theme || null,
   ```
2. `applyRow()` — ไม่ต้องแก้ตรรกะเลย: `hasOwnProperty(data,'category')` จะกลายเป็น true
   สำหรับแถวร่างเองโดยอัตโนมัติ → `bookKnown` ถูกต้องทันที และ `meta.category = data.category ?? 'anuchon'`
   ก็อ่านค่าที่ร่างเก็บไว้ได้เลย
   *(ทางเลือกที่เนี้ยบกว่า: ถ้าร่างเก็บ category เป็น null และมี `song_id` ให้ `resolveBook()`
   ทำงานต่อเหมือนเดิม — เงื่อนไข `if (!bookKnown.value && d.song_id)` ใน `loadDraft` ครอบให้แล้ว
   ถ้าเปลี่ยนเป็นเช็ค `meta.category == null` แทน)*
3. `saveDirect()` / `approve()` — ไม่ต้องแก้ ยังเป็นกติกาเดิม (เขียนเฉพาะค่าจริง)
4. อัปเดตเทสต์ `src/components/EditorMode.category.test.js` เคส "draft of a NEW song" ให้ยืนยันว่า
   ตอนนี้หมวดอยู่รอดข้ามการบันทึกร่างแล้ว
