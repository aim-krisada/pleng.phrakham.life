## ทำอะไรไป (ไฟล์เดียว — ตามขอบเขต)
`src/lib/songSearch.js` ขยาย · **ไม่แตะ** notation/lint/NoteRow/Guide/UI:

- **เนื้อ v2:** `lyricsText` อ่าน `content.arrangement[].syllables` แล้ว (เดิมอ่านแค่ v1 `lines`) — join พยางค์เป็นข้อความค้นได้ · token ขึ้นต้น `-` = ต่อคำเดิม (`'มั่น'+'-คง'` → `มั่นคง` ค้นเป็นคำเดียว) · ช่องว่าง (โน้ตค้าง/พัก) ข้าม · **ผลพลอยได้: snippet ในหน้ารายการเพลงโชว์เนื้อ v2 ได้แล้ว** (เดิมว่าง)
- **normalize ไทย:** `normalize()` = NFC (รวมสระ/วรรณยุกต์เป็นรูปมาตรฐาน) + lower-case + ยุบช่องว่าง · `compact()` = ตัดช่องว่างทิ้ง (ไทยไม่มีเว้นวรรคระหว่างคำ → คนพิมพ์เว้นวรรคมั่ว) ใช้ตอน fuzzy จะได้ไม่คิดค่าปรับต่อช่องว่าง
- **fuzzy:** `fuzzyDistance()` = approximate substring match (แก้ไขน้อยสุดของ pattern เทียบ substring ใดๆ ในข้อความ · DP O(text×pat) · early-exit เมื่อเกินงบ) · `fuzzyBudget()` = จำนวน typo ที่ทน ตามความยาว query (สั้น <4 ตัว = exact อย่างเดียว กันเจอมั่ว · ยาวขึ้นทนได้ถึง 3)
- **API เดิมคงไว้:** `filterSongs(songs, q)` ยังคืน array กรอง (คงลำดับ catalog) — ตอนนี้ fuzzy + เนื้อ v1/v2 · เพิ่ม `searchSongs(songs, q)` คืนผล **จัดอันดับ** (exact ก่อน fuzzy · `{song, score}`) ไว้ให้ UI picker รอบหน้าใช้ · `scoreSong()` = 0 (exact substring) หรือระยะแก้ไข (fuzzy) หรือ null (ไม่ match)

**เหตุผล design (KISS):** exact substring ลองก่อน (ถูก · score 0) → พลาดค่อย fuzzy เฉพาะที่พลาด → เร็วพอที่ 120 เพลง · query สั้นไม่ fuzzy กันผลลวง
