# wt-b027 — double-dot (`..` = ×1.75)

**branch:** `wt-b027` (ฐาน `studio-shell-redesign` @ `9baddb8`) · **backlog:** B027 (approved)
**Network URL (LAN test มือถือ):** http://192.168.1.124:5350/ · port 5350 · `--host`

## ทำอะไร
ขยายระบบความยาวโน้ต jianpu ให้รองรับ **จุดคู่** — พิมพ์ `..` ต่อท้ายโน้ต = ×1.75 ของค่าฐาน
(เดิมมีแค่จุดเดียว = ×1.5) อ้าง Wikipedia jianpu "two dots = +¾" (1 + ½ + ¼ = 1.75).

## การออกแบบ (สั้น)
- token เดิมมีแค่ `dotted` (boolean) → เพิ่ม **`dots` (0/1/2)** เป็นตัวหลัก · คง `dotted = dots > 0`
  ไว้เป็น alias เพื่อไม่ให้ที่อื่นพัง
- ค่าคูณความยาวรวมไว้ที่เดียว: **`DOT_FACTOR = [1, 1.5, 1.75]`** export จาก `notation.js`
  ใช้ทั้ง `beatCount` (นับจังหวะห้อง) และ `midi.tokenBeats` (ความยาวตอนเล่น) — SSOT เดียว
- **การแยกจุด augment กับจุดออกเทฟ (สำคัญ):** จุดที่ "นำหน้าเลขตัวถัดไป" = จุดออกเทฟต่ำของตัวนั้น
  ไม่ใช่ augment ของตัวนี้ · กฎ: ที่ **ท้าย token** เก็บจุด augment ได้ถึง 2 จุด · แต่ถ้าตามด้วยเลข
  ให้คงพฤติกรรมเดิม (1 จุด augment · ที่เหลือเป็นออกเทฟ) → ข้อมูลเก่าอย่าง `5..5` = "5. แล้ว .5" ไม่เปลี่ยน

## ไฟล์ที่แตะ (ตามขอบเขต brief)
- `src/lib/notation.js` — parse `..` (field `dots`) + `beatCount` (×1.75) + export `DOT_FACTOR`

- `src/views/Guide.vue` — เพิ่มแถวอธิบาย `5..` = ประจุดคู่ (×1.75)
