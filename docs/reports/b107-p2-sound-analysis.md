## 0. เครื่องยนต์ร่วม (ทั้ง 3 version ใช้เหมือนกัน)

**ที่มาของโน้ต:** โน้ตตัวเลข (movable-do) → MIDI + คอร์ด (lead-sheet) · แต่ละโน้ต = `{midi, beats, chord}`

**Voice-leading (เลือกเสียงคอร์ด):** เสียงบนของคอร์ด (3rd/5th/7th) วางในหน้าต่าง MIDI 48–67 โดยเลือก octave ที่**ใกล้คอร์ดก่อนหน้าที่สุด** (common tone ค้าง) → คอร์ดขยับน้อย ไม่กระโดด · เบส = รากในย่านต่ำ
```js
function voiceLed(ch, prevUp){ // เลือก octave ของแต่ละ pitch-class ให้ใกล้ค่าเฉลี่ยคอร์ดก่อนหน้า
  up = pcs.map(pc => nearestOctaveTo(pc, avg(prevUp)));  bass = root in low register; }
```

**Dynamics แกนหลัก (2 ตัวที่ใส่ครบทุก version):**
- **Metric accent** (เน้นจังหวะตกในห้อง 4/4): บีต1 ×1.0 · บีต3 ×0.9 · บีต2,4 ×0.8 · เศษ ×0.72
- **Melodic contour** (รูปทำนอง): ไต่ขึ้น ×+0.06 · โน้ตยอด ×+0.06 · ลง ×−0.04 · โน้ตยาว/จบวรรค ×−0.06

**Humanize (ใส่ครบทุก version · ค่าที่ P'Aim เคาะ):**
- **Velocity jitter** ±6% ต่อโน้ต · **Micro-timing** ±12 ms ต่อโน้ต (สุ่มเลื่อนเวลา)
```js
gain    = 0.35 * accent(beat) * contour(i) * (1 + 0.06*rnd());   // rnd() = −1..1
timeShift = 0.012 * rnd();                                        // ±12ms
```

## 1. 🥇 Version A — Grand Piano เดี่ยว (พอใจที่สุด)
**ไฟล์:** `docs/spikes/humanize-timbre-demo.html` · **เสียง:** Splendid Grand จริง (+ Felt = กรอง low-pass 1.5kHz)

### เทคนิคที่ใส่จริง ✅
| เทคนิค | รายละเอียด/ค่า |
|---|---|
| เสียงเปียโนจริง | Splendid Grand (PP velocity layer + makeup) |
| Humanize (vel+time) | ±6% / ±12ms |
| Metric accent + contour | ตามแกนร่วม §0 |
| **Pedal bass** | เบสรากค้างยาวเต็มคอร์ด (`fire(bass, bT, fullDur, cg*1.5)`) — ลุ่มลึก |
| **Arpeggio มือซ้าย** | broken chord ไล่ต่อบีต `seq=[up0,up1,up0,up1]` (โหมด "จัดเต็ม") |
| โหมดตรวจโน้ต | ปิดลูกเล่น = คอร์ดค้าง block ตรง ๆ (พี่เป้า) |
| Felt tone | low-pass 1.5kHz = นุ่ม (โหมดสงบ) |

### ทำไมเพราะสุด (สมมุติฐาน)
1. **sample คุณภาพสูง + timbre เดียว** — ไม่มีเสียงเครื่องอื่นมาชนย่าน
2. **เทคนิคเปียโน map กับ sample ตรง** — เปียโนออกแบบมาเล่น arp/pedal อยู่แล้ว
3. **จุดพลาดน้อยสุด** — ตัวแปรน้อย = คุมง่าย

### แต่ยังไม่ได้ใส่ (ออกแบบไว้แต่ปิดอยู่) ⬜
drop-2 / open voicing · **rubato ปลายวรรค** · **section dynamics** (verse เบา→chorus เต็ม) · embellishment สุ่ม (ประกายอ็อกเทฟ sparkle · เติมช่องคอร์ดยาว gapFill) · added tension (add9/maj7) · walking bass

## 2. 🥈 Version B — Guitar ไนลอนเดี่ยว
**ไฟล์:** `docs/spikes/guitar-solo-demo.html` · **เสียง:** ไนลอน CC จริง (48 ไฟล์ · custom buffer sampler · pitch-shift)

### เทคนิคที่ใส่จริง ✅
| ลาย | รายละเอียด/ค่า |
|---|---|
| **รูดจังหวะ (strum · default)** | จังหวะ **D _ D U _ U D U** · รูดลง = กวาดต่ำ→สูง 26ms/สาย (มีเบส · gain 0.30) · รูดขึ้น = สูง→ต่ำ 20ms/สาย (ไม่มีเบส · gain 0.18) |
| **เกาสาย (Travis)** | นิ้วโป้งเบสสลับ ราก/คู่5 ลงบีต + นิ้ว i-m-a ไล่ขึ้น 3 โน้ตในบีต (offset +0.25/+0.47/+0.69) |
| **กรีดรัว (rasgueado)** | กรีด 4 ที cascading ต้นคอร์ด (t+0/38/76/114ms · rake 12ms · gain ไล่ 0.12→0.27) + strum body เบา |
| **สไลด์เข้าโน้ต** | grace โน้ต −2 semitone ~55ms ก่อนโน้ตยาว (โอกาส 22%) |
| Humanize + accent + contour | ตามแกนร่วม |
| ปล่อยสายกังวานเอง | ไม่ตัดเสียง (ไนลอน decay ~2.4s) |

### จุดสังเกต
- ต่างจากเปียโนชัด (มีเสียงรูด/กรีด) — P'Aim เคยติว่า "เหมือนเทคนิคเปียโน" ตอนใช้ GM → **แก้โดยใส่ลายกีตาร์แท้ + ใช้ไฟล์จริง**
- ข้อจำกัด: sample เป็น pluck โน้ตเดี่ยว → เสียงรูดเป็นการยิงโน้ตเหลื่อม ไม่ใช่การกวาดสายจริง (coupling ระหว่างสายไม่มี)

### ยังไม่ได้ใส่ ⬜
hammer-on/pull-off · palm-mute (มือขวากันเสียง) · harmonics · การเปลี่ยนลายตามท่อน (section) · rubato · การจับ voicing ตามรูปคอร์ดจริงบนเฟร็ต (ตอนนี้ใช้ voice-leading เปียโน)

## 3. 🥉 Version C — รวมวง (แย่สุด · ต้องการคำแนะนำมากสุด)
**ไฟล์:** `docs/spikes/ensemble-real-demo.html` · **เสียง:** Grand + เชลโล CC + ไวโอลิน CC (จริงหมด)

### เทคนิคที่ใส่จริง ✅
| ชั้น | เครื่อง | ทำอะไร |
|---|---|---|
| **นำ (guide)** | เปียโน หรือ ไวโอลิน | ทำนองชิ้นเดียว + swell โน้ตยาว (+0.5s) + slide (ไวโอลิน) |
| **เคลื่อนไหว (motion)** | เปียโน | arpeggio ไล่คอร์ดต่อบีต + fill อ็อกเทฟบางบีต (chorus) |
| **โอบอุ้ม (foundation)** | เชลโล (เบส) + ไวโอลิน (pad) | เชลโล re-bow ทุก 3 บีต · ไวโอลินลากยาวเป็น pad (chorus เท่านั้น) |
| **มิติหน้า-หลัง** | — | เปียโน = near reverb (หน้า) · เชลโล/ไวโอลิน = far reverb ยาว (หลัง) |
| **ความหนาแน่นตามท่อน** | — | verse (46% แรก) = เปียโน+เชลโลโปร่ง · chorus = ไวโอลิน pad เอ่อเข้ามา |

### Balance (วัดจริงด้วย AnalyserNode)
| เครื่อง | peak | dB | บทบาท |
|---|---|---|---|
| เปียโน | 0.526 | **−5.6** | นำ (ดังสุด) |
| เชลโล | 0.144 | −16.8 | เบส (ใต้ ~11dB) |
| ไวโอลิน | 0.046 | −26.7 | pad (หลัง) |

### ทำไมแย่สุด (สมมุติฐาน — ให้ที่ปรึกษาช่วยยืนยัน/แก้)
1. **ตัวแปรเยอะสุด** → จุดพลาดเยอะ · เพิ่งเจอบั๊กเปียโนเงียบ + สายดังกลบ (แก้แล้วด้วยเครื่องวัด)
2. **sample คนละคุณภาพ/ระดับ** — เชลโล/ไวโอลิน CC (baked +9/+10dB · solo · ไม่ loop) vs เปียโน PD → timbre/หัวเสียงไม่กลมกลืน
3. **ไวโอลินเป็น pad ไม่สมจริง** — ใช้ไวโอลิน solo ลากยาวแทน string ensemble → เสียงบางกว่า pad จริง
4. **ยังใส่กฎไม่ครบ** (สำคัญ! ดู §5) — role-prominence (หลบพระเอก) · voice-leading ลึก · rubato · crescendo ยังไม่มี → เลยยัง "แบน/ไม่กลม" เทียบเดี่ยว

### ยังไม่ได้ใส่ ⬜ (เยอะสุด)
role-prominence (พระเอกเด่น→คนอื่นหลบ · call-response) · rubato/crescendo อัตโนมัติตามท่อน · drop-2/open · walking bass · voice-leading ที่ลึกกว่านี้ · pad ที่ crossfade เนียน (ตอนนี้ re-trigger ห้วน) · การเลือกเครื่องตามอารมณ์เพลง (auto-instrumentation)

## 5. ★ สิ่งที่ "ออกแบบไว้แล้ว แต่ยังไม่ได้เปิดในเดโม" (ตอบ P'Aim ตรง ๆ)
สเปก `docs/ds/instrument-arranger-p2.md` ออกแบบเทคนิคเหล่านี้ไว้ครบ แต่ **เดโมยังไม่ได้ wire** — คือ "ลูกเล่น" ที่ยังเหลือให้เปิด/ปั้น:

| เทคนิค (ออกแบบไว้) | เปียโน | กีตาร์ | รวมวง | หมายเหตุ |
|---|---|---|---|---|
| **Rubato** (ยืดจังหวะปลายวรรค) | ⬜ | ⬜ | ⬜ | กินใจขึ้นเยอะ · ยังไม่เปิดเลยสักตัว |
| **Section dynamics** (verse เบา→chorus เต็ม, ระดับเสียง) | ⬜ | ⬜ | ✅(density เท่านั้น) | เปียโน/กีตาร์ยังดังเท่ากันทั้งเพลง |
| **drop-2 / open voicing** | ⬜ | ⬜ | ⬜ | คอร์ดกว้าง โปร่ง อิ่ม |
| **Embellishment สุ่ม** (sparkle อ็อกเทฟ · gapFill เติมช่อง) | ⬜ | ⬜ | ✅(บางส่วน) | "ไม่เหมือนทุกครั้ง" เหมือนคนเล่น |
| **Role-prominence** (พระเอกเด่น→คนอื่นหลบ) | — | — | ⬜ | **หัวใจแก้เต็มวงจืด** ยังไม่มี |
| **Crescendo อัตโนมัติ** | ⬜ | ⬜ | ⬜ | ทวีอารมณ์ |
| **Walking bass** | ⬜ | ⬜ | ⬜ | เบสเดินเชื่อมคอร์ด |
| **Added tension** (add9/maj7) | ⬜ | ⬜ | ⬜ | สีสัน CCM (opt-in) |
| **Voice-leading ลึก** (score ทั้ง voicing) | basic | basic | basic | ตอนนี้เลือก per-note |

## 6. คำถามถึงที่ปรึกษา (จุดที่อยากได้คำแนะนำ)
1. **เต็มวงทำไมยังไม่กลม** — เป็นเพราะ (ก) ยังไม่มี role-prominence/call-response (ข) sample คนละคุณภาพ (ค) ไวโอลิน solo แทน pad จริง หรือ (ง) อื่น ๆ? ควรแก้อะไรก่อน?
2. **ลำดับความสำคัญของเทคนิคที่ยังปิด (§5)** — ถ้าเปิดเพิ่มได้ทีละตัว ตัวไหนให้ผลต่อความเพราะสูงสุด (โดยเฉพาะเต็มวง)?
3. **pad จริง** — เราไม่มี string-ensemble ที่ loop ได้ (ทุก sample เป็น one-shot · ไวโอลิน solo 14s ยาวสุด) → จะทำ "pad ค้างเนียน" ยังไงดี (crossfade re-trigger? layer หลายตัว?)
4. **timbre กลมกลืน** — เปียโน PD + เชลโล/ไวโอลิน CC หัวเสียงต่างกัน → มี EQ/reverb bus trick ให้ฟังเป็น "วงเดียวกัน" ไหม?
5. **เดี่ยว > รวม** — ที่ปรึกษาเห็นด้วยไหมว่าควรทำเดี่ยวให้สุดก่อน แล้วรวมค่อยเอา solo ที่ดีมาผสม (ตามที่คุยไว้)?
