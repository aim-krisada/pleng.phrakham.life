# DS — B045: แถบบนมือถือ (คืนที่ให้ชื่อเพลง)

> **ประเภท:** ข้อเสนอออกแบบ (SA) — **docs อย่างเดียว · ยังไม่โค้ด** · ต้องให้ P'Aim เคาะทิศก่อน (ขัด B009)
> **อ้างอิง:** backlog B045 · B009 (เคาะ "ไม่มี hamburger") · US/DS `ps3-shell` S4 · US/DS `ps3-dock` · img `docs/pm/realuse-assets/sing-redesign-annotated.png`

---

## สรุปสั้น (F60+)

มือถือ **ชื่อเพลงถูกตัดเหลือ "1. ..."** เพราะแถวบนแถวเดียวยัด 5 อย่าง: ชื่อแบรนด์ยาว + ชื่อเพลง + ฟอนต์ + ดาวน์โหลด + ล็อกอิน — สองตัวที่กินที่จริง ๆ คือ **ข้อความแบรนด์ยาว** กับ **ปุ่มฝั่งขวา 2–3 ปุ่ม**

- **ทบทวน B009:** เหตุผลเดิมที่ "ไม่มี hamburger" คือ *"▾ บอกว่าเป็นเมนูพอ"* — เป็นเรื่อง **สัญญะของปุ่มเมนู** (caret vs ไอคอนขีดสามขีด) ไม่ใช่เรื่อง "ต้องโชว์ชื่อแบรนด์เป็นตัวหนังสือ" → **ยังใช้ได้บน desktop** และ **ไม่ขัด** กับการย่อแบรนด์บนมือถือ ถ้าเรา **เก็บ ▾ ไว้**
- **SA แนะ:** (1) มือถือย่อแบรนด์เป็น **โลโก้เล็ก + ▾** (ไม่ใช่ hamburger เปล่า ๆ) → คืนที่ ~100px และ **ยังตรงหลัก B009** · (2) **ย้ายดาวน์โหลด (+ฟอนต์) ลง dock** ตาม note 2 → คืนอีก ~44–88px · รวมแล้วชื่อเพลงยาวเต็มบรรทัด · **desktop คงเดิมทุกอย่าง**
- **P'Aim เคาะ:** เอา **โลโก้+▾** (SA แนะ) หรือ **hamburger ☰** เต็มตัว (ตามที่วาดไว้) — ทั้งคู่ได้ชื่อเพลงยาวเท่ากัน ต่างกันที่เอกลักษณ์แบรนด์กับความสอดคล้อง B009

## 1. ปัญหา (จาก real-use มือถือ 9 ก.ค.)

แถบบนมือถือตอนนี้ = **แถวเดียว** อัดของ 5 อย่าง ทำให้ชื่อเพลงโดนบีบจนตัด:

```
[ เพลง.พระคำ.ชีวิต ▾ ] [ 1. ... ] [ อ ] [ ⬇ ] [ 👤 ]
   ~165px (ยาว)         โดนบีบ    ฟอนต์ ดาวน์โหลด ล็อกอิน
```

ผู้ที่กินที่แนวนอน:
| ตัว | กว้างโดยประมาณ | จำเป็นบนแถวบนไหม |
|---|---|---|
| ข้อความแบรนด์ "เพลง.พระคำ.ชีวิต" + ▾ | ~150–170px | **ไม่** — ผู้ใช้รู้อยู่แล้วว่าอยู่แอปไหน · ต้องการแค่ปุ่มเมนู |
| ชื่อเพลง | ยิ่งมากยิ่งดี | **ใช่** (คือพระเอกของหน้านี้) |
| ฟอนต์ (FontTool) | ~44px | ไม่ — ย้ายลง dock/เมนูการแสดงผลได้ |
| ดาวน์โหลด (DownloadTool) | ~44px | ไม่ — P'Aim อยากย้ายลง dock (note 2) |
| ล็อกอิน (ProfileTool) | ~44px | ใช่ — เล็ก ไอคอนเดียว |

> **หมายเหตุ implement:** โครง S4 ตั้งใจให้แถว 1 = `แบรนด์ · ชื่อเพลง · 👤` เท่านั้น แต่ของจริงเอา `.sb-right` (ฟอนต์+ดาวน์โหลด+ล็อกอิน) ไว้แถว 1 หมด → เบียดชื่อเพลง ส่วนหนึ่งของ B045 คือ **ทำให้แถว 1 เหลือของจำเป็นตามที่ S4 ตั้งใจ**

## 2. ทบทวน B009 — "ไม่มี hamburger" ยังใช้ได้บนมือถือไหม

**B009 เคาะไว้ว่า** (US/DS `ps3-shell` S4): มือถือ 2 แถว · ไอคอนล้วน · **ไม่มี hamburger** · เหตุผลบันทึกไว้: *"▾ บอกว่าเป็นเมนูพอ"*

**อ่านเจตนา:** เหตุผลนี้พูดถึง **สัญญะของตัวเปิดเมนู** — caret `▾` สื่อว่า "กดแล้วมีเมนู" อยู่แล้ว จึงไม่ต้องมีไอคอนขีดสามขีด (☰) มาซ้ำ · และกันปัญหา "hamburger เป็นตะกร้ารวมทุกอย่างจนหาไม่เจอ"

**เจตนานี้ *ไม่ได้* พูดถึงว่า** ต้องโชว์ชื่อแบรนด์เป็น **ตัวหนังสือยาว** — ตอนเคาะ B009 ยังไม่เจอปัญหาชื่อเพลงโดนตัดบนจอจริง จึงไม่ได้ชั่งน้ำหนักเรื่องนี้

**สรุปการทบทวน:**
- **Desktop:** เหตุผล B009 ยังใช้ได้เต็ม — ที่กว้าง โชว์ชื่อแบรนด์เต็ม + ▾ ดีต่อเอกลักษณ์ → **คงเดิม ไม่แตะ**
- **Mobile:** เป้าหมาย B009 (ให้เมนูอ่านง่าย ไม่ใช่ตะกร้ารวม) **ยังถือได้** ถ้าเรา **เก็บ ▾ ไว้** แล้วแค่ย่อ *ข้อความแบรนด์* เป็น *โลโก้* → นี่คือการ **แก้ตามเจตนา B009** ไม่ใช่การล้ม B009
- แต่ถ้า P'Aim อยากได้ **hamburger ☰ เต็มตัว** (ตามที่วาด) = **แก้ทิศ B009 บนมือถือ** → ต้องเคาะยืนยันชัด (ในที่นี้ ☰ ถือครองแค่ 4 ลิงก์หน้าเว็บ = ใช้ถูกจุด ไม่ใช่ตะกร้ารวม จึงพอรับได้ แต่ทิ้งสัญญะ ▾ ที่ B009 ตั้งใจ)

## 3. ทางเลือก

### (ก) มือถือ: แบรนด์ → hamburger ☰ (ตามที่ P'Aim วาด)
เปลี่ยน `[เพลง.พระคำ.ชีวิต ▾]` เป็นไอคอน `☰` เปิดเมนูหน้าเว็บชุดเดิม
- ✅ คืนที่มากสุด (~150px) · ชื่อเพลงยาวเต็ม · เป็น pattern ที่ผู้ใช้คุ้นมือ
- ✅ ที่นี่ ☰ ถือแค่ลิงก์หน้าเว็บ 4 อัน = ใช้ถูกจุด
- ❌ **ขัด B009 ตรง ๆ** (ทิ้งสัญญะ ▾ · ต้อง P'Aim เคาะกลับทิศ)
- ❌ ทิ้งเอกลักษณ์แบรนด์บนมือถือ (ไม่เห็นทั้งชื่อและโลโก้)

### (ข) มือถือ: แบรนด์ → โลโก้เล็ก + ▾  ⭐ SA แนะ
เปลี่ยนเป็น `[◈ ▾]` — favicon/โลโก้พระคำ.ชีวิต (มี asset อยู่แล้ว: `phrakham.life2/assets/favicon.ico`) + caret เดิม · กด = เมนูหน้าเว็บชุดเดิม
- ✅ คืนที่ ~100–110px (โลโก้ ~24px + ▾ แทนข้อความ ~160px)
- ✅ **ยังตรงหลัก B009** — เก็บ ▾ เป็นสัญญะเมนูไว้ (แก้ตามเจตนา ไม่ล้ม)
- ✅ คงเอกลักษณ์แบรนด์ (เห็นโลโก้)
- ⚠️ โลโก้อย่างเดียวอาจสื่อ "เมนู" อ่อนกว่า ☰ เล็กน้อย → **แก้ด้วยการคง ▾ ไว้ข้างโลโก้** (ตัว ▾ นี่แหละที่บอกว่ากดได้)

### (ค) ย้ายดาวน์โหลด (+ฟอนต์) ลง dock (ตาม note 2) — ทำคู่กับ ก/ข
เอา DownloadTool (และ FontTool) ออกจาก `.sb-right` แถวบน → ไปเป็นปุ่มใน **dock ล่าง**
- ✅ คืนที่แถวบนอีก ~44–88px · dock มี `download` ใน registry โหมด **edit อยู่แล้ว** (ps3-dock) → แค่เพิ่มเข้า registry โหมด **sing/print** ด้วย
- ✅ ดาวน์โหลด/ฟอนต์อยู่ใกล้บริบทเล่นเพลงมากกว่าอยู่บนหัว
- ⚠️ dock โผล่เฉพาะตอนเปิดเพลง (sing/sheet/print) — ถูกต้องแล้ว เพราะดาวน์โหลดมีความหมายเฉพาะตอนมีเพลงเปิด · หน้า `รายการเพลง` ไม่ต้องมี
- ⚠️ FontTool ถ้าไม่ย้าย dock อาจยุบเข้าเมนู "การแสดงผล" (display) ของ dock แทน — ให้ dev เลือกที่เหมาะ

### (ง) ชื่อเพลงขึ้นบรรทัดของตัวเองเต็มกว้าง
แถว 1 = แบรนด์·ปุ่ม, แถว 2 = ชื่อเพลงเต็มกว้าง, แถว 3 = โหมด
- ✅ ชื่อเพลงยาวสุด · ไม่ต้องแตะแบรนด์
- ❌ **กินแนวตั้ง 3 แถว** — มือถือตอนร้องเพลงพื้นที่จอมีค่า (ขัดเจตนา dock/B021 ที่พยายามคืนพื้นที่จอ) → **ไม่แนะ**

## 4. ข้อเสนอ SA

> **(ข) โลโก้เล็ก + ▾  +  (ค) ย้ายดาวน์โหลด/ฟอนต์ลง dock**

เหตุผล:
1. **แก้ปัญหาได้จริง** — สองตัวรวมกันคืนที่ ~150–190px ชื่อเพลงยาวเต็มบรรทัดเท่ากับทางเลือก hamburger
2. **ไม่ต้องล้ม B009** — เก็บสัญญะ ▾ ไว้ = แก้ตามเจตนาเดิม (ต่างจาก ก ที่ต้องเคาะกลับทิศ)
3. **desktop ไม่กระทบ** — เปลี่ยนเฉพาะใน `@media (max-width: 760px)`
4. **ใช้ของที่มีอยู่** — โลโก้ favicon และ dock `download` registry มีอยู่แล้ว

ผลลัพธ์แถวบนมือถือ:
```
ก่อน:  [ เพลง.พระคำ.ชีวิต ▾ ][ 1. ... ][ อ ][ ⬇ ][ 👤 ]
หลัง:  [ ◈▾ ][ 1. ชื่อเพลงยาวเต็มบรรทัดได้แล้ว        ][ 👤 ]
        โลโก้                                          ล็อกอิน
       (ดาวน์โหลด/ฟอนต์ → ปุ่มใน dock ล่าง)
```

## 5. สิ่งที่ P'Aim ต้องเคาะ

1. **ตัวเปิดเมนูแบรนด์บนมือถือ** — เลือก: **(ข) โลโก้ + ▾** (SA แนะ · ตรง B009) · หรือ **(ก) hamburger ☰** (ตามที่วาด · แก้ทิศ B009)
2. **ย้ายดาวน์โหลดลง dock** — ยืนยัน (note 2) → dev เพิ่ม `download` เข้า registry โหมด sing/print
3. **FontTool** — ย้ายลง dock ด้วย หรือยุบเข้าเมนู "การแสดงผล" ของ dock?

หลัง P'Aim เคาะ → SA เขียน AC ลง `ps3-shell` S4 (amend) แล้วส่งต่อสายโค้ด (session อื่น) — **SA ไม่แตะ ShellBar เอง**

## 6. Responsive / WCAG / IA

- **แถบบนเดียวทุกโหมด (S1) คงไว้** — ข้อเสนอนี้แค่ย้าย/ย่อของในแถบเดียวเดิม ไม่แตกแถบต่อโหมด
- **WCAG 2.2 AA — ปุ่มแตะ ≥44px:** โลโก้+▾ ต้องเป็น hit target เดียว ≥44×44px · ปุ่มดาวน์โหลด/ฟอนต์ใน dock ≥44px · ล็อกอิน 👤 ≥44px
- โลโก้ต้องมี `aria-label="เมนู · หน้าแรก"` (หรือแยก: โลโก้=ไปหน้าแรก, ▾=เปิดเมนู — สอง hit target เหมือน desktop) · **แนะ:** คงพฤติกรรม desktop (แบรนด์=ลิงก์หน้าแรก, ▾=เมนู) เพื่อสอดคล้อง
- **โฟกัสชัด · click-outside/Esc ปิด** — ใช้ pattern popover เดิมของ shell
- **Breakpoint:** ใช้เส้นเดิม `max-width: 760px` (ที่ S4 ใช้อยู่) — ไม่เพิ่ม breakpoint ใหม่
