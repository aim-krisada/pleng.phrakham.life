### 🔴 AC-1.7 — event log วัด Cancellation Rate (NOT PROVEN #2 · ความเสี่ยงอันดับ 1 จากรอบ Pro)

**ต้องใส่ตั้งแต่ CP-1 ไม่ใช่มาเติมทีหลัง** — ไม่มี log = วัดความเสี่ยงที่ Pro ชี้ว่าใหญ่ที่สุดไม่ได้เลย

**เก็บ 1 record ต่อ 1 "engaged open"** (localStorage/in-memory · ⛔ ไม่มีบัญชี ไม่มี PII · ตรง mission):

| field | ค่า | ใช้ทำอะไร |
|---|---|---|
| `opener` | `'ctrlk'` \| `'bracket'` \| `'button'` \| `'ctrlaltk'` | รู้ว่าประตูไหนถูกยกเลิกบ่อย |
| `engaged` | bool — จริงเมื่อ **arm แถว (`↓`/แตะ) หรือพิมพ์ ≥1 อักขระค้น** | **นิยาม denominator** (ดูด้านล่าง) |
| `outcome` | `'commit'` \| `'cancel_esc'` \| `'cancel_clickout'` \| `'cancel_type_through'` | แยกยกเลิกจากสำเร็จ |
| `open_ms` | ms ตั้งแต่เปิดจนปิด | **Time-to-Execute** (median ของ commit) |
| `query_len` | ความยาวคำค้นตอนปิด | 0 = เปิดแล้วปิดเปล่า |
| `armed` | bool — เคยมีแถวถูกเลือกไหม | แยก "เปิดแล้วงง" ออกจาก "เปิดแล้วเลือก" |
| `committed_id` | id คำสั่ง หรือ null | รู้ว่าอันไหนถูกใช้ (ป้อนโซน "ใช้ล่าสุด" §4 ด้วย) |
| `rows_at_close` | จำนวนแถวที่แสดงตอนปิด | จับ search collision (§10.1 Miller) |

**นิยามที่ต้องเป๊ะ (กันนับเพี้ยน):**
- **Cancellation Rate = `engaged opens ที่ outcome≠commit` ÷ `engaged opens ทั้งหมด`**
- ⛔ **`[` ที่ผู้ใช้แค่พิมพ์ผ่าน (`cancel_type_through`) โดยไม่เคย arm/ไม่เคยพิมพ์ค้น = `engaged:false` → ไม่นับเป็นทั้งตัวตั้งและตัวหาร**
  (ไม่งั้น autocomplete ที่เด้งแล้วผู้ใช้พิมพ์ต่อจะถูกนับเป็น "ยกเลิก" ทั้งที่ทำงานถูก — จะเห็น cancel rate สูงหลอก)
- เกณฑ์เตือน: **Cancellation Rate > ~20%** = ช้า/ขวางเกินไปสำหรับผู้ใช้ (ตัวเลขจากรอบ Pro — เป็นสัญญาณ ไม่ใช่เส้นตายมาตรฐาน)
- ✅ **พิสูจน์ว่า log ทำงาน:** จำลอง เปิด-commit / เปิด-Esc / `[`-พิมพ์ผ่าน → อ่านค่ากลับได้ครบ 8 field · สูตร cancel-rate คำนวณถูกและ**ไม่นับ type-through**

**AC-1.8 — ⛔ ไม่แตะข้อมูลเพลง** (palette เรียกทางเดินแทรกที่มีอยู่ · CP-0 การันตี dispatch เดียว)
