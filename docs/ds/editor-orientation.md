## 1 · D3 ⭐ ตัวตนมาจากข้อมูล — **หัวใจของงานทั้งใบ**

### ผิดตรงไหน (ของจริงวันนี้)
`reviewingDraft` เป็น `ref` ที่ **มีคนเซ็ตให้ที่เดียวในไฟล์** (`loadDraft():1219`) และ**ถูกล้างที่ 3 ที่** (1122 · 1167 · 2204)
→ **มันไม่ได้แปลว่า "ร่างนี้รอตรวจอยู่" · มันแปลว่า "คุณบังเอิญเดินเข้าประตูที่ถูกต้อง"**

```js
// วันนี้ — ประตูเป็นคนกำหนดความจริง
const reviewingDraft = ref(null)                                    // 151
reviewingDraft.value = isApprover.value && d.status === 'pending' ? d : null   // 1219 — ประตูเดียว
reviewingDraft.value = null                                         // 1122 · 1167 · 2204 — ประตูอื่นล้างทิ้ง
```

### แก้ยังไง — ให้ข้อมูลเป็นคนกำหนด
```js
// เสนอ: ร่างที่รอตรวจของ "เพลงที่เปิดอยู่" = ข้อเท็จจริงที่คำนวณได้ ไม่ใช่ผลข้างเคียงของการนำทาง
const openDraft = ref(null)   // ร่างที่กำลังแก้ตรง ๆ (มาจากกล่องร่าง) — คงพฤติกรรมเดิม

const pendingForThisSong = computed(() =>
  pendingDrafts.value.find((d) => d.song_id === editingId.value) || null
)

// "กำลังตรวจร่างคนอื่นอยู่" = เปิดร่างนั้นอยู่จริง + เป็นของคนอื่น + รอตรวจ
const reviewingDraft = computed(() =>
  isApprover.value && openDraft.value?.status === 'pending' &&
  openDraft.value.author_id !== session.value?.user?.id
    ? openDraft.value : null
)
```
- **แก้บั๊กขอบ #5 ฟรี** (approver เปิดร่างตัวเอง → ไม่ขึ้น *"กำลังตรวจฉบับร่างของ [ตัวเอง]"* อีก) เพราะเช็ก `author_id` แล้ว
- **`pendingForThisSong` = ตัวปลดล็อก D4** — ข้อมูลที่โหลดมาแล้วตั้งแต่บรรทัด 1212 แต่ไม่เคยมีใครถามมัน

**⚠️ dev ระวัง:** `loadDrafts()` ยิงเฉพาะตอนล็อกอิน/เปิดกล่อง → **ถ้า `pendingDrafts` ยังว่างตอน `loadSong()` เตือนจะไม่ขึ้น** → ต้อง `await loadDrafts()` ให้เสร็จก่อน หรือ `watch(editingId)` แล้วค่อยเช็ก
**ห้าม hardcode:** นับสดจาก DB เสมอ (`board.md` เคยเขียน `verified 8/116` → วัดจริงวันนี้ `20/104` — **เน่าใน 4 วัน**)

### AC
- เปิดเพลงที่มีร่าง `pending` ผ่าน **`เลือกเพลง` · URL ตรง · รีเฟรช · กล่องร่าง** → **ได้ผลเหมือนกันทั้ง 4 ทาง**
- unit test ครบทั้ง 9 สถานะใน US §4 · **derive จาก fixture ห้ามฝังเลข**

## 2 · D4 เตือนก่อนทับ (ทางเดินสู่ความเสียหายใน US §2)

**ทันทีที่เปิดเพลงที่มี `pendingForThisSong`** → แถบสีเตือน **inline** (ไม่ใช่ toast — M3 ไม่มี banner component แล้ว และ snackbar หายเอง → **สถานะถาวรต้องอยู่ในหน้า**):

```
⚠️ โมส่งร่างของเพลงนี้มารอตรวจ (2 วันก่อน)   [ ดูร่างของโม ]  [ แก้ฉบับเผยแพร่ต่อ ]
```

- **2 ปุ่ม = 2 ทางที่ชัด** (M3 Dialogs: สูงสุด 2 action · หนึ่งยืนยัน หนึ่งปิด)
- ถ้าเลือก `แก้ฉบับเผยแพร่ต่อ` → แถบ**ยังอยู่** (ไม่ปิดตัวเอง) เพราะความจริงยังไม่เปลี่ยน
- กด `เผยแพร่` ทั้งที่มีร่างค้าง → **ยืนยันที่บอกผลจริง** ห้ามถาม *"แน่ใจไหม?"* ลอย ๆ (M3 Dialogs ห้ามพาดหัวกำกวม):
  ```
  เผยแพร่ทับฉบับปัจจุบัน — ร่างของโมที่รอตรวจอยู่จะยังไม่ถูกอนุมัติ
  [ ยกเลิก ]  [ เผยแพร่ทับ ]
  ```
  (**HIG Alerts:** เตือนเฉพาะการกระทำที่**ย้อนไม่ได้และไม่ได้ตั้งใจ** — ตรงนี้เข้าเงื่อนไขเป๊ะ)

**AC:** เส้นทางใน US §2 เดินซ้ำแล้ว **ต้องมีจุดหยุดอย่างน้อย 1 จุดที่บอกความจริง** · `aria-live="polite"` (WCAG 4.1.3)

## 3 · D2 ปุ่มหลักพูดความจริง (WCAG 3.2.4 · 2.4.6 · 3.3.2 ระดับ A)

| สถานะ | วันนี้ (1691) | เสนอ |
|---|---|---|
| ตรวจร่างคนอื่น | `✅ อนุมัติ` | **`✅ อนุมัติร่างของโม`** |
| approver + เพลงเผยแพร่ | `✅ เผยแพร่` | **`✅ เผยแพร่ทับฉบับปัจจุบัน`** |
| approver + มีร่างคนอื่นค้าง | `✅ เผยแพร่` ← **อันตราย** | **`⚠️ เผยแพร่ทับ (มีร่างรอตรวจ)`** |
| ไม่ใช่ approver | `📨 ส่งตรวจ` | คงเดิม (ถูกอยู่แล้ว) |

- **ห้ามป้ายเดียวกันให้ผลต่างกัน** — `save()` (1414) กับ `primaryAction()` (1692) **ตัดสินใจคนละแบบในสถานะเดียวกัน** (`save()` → `saveDraft('pending')` · `primaryAction()` → `approve()`) → **dev ต้องรวมให้เหลือแหล่งความจริงเดียว** ไม่ใช่ 2 ฟังก์ชันที่เถียงกันเอง
- ป้ายยาวขึ้น → **`ui-standards` §2: ที่แคบ = ปรับ layout ไม่ใช่ตัดจนอ่านไม่ออก** · มือถือ 360px ตัดเหลือ `อนุมัติร่างของโม` ได้ **ห้ามตัดเหลือ `อนุมัติ`** (ข้อมูลสำคัญที่สุดคือ *ของใคร*)

## 4 · D1 แถบระบุตัวตน — 1 แถบ · 1 บ้าน · ทุกโหมด 🚩 *รอ P'Aim เคาะ*

**ทำไมต้องอยู่บน shell:** พี่เปาเจอปัญหาใน `แผ่นเพลง`/`ฝึกร้อง` ด้วย (issues16 = 2 หน้าต่างเทียบกัน) และ **โหมดพวกนั้นไม่รู้จักคำว่า "ร่าง" เลย** (grep ยืนยัน = ว่าง)
**ยึด `Studio.vue:196` ของทีมเอง:** *"lives on the shell so it works in EVERY mode"* · **Apple HIG:** title *"differentiates between the content of multiple open windows"* · **M3:** headline + subtitle = ช่องที่มีอยู่แล้วและว่างอยู่

### หน้าตา (1 บรรทัด · sticky ใต้ ShellBar · ทุกโหมด)
```
┌────────────────────────────────────────────────────────────┐
│ 📄 21. เมื่อข้าได้พบพระองค์ครั้งแรก · ร่างของโม — รอตรวจ  ● ยังไม่บันทึก │
└────────────────────────────────────────────────────────────┘
   ↑ไอคอน  ↑เพลงอะไร (ข้อ1)      ↑ฉบับของใคร (ข้อ2)      ↑สถานะ (ข้อ3)
```

| ฉบับ | ไอคอน | คำ | สี |
|---|---|---|---|
| เผยแพร่แล้ว | `check-circle` | `ฉบับเผยแพร่` | เขียว/กลาง |
| ร่างของฉัน | `file-pen` | `ร่างของฉัน` | น้ำเงิน |
| **ร่างคนอื่นรอตรวจ** | `search` | **`ร่างของโม — รอตรวจ`** | **ส้ม/เน้น** |
| เพลงใหม่ | `file-plus` | `เพลงใหม่ · ยังไม่บันทึก` | เทา |

**กติกา (`ui-standards` + WCAG):**
- **ไอคอน + คำ + สี เสมอ — ห้ามพึ่งสีอย่างเดียว** (WCAG 1.4.1) ← **ข้อนี้คือหัวใจของ AC-5**: พี่เปาวาง 2 หน้าต่างข้างกัน ต้องแยกออก**ด้วยหางตา**
- `position: sticky` จริง (`.review-banner:3375` วันนี้**ไม่มี** → เลื่อนแล้วหาย)
- **1 บรรทัด ไม่ห่อ** (`ui-standards` §2 list row) · ชื่อยาว → truncate + `title` · **ห้ามตัด "ของใคร" ทิ้ง**
- คอนทราสต์ ≥ 4.5:1 · ขยาย 200% ไม่แตก (WCAG 1.4.4) · 360/412px ไม่ล้น
- `aria-live="polite"` ตอนสลับฉบับ (WCAG 4.1.3) · **ไม่พิมพ์** (`no-print`)
- **ไม่ใช่ component ใหม่** — ยกเนื้อจาก `.review-banner` (2406) ที่พิสูจน์แล้วว่าดึงชื่อคนถูก

**🚩 ต้องเคาะ:** อยู่บน `Studio.vue` = แตะไฟล์ที่ `mission.md:44` ห้ามแตะ · **สาย recycle-bin §6.1(ข) ติดข้อเดียวกัน → เคาะทีเดียวได้ 2 สาย**
**ถ้าไม่เคาะ:** วางในหน้าแก้ไขอย่างเดียวไปก่อน (แก้ได้ ~70% — ขาด `แผ่นเพลง`/`ฝึกร้อง` = ขาดจอที่ issues16 ถ่ายมาพอดี)

## 7 · D6 ประตูลบ (issues10) 🚩 *รอ recycle-bin*

**ปุ่มลบมีอยู่แล้ว** (`EditorMode.vue:2271`) · **พี่เปาเป็น approver = สิทธิ์ผ่านอยู่แล้ว** · **สิ่งที่บล็อกเขาคือ "หาไม่เจอ"** — ปุ่มโผล่เฉพาะ**โหมดแก้ไข + หลังโหลดเพลงเข้า editor** แต่เขารีวิวจากหน้ารายการ → **ไม่มีวันเจอ**

**มาตรฐานหนุนเต็ม:** **M3 Top app bar:** *"Avoid placing an overflow menu in the app bar when possible."* · **Apple Toolbars:** *"Prioritize **less important** actions for inclusion in the More menu"* + จัดกลุ่ม action สำคัญ *"in dedicated, familiar, and visually distinct sections"*
→ **"ลบเพลงทั้งเพลง" ไม่ใช่ less important** — มันไม่ควรอยู่ใน `จัดการ ▾` ตั้งแต่แรก

**เสนอ:** ประตูระดับ shell คู่กับ `สร้างเพลงใหม่` / `เลือกเพลง` (ตรงกับที่ P'Aim พูดเอง: *"อยู่คู่กับ สร้างเพลงใหม่/ตรวจแล้ว จะง่ายที่สุด"*)
**⚠️ `ui-standards` §2 single-source: ย้าย ไม่ใช่เพิ่ม** — ถอดออกจาก `จัดการ ▾`
**⚠️ ลำดับบังคับ:** สาย recycle-bin ยืนยันว่า **"สร้างกำแพงก่อนเจาะประตู"** → **D6 ต้องรอเฟส 1-2 ของเขา** (ถังขยะ+กู้คืนพร้อมใช้) **ไม่งั้นเราทำให้ปุ่มลบถาวรหาง่ายขึ้น = เร่งความเสียหาย**

## 8 · เลือกเพลง: การ์ด vs ComboSelect — **ต่างกันถูกแล้ว ผมไม่ทำให้เหมือนกัน**

**PM verify มาว่า `SongList`/`SongView` = การ์ด · `Studio`/`EditorMode` = ComboSelect · "ไม่เคยมีใครตัดสินใจ โตมาคนละทาง"**
**คำตัดสินผม: โตมาคนละทางจริง แต่ดันโตมาถูก — และผมมีมาตรฐานหนุน ไม่ใช่ปล่อยผ่าน**

| งาน | คนใช้ | รู้ไหมว่าจะเอาเพลงไหน | เครื่องมือที่ถูก |
|---|---|---|---|
| หาเพลงมาร้อง | Tier 0 | **ไม่รู้ — มาเปิดดูว่ามีอะไร** | **การ์ด** (browse · **NN/g #6 recognition**) |
| พิมพ์เพลงที่ 80 | พี่เปา | **รู้อยู่แล้ว** | **ช่องพิมพ์** (recall · เร็วกว่ามาก) |

**NN/g #6:** *"Minimize the user's memory load by making elements, actions, and options visible."* — recognition สำหรับคนที่**ไม่รู้ว่าจะหาอะไร** · แต่พี่เปา**รู้อยู่แล้ว** → บังคับให้เขา browse การ์ด 124 ใบ = **ทำให้ช้าลง**
**WCAG 3.2.4 ไม่ได้ถูกละเมิด** — *"Components that have the **same functionality**… identified consistently"* → **คนละ functionality** (สำรวจ vs กระโดดตรง) → ยอมให้ต่างได้

> **สิ่งที่ต้องเหมือนกันคือ "แถบระบุตัวตนหลังเปิดเพลงแล้ว" (D1) ไม่ใช่ "วิธีเลือกเพลง"**
> **P'Aim พูดถูกว่ามันไม่เหมือนกัน — แต่ตัวที่ทำให้เขางงจริงคือจอหลังจากนั้น ที่ไม่บอกว่าเขาเปิดอะไรมา**

**สิ่งเดียวที่ต้องแก้:** ComboSelect **ต้องโชว์ป้ายฉบับด้วย** (`21. ชื่อเพลง` · `ร่างของโม — รอตรวจ`) เพราะกล่องร่างทำถูกอยู่แล้ว (`issues10` โชว์ `— โดย k.pituck`) แต่ ComboSelect ไม่โชว์ → **ยกของดีที่มีอยู่แล้วมาใช้ ไม่ต้องคิดใหม่**
