## 1. 🔴 หัวใจของงาน — "อัปเดตหากัน" ไม่ใช่ปัญหา sync แต่เป็นปัญหา **โมเดล**

### ทำไมวันนี้ยังไม่เชื่อมกัน
ทั้งสองขาแก้ **ข้อมูลคนละที่** — ไม่ใช่สองมุมมองของของสิ่งเดียวกัน:

| ขา | เขียนลงที่ไหน |
|---|---|
| ลากวางในแผงโครงเพลง | `arrangement[]` (ลำดับท่อน) |
| สัญลักษณ์ในโน้ต (`***` · `‖:` `:‖` · `1.` `2.` · จบเพลง) | item ในบรรทัด ใน `stanzas[].lines` |

> ⛔ **ถ้าเขียนโค้ด "ซิงก์ 2 ที่" = จะไล่ซิงก์ไปตลอดชีวิตและมันจะเพี้ยนแน่นอน** —
> ทุกฟีเจอร์ใหม่ต้องเพิ่มเส้นซิงก์อีกเส้น · และมี 2 ที่ที่อ้างความจริงคนละอย่างเมื่อไหร่ ก็มีบั๊กเมื่อนั้น

### 📌 หลักที่ใช้: **หนึ่งความจริง สองภาพฉาย** (one source, two projections)

```
        ┌──────────── ผู้ใช้แก้ได้ 2 ทาง ────────────┐
        │                                            │
   ลากวางในแผง                            พิมพ์สัญลักษณ์ในโน้ต
        │                                            │
        ▼                                            ▼
   เขียนลง arrangement[]                    เขียนลง item ในห้อง
   (ลำดับของท่อน)                          (สิ่งที่เกิด "ที่ห้อง")
        │                                            │
        └──────────────┬─────────────────────────────┘
                       ▼
            resolvePlayOrder()  ← ไทม์ไลน์ที่กางแล้ว (มีอยู่แล้ว)
                       │
        ┌──────────────┴─────────────┐
        ▼                            ▼
   แผงโครงเพลง render            แผ่นเพลง render
```

**กฎเหล็ก 3 ข้อ:**
1. **ข้อเท็จจริงแต่ละอย่างมีเจ้าของเดียว** — "ท่อนไหนมาก่อนหลัง" เจ้าของคือ `arrangement[]` ·
   "ห้องนี้ย้อน/จบ/เป็นรอบ 1" เจ้าของคือ item ในห้อง · **ห้ามเก็บซ้ำ 2 ที่เด็ดขาด**
2. **ทางเข้าแก้มีได้หลายทาง แต่ปลายทางเขียนที่เจ้าของเสมอ** — กด "ซ้ำ" จากแผง = เขียนลง item ในห้อง
   (ไม่ใช่เก็บ flag ไว้ที่แถวในแผง) · ลากในแผง = เขียน `arrangement[]`
3. **ทั้ง 2 หน้าจอ render จากไทม์ไลน์ที่ resolve แล้วเสมอ** → แก้ทางไหนอีกทางเปลี่ยนตามทันที
   **โดยไม่มีโค้ดซิงก์แม้แต่บรรทัดเดียว**

> **นี่คือ pattern เดียวกับ Figma (layers panel ↔ canvas) และ VS Code (outline ↔ editor)** —
> ไม่มีใครเขียน syncer ระหว่างสองแผงนั้น เพราะทั้งคู่คือภาพฉายของ document เดียว

### สิ่งที่ต้องมีก่อน (มาจาก spike แล้ว — อยู่ในเอกสารพี่น้อง)
- **G8a รวม 2 ชั้นขยาย** (`expandRepeats` หน่วยห้อง ↔ `resolvePlayOrder` หน่วยบรรทัด) เป็นลิสต์เดียว
  → **ถ้าไม่รวม ไทม์ไลน์กลางไม่มีจริง = สถาปัตยกรรมข้างบนตั้งไม่ได้**
- **G18 อุดรูข้อมูลหาย** (`EditorMode.vue:99-121` ทิ้ง item ที่ไม่รู้จักตอนเซฟ) →
  ไม่อุดก่อน = ใส่ `‖:` แล้วเซฟ **หายเงียบ**

## 2. หน้าตาแผงโครงเพลง (เดสก์ท็อป)

ต่อยอดจากแถวเดิม — **เพิ่ม 1 แถบสัญลักษณ์ต่อท่อน** ไม่ย้ายอะไรของเดิม

```
┌─ โครงเพลง ────────────────────────────────── [＋ ท่อน] ┐
│                                                          │
│ ⠿  ♦ ข้อ 1            ทำนอง A ▾      28/28 ✓     ⋯      │
│      ‖:  ปักต้นซ้ำ   ⟲ ซ้ำ 2 รอบ   1. 2.        ← แถบใหม่│
│                                                          │
│ ⠿  ♦ รับ  (ร้องรับทุกข้อ)  ทำนอง B ▾   33/33 ✓   ⋯      │
│      ***  ท่อนรับ                                       │
│                                                          │
│ ⠿  ♦ ข้อ 2            ทำนอง A ▾      28/33 ✗     ⋯      │
│                                                          │
│ ⠿  ♦ ข้อ 3            ทำนอง A ▾      28/28 ✓     ⋯      │
│      ‖  จบเพลง                                          │
│                                                          │
├─ ▸ ทำนอง (โน้ต)  A · B                        [＋ ทำนอง]│
└──────────────────────────────────────────────────────────┘
```

**สิ่งที่เพิ่มจากของเดิม = แถบสัญลักษณ์ใต้ชื่อท่อน เท่านั้น**
- แสดง **เฉพาะสัญลักษณ์ที่ท่อนนั้นมีจริง** — ท่อนที่ไม่มีอะไร ไม่มีแถบ (จอไม่รก)
- ทุกชิปกดได้ → **กระโดดไปที่ห้องนั้นในแผ่นโน้ต** พร้อมไฮไลต์ (ไม่ใช่เปิด dialog)
- ปุ่ม `⋯` = เพิ่มสัญลักษณ์ที่ยังไม่มี (เมนูเดียวกับป๊อปอัป `?` ในเอกสารพี่น้อง §4.2)

### ทำไมไม่ทำให้แผงแก้สัญลักษณ์ได้เต็มที่
เพราะ **`‖:` `1.` `2.` ผูกกับ "ห้อง" ไม่ใช่ "ท่อน"** — ท่อนหนึ่งมีหลายห้อง ·
ถ้าให้แผงตั้งค่าได้เอง ต้องมีตัวเลือก "ห้องไหน" ในแผง = ทำแผงให้เป็นตัวแก้โน้ตอีกตัว
→ **แผง = เห็นภาพรวม + กระโดดไปแก้ · แผ่นโน้ต = ที่แก้จริง** (หน้าที่ชัด ไม่ทับกัน)

### ลากวาง — คงของเดิม + เติมมาตรฐาน
ของเดิมมี DnD + pointer + ▲▼ + aria-live แล้ว · **เพิ่ม 2 อย่างให้ครบมาตรฐาน:**
- **ลากด้วยคีย์บอร์ดได้** — โฟกัสที่ grip แล้ว `Space` ยก · `↑↓` ย้าย · `Space` วาง · `Esc` ยกเลิก
  (WCAG 2.2 **2.5.7 Dragging Movements** บังคับว่าทุกการลากต้องมีทางที่ไม่ต้องลาก —
  ▲▼ ที่มีอยู่ผ่านข้อนี้แล้ว แต่ pattern ยก-ย้าย-วาง ทำให้ลำดับยาวๆ เร็วกว่ามาก)
- **เส้นบอกจุดวาง (drop indicator) ชัดเจน** + `aria-live` บอกผลลัพธ์ (มีแล้ว — คงไว้)

## 3. ขาที่ 2 — สัญลักษณ์ในตัวโน้ต

**ใช้คำสั่ง `[…]` จาก `note-symbol-set.md` §3.1 ทั้งชุด ไม่คิดใหม่:**
`[ซ้ำ 2 รอบ]` · `[เริ่มซ้ำ]` · `[รอบ 1]` `[รอบ 2,3]` · `[จบ]` · `[รับ]` · `[ย้อนต้นเพลง]` …

**ตอบคำถามพี่เปาโดยตรง:**

| คำถาม | คำตอบ |
|---|---|
| *"ตัวจบเพลง กับ ตัวย้อน อยู่ตรงไหน"* | พิมพ์ `[จบ]` / `[ซ้ำ]` **ที่ห้องนั้นในแผ่นโน้ต** · เห็นผลเป็น `‖` / `:‖` ทันที · และโผล่เป็นชิปในแผงโครงเพลงเอง |
| *"ท่อนรับ / ท่อน hook ทำยังไง"* | พิมพ์ `[รับ]` → ได้ `***` เหมือนเดิม + ติ๊ก "ร้องรับทุกข้อ" ให้เอง (`afterEachVerse` มีอยู่แล้ว) · **`***` ของเดิมยังใช้ได้ ไม่พัง** |
| *"เส้นย้อนจบ 1-2 ต้องยาวคลุมทุกห้องที่เล่นจริง"* | **บั๊กการวาด — แก้แยก ดู §4** |

## 4. 🐛 เส้น volta ต้องคลุมทุกห้องที่เล่นจริง

**อาการ:** `volta` เก็บเป็น flag **ต่อห้อง** (`{type:'volta',num}`) แล้ววาดกล่องต่อห้องนั้นห้องเดียว
→ รอบจบที่ยาว 3 ห้อง ได้กล่องแค่ห้องแรก · **ผิดมาตรฐานสากล** (Gould: วงเล็บ volta คลุม
ทุกห้องของรอบนั้น และปิดหัวท้าย)

**แก้:** วาดจาก **ช่วงห้องที่ต่อเนื่องกันซึ่งมี `volta` เลขเดียวกัน** ไม่ใช่ต่อห้อง
- เป็นงาน **render ล้วน** ไม่แตะข้อมูล (`SongSheet.vue`)
- ห้องท้ายของช่วงเป็นตัวปิดวงเล็บ (ขาลง) · ห้องแรกมีเลขกำกับ
- ⚠️ **ต้องแก้ที่การพิมพ์ด้วย** ไม่ใช่แค่บนจอ (`docs/reports/b116-print-beams.md` มีบทเรียนว่า
  print path แยกจากจอ) — **verify จาก PDF จริง ไม่ใช่จาก DOM** (memory `feedback_verify_print_from_pdf`)

## 5. ปักหมุดเครื่องมือ + แบ่งหมวด

**ของเดิมมี 📌 อยู่แล้ว → ต่อยอด ไม่สร้างระบบใหม่**

พี่เปาขอ: ปักหมุดของที่ใช้บ่อยเอง + แบ่งหมวดเพราะเครื่องมือเยอะ (เขาเสนอเป็นแท็บ)

**ฟันธง: หมวด = แท็บ + แถวปักหมุดอยู่เหนือแท็บเสมอ**

```
┌────────────────────────────────────────────┐
│ 📌  ⟲ซ้ำ   1.2.   ‖จบ   ***รับ        ✎แก้ │ ← ปักหมุดเอง เห็นตลอด
├────────────────────────────────────────────┤
│ [ โครงสร้าง ] [ วนซ้ำ ] [ คอร์ด ] [ แสดงผล ]│ ← หมวด
│  … เครื่องมือของหมวดที่เลือก …              │
└────────────────────────────────────────────┘
```
- **แถวปักหมุด = อยู่เหนือแท็บ ไม่ใช่ในแท็บ** — ของที่ใช้บ่อยต้องไม่หายเมื่อสลับหมวด
  (นี่คือเหตุผลทั้งหมดของการปักหมุด)
- ปักหมุด/ถอนหมุด = กดค้าง หรือเมนู `⋯` · **ห้ามใช้ hover เป็นทางเดียว**
  (เครื่อง P'Aim รายงาน `hover:none` ทั้งที่มีเมาส์ — memory `feedback_hover_media_hides_on_touchscreen_laptop`)
- จำค่าไว้ต่อผู้ใช้ (localStorage) · มีปุ่มคืนค่าเริ่มต้น
- **ทุกปุ่มยังโชว์คีย์ลัดของตัวเองตาม `note-symbol-set.md` §4.1** (key-equivalent annotation)

## 6. มือถือ (ระบุตามที่สั่ง — แต่โฟกัสเดสก์ท็อปก่อน)

| | เดสก์ท็อป | มือถือ |
|---|---|---|
| แผงโครงเพลง | คอลัมน์ข้างถาวร | **แผ่นเลื่อนขึ้นเต็มความกว้าง** (ตาม `docs/ux-platform-patterns.md`: mobile = full-screen ไม่ใช่ popup ลอย) |
| จัดลำดับ | ลาก / ▲▼ / คีย์บอร์ด | **▲▼ เป็นหลัก** — ลากบนลิสต์ที่สกรอลล์ได้บนจอสัมผัสพลาดง่าย · ลากยังใช้ได้ผ่าน grip เดิม |
| ชิปสัญลักษณ์ | กด → กระโดดไปห้องนั้น | เหมือนกัน + ปิดแผ่นเลื่อนอัตโนมัติ |
| แถวปักหมุด | เหนือแท็บ | เหนือแป้นพิมพ์ (ของเดิมใช้ `visualViewport` อยู่แล้ว) |
