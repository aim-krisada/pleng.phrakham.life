## 1 · รากปัญหา — ทำไม 3 โหมดไม่เหมือนกัน
**แถบควบคุมมาจากคนละที่:**
- **ฝึกร้อง:** `SongViewer` ส่ง custom control เต็มกว้าง `SingTransport` (`region:'top'`) → StudioDock **ซ่อน chrome ตัวเอง** → SingTransport **วาด grip + ⚙ + แผง เอง**
- **แผ่นเพลง / แก้ไข:** ใช้ **chrome ของ StudioDock** (grip ซ้าย · คลัสเตอร์ขวา blend/customize/⋯ · ไม่มี ⚙)

ผล = **โค้ด chrome 2 ชุด** วาง grip/⚙/แผง คนละที่ → drift (ตารางเทียบ = ภาคผนวก)

## 2 · โครงที่เคาะแล้ว — "ช่องไอคอน (slot) เดียว"

ทุกโหมดใช้ StudioDock โครง slot เดียว · ทุก element = ช่องขนาดเท่ากันในกริดเดียว:

```
  (บรรทัดล้น — element เต็มกว้าง เช่น timeline/คีย์ ขึ้นแถวเอง)
┌──────────────────────────────────────────────────┐
│ [⁝⁝ grip]   ●  ●  ●  ●  (ปุ่มตามต้องใช้)   [⋯] [⚙] │  ← แถวล่างสุด: grip ซ้ายสุด · ⋯+⚙ ขวาสุด
└──────────────────────────────────────────────────┘
```

- **R1 · grip = ซ้ายสุด · ⚙ เฟือง = ขวาสุด** — ทั้งคู่อยู่ **แถวล่างสุดเสมอ** ทุกโหมด ทุกขนาดจอ (ยึด 2 มุม)
- **R2 · ระหว่าง 2 มุม = ปุ่มตามต้องใช้** (จัดกลาง)
- **R3 · ทุกอย่างนับเป็น "ช่อง (slot)" เท่ากัน** — ปุ่มปกติ = 1 · **timeline = ไอคอนยาว (หลายช่อง · อยู่ในกริดปกติ แชร์แถวได้ถ้าพอ)** · **เลือกท่อน = ปุ่มกว้าง 2 ช่อง**
- **R4 · ล็อคจำนวนช่อง/บรรทัด — มือถือ 6 · เดสก์ท็อป 12** (เดสก์ท็อปพอดี max-width 700px) [เคาะแล้ว]
- **R5 · ปุ่มล้น cap → พับเข้า ⋯ (คง overflow D3 เดิม)** · **⋯ วางก่อน ⚙ บนแถวล่าง** → ขวาสุด = `[⋯] [⚙]` [เคาะ: เก็บ ⋯ ไว้]
- **R6 · แผงเฟือง ⚙ = แบบหน้าฝึกร้องเป๊ะ ทุกโหมด** — รายการทุก control + เลื่อนลำดับ ▲▼ + 📌 pin/unpin = **บ้านเดียวของทุกปุ่ม** · **ความโปร่ง + จัดปุ่ม ยุบเข้าแผงนี้** (เลิกปุ่ม chrome แยกขวา เหลือแค่ ⋯)
- **R7 · grip** แตะ = ยุบ/กาง · ลาก = ย้ายทั้งแถบ · ยุบ→FAB (คง `combinedDown/Move/Up` เดิม)
- **R8 · แถวคีย์ jianpu (แก้ไข)** = แถวพิเศษเต็มกว้าง วางเหนือกริด (คงเดิม) — โครง grip/⚙ ยังเหมือนกัน

### วางจริงต่อโหมด
| โหมด | เหนือกริด | แถวล่าง (คงที่) |
|---|---|---|
| 🎤 ฝึกร้อง | (timeline ล้นก็ขึ้นแถวเอง) | `[grip] [timeline ยาว] ◀ ▶ ▶ 🔁 [เลือกท่อน·2] Aa [⋯] [⚙]` |
| 📄 แผ่นเพลง | — | `[grip]  🖨 พิมพ์  [⚙]` |
| ✏️ แก้ไข | `[แถวคีย์ jianpu เต็มกว้าง]` | `[grip] ↶ ↷ ▶ ⛭ [⋯] [⚙]` |

### ทำไมดี / ต้นทุน
| ✔ ดี | ✖ ต้นทุน |
|---|---|
| เบสโค้ดเดียว · โครงเดียว แก้ง่ายแค่เพิ่ม/ลด element · grip/⚙ ยึด 2 มุมคาดเดาได้ · แผง ⚙ บ้านเดียวทุกโหมด · cap 6/12 = packing ง่าย deterministic | refactor ให้ SingTransport เลิกวาด chrome เอง → ใช้ grip/⚙/panel/packing ของ StudioDock ชุดเดียว · ทับ B043/dock-polish → PM จัดคิว worktree |

## 2b · ปุ่มแต่ละโหมด + โมเดลแผงเฟือง (P'Aim ถาม 10 ก.ค.)

### โมเดลแผงเฟือง ⚙ = "บ้าน" (แนะนำ · แบบ SingTransport เดิม)
- แผง ⚙ = รายการ **ทุกปุ่มปรับได้** ของโหมดนั้น (บ้านเต็ม) · แต่ละแถว: ไอคอน · ชื่อ · ตัวปรับค่า inline · ▲▼ เลื่อนลำดับ(ถ้าปัก) · 📌
- 📌 ติด = **โผล่บนแถบด้วย** (ทางลัด) · 📌 ไม่ติด = อยู่ในแผงอย่างเดียว
- **ปุ่มที่ปักแล้วยังอยู่ในแผง** — เพราะที่ถอนพิน/เลื่อนลำดับอยู่ในแผง (แถบ = ทางลัดเฉยๆ) · 📌 ที่ติด = ป้ายว่า "ตัวนี้ขึ้นแถบแล้ว"
- **ความโปร่งแถบ ก็มี 📌** เหมือนทุกปุ่ม (ตอบ P'Aim: ใช่) · ปักแล้วขึ้นแถบเป็นสไลเดอร์เล็ก
- ทางเลือก B (ไม่แนะนำ): ปัก = "ย้าย" ออกจากแผงขึ้นแถบ (แผงเหลือเฉพาะยังไม่ปัก) · ถอนต้องมีโหมดจัดปุ่มบนแถบ → งงกว่าสำหรับอาสาสมัคร

### ปุ่มแต่ละโหมด
Fixed chrome ทุกโหมด: **grip** (ซ้ายสุด) · **⚙** (ขวาสุด · เปิดแผงบ้าน) · **⋯** (ขวา ก่อน ⚙ · โผล่เมื่อปุ่มล้น cap)

| โหมด | บนแถบ เสมอ | ปักได้ (default ปัก \*) | ในแผง ⚙ (ปรับได้ครบ) |
|---|---|---|---|
| 🎤 ฝึกร้อง | timeline · เลือกท่อน · ◀ ▶/⏸ ▶ 🔁 · **Aa** (ถาวร ปักถอนไม่ได้) | คอร์ด · ความเร็ว\* · คีย์\* · แสดงผล\* · ความโปร่ง | แสดงผล · คอร์ด · คีย์ · ความเร็ว · ความโปร่ง · ดาวน์โหลด · พิมพ์/PDF |
| 📄 แผ่นเพลง | 🖨 พิมพ์ (prime) · **Aa** (ถาวร) | แสดงผล · คอร์ด · คีย์ · ความโปร่ง | แสดงผล · คอร์ด · คีย์ · ความโปร่ง · พิมพ์ |
| ✏️ แก้ไข | แถวคีย์ jianpu (พิเศษ) · ↶ ↷ · ▶ ฟังท่อน · ⏵ ฟังทั้งเพลง · ■ หยุด(ตอนเล่น) · ส่งตรวจ (prime) · ดาวน์โหลด | (ปุ่มไหนโชว์บนแถบ = จัดในแผง) | จัดปุ่ม (เพิ่ม/เอาออก/เลื่อน) · ความโปร่ง · บันทึกร่าง (ถ้าล็อกอิน) |

\* default ปักบนแถบ · **Aa** = ถาวร (คนอ่านต้องมีเสมอ · ไม่นับเป็น pin) · แก้ไข**ไม่มี** คอร์ด/คีย์/ความเร็ว (โหมดแก้โน้ต ไม่ใช่เล่น) · ชุดจริงอ้าง `SongViewer.settingDescs` + `EditorMode.DOCK_DEFAULT`

## 3 · AC (Acceptance Criteria) — พร้อมจ่าย dev
1. **โครงเดียว:** 3 โหมดใช้ StudioDock โครง slot เดียว — grip = slot ซ้ายสุดของแถวล่าง · ⚙ = slot ขวาสุดของแถวล่าง **เสมอ** ทุกโหมด ทุกขนาดจอ
2. **slot uniform:** ทุก element วางในกริด slot เท่ากัน · ปุ่ม=1 · timeline=ไอคอนยาว(หลาย slot, อยู่ในกริด แชร์แถวได้) · เลือกท่อน=2 slot
3. **row cap:** มือถือ ≤6 slot/แถว · เดสก์ท็อป ≤12 (หรือพอดี max-width 700px) — เกิน = ปุ่มท้ายพับเข้า ⋯
4. **overflow:** ปุ่มเกิน cap → ⋯ popover (คง D3) · ⋯ วางก่อน ⚙ บนแถวล่าง · element เต็มกว้าง (คีย์ jianpu) ขึ้นแถวเอง
5. **แผงเฟือง ⚙:** รายการทุก control + เลื่อนลำดับ ▲▼ + 📌 pin/unpin แบบฝึกร้อง **ทุกโหมด** · ความโปร่ง + จัดปุ่ม อยู่ในแผงนี้ (ไม่มีปุ่ม chrome แยกขวา ยกเว้น ⋯)
6. **grip:** แตะ=ยุบ/กาง · ลาก=ย้ายทั้งแถบ (คง `combinedDown/Move/Up`) · ยุบ→FAB เหมือนเดิม
7. **คีย์ jianpu (แก้ไข):** แถวพิเศษเต็มกว้างเหนือกริด คงเดิม
8. **localStorage:** pin order + ลำดับปุ่ม per-mode (คงคีย์เดิม `pleng.dock.<mode>.tools` + `pleng.dock.sing.pins`) · collapse/alpha/pos ร่วมทุกโหมด (คงเดิม)
9. **WCAG 2.2 AA:** tap target ≥44 · aria-label ทุกปุ่ม · แผง/popover ปิดด้วย Esc/คลิกนอก · clamp ไม่หลุดขอบ (คงเดิม)
10. **เบสโค้ดเดียว:** SingTransport เลิกวาด chrome เอง — grip/⚙/panel/packing มาจาก StudioDock ชุดเดียว · เพิ่ม/ลดปุ่ม = แก้ list ของ element เท่านั้น
