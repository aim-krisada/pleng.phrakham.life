## 1 · รากปัญหา — ทำไม 3 โหมดไม่เหมือนกัน

**แถบควบคุมมาจากคนละที่:**
- **ฝึกร้อง (sing):** `SongViewer` ส่ง 1 custom control เต็มกว้าง = `SingTransport` (`region:'top'`). เพราะ dock ไม่มีปุ่มแถวปกติ (`rowTools` ว่าง) StudioDock จึง **ซ่อน chrome ของตัวเอง** → `SingTransport` **วาด grip + ⚙ + แผง เอง**
- **แผ่นเพลง (print):** `Studio` ส่ง 1 ปุ่ม `พิมพ์` → ใช้ **chrome ของ StudioDock** (grip `sd-combined` + คลัสเตอร์ขวา blend/customize/⋯)
- **แก้ไข (edit):** `EditorMode` ส่ง tools + แถวคีย์ jianpu → ใช้ **chrome ของ StudioDock** เหมือน print

ผลคือ **มีโค้ด "chrome ของ dock" 2 ชุด** (SingTransport ชุดนึง · StudioDock ชุดนึง) วาง grip/⚙/แผง คนละที่ → drift

### ตารางเทียบ (ของจริง)
| องค์ประกอบ | 🎤 ฝึกร้อง | 📄 แผ่นเพลง | ✏️ แก้ไข |
|---|---|---|---|
| แถบมาจาก | SingTransport (วาดเอง) | StudioDock chrome | StudioDock chrome |
| grip | บนซ้าย แถว 1 (นอกกรอบ) | ซ้ายแถวเดียว (`sd-combined`) | ซ้ายแถว tools |
| ⚙ ตั้งค่า | **ล่างซ้าย** — 1 ปุ่ม = บ้านของทุกอย่าง (`.mp-more`) | **ขวา** — ไม่มี ⚙ · blend อย่างเดียว | **ขวา** — ไม่มี ⚙ · 3 ปุ่ม (blend/customize/⋯) |
| timeline | มี · แถว 1 · ~200px **ไม่เต็ม** (แชร์แถวกับ ☰ เลือกท่อน) | ไม่มี | ไม่มี |
| ปุ่มหลัก | transport กลางๆ แถว 2 | พิมพ์ ชิดซ้าย | tools ชิดซ้าย |
| ปุ่มลัด (pins) | ขวาของ transport | — | — |
| แผงกาง | ปักซ้าย (`left:0`) · **ลากได้** · กางขึ้น · clamp | ปักขวา (`right:8px`) · ลากไม่ได้ | ปักขวา · ลากไม่ได้ |

### ตรงกับ 5 ประเด็นที่ P'Aim ยก
1. **⚙ อยู่ตรงไหน** → ตอนนี้ ฝึกร้อง=ซ้าย · print/edit=ขวา (แถมไม่ใช่ ⚙ แต่เป็น 3 ปุ่ม chrome แยก)
2. **grip กับ ⚙ แนวเดียวกัน** → ฝึกร้องตรง (grip แถว1-ซ้าย, ⚙ แถว2-ซ้าย) · print/edit ไม่ตรง (grip ซ้าย, settings ขวา)
3. **timeline เต็มกว้าง** → มีแค่ฝึกร้อง และยังไม่เต็ม
4. **พฤติกรรมแผง** → ฝึกร้อง ปักซ้าย+ลากได้ · print/edit ปักขวา+ลากไม่ได้
5. **center align** → transport กลางใน sing · tools ชิดซ้ายใน print/edit

## 2 · ที่เสนอ — "dock grammar" เดียว

โครงเดียวกันทุกโหมด (ภาพร่างข้อ 2 ในไฟล์ HTML):

```
┌───────────────────────────────────────────┐
│ ⁝⁝                                          │  R1 grip: มุมบนซ้าย เสมอ
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ │  R3 timeline: แถวตัวเอง เต็มกว้าง (เฉพาะโหมดมี playback)
│ ⚙        ◀  ▶/⏸  ▶  🔁          [ pins ]   │  R2 ⚙ ซ้าย · R4 หลักกลาง · R5 pins ขวา
└───────────────────────────────────────────┘
```

- **R1 · grip** = มุมบนซ้ายของแถบ เสมอ ทุกโหมด (ขอบซ้าย = padding ของแถบ)
- **R2 · ⚙ ตั้งค่า** = ล่างซ้าย ใต้ grip พอดี → **grip กับ ⚙ ขอบซ้ายตรงกัน** (ตามที่ P'Aim ขอ) · ⚙ = **บ้านเดียว** ของปุ่มรองทุกโหมด: ความโปร่ง/จัดปุ่ม/ดูเพิ่ม ยุบเข้ามาใน ⚙ (แบบที่ฝึกร้องทำอยู่แล้ว)
- **R3 · timeline** = แถวของตัวเอง เต็มความกว้าง ใต้ grip · เฉพาะโหมดที่มี playback (ฝึกร้องตอนนี้ · "ฟังท่อน" ของแก้ไขต่อยอดทีหลังได้) · "เลือกท่อน" ออกจากแถว timeline
- **R4 · ปุ่มหลัก** = กึ่งกลางแถวหลัก
- **R5 · ปุ่มลัด (pins)** = ชิดขวาแถวหลัก
- **R6 · แผง ⚙** = กางขึ้นจากปุ่ม ⚙ (ปักซ้าย) · หัวแผงลากได้ · clamp ไม่หลุดขอบ · 📌 pin ดันปุ่มจากแผงขึ้นแถบได้ — **พฤติกรรมเดียวทุกโหมด**

### ข้อดี / ข้อเสีย
| ✔ ข้อดี | ✖ ข้อเสีย / ต้นทุน |
|---|---|
| โมเดลเดียวในหัวผู้ใช้ · grip+⚙ ขอบซ้ายตรงทุกโหมด · timeline เต็มแถว · "บ้านเดียว" ⚙ เลิกงง 3-ปุ่ม-vs-1-⚙ · แผงกางเหมือนกันหมด | print/edit: จัดปุ่ม/⋯ ลึกลง 1 แตะใน ⚙ (2 โหมดนี้ปุ่มน้อย ไม่ค่อยกระทบ) · **ต้อง refactor ให้ SingTransport + StudioDock ใช้โครง dock เดียว** (ตอนนี้ sing เลี่ยง chrome ของ dock ทั้งหมด → grip/แผง/clamp ซ้ำ 2 ที่) = งาน build จริง |

## 3 · คำถามให้ P'Aim เคาะ (ก่อนเขียน AC)

| # | คำถาม | SA แนะนำ |
|---|---|---|
| Q1 | ⚙ **ซ้ายล่าง** (ตรง grip) หรือ **ขวาล่าง** (นิ้วโป้งขวาถึงบนมือถือ)? | **ซ้ายล่าง** — ตรงกับที่พี่ขอ · เหตุผลเดียวจะเลือกขวา = เอื้อมมือถือ |
| Q2 | print/edit: ความโปร่ง+จัดปุ่ม+⋯ **ยุบเข้า ⚙** หรือแยกปุ่มขวาเหมือนเดิม? | **ยุบเข้า ⚙** — ให้เหมือนฝึกร้อง |
| Q3 | ปุ่มหลัก **กึ่งกลาง** ทุกโหมด หรือ **ชิดซ้ายหลัง ⚙** (มือคุ้นตำแหน่งปุ่มแก้ไข)? "สม่ำเสมอ" = เหมือนเป๊ะ หรือ ไวยากรณ์เดียว? | ฝึกร้อง=กลาง · แก้ไข=ชิดซ้าย (tools ≠ playback) — ขอพี่เคาะนิยาม |
| Q4 | timeline เต็มแถว → "เลือกท่อน" ไปแถวหลัก(ขวา) หรือเข้า ⚙? | ปุ่มเล็กแถวหลัง(ขวา) — ยังเห็นเร็ว |
| Q5 | ทำ **โครง dock เดียว** (sing เลิกวาด chrome เอง) เพื่อการันตี R1–R6? | ใช่ — ไม่งั้น drift อีก |
