# Brief — B098: แยกเครื่องมือ "โน้ต" กับ "ห้อง (bar)" — คัดลอก/ลบ ครบ 4 การกระทำ

**ฐาน:** `studio-shell-redesign` · **worktree/branch ใหม่:** `b098-note-bar-tools` (แตกจากฐาน · เช็ก `git branch --show-current` ก่อน commit)
**สั่งโดย:** PM (pm11) · ที่มา: P'Aim 12 ก.ค.

## อาการปัจจุบัน (P'Aim รายงาน)
ในหน้าแก้ไข ปุ่ม **✕** กับปุ่ม **ถังขยะ** ทำงาน **เหมือนกันเป๊ะ = ลบทั้งห้อง (bar)** — ซ้ำซ้อน

## สิ่งที่ต้องได้ (requirement — P'Aim เคาะ)
แยกเครื่องมือเป็น **2 ระดับ** ให้ครบ **4 การกระทำ**:
| ระดับ | คัดลอก | ลบ |
|---|---|---|
| **โน้ต** (เฉพาะตัวโน้ตที่เลือก) | คัดลอกโน้ต | ลบโน้ต |
| **ห้อง / bar** (ทั้งห้อง) | คัดลอกห้อง | ลบห้อง |
- "ห้อง" = bar (ช่องห้องเพลง · ยืนยันโดย P'Aim) · ไม่ใช่ทั้งบรรทัด
- ต้องชัดว่าอันไหนจัดการ "โน้ตตัวเดียว" อันไหน "ทั้งห้อง" — ผู้ใช้ต้องไม่สับสน

## เพิ่มเติม (P'Aim 12 ก.ค. · รวมเข้างานนี้เพราะโซนเดียวกัน)
- **หลังกด "เพิ่มห้อง" → ให้ cursor เด้งไปอยู่ในช่องโน้ตของห้องใหม่อัตโนมัติ พร้อมพิมพ์โน้ตต่อได้ทันที** (ไม่ต้องเอาเมาส์ไปคลิกช่องเอง) · ครอบใน verify + test ด้วย

## ขอบเขตงาน (สายที่รับ = วิเคราะห์+ออกแบบ+แก้+verify เองครบ)
- วิเคราะห์ UI เครื่องมือโน้ต/ห้อง ปัจจุบันในหน้าแก้ไข (ปุ่ม ✕/ถังขยะ ทำอะไรจริง) แล้วออกแบบที่วาง 4 การกระทำนี้
- **KISS — ปรับ/ต่อยอดของเดิม ไม่รื้อทำใหม่** (ถ้าจะรื้อใหญ่ ping PM ก่อน) · คงพฤติกรรม responsive/มือถือเดิม (B092: มือถือเก็บ สำเนา/ลบ ใน ⋯) ให้ยัง work
- ตาม `docs/ui-standards.md` (a11y: label/aria ชัด · target size) · ป้ายภาษาไทยอ่านรู้เรื่อง
