## สิ่งที่ต้องส่งมอบ

### 1. ✅ ขอบเขต — **P'Aim เคาะแล้ว (17 ก.ค.): เอาครบทั้ง 3 ข้อ**

> **1. ต้องใช้งาน คนไม่รู้ดนตรีต้องทำได้ด้วย**
> **2. ต้องวาดให้ถูกต้องทุกที่**
> **3. ต้องเล่นให้ถูกต้องด้วย**
> *"ให้ SA ทำให้ครบนะ"*

**→ คำถาม "UI ก่อน แล้วเล่นทีหลัง" = ตกไป · ออกแบบให้ครบทั้ง 3 ตั้งแต่แรก** (ระบบสัญลักษณ์ที่ทำครึ่ง ๆ แย่กว่าไม่มี — UI สัญญาแล้วเล่นไม่ตาม = พี่เปาเลิกเชื่อทั้งระบบ)

**ข้อ 2 "ทุกที่" = ต้องไล่ให้ครบ ห้ามตกที่ไหน** — จอ · **พิมพ์ A4** · **PDF export** · แถบแก้ไข · (ที่ไหนอีก? **ไล่เอง อย่าเดา**)
**ข้อ 3 "เล่นถูก" = ไม่ใช่แค่เสียง** — เสียงเล่น · **คาราโอเกะ/ไฮไลต์ต้องตามลำดับที่กระโดด** · timeline dots · **MP3 export ต้องตรงกับที่ฟังสด**
**ข้อ 1 "คนไม่รู้ดนตรีทำได้" = ข้อที่ยากที่สุด** ไม่ใช่ข้อที่ง่ายที่สุด — **นี่คือหัวใจ ไม่ใช่ของแถม**

**⚠️ นี่คืองานใหญ่ — ถ้าต้องแบ่งเฟส แบ่งได้ แต่ต้อง *ออกแบบ* ครบทั้ง 3 ก่อน แล้วเสนอลำดับให้ PM/P'Aim เคาะ · ห้ามตัดข้อใดข้อหนึ่งทิ้งเงียบ ๆ**
**ข่าวดี: สถาปัตยกรรมเผื่อไว้แล้ว** — `resolvePlayOrder` คืน ranges · `midi.js expandRepeats` ทำ `‖: :‖` ระดับห้องอยู่แล้ว · คอมเมนต์ `songModel.js:185` บอกเองว่าเป็น "dispatch seam" ของ D.C./D.S./Coda **และ display pass ไม่ต้องเปลี่ยน**
- **เพลงเก่า:** ตอนนี้มีเพลงไหนพิมพ์ข้อความในช่องป้ายไว้แล้วบ้าง? (**นับจริง ห้ามเดา**) → เปลี่ยนเป็นตัวเลือกแล้วของเก่าจะเป็นยังไง? (**ห้ามทำข้อมูลเก่าหาย**)
- **ขอบเขตสัญลักษณ์:** เอาแค่ Fine/D.C./D.S./Segno หรือรวม Coda (𝄌) ด้วย?

### 2. ⭐ ออกแบบ UI ที่ **อธิบายตัวเองได้** — พี่เปาต้องไม่ต้องรู้ทฤษฎีดนตรี
**P'Aim อธิบายไว้เอง (ใช้ภาษานี้เลย — นี่คือ SSOT ของถ้อยคำ):**

> เพลง = **เส้นทางรถไฟฟ้า 4 สถานี** · **Fine** = ป้าย "ปลายทาง" (แต่จบได้ต่อเมื่อวนกลับมาเจอรอบสอง) · **D.C.** = ปุ่ม **"Reset"** กลับไปหน้าแรกสุด · **D.S.** = ปุ่ม **"Skip"** วาร์ปกลับไปกลางเรื่อง (จุดที่มีป้าย 𝄋)
> **D.C. al Fine** = นั่งไปสุดสาย → **วาร์ปกลับสถานี 1** → นั่งต่อ → **ถึงป้าย Fine ลงรถ จบ**
> **D.S. al Fine** = นั่งไปสุดสาย → **วาร์ปกลับสถานี 3 (จุด 𝄋)** ข้ามอินโทร → นั่งต่อ → **เจอ Fine จบ**

**ข้อกำหนดของ UI:**
- **เลือกจากรายการ ไม่ใช่พิมพ์** · แต่ละตัวเลือกมี **คำอธิบายภาษาคน** (ห้ามมีแค่ "D.C. al Fine" ลอย ๆ)
- **ตอบคำถามพี่เปาให้ได้: "กดแล้วจะไปโชว์ตรงไหน"** → ต้องเห็นผลก่อนกด/หลังกด
- **บอกลำดับการเล่นที่ได้เป็นภาษาคน** เช่น *"เล่น: ท่อน 1 → 2 → 3 → กลับไปท่อน 1 → จบที่ท่อน 2"* ← **นี่คือของที่ทำให้ "อ่านแล้วงง" หายไป**
- **กันการเลือกที่เป็นไปไม่ได้:** เลือก D.S. แต่ไม่มี Segno · เลือก D.C. al Fine แต่ไม่มี Fine → **เตือน/ปิดตัวเลือก อย่าปล่อยให้เลือกแล้วเงียบ**
- **issues11 บอกว่าต้องเลื่อนขึ้นบนสุดถึงกดได้** → **แก้เรื่องตำแหน่ง/การเข้าถึงด้วย** (อยู่ในเมนู "..." ของบรรทัด)
- **มาตรฐานสากล:** สัญลักษณ์ที่พิมพ์บนแผ่นต้องถูกต้อง (𝄋 Segno · 𝄌 Coda · Fine · D.C./D.S.) — **UI เป็นภาษาคนได้ แต่แผ่นที่พิมพ์ต้องเป็นสากล** (`docs/ui-standards.md` · WCAG)
- **⛔ refine ไม่ใช่ redesign** — ต่อยอดเมนูบรรทัดเดิม ไม่รื้อ editor

### 3. เสนอ (ไม่ต้อง implement)
- โครงข้อมูล: เก็บเป็นอะไรแทน free text · **ต่อกับ `resolvePlayOrder` ยังไง** (คอมเมนต์เขาบอกจุดต่อไว้แล้ว)
- **ผลต่อการพิมพ์แผ่น A4 + คาราโอเกะ/ไฮไลต์** (ป้ายทำให้ลำดับเล่นเปลี่ยน → ไฮไลต์ต้องตามถูก)
- ⚠️ **คู่มือ `/#/notation` ที่เพิ่งเผยแพร่ ไม่มีเรื่องนี้** → ถ้าทำ ต้องเพิ่มหัวข้อ (**ใช้ภาษารถไฟฟ้าของ P'Aim**)
