## B034 — ปุ่มหุบ dock กดแล้วกางกลับไม่ได้ (desktop) · รุนแรง สูง · ชัดเจน
- **อาการ:** กดปุ่มหุบ → `collapsed=true` ✓ แต่ปุ่มเดิมยังโชว์ตัวเดียว กดซ้ำ = หุบซ้ำ ไม่กาง → dock ค้างหุบ ต้อง refresh (เฉพาะ desktop · มือถือใช้ `.sd-tab` กางได้ปกติ)
- **ต้นเหตุ (sa-ps3):** `StudioDock.vue` ปุ่มหุบ (~บรรทัด 292–297) `@click.stop="collapse"` เรียก `collapse()` เสมอ ไม่มี expand · ปุ่มไม่มีคลาส `hideoncol` เลยเป็นปุ่มเดียวที่เหลือตอนหุบ · ทางกางจริง `onDockClick` (~บรรทัด 157) ถูก `.stop` บัง + ปุ่มกินพื้นที่แถบเกือบหมด
- **แนวแก้:** ทำปุ่มหุบให้ **toggle** — ตอน `collapsed` ให้ `expand()` · หรือ ตอนหุบซ่อนปุ่มหุบแล้วโชว์ปุ่ม "กาง" แทน (ให้ desktop กางคืนได้ทุกกรณี)

## B033 — คีย์โน้ต 21 ตัวยัดแถวเดียว เล็ก/กดยาก (มือถือ ~14px) · รุนแรง สูง
- **P'Aim เคาะ:** แตกแถวคีย์โน้ตเป็น **2 แถว** · แถวปุ่มเครื่องมือ (tools) เดิม**คงไว้** → dock รวม **3 แถว**
- **เลย์เอาต์เป้าหมาย:**
  ```
  แถว 1 (คีย์)      1 2 3 4 5 6 7 0 – ~        ← ตัวเลข/โน้ต + พัก(0) + ขีด + ไท
  แถว 2 (คีย์)      . ' _ ^ ( ) { } # b n       ← จุด/อ็อกเทฟ + วงเล็บ + # b n
  แถว 3 (เครื่องมือ) ✋ ↶ ↷ ▷ ⬇ … หุบ            ← ของเดิม ไม่เปลี่ยน
  ```
  - เก็บ **ครบทั้ง 21 คีย์** (อย่าตัดทิ้ง) · จัดกลุ่ม "ตัวเลขบน–เครื่องหมายล่าง" · ปุ่มใหญ่ขึ้น กดง่ายบนมือถือ
  - ใช้สไตล์ปุ่มอ้างอิง `docs/design/ps3-dock-prototype.html` (`.key`) · **หมายเหตุ:** prototype เดิมมีแค่ 11 คีย์แถวเดียว — ของจริง 21 ตัว 2 แถวนี้ = ส่วนขยายที่ตกลงกันใหม่ (prototype ยังไม่ได้อัปเดต)
- **ต้นเหตุ (sa-ps3):** `StudioDock.vue` `.sd-keys{flex-wrap:nowrap} .sd-key{flex:1 1 0}` (บรรทัดเดียว ไม่ scroll) + palette 21 คีย์ส่งจาก `EditorMode`
- **แนวแก้:** ให้ `.sd-keys` เป็น 2 แถว (เช่น grid 2 rows / wrap ตามกลุ่ม) · คีย์แต่ละตัวใหญ่พอกดบนมือถือ
